# Angular ボタングループ コンポーネント概要

<div class="highlight">
Angular Button Group コンポーネントは、ボタンを水平/垂直配置、単一/複数選択、切り替え機能を備えたスタイル付きボタングループに整理するために使用されます。
</div>

## Ignite UI for Angular ボタングループを使い始める

Ignite UI for Angular ボタングループ コンポーネントを使い始めるには、まず Ignite UI for Angular をインストールする必要があります。既存の Angular アプリケーションで、次のコマンドを入力します:

```cmd
ng add igniteui-angular
```

Ignite UI for Angular の完全な紹介については、[*使い始める*](/general/getting-started)トピックをお読みください。

次のステップは、`IgxButtonGroupModule` を **app.module.ts** ファイルにインポートすることです。

```typescript
// app.module.ts

...
import { IgxButtonGroupModule } from 'igniteui-angular/button-group';
// import { IgxButtonGroupModule } from '@infragistics/igniteui-angular'; for licensed package

@NgModule({
    ...
    imports: [..., IgxButtonGroupModule],
    ...
})
export class AppModule {}
```

または、`16.0.0` 以降では、`IgxButtonGroupComponent` をスタンドアロン依存関係としてインポートするか、[`IGX_BUTTON_GROUP_DIRECTIVES`](https://github.com/IgniteUI/igniteui-angular/blob/master/projects/igniteui-angular/button-group/src/button-group/public_api.ts) トークンを使用してコンポーネントとそのすべてのサポーティング コンポーネントおよびディレクティブをインポートできます。

```typescript
// home.component.ts

...
import { IGX_BUTTON_GROUP_DIRECTIVES } from 'igniteui-angular/button-group';
import { IgxIconComponent } from 'igniteui-angular/icon';
// import { IGX_BUTTON_GROUP_DIRECTIVES, IgxIconComponent } from '@infragistics/igniteui-angular'; for licensed package

@Component({
    selector: 'app-home',
    template: `
    <igx-buttongroup>
        <button igxButton>
            <igx-icon>format_align_left</igx-icon>
        </button>
        <button igxButton>
            <igx-icon>format_align_center</igx-icon>
        </button>
        <button igxButton>
            <igx-icon>format_align_right</igx-icon>
        </button>
        <button igxButton selected>
            <igx-icon>format_align_justify</igx-icon>
        </button>
    </igx-buttongroup>
    `,
    styleUrls: ['home.component.scss'],
    standalone: true,
    imports: [IGX_BUTTON_GROUP_DIRECTIVES, IgxIconComponent]
    /* or imports: [IgxButtonGroupComponent, IgxButtonDirective, IgxIconComponent] */
})
export class HomeComponent {}
```

<DocsAside type="info">
このコンポーネントはMaterial Iconsを使用しています。以下のリンクを`index.html`に追加してください:`<link href="https://fonts.googleapis.com/icon?family=Material+Icons" rel="stylesheet">`
</DocsAside>

Ignite UI for Angular ボタングループ モジュールまたはディレクティブをインポートしたので、`igx-buttongroup` およびそのボタンの基本的な構成を開始できます。

## Angular ボタングループ コンポーネントの使用

### ボタングループの追加

<ApiLink type="ButtonGroup" label="igx-buttongroup" /> セレクターを使用してボタンをラップし、ボタングループに表示します。デフォルトで選択されるボタンが必要な場合は、`selected` プロパティを使用します:

```html
{/* sample.component.html */}
<igx-buttongroup>
  <button igxButton>
    <igx-icon>format_align_left</igx-icon>
  </button>
  <button igxButton>
    <igx-icon>format_align_center</igx-icon>
  </button>
  <button igxButton>
    <igx-icon>format_align_right</igx-icon>
  </button>
  <button igxButton selected>
    <igx-icon>format_align_justify</igx-icon>
  </button>
</igx-buttongroup>
```

### 選択

`igx-buttongroup` 選択を構成するために、その <ApiLink type="ButtonGroup" member="selectionMode" label="selectionMode" /> プロパティを使用できます。このプロパティは次の 3 つのモードを受け入れます:

- **single** - ボタングループのデフォルト選択モード。単一のボタンはユーザーによって選択/非選択にすることができます。
- **singleRequired** - ラジオグループの動作を模倣します。1 つのボタンのみを選択できます。初期選択後は、ユーザー操作による非選択は不可能です。
- **multi** - グループ内の複数のボタンを選択および非選択にすることができます。

### カスタム トグル ボタン

<ApiLink type="ButtonGroup" member="values" label="values" /> 入力プロパティを使用して、ボタングループ内にカスタマイズされたボタンの配列を設定します。

```typescript
// sample.component.ts

interface IButton {
    ripple?: string;
    label?: string;
    disabled?: boolean;
    togglable?: boolean;
    selected?: boolean;
    color?: string;
    icon?: string;
}

class ToggleButton {
    private ripple: string;
    private label: string;
    private disabled: boolean;
    private togglable: boolean;
    private selected: boolean;
    private color: string;
    private icon: string;

    constructor(obj?: IButton) {
        this.ripple = obj.ripple || 'gray';
        this.label = obj.label;
        this.selected = obj.selected || false;
        this.togglable = obj.togglable || true;
        this.disabled = obj.disabled || false;
        this.color = obj.color;
        this.icon = obj.icon;
    }
}
...
public bordersButtons: ToggleButton[];

public ngOnInit() {
    this.bordersButtons = [
        new ToggleButton({
            icon: 'border_top',
            selected: true
        }),
        new ToggleButton({
            icon: 'border_right',
            selected: false
        }),
        new ToggleButton({
            icon: 'border_bottom',
            selected: false
        }),
        new ToggleButton({
            icon: 'border_left',
            selected: false
        })
    ];
}
...
```

```html
{/* sample.component.html */}
<igx-buttongroup [selectionMode]="'multi'" [values]="bordersButtons"></igx-buttongroup>
```

## スタイリング

### ボタングループ テーマ プロパティ マップ

`$item-background` プロパティに値を設定すると、以下の表に記載されている関連するすべての従属プロパティが自動的に更新され、視覚的な一貫性が保たれます。テーブルは、プライマリ プロパティをカスタマイズするときに影響を受けるプロパティを示しています。

| プライマリ プロパティ         | 従属プロパティ                    | 説明                                      |
| ----------------------------- | --------------------------------- | ----------------------------------------- |
| **$item-background**          | $item-hover-background            | アイテムのホバー背景色。                  |
|                               | $item-selected-background         | 選択されたアイテムの背景色。              |
|                               | $item-focused-background          | フォーカスされたアイテムの背景色。        |
|                               | $disabled-background-color        | 無効なアイテムの背景色。                  |
|                               | $item-border-color                | アイテムの境界線の色。                    |
|                               | $item-text-color                  | アイテムのテキスト色。                    |
|                               | $idle-shadow-color                | アイテムのアイドル シャドウ色。           |
| **$item-hover-background**    | $item-selected-hover-background   | 選択されたアイテムのホバー背景色。        |
|                               | $item-focused-hover-background    | フォーカスされたホバー背景色。            |
|                               | $item-hover-text-color            | ホバーされたアイテムのテキスト色。        |
|                               | $item-hover-icon-color            | ホバーされたアイテムのアイコン色。        |
| **$item-selected-background** | $item-selected-focus-background   | 選択されたアイテムのフォーカス背景色。    |
|                               | $disabled-selected-background     | 無効な選択背景色。                        |
|                               | $item-selected-text-color         | 選択されたアイテムのテキスト色。          |
|                               | $item-selected-icon-color         | 選択されたアイテムのアイコン色。          |
|                               | $item-selected-hover-text-color   | 選択されたホバーアイテムのテキスト色。    |
|                               | $item-selected-hover-icon-color   | 選択されたホバーアイテムのアイコン色。    |
| **$item-border-color**        | $item-hover-border-color          | ホバーされたアイテムの境界線の色。        |
|                               | $item-focused-border-color        | フォーカスされたアイテムの境界線の色。    |
|                               | $item-selected-border-color       | 選択されたアイテムの境界線の色。          |
|                               | $item-selected-hover-border-color | 選択されたホバーアイテムの境界線の色。    |
|                               | $item-disabled-border             | 無効なアイテムの境界線の色。              |
|                               | $disabled-selected-border-color   | 無効な選択アイテムの境界線の色。          |

ボタングループのスタイリングを開始するには、すべてのテーマ関数とコンポーネント mixins が存在する `index` ファイルをインポートする必要があります:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

最も単純なアプローチに従うと、<ApiLink kind="sass" module="themes" type="function-button-group-theme" label="button-group-theme" code={true} /> を拡張し、`$item-background` のみを指定する新しいテーマを作成します。この値に基づいて、テーマは自動的に適切な状態色、コントラスト フォアグラウンド、およびボタングループの境界線を計算します。もちろん、必要に応じて、テーマ パラメーターをカスタム値でオーバーライドすることもできます。

```scss
$custom-button-group: button-group-theme(
  $item-background: #57a5cd,
);
```

ご覧のとおり、`button-group-theme` は、そのアイテムの基本的なスタイリングに役立つパラメーターを公開しています。さらに詳しく調べて、ボタン固有のパラメーターを変更したい場合は、`button-theme` を拡張し、それぞれのボタングループ クラスの下でスコープする新しいテーマを作成できます。

最後のステップは、コンポーネントのテーマをインクルードすることです。

```scss
:host {
  @include tokens($custom-button-group);
}
```

### Tailwind でのスタイリング

カスタム Tailwind ユーティリティ クラスを使用して `button-group` をスタイリングできます。まず [Tailwind を設定](/themes/misc/tailwind-classes)してください。

グローバル スタイルシート内の tailwind インポートと共に、次のようにして目的のテーマ ユーティリティを適用できます:

```scss
@import "tailwindcss";
...
@use 'igniteui-theming/tailwind/utilities/material.css';
```

ユーティリティ ファイルには、`light` と `dark` テーマ バリアントの両方が含まれています。

- ライト テーマには `light-*` クラスを使用します。
- ダーク テーマには `dark-*` クラスを使用します。
- プレフィックスの後にコンポーネント名を追加します。例: `light-button-group`、`dark-button-group`。

適用すると、これらのクラスは動的テーマの計算を有効にします。ここから、`arbitrary properties` を使用して生成された CSS 変数をオーバーライドできます。コロンの後に、有効な CSS カラー形式 (HEX、CSS 変数、RGB など) を指定します。

<ApiLink kind="sass" module="themes" type="function-button-group-theme" label="button-group-theme" /> でプロパティの完全なリストを見つけることができます。構文は以下の通りです:

```html
<igx-button-group
class="!light-button-group ![--item-background:#7B9E89]"
>
...
</igx-button-group>
```

<DocsAside type="info">
コンポーネントのデフォルトテーマによってオーバーライドされないようにするために、感嘆符(`!`)は必須です。Tailwindはスタイルをレイヤーで適用し、これらのスタイルを重要としてマークしないと、コンポーネントのデフォルトテーマによってオーバーライドされます。
</DocsAside>
