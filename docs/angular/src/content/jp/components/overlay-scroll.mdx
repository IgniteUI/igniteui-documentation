# スクロール ストラテジ (スクロール方法)

スクロール方法は指定された `IgxOverlayService` でスクロールを処理する方法を決定します。4 つのスクロール方法があります。
1. **NoOperation** -  何もしません。
2. **Block** - イベントはキャンセルされ、コンポーネントはウィンドウと共にスクロールしません。
3. **Close** - 許容値を使用して許容範囲を超えた場合にスクロールで展開したコンポーネントを閉じます。
4. **Absolute** - すべてをスクロールします。

1. **NoOperation** - does nothing.
2. **Block** - the event is canceled and the component does not scroll with the window.
3. **Close** - uses a tolerance and closes an expanded component upon scrolling if the tolerance is exceeded.
4. **Absolute** - scrolls everything.

## 使用方法

各スクロール方法は以下のメソッドがあります。
- <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" member="initialize" /> - スクロール方法を初期化します。ドキュメントへの参照、オーバーレイ サービスへの参照、および描画されるコンポーネントの id が必要です。
- <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" member="attach" /> - スクロール方法を指定した要素またはドキュメントにアタッチします。
- <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" member="detach" /> - スクロール方法をデタッチします。

- <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" member="initialize" /> - initializes the scroll strategy. It needs a reference of the document, the overlay service and the id of the component rendered
- <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" member="attach" /> - attaches the scroll strategy to the specified element or to the document
- <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" member="detach" /> - detaches the scroll strategy

```typescript
this.scrollStrategy.initialize(document, overlayService, id);
this.scrollStrategy.attach();
this.scrollStrategy.detach();
```

### 作業の開始

ポジション ストラテジは、<ApiLink type="OverlayService" suffix={false} member="attach" label="overlay.attach()" /> メソッドが呼ばれたときに <ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" /> パラメーターのプロパティとして渡されます。

```typescript
// Initializing and using overlay settings
const overlaySettings: OverlaySettings = {
    positionStrategy: new GlobalPositionStrategy(),
    scrollStrategy: new AbsoluteScrollStrategy(), //Passes the scroll strategy
    modal: true,
    closeOnOutsideClick: true
}
const overlayId = overlay.attach(dummyElement, overlaySettings); 
```

<div class="divider"></div>

オーバーレイで使用するスクロール方法の変更は、オーバーレイに渡される <ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" /> オブジェクトの <ApiLink kind="interface" prefixed={false} suffix={false} type="IScrollStrategy" /> プロパティをオーバーライドします。ストラテジがすでにアタッチされている場合は、以前に生成された ID をデタッチする必要があります:

```typescript
// overlaySettings is an existing object of type OverlaySettings
// to override the scroll strategy
const newOverlaySettings = Object.assing({}, overlaySettings);
Object.assing(newOverlaySettings, {
    scrollStrategy: new CloseScrollStrategy()
})
const overlayId = overlay.attach(dummyElement, newOverlaySettings);
overlay.show(overlayId);
```

## モーダル

<ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" /> オブジェクトにブール値 (<ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" member="modal" />) を渡すことができます。これはオーバーレイが表示される方法を制御します。
- <ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" member="modal" /> プロパティが `false` の場合、要素は DOM 前景にアタッチされますが、すべての要素がまだアクティブで操作可能 (スクロール、クリックなど) です。
- <ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" member="modal" /> プロパティが `true` の場合、要素が DOM 前景にアタッチされ、背景の要素が非アクティブなため操作不能となり、すべてのイベントが中止されます。

- If the <ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" member="modal" /> property is `false`, the element will be attached to the DOM foreground but everything will still be active and interactable - e.g. scrolling, clicking, etc.
- If the <ApiLink kind="interface" prefixed={false} suffix={false} type="OverlaySettings" member="modal" /> property is `true`, the element will be attached to the DOM foreground and an overlay blocker will wrap behind it, stopping propagation of all events:
