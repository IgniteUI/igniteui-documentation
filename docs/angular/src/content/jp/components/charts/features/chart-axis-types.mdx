# Angular 軸タイプ

Ignite UI for Angular カテゴリ チャートは、<ApiLink type="CategoryXAxis" /> および <ApiLink type="NumericYAxis" /> タイプを 1 つだけ使用します。同様に、Ignite UI for Angular ファイナンシャル チャートは、<ApiLink type="TimeXAxis" /> タイプと <ApiLink type="NumericYAxis" /> タイプを 1 つだけ使用します。ただし、Ignite UI for Angular データ チャートは複数の軸タイプをサポートしており、[軸の位置](./chart-axis-layouts.mdx#軸位置の例)を設定してチャートの任意の側に配置したり、[軸交差](./chart-axis-layouts.mdx#軸交差の例)プロパティを使用してチャートの内部に配置したりできます。このトピックでは、相互に互換性のある軸とシリーズ、および固有の軸に対するいくつかの特定のプロパティについて、それぞれについて説明します。

## デカルト軸

デカルト軸を持つ <ApiLink type="DataChart" /> では 3 つのタイプの X 軸を使用して水平 (X 軸) および垂直 (X 軸) 方向にデータをプロットすることが可能です。
(<ApiLink type="CategoryXAxis" />、<ApiLink type="NumericXAxis" />、および <ApiLink type="TimeXAxis" />) と 2 つのタイプの Y 軸 (<ApiLink type="CategoryYAxis" /> および <ApiLink type="NumericYAxis" />)。

### カテゴリ X 軸

<ApiLink type="CategoryXAxis" /> は、そのデータを一連のカテゴリ データ項目として扱います。文字列や数値など、ほぼすべてのタイプのデータを表示できます。この軸に数値をプロットする場合、この軸は離散軸であり、連続ではないことに注意してください。これは、各カテゴリ データ項目がその前のデータ項目から等距離に配置されることを意味します。また、項目は軸のデータ ソースに表示される順序でプロットされます。

<ApiLink type="CategoryXAxis" /> では、データをプロットするために <ApiLink type="DataSource" /> と <ApiLink type="NumericYAxis" member="label" label="Label" /> を提供する必要があります。通常、次のタイプの系列をプロットするために <ApiLink type="NumericYAxis" /> と共に使用されます。

### カテゴリ Y 軸

<ApiLink type="CategoryYAxis" /> は、上記の <ApiLink type="CategoryXAxis" /> と非常によく似た働きをしますが、水平ではなく垂直に配置されます。また、この軸では、データをプロットするために <ApiLink type="DataSource" /> と <ApiLink type="NumericYAxis" member="label" label="Label" /> を提供する必要があります。<ApiLink type="CategoryYAxis" /> は通常 <ApiLink type="NumericXAxis" />  と組み合わせて次のタイプのシリーズをプロットするために使用されます。

- <ApiLink type="BarSeries" />
- <ApiLink type="StackedBarSeries" />
- <ApiLink type="Stacked100BarSeries" />

### 数値 X 軸

<ApiLink type="NumericXAxis" /> は、そのデータを連続的に変化する数値データ項目として扱います。この軸のラベルは、X 軸に沿って水平に配置されます。<ApiLink type="NumericXAxis" /> ラベルの位置は、<ApiLink type="NumericYAxis" /> と組み合わせた場合にサポートされるさまざまな [ScatterSeries](../types/scatter-chart.mdx) (散布シリーズ) の <ApiLink type="BubbleSeries" member="xMemberPath" label="XMemberPath" /> プロパティに依存します。または、<ApiLink type="CategoryXAxis" /> と組み合わせた場合、これらのラベルは、<ApiLink type="BarSeries" />、<ApiLink type="StackedBarSeries" />、および <ApiLink type="Stacked100BarSeries" /> の <ApiLink type="AreaSeries" member="valueMemberPath" label="ValueMemberPath" /> に対応して配置されます。

### 時間 X 軸

<ApiLink type="TimeXAxis" /> は、そのデータを、日付でソートされた一連のデータ項目として扱います。この軸タイプのラベルは日付であり、日付間隔に従ってフォーマットおよび配置できます。この軸の日付範囲は、<ApiLink type="TimeXAxis" member="dateTimeMemberPath" label="DateTimeMemberPath" /> を使用してマップされたデータ列の日付値によって決定されます。これは、<ApiLink type="DataSource" /> とともに、この軸タイプでデータをプロットするために必要です。

<ApiLink type="TimeXAxis" /> は、<ApiLink type="FinancialChart" /> コンポーネントの X 軸タイプです。

#### 時間 X 軸のブレーク

<ApiLink type="TimeXAxis" /> には、<ApiLink type="TimeXAxis" member="breaks" label="Breaks" /> (ブレーク) を使用してデータの間隔を除外するオプションがあります。その結果、ラベルとプロットされたデータは除外された間隔では表示されません。たとえば、勤務日/休業日、休日、週末などです。<ApiLink type="TimeAxisBreak" /> のインスタンスを軸の <ApiLink type="TimeXAxis" member="breaks" label="Breaks" /> コレクションに追加し、一意の <ApiLink type="TimeAxisBreak" member="start" label="Start" />、<ApiLink type="TimeAxisBreak" member="end" label="End" />、および <ApiLink type="NumericYAxis" member="interval" label="Interval" /> を使用して構成できます。

#### 時間 X 軸の書式設定

<ApiLink type="TimeXAxis" /> には、<ApiLink type="TimeAxisLabelFormat" /> オブジェクトのコレクションを表す <ApiLink type="TimeXAxis" member="labelFormats" label="LabelFormats" /> プロパティがあります。コレクションに追加された各 <ApiLink type="TimeAxisLabelFormat" /> は、一意の <ApiLink type="TimeAxisLabelFormat" member="format" label="Format" /> (書式) と <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" /> (範囲) を割り当てる役割を果たします。これは、データを年からミリ秒にドリルダウンし、チャートに表示される時間の範囲に応じてラベルを調整する場合に特に役立ちます。

<ApiLink type="TimeAxisLabelFormat" /> の <ApiLink type="TimeAxisLabelFormat" member="format" label="Format" /> プロパティは、特定の表示範囲に使用する形式を指定します。<ApiLink type="TimeAxisLabelFormat" /> の <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" /> プロパティは、軸ラベルの形式が別の形式に切り替わる表示範囲を指定します。たとえば、範囲が 10 日と 5 時間に設定された 2 つの <ApiLink type="TimeAxisLabelFormat" /> 要素がある場合、軸の表示範囲が 10 日未満になるとすぐに、5 時間形式に切り替わります。

#### 時間 X 軸の間隔

<ApiLink type="TimeXAxis" /> は、カテゴリ軸と数値軸の従来の <ApiLink type="NumericYAxis" member="interval" label="Interval" /> プロパティを <ApiLink type="TimeAxisInterval" /> 型の <ApiLink type="TimeXAxis" member="intervals" label="Intervals" /> コレクションに置き換えます。コレクションに追加された各 <ApiLink type="TimeAxisInterval" /> は、一意の <ApiLink type="NumericYAxis" member="interval" label="Interval" />、<ApiLink type="TimeAxisLabelFormat" member="range" label="Range" />、および <ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" /> を割り当てる役割を果たします。これは、データを年単位からミリ秒単位にドリルダウンして、チャートに表示される時間の範囲に応じてラベル間に一意の間隔を設ける場合に特に役立ちます。これらのプロパティの説明は次のとおりです。

- <ApiLink type="NumericYAxis" member="interval" label="Interval" />: 使用する間隔を指定します。<ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" /> プロパティに関連付けられています。たとえば、<ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" /> が `Days` に設定されている場合、<ApiLink type="NumericYAxis" member="interval" label="Interval" /> で指定される数値は日数になります。
- <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" />: 軸間隔が別の間隔に切り替わる可視範囲を指定します。たとえば、範囲が 10 日間および 5 時間に設定された 2 つの TimeAxisInterval がある場合、軸の表示範囲が 10 日間より短くなる際に 5 時間範囲の間隔に変更します。
- <ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" />: <ApiLink type="NumericYAxis" member="interval" label="Interval" /> プロパティの時間単位を指定します。

## 極座標軸

極座標軸を持つ <ApiLink type="DataChart" /> により、チャートの中心から外側 (半径軸) およびチャートの中心の周り (角度軸) にデータをプロットできます。

### カテゴリ角度軸

<ApiLink type="CategoryAngleAxis" /> は、そのデータを一連のカテゴリ データ項目として扱います。この軸のラベルは、その順序での位置に従って円の端に沿って配置されます。この軸のタイプでは、数字、文字列などのほぼすべてのデータのタイプを表示できます。

<ApiLink type="CategoryAngleAxis" /> は通常、[ラジアル シリーズ](../types/radial-chart.mdx)をプロットするために <ApiLink type="NumericRadiusAxis" /> と共に使用されます。

### 比例カテゴリ角度軸

<ApiLink type="ProportionalCategoryAngleAxis" /> は、そのデータを一連のカテゴリ データ項目として扱います。この軸のラベルは、シーケンス内の位置に応じて円の端に沿って配置されます。この軸の種類では、数字、文字列などのほぼすべてのデータのタイプを表示できます。

<ApiLink type="ProportionalCategoryAngleAxis" /> は通常、<ApiLink type="NumericRadiusAxis" /> と一緒に使用され、円チャートをプロットします (例: [ラジアル シリーズ](../types/radial-chart.mdx))。

### 数字角度軸

<ApiLink type="NumericAngleAxis" /> は、そのデータを連続的に変化する数値データ項目として扱います。この軸領域のラベルは、円形プロットの中心から始まる半径線に沿って配置されます。<ApiLink type="NumericAngleAxis" /> のラベルの位置は、[極座標シリーズ](../types/polar-chart.mdx) オブジェクトの <ApiLink type="BubbleSeries" member="radiusMemberPath" label="RadiusMemberPath" /> プロパティまたは[ラジアル シリーズ](../types/radial-chart.mdx) オブジェクトの <ApiLink type="AreaSeries" member="valueMemberPath" label="ValueMemberPath" /> プロパティを使用してマップされたデータ列の値によって異なります。

<ApiLink type="NumericAngleAxis" /> は、<ApiLink type="CategoryAngleAxis" /> と共に使用して[ラジアル シリーズ](../types/radial-chart.mdx)をプロットするか、<ApiLink type="NumericRadiusAxis" /> と共に使用して[極座標シリーズ](../types/polar-chart.mdx)をプロットすることができます。

### 数字半径軸

<ApiLink type="NumericRadiusAxis" /> は、データを連続的に変化する数値データ項目として扱います。この軸のラベルは、円形プロットの周りに配置されます。ラベルの位置は、対応する極座標シリーズの `AngleMemberPath` プロパティを使用してマップされたデータ列の値によって異なります。

<ApiLink type="NumericRadiusAxis" /> を <ApiLink type="NumericRadiusAxis" /> と共に使用して、[極座標シリーズ](../types/polar-chart.mdx)をプロットできます。
