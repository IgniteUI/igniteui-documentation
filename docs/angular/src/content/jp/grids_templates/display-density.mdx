# Angular {ComponentTitle} のサイズ

**{ComponentName}** デザインは[マテリアル デザイン ガイドライン](https://material.io/design)に基づきます。現在、それぞれ small (小)、medium (中)、または large (大) のビューを表示する、事前定義されたサイズ オプションのセットから選択するオプションが提供されています。マテリアル UI テーブル/マテリアル UI グリッドに適切なサイズを選択することで、量の多いコンテンツを操作する際のユーザー エクスペリエンスを飛躍的に向上させることができます。

## 使用方法

上記デモで示されるように、<ApiLink type="{ComponentApiType}" /> には 3 つのサイズ オプション (**small**、**medium**、**large**) が用意されています。以下のコード スニペットは、size を設定する方法を示します。

```html
<{ComponentSelector} #{ComponentObjectRef} [data]="data" style="--ig-size: var(--ig-size-small)">
</{ComponentSelector}>
```

以下のコード スニペットは、size を設定する方法を示します。各オプションを {ComponentTitle} に反映する方法を紹介します。サイズ間で切り替える際に各 {ComponentTitle} 要素の高さとそのパディングが変更されます。カスタムの列 <ApiLink type="Column" member="width" label="**width**" /> を適用する場合、左右のパディングより大きくする必要があることに注意してください。
- **--ig-size-large** - これはデフォルトの {ComponentTitle} サイズです。密度が最も低く、行の高さが `50px` です。左右のパディングは `24px` です。最小列 <ApiLink type="Column" member="width" label="width" /> は `80px` です。
- **--ig-size-medium** - これは行の高さが `40px` の中間サイズです。左右のパディングは `16px` です。最小列 <ApiLink type="Column" member="width" label="width" /> は `64px` です。
- **--ig-size-small** - これは行の高さが `32px` の最小サイズです。左右のパディングは `12px` です。最小列 <ApiLink type="Column" member="width" label="width" /> は `56px` です。

<DocsAside type="info" title="情報">
現在サイズはオーバーライド**できません**。
</DocsAside>

引き続きサンプルを使用して、各サイズの適用方法について説明します。最初に各サイズを切り替えるボタンを追加します。

```html

```

```typescript
@ViewChild(IgxButtonGroupComponent) public buttonGroup: IgxButtonGroupComponent;
public size = 'small';
public sizes;

public ngOnInit() {
    this.sizes = [
        {
            label: 'small',
            selected: this.size === 'small',
            togglable: true
        },
        {
            label: 'medium',
            selected: this.sie === 'medium',
            togglable: true
        },
        {
            label: 'large',
            selected: this.size === 'large',
            togglable: true
        }
    ];
}
```

最後にサイズを適用するためのロジックを実装します。

```typescript
@ViewChild('{ComponentObjectRef}', { read: {ComponentName}Component })
public {ComponentObjectRef}: {ComponentName}Component;

public selectSize(event: any) {
    this.size = this.sizes[event.index].label;
}

@HostBinding('style.--ig-size')
protected get sizeStyle() {
    return `var(--ig-size-${this.size})`;
}
```

<ApiLink type="{ComponentApiType}" /> の {ComponentTitle} 行の高さを変更するその他のオプションに <ApiLink type="{ComponentApiType}" member="rowheight" label="rowHeight" /> プロパティがあります。このプロパティと `--ig-size` CSS 変数 オプションが {ComponentTitle} レイアウトにどのように動作に影響するかを以下で確認できます。

以下を確認してください。
- **<ApiLink type="{ComponentApiType}" member="rowheight" label="rowHeight" /> が指定されている場合**、`--ig-size` CSS 変数は行の高さに**影響しません**。
- `--ig-size` は、上記の理由により**残りすべてのグリッド要素に影響します**。

サンプル機能を拡張して <ApiLink type="{ComponentApiType}" member="rowHeight" label="rowHeight" /> プロパティを {ComponentTitle} に追加します。

 ```html
 <{ComponentSelector} #{ComponentObjectRef} [data]="data" [rowHeight]="'80px'" width="100%" 
 height="550px" [allowFiltering]="true">
 ..............
 </{ComponentSelector}>
 ```
