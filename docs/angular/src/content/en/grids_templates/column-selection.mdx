# Angular {ComponentTitle} Column Selection

The Column selection feature provides an easy way to select an entire column with a single click. It emphasizes the importance of a particular column by focusing the header cell(s) and everything below. The feature comes with a rich `API` that allows for manipulation of the selection state, data extraction from the selected fractions and data analysis operations and visualizations.

## Basic usage

The column selection feature can be enabled through the <ApiLink type="{ComponentApiType}" member="columnSelection" label="columnSelection" /> input, which takes <ApiLink kind="enum" type="GridSelectionMode" /> values.

## Interactions

The default selection mode is `none`. If set to `single` or `multiple` all of the presented columns will be <ApiLink type="Column" member="selectable" label="selectable" />. With that being said, in order to select a column, we just need to click on one, which will mark it as <ApiLink type="Column" member="selected" label="selected" />. If the column is not selectable, no selection style will be applied on the header, while hovering.

<Aside type="note">
[`Multi-column Headers`](multi-column-headers) don't reflect on the <ApiLink type="Column" member="selectable" label="selectable" /> input. The <ApiLink type="ColumnGroup" /> is <ApiLink type="Column" member="selectable" label="selectable" />, if at least one of its children has the selection behavior enabled. In addition, the component is marked as <ApiLink type="ColumnGroup" member="selected" label="selected" /> if all of its `selectable` descendants are <ApiLink type="Column" member="selected" label="selected" />.
</Aside>

## Keyboard combinations

<Aside type="note">
The keyboard combinations are available only when the grid <ApiLink type="{ComponentApiType}" member="columnselection" label="columnSelection" /> input is set to `multiple`.
</Aside>

There are two scenarios for keyboard navigation of the **Column Selection** feature:

- Multi-column selection - holding <kbd>ctrl</kbd> + <kbd>click</kbd> on every **selectable** header cell.
- Range column selection - holding <kbd>shift</kbd> + <kbd>click</kbd> selects all **selectable** columns in between.

## API manipulations

The **API** provides some additional capabilities when it comes to the **non-visible** columns such that, every **hidden** column could be marked as <ApiLink type="Column" member="selected" label="selected" /> by setting the corresponding **setter**.

<Aside type="note">
The above statement also applies to the <ApiLink type="ColumnGroup" />, except that when the <ApiLink type="ColumnGroup" member="selected" label="selected" /> property is changed it changes the state of its descendants.
</Aside>

More information regarding the API manipulations could be found in the [`API References`](#api-references) section.

## Styling

Before diving into the styling options, the core module and all component mixins need to be imported.

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

<Aside type="note">
Please note that [`row selection`](row-selection) and [`column selection`](column-selection) can't be manipulated independently. They depend on the same `variables`.
</Aside>

With that being said, let's move on and change the **selection** and **hover** styles.<br/>
Following the simplest approach, let's define our custom **theme**.

<ComponentBlock for="TreeGrid">

```scss
$custom-grid-theme: grid-theme(
  $row-selected-background: #011627,
  $row-selected-text-color: #ecaa53,
  $row-selected-hover-background: #011627,
  $header-selected-text-color: #ecaa53,
  $header-selected-background: #011627,
  $expand-icon-color: #ecaa53,
  $expand-icon-hover-color: #b64b80
);
```

</ComponentBlock>
<ComponentBlock for="Grid, HierarchicalGrid, PivotGrid">

```scss
$custom-grid-theme: grid-theme(
  $row-selected-background: #011627,
  $row-selected-text-color: #ecaa53,
  $row-selected-hover-background: #011627,
  $header-selected-text-color: #ecaa53,
  $header-selected-background: #011627
);
```

</ComponentBlock>


The [`grid-theme`]({environment:sassApiUrl}/themes#function-grid-theme) accepts several parameters but those are the five responsible for changing the appearance of all selected columns:

- **$row-selected-background** - sets the background of the selected fraction.
- **$row-selected-text-color** - sets the text color of the selected fraction
- **$row-selected-hover-background** - sets the color of the hovered cell or bunch of cells.
- **$header-selected-text-color** - sets the text color of the selected column header
- **$header-selected-background** - sets the background color of the selected column header.

### Using CSS Variables

The last step is to include the custom `igx-grid` theme.

```scss
:host {
  @include tokens($custom-grid-theme)
}
```
