# Angular {ComponentTitle} Size

**{ComponentName}** design is based on [Material Design Guidelines](https://material.io/design). We currently provide an option to choose between predefined set of size options that will bring a small, medium, or large view respectively. By selecting the right size for your Material UI table / Material UI grid you can significantly improve the user experience when interacting with large amounts of content.

## Usage

As you can see in the demo above, the <ApiLink type="{ComponentApiType}" /> provides three size options: **small**, **medium** and **large**. The code snippet below shows how to set size:

```html
<{ComponentSelector} #{ComponentObjectRef} [data]="data" style="--ig-size: var(--ig-size-small)">
</{ComponentSelector}>
```

And now let's see in details how each option reflects on the {ComponentTitle} component. When you switch between different sizes the height of each {ComponentTitle} element and the corresponding paddings will be changed. Also if you want to apply custom column <ApiLink type="Column" member="width" label="**width**" />, please consider the fact that it must be bigger than the sum of left and right padding.

- **--ig-size-large** - this is the default {ComponentTitle} size with the lowest intense and row height equal to `50px`. Left and Right paddings are `24px`; Minimal column <ApiLink type="Column" member="width" label="width" /> is `80px`;
- **--ig-size-medium** - this is the middle size with `40px` row height. Left and Right paddings are `16px`; Minimal column <ApiLink type="Column" member="width" label="width" /> is `64px`;
- **--ig-size-small** - this is the smallest size with `32px` row height. Left and Right paddings are `12px`; Minimal column <ApiLink type="Column" member="width" label="width" /> is `56px`;

<DocsAside type="info">
Please keep in mind that currently you **can not** override any of the sizes.
</DocsAside>

Let's now continue with our sample and see in action how each size is applied. Let's first add a button which will help us to switch between each size:

```html

</div>
```

```typescript
@ViewChild(IgxButtonGroupComponent) public buttonGroup: IgxButtonGroupComponent;

public sizes;

public ngOnInit() {
    this.sizes = [

            label: 'small',
            selected: this.size === 'small',
            togglable: true

        {
            label: 'medium',
            selected: this.sie === 'medium',
            togglable: true
        },
        {
            label: 'large',
            selected: this.size === 'large',
            togglable: true
        }
    ];
}
```

Finally, let's provide the necessary logic in order to actually apply the size:

```typescript
@ViewChild('{ComponentObjectRef}', { read: {ComponentName}Component })
public {ComponentObjectRef}: {ComponentName}Component;

public selectSize(event: any) {
    this.size = this.sizes[event.index].label;
}

@HostBinding('style.--ig-size')
protected get sizeStyle() {
    return `var(--ig-size-${this.size})`;
}
```

Another option that <ApiLink type="{ComponentApiType}" /> provides for you, in order to be able to change the height of the rows in the {ComponentTitle}, is the property <ApiLink type="{ComponentApiType}" member="rowheight" label="rowHeight" />. So let's see in action how this property affects the {ComponentTitle} layout along with the `--ig-size` CSS variable.

Please keep in mind the following:

- `--ig-size` CSS variable will have **NO** impact on row height **if there is <ApiLink type="{ComponentApiType}" member="rowheight" label="rowHeight" /> specified**;
- `--ig-size` will **affect all of the rest elements in the {ComponentTitle}**, as it has been described above;

And now we can extend our sample and add <ApiLink type="{ComponentApiType}" member="rowHeight" label="rowHeight" /> property to the {ComponentTitle}:

 ```html
 <{ComponentSelector} #{ComponentObjectRef} [data]="data" [rowHeight]="'80px'" width="100%" 
 height="550px" [allowFiltering]="true">
 ..............
 </{ComponentSelector}>
 ```
