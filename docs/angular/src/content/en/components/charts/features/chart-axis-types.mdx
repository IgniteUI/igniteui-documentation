# Angular Axis Types

The Ignite UI for Angular Category Chart uses only one <ApiLink type="CategoryXAxis" /> and one <ApiLink type="NumericYAxis" /> type. Similarly, Ignite UI for Angular Financial Chart uses only one <ApiLink type="TimeXAxis" /> and one <ApiLink type="NumericYAxis" /> types. However, the Ignite UI for Angular Data Chart provides support for multiple axis types that you can position on any side of the chart by setting [axis location](chart-axis-layouts.md#axis-locations-example) or even inside of the chart by using [axis crossing](chart-axis-layouts.md#axis-crossing-example) properties. This topic goes over each one, which axes and series are compatible with each other, and some specific properties to the unique axes.

## Cartesian Axes

The <ApiLink type="DataChart" /> with Cartesian Axes, allows you to plot data in horizontal (X-axis) and vertical (X-axis) direction with 3 types of X-Axis
(<ApiLink type="CategoryXAxis" />, <ApiLink type="NumericXAxis" />, and <ApiLink type="TimeXAxis" />) and 2 types of Y-Axis (<ApiLink type="CategoryYAxis" /> and <ApiLink type="NumericYAxis" />).

### Category X-Axis

The <ApiLink type="CategoryXAxis" /> treats its data as a sequence of categorical data items. It can display almost any type of data including strings and numbers. If you are plotting numbers on this axis, it is important to keep in mind that this axis is a discrete axis and not continuous. This means that each categorical data item will be placed equidistant from the one before it. The items will also be plotted in the order that they appear in the axis' data source.

The <ApiLink type="CategoryXAxis" /> requires you to provide a <ApiLink type="DataSource" /> and a <ApiLink type="NumericYAxis" member="label" label="Label" /> in order to plot data with it. It is generally used with the <ApiLink type="NumericYAxis" /> to plot the following type of series:

### Numeric X-Axis

The <ApiLink type="NumericXAxis" /> treats its data as continuously varying numerical data items. Labels on this axis are placed horizontally along the X-Axis. The location of the <ApiLink type="NumericXAxis" /> labels depends on the <ApiLink type="BubbleSeries" member="xMemberPath" label="XMemberPath" /> property of the various [Scatter Series](../types/scatter-chart.md) that it supports if combined with a <ApiLink type="NumericYAxis" />. Alternatively, if combined with the <ApiLink type="CategoryXAxis" />, these labels will be placed corresponding to the <ApiLink type="AreaSeries" member="valueMemberPath" label="ValueMemberPath" /> of the <ApiLink type="BarSeries" />, `RangeBarSeries`, <ApiLink type="StackedBarSeries" />, and <ApiLink type="Stacked100BarSeries" />.

The <ApiLink type="NumericXAxis" /> is compatible with the following type of series:

- <ApiLink type="BarSeries" />

### Time X Axis

The <ApiLink type="TimeXAxis" /> treats its data as a sequence of data items, sorted by date. Labels on this axis type are dates and can be formatted and arranged according to date intervals. The date range of this axis is determined by the date values in a data column that is mapped using its <ApiLink type="TimeXAxis" member="dateTimeMemberPath" label="DateTimeMemberPath" />. This, along with a <ApiLink type="DataSource" /> is required to plot data with this axis type.

The <ApiLink type="TimeXAxis" /> is the X-Axis type in the <ApiLink type="FinancialChart" /> component.

#### Breaks in Time X Axis

The <ApiLink type="TimeXAxis" /> has the option to exclude intervals of data by using <ApiLink type="TimeXAxis" member="breaks" label="Breaks" />. As a result, the labels and plotted data will not appear at the excluded interval. For example, working/non-working days, holidays, and/or weekends. An instance of <ApiLink type="TimeAxisBreak" /> can be added to the <ApiLink type="TimeXAxis" member="breaks" label="Breaks" /> collection of the axis and configured by using a unique <ApiLink type="TimeAxisBreak" member="start" label="Start" />, <ApiLink type="TimeAxisBreak" member="end" label="End" /> and <ApiLink type="NumericYAxis" member="interval" label="Interval" />.

#### Formatting in Time X Axis

The <ApiLink type="TimeXAxis" /> has the <ApiLink type="TimeXAxis" member="labelFormats" label="LabelFormats" /> property, which represents a collection of <ApiLink type="TimeAxisLabelFormat" /> objects. Each <ApiLink type="TimeAxisLabelFormat" /> added to the collection is responsible for assigning a unique <ApiLink type="TimeAxisLabelFormat" member="format" label="Format" /> and <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" />. This can be especially useful for drilling down data from years to milliseconds and adjusting the labels depending on the range of time shown by the chart.

The <ApiLink type="TimeAxisLabelFormat" member="format" label="Format" /> property of the <ApiLink type="TimeAxisLabelFormat" /> specifies what format to use for a particular visible range. The <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" /> property of the <ApiLink type="TimeAxisLabelFormat" /> specifies the visible range at which the axis label formats will switch to a different format. For example, if you have two <ApiLink type="TimeAxisLabelFormat" /> elements with a range set to 10 days and another set to 5 hours, then as soon as the visible range of the axis becomes less than 10 days, it will switch to 5-hour format.

#### Intervals in Time X Axis

The <ApiLink type="TimeXAxis" /> replaces the conventional <ApiLink type="NumericYAxis" member="interval" label="Interval" /> property of the category and numeric axes with an <ApiLink type="TimeXAxis" member="intervals" label="Intervals" /> collection of type <ApiLink type="TimeAxisInterval" />. Each <ApiLink type="TimeAxisInterval" /> added to the collection is responsible for assigning a unique <ApiLink type="NumericYAxis" member="interval" label="Interval" />, <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" /> and <ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" />. This can be especially useful for drilling down data from years to milliseconds to provide unique spacing between labels depending on the range of time shown by the chart. A description of these properties is below:

- <ApiLink type="NumericYAxis" member="interval" label="Interval" />: This specifies the interval to use. This is tied to the <ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" /> property. For example, if the <ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" /> is set to `Days`, then the numeric value specified in <ApiLink type="NumericYAxis" member="interval" label="Interval" /> will be in days.
- <ApiLink type="TimeAxisLabelFormat" member="range" label="Range" />: This specifies the visible range at which the axis interval will switch to a different interval. For example, if you have two TimeAxisInterval with a range set to 10 days and another set to 5 hours, as soon as the visible range in the axis becomes less than 10 days it will switch to the interval whose range is 5 hours.
- <ApiLink type="TimeAxisInterval" member="intervalType" label="IntervalType" />: This specifies the unit of time for the <ApiLink type="NumericYAxis" member="interval" label="Interval" /> property.

## Polar Axes

The <ApiLink type="DataChart" /> with Polar Axes, allows you to plot data outwards (radius axis) from center of the chart and around (angle axis) of center of the chart.

### Category Angle Axis

The <ApiLink type="CategoryAngleAxis" /> treats its data as a sequence of category data items. The labels on this axis are placed along the edge of a circle according to their position in that sequence. This type of axis can display almost any type of data including strings and numbers.

The <ApiLink type="CategoryAngleAxis" /> is generally used with the <ApiLink type="NumericRadiusAxis" /> to plot [Radial Series](../types/radial-chart.md).

The labels on this axis are placed along the edge of a circle according to their position in that sequence. This type of axis can display almost any type of data including strings and numbers.

The <ApiLink type="ProportionalCategoryAngleAxis" /> is generally used with the <ApiLink type="NumericRadiusAxis" /> to plot a pie chart eg. [Radial Series](../types/radial-chart.md).

### Numeric Angle Axis

The <ApiLink type="NumericAngleAxis" /> treats its data as continuously varying numerical data items. The labels on this axis area placed along a radius line starting from the center of the circular plot. The location of the labels on the <ApiLink type="NumericAngleAxis" /> varies according to the value in the data column mapped using the <ApiLink type="BubbleSeries" member="radiusMemberPath" label="RadiusMemberPath" /> property of the [Polar Series](../types/polar-chart.md) object or the <ApiLink type="AreaSeries" member="valueMemberPath" label="ValueMemberPath" /> property of the [Radial Series](../types/radial-chart.md) object.

The The <ApiLink type="NumericAngleAxis" /> can be used with either the <ApiLink type="CategoryAngleAxis" /> to plot [Radial Series](../types/radial-chart.md) or with the <ApiLink type="NumericRadiusAxis" /> to plot [Polar Series](../types/polar-chart.md) respectively.

The <ApiLink type="NumericRadiusAxis" /> treats the data as continuously varying numerical data items. The labels on this axis are placed around the circular plot. The location of the labels varies according to the value in a data column mapped using the `AngleMemberPath` property of the corresponding polar series.

The <ApiLink type="NumericRadiusAxis" /> can be used with the <ApiLink type="NumericRadiusAxis" /> to plot [Polar Series](../types/polar-chart.md).
