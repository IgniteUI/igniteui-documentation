# Angular Chart Highlight Filter

The Ignite UI for Angular Chart components support a data highlighting overlay that can enhance the visualization of the series plotted in those charts by allowing you to view a subset of the data plotted. When enabled, this will highlight a subset of data while showing the total set with a reduced opacity in the case of column and area series types, and a dashed line in the case of line series types. This can help you to visualize things like target values versus actual values with your data set. This feature is demonstrated in the following example:

<Sample src="/charts/data-chart/chart-highlight-filter-multiple-series" height={500} alt="Angular Highlight Filter Example" />

Note that data highlighting feature is supported by the <ApiLink type="DataChart" /> and <ApiLink type="CategoryChart" />, but it is configured in different ways in those controls due to the nature of how those controls work. One thing remains constant with this feature though, in that you need to set the <ApiLink type="CategoryChart" member="highlightedValuesDisplayMode" label="HighlightedValuesDisplayMode" /> property to `Overlay` if you want to see the highlight. The following will explain the different configurations for the highlight filter feature.

## Using Highlight Filter with DataChart

In the <ApiLink type="DataChart" />, much of the highlight filter API happens on the series themselves, mainly by setting the <ApiLink type="CategoryChart" member="highlightedDataSource" label="HighlightedItemsSource" /> property to a collection representing a subset of the data you want to highlight. The count of the items in the <ApiLink type="CategoryChart" member="highlightedDataSource" label="HighlightedItemsSource" /> needs to match the count of the data bound to the <ApiLink type="CategoryChart" member="DataSource" label="ItemsSource" /> of the series that you are looking to highlight, and in the case of category series, it will use the `ValueMemberPath` that you have defined as the highlight path by default. The sample at the top of this page uses the <ApiLink type="CategoryChart" member="highlightedDataSource" label="HighlightedItemsSource" /> in the <ApiLink type="DataChart" /> to show the overlay.

In the case that the schema does not match between the <ApiLink type="CategoryChart" member="highlightedDataSource" label="HighlightedItemsSource" /> and the <ApiLink type="CategoryChart" member="DataSource" label="ItemsSource" /> of the series, you can configure this using the `HighlightedValueMemberPath` property on the series. Additionally, if you would like to use the <ApiLink type="CategoryChart" member="DataSource" label="ItemsSource" /> of the series itself as the highlight source and have a path on your data item that represents the subset, you can do this. This is done by simply setting the `HighlightedValueMemberPath` property to that path and not providing a <ApiLink type="CategoryChart" member="highlightedDataSource" label="HighlightedItemsSource" />.

The reduced opacity of the column and area series types is configurable by setting the <ApiLink type="Series" member="highlightedValuesFadeOpacity" label="HighlightedValuesFadeOpacity" /> property on the series. You can also set the <ApiLink type="CategoryChart" member="highlightedValuesDisplayMode" label="HighlightedValuesDisplayMode" /> property to `Hidden` if you do not wish to see the overlay at all.

The part of the series shown by the highlight filter will be represented in the legend and tooltip layers of the chart separately. You can configure the title that this is given in the tooltip and legend by setting the <ApiLink type="Series" member="highlightedTitleSuffix" label="HighlightedTitleSuffix" />. This will append the value that you provide to the end of the <ApiLink type="CategoryChart" member="chartTitle" label="Title" /> of the series.

If the <ApiLink type="DataLegend" /> or <ApiLink type="DataToolTipLayer" /> is used then the highlighted series will appear grouped. This can be managed by setting the <ApiLink type="Series" member="highlightedValuesDataLegendGroup" label="HighlightedValuesDataLegendGroup" /> property on the series to categorize them appropriately.

## Using Highlight Filter in CategoryChart

The <ApiLink type="CategoryChart" /> highlight filter happens on the chart by setting the <ApiLink type="CategoryChart" member="initialHighlightFilter" label="InitialHighlightFilter" /> property. Since the <ApiLink type="CategoryChart" /> takes all of the properties on your underlying data item into account by default, you will need to define the <ApiLink type="CategoryChart" member="initialGroups" label="InitialGroups" /> on the chart as well so that the data can be grouped and aggregated in a way that you can have a subset of the data to filter on. You can set the <ApiLink type="CategoryChart" member="initialGroups" label="InitialGroups" /> to a value path in your underlying data item to group by a path that has duplicate values.
