# Transaction Service

The <ApiLink kind="interface" type="TransactionService" label="Transaction Service" /> is an injectable middleware (through [Angular's DI](https://angular.io/guide/dependency-injection)) that a component may use to accumulate changes without immediately affecting the underlying data.

        alt="Transaction Service Architecture"

<DocsAside type="info">
The data transformation from the schema above is not mandatory. You do not need to use a pipe in order to use the <ApiLink kind="interface" type="TransactionService" label="Transaction Service" />.
</DocsAside>

The <ApiLink kind="interface" type="TransactionService" label="Transaction Service" /> allows adding transactions. After at least one transaction is added, you may commit or clear all the changes or the changes for a single record only. As it keeps a detailed log, it can also execute undo and redo operations.

Every time you execute an operation (<ApiLink kind="interface" type="Transaction" />), it is added to the transaction log and undo stack. All the changes in the transaction log are then accumulated per record. From that point, the service maintains an aggregated **state** that consists only of add/update/delete operations for unique records. This is based on a <ApiLink type="State" kind="interface" /> interface which has three properties: `recordRef`, `type` and `value`.

We have built three classes on top of the <ApiLink kind="interface" type="TransactionService" label="Transaction Service" /> that provide users with the ability to commit all changes they have made, or only changes made to a specific record, at once. Those classes are <ApiLink type="BaseTransactionService" label="igxBaseTransactionService" />, <ApiLink type="TransactionService" /> and `IgxHierarchicalTransactionService`.

The <ApiLink type="TransactionService" /> and `IgxHierarchicalTransactionService` are fully integrated with our <ApiLink type="Grid" label="igxGrid" />, <ApiLink type="HierarchicalGrid" label="igxHierarchicalGrid" /> and <ApiLink type="TreeGrid" label="igxTreeGrid" /> components. You can find detailed examples of using those components with transactions enabled in the following topics:

- [igxGrid Batch Editing and Transactions](/grid/batch-editing)
- [igxHierarchicalGrid Batch Editing and Transactions](/hierarchicalgrid/batch-editing)
- [igxTreeGrid Batch Editing and Transactions](/treegrid/batch-editing)

A more detailed overview of the opportunities that the <ApiLink kind="interface" type="TransactionService" label="Transaction Service" /> provides can be found in our ["Building a transaction service for managing large scale editing experiences" blog](https://blog.angular.io/building-a-transaction-service-for-managing-large-scale-editing-experiences-ded666eafd5e)
