# Angular Toolbar Overview

The Angular Toolbar component is a companion container for UI operations to be used primarily with our charting components. The toolbar will dynamically update with a preset of properties and tool items when linked to our <ApiLink pkg="charts" type="DataChart" /> or <ApiLink pkg="charts" type="CategoryChart" /> components. You'll be able to create custom tools for your project allowing end users to provide changes, offering an endless amount of customization.

## Dependencies


Install the Ignite UI for Angular layouts, inputs, charts and core packages:

```cmd
npm install igniteui-angular-layouts
npm install igniteui-angular-inputs
npm install igniteui-angular-charts
npm install igniteui-angular-core
```

The following modules are required when using the <ApiLink pkg="layouts" type="Toolbar" exclude="React,WebComponents" /> with the <ApiLink pkg="charts" type="DataChart" /> component and it's features.



```ts
import { IgxToolbarModule } from 'igniteui-angular-layouts';
import { IgxDataChartToolbarModule, IgxDataChartCoreModule, IgxDataChartCategoryModule, IgxDataChartAnnotationModule, IgxDataChartInteractivityModule, IgxDataChartCategoryTrendLineModule  } from 'igniteui-angular-charts';

@NgModule({
    imports: [
        // ...
        IgxToolbarModule,
        IgxDataChartToolbarModule,
        IgxDataChartCoreModule,
        IgxDataChartCategoryModule,
        IgxDataChartAnnotationModule,
        IgxDataChartInteractivityModule,
        IgxDataChartCategoryTrendLineModule
        // ...
    ]
})
export class AppModule {}
```

## Usage

### Tool Actions

The following is a list of the different <ApiLink pkg="layouts" type="ToolAction" exclude="React,WebComponents" /> items that you can add to the Toolbar.

- <ApiLink pkg="layouts" type="ToolActionButton" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionIconButton" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionIconMenu" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionLabel" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionNumberInput" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionRadio" exclude="React,WebComponents" />
- <ApiLink pkg="layouts" type="ToolActionSubPanel" exclude="React,WebComponents" />

Each of these tools exposes an `OnCommand` event that is triggered by mouse click. Note, the <ApiLink pkg="layouts" type="ToolActionIconMenu" exclude="React,WebComponents" /> is a wrapper for other tools that can also be wrapped inside a <ApiLink pkg="layouts" type="ToolActionIconMenu" exclude="React,WebComponents" />.

New and existing tools can be repositioned and marked hidden using the <ApiLink pkg="layouts" type="ToolAction" member="overlayId" label="OverlayId" exclude="React,WebComponents" />, <ApiLink pkg="layouts" type="ToolAction" member="beforeId" label="BeforeId" exclude="React,WebComponents" /> and <ApiLink pkg="layouts" type="ToolAction" member="afterId" label="AfterId" exclude="React,WebComponents" /> properties on the <ApiLink pkg="layouts" type="ToolAction" exclude="React,WebComponents" /> object. ToolActions also expose a <ApiLink pkg="layouts" type="ToolAction" member="visibility" label="Visibility" exclude="React,WebComponents" /> property.

The following example demonstrates a couple of features. First you can group tools together in the <ApiLink pkg="layouts" type="ToolActionSubPanel" exclude="React,WebComponents" /> including hiding built in tools such as the **ZoomReset** and **AnalyzeMenu** menu tool actions. In this example a new instance of the **ZoomReset** tool action within the **ZoomMenu** by using the the <ApiLink pkg="layouts" type="ToolAction" member="afterId" label="AfterId" exclude="React,WebComponents" /> property and assigning that to **ZoomOut** to be precise with it's placement. It is also highlighted via the <ApiLink pkg="layouts" type="ToolAction" member="isHighlighted" label="IsHighlighted" exclude="React,WebComponents" /> property on the tool.

### Angular Data Chart Integration

The Angular Toolbar contains a <ApiLink pkg="layouts" type="Toolbar" member="Target" label="Target" exclude="React,WebComponents" /> property. This is used to link a component, such as the <ApiLink pkg="charts" type="DataChart" /> as shown in the code below:




```html
  <div class="legend">
    <igx-toolbar
      name="toolbar"
      [target]="chart"
      #toolbar>
    </igx-toolbar>
  </div>
  <div class="container fill">
    <igx-data-chart
    name="chart"
    #chart>
  </igx-data-chart>
```

Several pre-existing <ApiLink pkg="layouts" type="ToolAction" exclude="React,WebComponents" /> items and menus become available when the <ApiLink pkg="charts" type="DataChart" /> is linked with the Toolbar. Here is a list of the built-in Angular <ApiLink pkg="charts" type="DataChart" /> Tool Actions and their associated <ApiLink pkg="layouts" type="ToolAction" member="overlayId" label="OverlayId" exclude="React,WebComponents" />:

Zooming Actions

- `ZoomMenu`: A <ApiLink pkg="layouts" type="ToolActionIconMenu" exclude="React,WebComponents" /> that exposes three <ApiLink pkg="layouts" type="ToolActionLabel" exclude="React,WebComponents" /> items to invoke the <ApiLink pkg="charts" type="DomainChart" member="zoomIn" label="ZoomIn" /> and <ApiLink pkg="charts" type="DomainChart" member="zoomOut" label="ZoomOut" /> methods on the chart for increasing/decreasing the chart's zoom level including `ZoomReset`, a <ApiLink pkg="layouts" type="ToolActionLabel" exclude="React,WebComponents" /> that invokes the <ApiLink pkg="charts" type="DomainChart" member="resetZoom" label="ResetZoom" /> method on the chart to reset the zoom level to it's default position.

Trend Actions

- `AnalyzeMenu`: A <ApiLink pkg="layouts" type="ToolActionIconMenu" exclude="React,WebComponents" /> that contains several options for configuring different options of the chart.
- `AnalyzeHeader`: A sub section header.
  - `LinesMenu`: A sub menu containing various tools for showing different dashed horizontal lines on the chart.
  - `LinesHeader`: A sub menu section header for the following three tools:
    - `MaxValue`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that displays a dashed horizontal line along the yAxis at the maximum value of the series.
    - `MinValue`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that displays a dashed horizontal line along the yAxis at the minimum value of the series.
    - <ApiLink pkg="charts" kind="enum" type="ValueLayerValueMode" member="Average" label="Average" prefixed={false} />:  A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that displays a dashed horizontal line along the yAxis at the average value of the series.
  - `TrendsMenu`: A sub menu containing tools for applying various trendlines to the <ApiLink pkg="charts" type="DataChart" /> plot area.
  - `TrendsHeader`: A sub menu section header for the following three tools:
    - **Exponential**: A <ApiLink pkg="layouts" type="ToolActionRadio" exclude="React,WebComponents" /> that sets the <ApiLink pkg="charts" type="DomainChart" member="trendLineType" label="TrendLineType" /> on each series in the chart to **ExponentialFit**.
    - **Linear**: A <ApiLink pkg="layouts" type="ToolActionRadio" exclude="React,WebComponents" /> that sets the <ApiLink pkg="charts" type="DomainChart" member="trendLineType" label="TrendLineType" /> on each series in the chart to **LinearFit**.
    - **Logarithmic**: A <ApiLink pkg="layouts" type="ToolActionRadio" exclude="React,WebComponents" /> that sets the <ApiLink pkg="charts" type="DomainChart" member="trendLineType" label="TrendLineType" /> on each series in the the chart to **LogarithmicFit**.
- `HelpersHeader`: A sub section header.
  - `SeriesAvg`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that adds or removes a <ApiLink pkg="charts" type="ValueLayer" /> to the chart's series collection using the <ApiLink pkg="charts" kind="enum" type="ValueLayerValueMode" prefixed={false} /> of type <ApiLink pkg="charts" kind="enum" type="ValueLayerValueMode" member="Average" label="Average" prefixed={false} />.
  - `ValueLabelsMenu`: A sub menu containing various tools for showing different annotations on the <ApiLink pkg="charts" type="DataChart" />'s plot area.
  - `ValueLabelsHeader`: A sub menu section header for the following tools:
    - `ShowValueLabels`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that toggles data point values by using a <ApiLink pkg="charts" type="CalloutLayer" />.
    - `ShowLastValueLabel`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that toggles final value axis annotations by using a <ApiLink pkg="charts" type="FinalValueLayer" />.
- `ShowCrosshairs`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that toggles mouse-over crosshair annotations via the chart's <ApiLink pkg="charts" type="DomainChart" member="crosshairsDisplayMode" label="CrosshairsDisplayMode" /> property.
- `ShowGridlines`: A <ApiLink pkg="layouts" type="ToolActionCheckbox" exclude="React,WebComponents" /> that toggles extra gridlines by applying a `MajorStroke` to the X-Axis.

Save to Image Action

- `CopyAsImage`: A <ApiLink pkg="layouts" type="ToolActionLabel" exclude="React,WebComponents" /> that exposes an option to copy the chart to the clipboard.
- `CopyHeader`: A sub section header.

### SVG Icons

When adding tools manually, icons can be assigned using the `RenderIconFromText` method. There are three parameters to pass in this method. The first is the icon collection name defined on the tool eg. <ApiLink pkg="layouts" type="ToolAction" member="iconCollectionName" label="IconCollectionName" exclude="React,WebComponents" />. The second is the name of the icon defined on the tool eg. <ApiLink pkg="layouts" type="ToolAction" member="iconName" label="IconName" exclude="React,WebComponents" />, followed by adding the SVG string.

### Data URL Icons

Similarly to adding svg, you can also add an Icon image from a URL via the <ApiLink pkg="layouts" type="Toolbar" member="registerIconFromDataURL" label="RegisterIconFromDataURL" exclude="React,WebComponents" />. The method's third parameter would be used to enter a string URL.

The following snippet shows both methods of adding an Icon.



```html
<igx-tool-action-label

    iconName="CustomIcon"
    iconCollectionName="CustomCollection">
```

### Vertical Orientation

By default the Angular Toolbar is shown horizontally, but it also has the ability to shown vertically by setting the <ApiLink pkg="layouts" type="Toolbar" member="orientation" label="Orientation" exclude="React,WebComponents" /> property.



```html
<igx-toolbar orientation="Vertical" />
```

### Color Editor

You can add a custom color editor tool to the the Angular Toolbar, which will also work with the Command event to perform custom styling to your application.



```html
<igx-toolbar
  name="toolbar"
  #toolbar>
      <igx-tool-action-color-editor
      title="Series Brush"
      name="colorEditorTool"
      #colorEditorTool>
      </igx-tool-action-color-editor>
</igx-toolbar>
```
