# Ignite UI CLI MCP

<p class="highlight">Ignite UI CLI MCP е <a href="https://modelcontextprotocol.io/" target="_blank">Протокол за контекст на модела</a> (MCP) сървър, който позволява на AI асистентите да създават проекти, модифицират съществуващи приложения, създават и актуализират компоненти и отговарят на въпроси относно документацията за приложения на Ignite UI for Angular. Свързване на Ignite UI CLI MCP към вашия редактор, GitHub хранилище или desktop AI клиент и опишете какво искате - асистентът използва CLI инструментите за вас.</p>

<igc-divider></igc-divider>

## Преглед

Ignite UI CLI MCP дава на AI помощниците преки достъп до Ignite UI CLI проектни скелети, генериране на компоненти, модификация на проекти и работни потоци с информация за документация чрез чат или режим на агент. Сървърът работи наряду с [Ignite UI Theming MCP](./theming-mcp.md). CLI MCP обработва работни потоци на проекти и компоненти, докато Theming MCP обработва палети, теми, токени и стилизирането. Повечето екипи свързват и двата сървъра в една и съща AI клиентска сесия.

Препоръчаният път за настройка е да започнете с Ignite UI CLI първо. Този път създава проекта, инсталира необходимите пакети и ви подсказва да изберете кои AI клиенти и агенти да конфигурирате. Можете също така да започнете от празна папка и да оставите помощника да създаде проекта чрез MCP, или да свържете MCP с проект, който вече съществува.

**Примерни подкани за опит веднага след свързване:**

> *"Създайте нов Ignite UI for Angular проект в тази папка, използвайте препоръчаните стойности по подразбиране и добавете стартова страница с примерни данни."*

> *"Добавете нова grid страница към този проект, свържете я с примерни данни и запазете текущата структура на навигацията."*

> *"Какви свойства и събития разкрива комбо компонентът и кои от тях са най-полезни за падащ список с търсене?"*

> *"Актуализирайте съществуващия проект, за да добавите страничен формат на навигация и запазите текущите страници и маршрути."*

## Предварителни изисквания

Преди да конфигурирате MCP сървъра, уверете се, че имате:

- **Node.js** (v18 или по-нова) инсталиран - това предоставя командата `npx`, използвана за стартиране на сървъра.
- **AI клиент с поддръжка на MCP** - например VS Code с GitHub Copilot, облачен агент на GitHub Copilot, Cursor, Claude Desktop, Claude Code или JetBrains IDE с приставката AI Assistant.
- Интернет достъп за разрешаване на пакети `npx` първия път, когато сървърът стартира.
- Един от следните начални пункти:
  - празна папка за нов проект
  - проект, който сте създали с Ignite UI CLI
  - съществуващ Ignite UI for Angular проект, върху който искате да продължите работата

Ако искате инструменти за теми в една и съща клиентска сесия и `igniteui-theming` вече не е наличен във вашия проект, изпълнете:

```bash
npm install igniteui-theming
```

## Конфигурирайте Ignite UI CLI MCP за вашия AI клиент

MCP сървърът се стартира чрез `npx`. Не е необходима отделна инсталация, освен Node.js и достъп до пакета `igniteui-cli`.

Каноничната команда за стартиране е:

```bash
npx -y igniteui-cli mcp
```

<DocsAside type="info">

Флагът `-y` казва на `npx` да автоматично потвърди подканата за свалянето на пакета, така че сървърът да може да стартира без ръчна намеса.

</DocsAside>

### Изберете път за настройка

Можете да започнете с Ignite UI CLI MCP по три начина:

> **Препоръчано - CLI първо**
> Създайте проекта с Ignite UI CLI първо, като използвате `ig new` или съответната команда `npx --package igniteui-cli igniteui new`. Това е най-лесната настройка, защото Ignite UI CLI скелетира проекта, инсталира необходимите пакети и ви подсказва да изберете кои AI клиенти да конфигурирате за MCP и кои агенти да конфигурирате файлове с умения за. След това трябва само да преглед­не генерираната конфигурация и да отворите проекта в вашия AI клиент.

> **Празна папка**
> Начнете с напълно празна папка, добавете MCP конфигурацията ръчно и след това попросете помощника да създаде проекта чрез чат. Този път е полезен, когато искате MCP да управлява потока на създаване на проекта от началото вместо да пускате CLI самостоятелно първо.

> **Съществуващ проект**
> Добавете MCP конфигурацията към проект, който вече имате, и продължете работата в текущата кодова база. Изпълнете `ig ai-config` (или `ng generate @igniteui/angular-schematics:ai-config` за Angular проекти), за да напишете MCP конфигурация за вашите избрани AI клиенти и копирайте Agent Skills във вашия проект. Командата ви подсказва да изберете кои клиенти и агенти да конфигурирате.

И трите пътя използват същите MCP сървъри. Разликата е само как проектът се подготвя преди да започнете да задавате подкани:

- в пътя **CLI-първо**, Ignite UI CLI създава проекта и подготвя първата MCP конфигурация за вас
- в пътя **празна-папка**, вие създавате MCP конфигурацията първо и оставяте помощника да създаде проекта след това
- в пътя **съществуващ-проект**, изпълнете `ig ai-config`, за да напишете MCP конфигурация за вашите избрани клиенти и автоматично копирайте Agent Skills

Във всички случаи, след като MCP сървърите са свързани и видими в вашия AI клиент, помощникът може да продължи работата в една и съща сесия.

Ако създавате проект с Ignite UI CLI първо, можете да пускате CLI по един от тези начини:

- **Глобална инсталация**

  ```bash
  npm install -g igniteui-cli
  ```

  Това ви дава командата `ig` в което и да е терминално сесия и е най-ясният вариант, ако планирате да създавате и скелетирате проекти редовно.

- **Без глобална инсталация**

  ```bash
  npx --package igniteui-cli igniteui new
  ```

  Това пуска CLI чрез `npx` вместо глобална команда `ig`.

За пътя **CLI-първо**, можете да създадете проекта в насочен режим или със преки команди.

Използвайте насочения режим, когато искате CLI-то да ви преведе през наличните опции:

```bash
ig new
```

Съответната форма на `npx`:

```bash
npx --package igniteui-cli igniteui new
```

Използвайте преки команди, когато вече знаете настройките на проекта:

```bash
ig new my-app --framework=angular --type=igx-ts --template=empty
```

Съответната форма на `npx`:

```bash
npx ig new my-app --framework=angular --type=igx-ts --template=empty
```

В насочения режим Ignite UI CLI подсказва за име на проект, рамка, шаблон, тема и дали да добави компонент или да завърши настройката. В преките режим предоставяте рамката и всички поддържани опции в самата команда.

### VS Code

GitHub Copilot в VS Code поддържа MCP сървъри чрез конфигурационен файл на ниво работно пространство. Изпълнете `ig ai-config` (или `ng generate @igniteui/angular-schematics:ai-config`) от коренът на вашия проект, за да генерирате този файл автоматично. За да го конфигурирате ръчно, създайте или редактирайте `.vscode/mcp.json` в коренът на вашия проект:

```json
{
  "servers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    },
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

Ако сте създали проекта с Ignite UI CLI първо, прегледайте генерирания `.vscode/mcp.json` и потвърдете, че двата записа присъстват.

След като е запазено, отворете панела за чат на GitHub Copilot, превключете в режим **Agent**, и инструментите на Ignite UI CLI MCP ще бъдат налични.

<DocsAside type="info">

Поддръжката на MCP в VS Code изисква GitHub Copilot и VS Code 1.99 или по-нова версия.

</DocsAside>

### Cursor

Cursor поддържа конфигурацията на MCP, ограничена до проект. Създайте или редактирайте `.cursor/mcp.json` в коренът на вашия проект:

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    },
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

Сървърите ще бъдат подхвърлени автоматично, когато отворите нова Cursor чат сесия.

<DocsAside type="info">

Можете също така да конфигурирате MCP сървъри глобално чрез **Settings → MCP** в Cursor.

</DocsAside>

### Claude Desktop

Добавете сървърите към вашия конфигурационен файл на Claude Desktop:

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    },
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

Рестартирайте Claude Desktop след запазването. Ще видите индикатор на MCP сървър (икона на плъзгач) в областта за въвеждане на чат, потвърждаващ, че MCP инструментите са активни.

### Claude Code

Claude Code поддържа MCP сървъри чрез своя CLI и конфигурационен файл `.mcp.json` ограничен до проект. За да споделите конфигурацията със вашия екип, създайте или редактирайте `.mcp.json` в коренът на вашия проект:

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    },
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

Алтернативно, можете да добавите сървърите чрез командния ред само за вашата локална среда:

```bash
claude mcp add igniteui-cli -- npx -y igniteui-cli mcp
claude mcp add igniteui-theming -- npx -y igniteui-theming igniteui-theming-mcp
```

Използвайте командата `/mcp` вътре в Claude Code, за да проверите дали сървърите са свързани.

### JetBrains IDE

JetBrains AI Assistant поддържа MCP сървъри чрез настройките на IDE:

1. Отворете **Settings** (или **Preferences** на macOS).

2. Преминете към **Tools → AI Assistant → Model Context Protocol (MCP)**.

3. Кликнете на **+ Add** и изберете **As JSON** или използвайте полетата на формуляра.

4. Въведете следната конфигурация:

   ```json
   {
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    },
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
   ```

5. Кликнете на **OK** и рестартирайте AI Assistant.

<DocsAside type="info">

Поддръжката на MCP изисква приставката AI Assistant да бъде инсталирана и включена във вашия JetBrains IDE.

</DocsAside>

### Други MCP клиенти

За всеки друг MCP-съвместим клиент, използвайте транспорт STDIO с тези команди за стартиране:

```bash
npx -y igniteui-cli mcp
npx -y igniteui-theming igniteui-theming-mcp
```

### GitHub

Облачният агент на GitHub Copilot поддържа конфигурацията на MCP на ниво хранилище. В вашето хранилище:

1. Отворете главната страница на хранилището.
2. Кликнете на **Settings**.
3. В страничното меню, отидете на **Copilot** след това **Cloud agent**.
4. Поставете вашия JSON в раздела **MCP configuration**.
5. Кликнете на **Save**.

Използвайте конфигурация на хранилище като тази:

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "type": "local",
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"],
      "tools": ["*"]
    },
    "igniteui-theming": {
      "type": "local",
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"],
      "tools": ["*"]
    }
  }
}
```

Тази настройка е полезна, когда искате облачният агент на GitHub да използва същите инструменти на CLI MCP и Theming MCP директно от контекста на хранилището.

## Налични инструменти

MCP сървърът разкрива набор от инструменти, които AI използва автоматично на базата на вашите подкани. Никога не призовавате тези инструменти директно. Опишете какво искате в чат и помощникът избира правилния. Таблицата е тук, така че да знаете какво е налично, преди да започнете.

За да получите живия списък с текущи параметри, попросете:

> *"Какви инструменти предоставя Ignite UI CLI MCP?"*

Ето кратък преглед на всеки инструмент:

| Инструмент     | Описание                                                                                                                                                          |
| -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `list_components` | Показва налични документи на Ignite UI компоненти за рамка. Приема незадължителен филтър на ключови думи (съвпадение на подстринг без разлика между главни и малки букви срещу име, ключови думи и резюме). |
| `get_doc` | Получава пълното съдържание на документ на определен компонент по име в kebab-case (например `grid-editing`, `combo-overview`). Включва примери на код, таблици и връзки.      |
| `search_docs` | Търсене в пълния текст на документите на Ignite UI за рамка. Връща до 20 класирани резултата с отрывки.                                                               |
| `get_project_setup_guide` | Връща ръководството за настройка на проект за създаване на нов проект в определена рамка, включително стъпки на CLI и инструкции за инсталация.                                    |
| `search_api` | Търси записи на API по ключова дума или име на компонент в Angular, React, Blazor и Web Components.                                                                  |
| `get_api_reference` | Връща пълната справка на API за определен компонент или клас, включително свойства, методи и събития. Поддържа Angular, React, Blazor и Web Components.     |

На високо ниво инструментите на CLI MCP помагат с:

- създаване на нови проекти
- работа в съществуващи проекти
- добавяне и модификация на компоненти
- актуализиране на структура и конфигурация на проект
- отговори на въпроси за документация и API

<DocsAside type="info">

Търсенето на рамки използва префиксите на компоненти: `for Angular`, `for React`, `for Web Components`, `for Blazor`. Помощникът автоматично избира правилната рамка от вашите отворени файлове или контекст на подкана.

</DocsAside>

Сървърът за теми добавя работни потоци за стилизиране, теми, палети и токени към една и съща клиентска сесия.

## Общи работни потоци

Следните сценарии за настройка показват кога да използвате всеки начален пункт.

### Настройка CLI-първо

Създайте проекта с Ignite UI CLI първо, когато искате най-бързата насочена настройка. `ig new` ви подсказва да изберете кои AI клиенти да конфигурирате за MCP и кои агенти да конфигурирате файлове с умения за.

Примерни сценарии:

- *"Искам да създам нов проект с Ignite UI CLI първо, отворя генерирания проект в VS Code и продължа оттам с MCP."*
- *"Вече знам, че искам React проект, затова искам да го създам със CLI и след това да използвам MCP, за да добавя страници и компоненти."*

### Настройка на празна папка

Начнете от празна папка, когато искате помощникът да създаде проекта от чат, след като сами добавите MCP конфигурацията.

Примерни сценарии:

- *"Имам напълно празна папка и искам помощникът да създаде целия проект от чат, след като MCP е свързан."*
- *"Не искам да пускам Ignite UI CLI ръчно първо. Искам MCP да управлява първата стъпка на създаване на проект."*

### Настройка на съществуващ проект

Свържете MCP с съществуващ проект, когато искате да запазите текущата кодова база и да използвате помощника за промени на проекта, работа с компоненти и въпроси за документация.

Примерни сценарии:

- *"Вече имам проект и само искам да добавя MCP, така че помощникът да може да ми помогне да актуализирам страници и компоненти."*
- *"Проектът вече съществува и главно искам да задавам въпроси за документация и API, докато работя в текущата кодова база."*

## Отстраняване на проблеми

**`npx` не е разпознато**

Node.js не е инсталиран или не е наличен в текущата терминална среда. Инсталирайте Node.js от [nodejs.org](https://nodejs.org) и проверете със `node --version`.

**`ig` не е разпознато**

Ако искате да използвате глобалната команда `ig`, първо инсталирайте Ignite UI CLI със `npm install -g igniteui-cli`. Ако не искате глобална инсталация, използвайте формата `npx --package igniteui-cli igniteui ...` вместо това.

**Инструментите на MCP не се появяват след запазването на конфигурацията**

Преизкарайте работното пространство, отворите отново редактора или рестартирайте AI клиента. Някои клиенти изискват пълен рестарт, за да детектират нови MCP конфигурационни файлове.

**Един сървър не стартира**

Проверете дали съдържанието на конфигурацията съответства точно на примерите, включително имена на ключове и ред на аргументите.

**Проектът е създан, но MCP не е конфигуриран за един от вашите AI клиенти**

`ig new` ви подсказва да изберете кои клиенти да конфигурирате. Ако сте прескочили клиент по време на създаване на проект, изпълнете `ig ai-config` и изберете липсващите клиенти, когда ви бъдат подсказани.

**Помощникът работи в грешната папка или не може да намери файловете на проекта**

Убедете се, че AI клиентът е насочен към коренът на проекта. Ако използвате редактор-базиран клиент, отворете папката на коренът на проекта. Ако използвате настолен или чат-първо клиент, направете правилната папка на проекта достъпна като контекст на сесия.

**Настройката на празна папка не се държи както е очаквано**

Проверете дали папката наистина е празна преди да начнете. Ако папката вече съдържа файлове на проект, използвайте пътя на съществуващ проект вместо да я третирате като ново празно работно пространство.

**Конфигурацията на GitHub MCP е отхвърлена**

Валидирайте, че JSON използва структура `mcpServers` и че всеки локален запис на сървър включва `type`, `command` и `args`. Ако включите полето `tools`, използвайте валидни имена на инструменти или `["*"]`.
