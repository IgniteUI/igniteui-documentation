# AI-Подпомогнат Entwicklung с Ignite UI

Ignite UI for Angular осигурява пълен AI toolchain - Agent Skills, Ignite UI CLI MCP сървъра, Ignite UI Theming MCP сървъра и MAKER MCP сървъра - който предоставя на AI кодирущите асистенти правилни component API, пътища на импортиране и design tokens. Agent Skills са пакети с инструкции, които притежават разработчици и дефинират как AI агентите използват Ignite UI в конкретен проект. CLI MCP сървърът (`igniteui-cli`) излага Ignite UI CLI scaffolding, управление на компоненти и документационни инструменти на активната AI агентна сесия чрез Model Context Protocol. Theming MCP сървърът (`igniteui-theming`) излага Ignite UI Theming Engine като изпълнимо агентно контекст. MAKER MCP (`@igniteui/maker-mcp`) е многоагентен AI оркестрационен MCP сървър от Infragistics, който разлага сложни задачи в валидирани, изпълними планове стъпки, използвайки консенсус базиран алгоритъм на гласуване между множество AI агенти. Skills, CLI MCP и Theming MCP - и трите се конфигурират с една команда: `npx igniteui-cli ai-config`

MCP сървърите и Agent Skills служат различни цели и имат различни предусловия:

| Компонент            | Какво предоставя                                             | Изисква ли Ignite UI инсталиран? |
| -------------------- | ------------------------------------------------------------ | -------------------------------- |
| Ignite UI MCP сървър | Запитване на документация, API референция, scaffolding инструменти | Не                               |
| Theming MCP сървър   | Design tokens, palette инструменти, WCAG контрастна валидация | Не                               |
| Agent Skills         | Инструкционни пакети на ниво проект за правилно използване на компоненти | Да                               |

Можете да започнете оценяване на Ignite UI AI помощта само със MCP сървърите - Ignite UI не трябва да бъде инсталиран в проекта. Agent Skills стават достъпни, щом инсталирате Ignite UI пакети.

CLI MCP сървърът изисква STDIO транспорт; HTTP-базирани MCP клиенти не се поддържат. Agent Skills и CLI MCP сървърът не модифицират файлове на проекта автономно - те излагат инструменти и инструкции на активния AI агент, който действа върху подкани на разработчика.

## Конфигуриране на AI Toolchain

Изпълнете тази команда от корена на вашия съществуващ Angular, React, Blazor или Web Components проект. Тя конфигурира MCP сървърите, копира файлове на навици, специфични за framework, в директорията с навици на всеки агент и настройва инструкционни файлове. Използвайте `--assistants` за избор на които кодирущи асистенти получават MCP конфигурация и `--agents` за избор на които агенти получават skill файлове. Ако не са предоставени параметри, командата влиза в интерактивен режим, подканвайки ви да изберете асистенти и агенти. Съществуващите файлове се актуализират само ако съдържанието им се е променило.

```bash
npx igniteui-cli ai-config
```

<DocsAside type="info">
Без фиксирана версия, `npx` може да изтегли по-ста версия на CLI, която не разпознава подкомандата `ai-config` и вместо това ще стартира интерактивен подканване за създаване на проект, което ще създаде нов проект вътре в съществуващия ви. Уверете се, че сте инсталирали CLI версия 16.x.
</DocsAside>

Ако `ai-config` не може да открие framework от вашите файлове на проекта, подканва ви да изберете един - така че командата работи дори от проект, където все още няма инсталиран Ignite UI пакет.

След като командата завърши, стартирайте MCP сървърите във вашия AI клиент. Сървърите са конфигурирани, но все още не работят - клиентът трябва да стартира всеки сървър, преди неговите инструменти да бъдат достъпни на агента.

### Какво да очаквате

Ако Ignite UI е **не** инсталиран в проекта:

<DocsAside type="info">
Ignite UI MCP сървъри конфигурирани за вашите избрани клиенти
Нямат намерени AI файлове със умения. Уверете се, че пакетите са инсталирани (`npm install`) и вашите Ignite UI пакети са актуални.
</DocsAside>

MCP сървърите са готови за използване. Skills ще бъдат добавени автоматично следващия път когато изпълните `ai-config` след инсталиране на Ignite UI.

Ако Ignite UI **е** инсталиран в проекта:

<DocsAside type="info">
Ignite UI MCP сървъри конфигурирани за вашите избрани клиенти
Умения на Agent копирани в директориите за умения на вашите избрани агенти
</DocsAside>

Както MCP сървърите, така и Skills се конфигурират.

### Стартиране на сървърите

**VS Code с GitHub Copilot:**

Отворете `.vscode/mcp.json`. VS Code показва вътрешно **Start** бутон над всеки запис на сървъра. Кликнете **Start** за както `igniteui` така и `igniteui-theming`. След стартиране, VS Code показва наличния брой инструменти до всеки сървър (например, *"13 tools | 1 prompt"*). Алтернативно, изпълнете **MCP: List Servers** от Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`), изберете всеки сървър и изберете **Start**.

**Cursor:**

Отворете **Settings → MCP**, намерете `igniteui` и `igniteui-theming` записи и превключете всеки един. Cursor стартира всеки сървър веднага и показва броя му на инструменти.

**Claude Code:**

Изпълнете `claude mcp list` за потвърждение, че оба сървъра са регистрирани. Claude Code стартира MCP сървърите автоматично когато техните инструменти са за пръв път призовани - не е необходима ръчна стъпка за стартиране.

**JetBrains AI Assistant:**

Отворете **Settings → Tools → AI Assistant → Model Context Protocol (MCP)**. Кликнете на икона play до всеки запис на сървъра за стартиране.

**Claude Desktop:**

Затворете и преоткрийте Claude Desktop. Сървърите се стартират автоматично при стартиране.

### Инсталиране на Ignite UI и добавяне на Skills по-късно

Ако сте изпълнили `ai-config` без инсталиран Ignite UI и желаете да добавите Skills, инсталирайте Ignite UI пакета за вашия framework и преизпълнете командата:

```bash
npm install igniteui-angular
npx igniteui-cli@latest ai-config
```

Командата открива, че Skills са сега достъпни и копира техния. MCP конфигурационни файлове за вашите избрани клиенти остават без промяна (вече са актуални).

## AI Toolchain на един поглед

AI toolchain на Ignite UI се състои от три независимо използвеми слоя. Всеки слой може да бъде включен самостоятелно; те са проектирани да работят заедно.

| Слой                                | Какво предоставя                                                                                            | Собственик   | Frameworks                             |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------ | -------------------------------------- |
| Agent Skills                        | Инструкционни пакети, които притежават разработчици: пътища на импортиране, модели на компоненти, потоци на решение, проектни конвенции | Разработчик    | Angular, React, Web Components, Blazor |
| CLI MCP сървър (`igniteui-cli`)     | Scaffolding на проект, управление на компоненти, документация и API запитване чрез MCP                            | Infragistics | Angular, React, Web Components, Blazor |
| Theming MCP сървър (`igniteui-theming`) | Design tokens, определения на палета, генериране на CSS custom свойства, WCAG AA контрастна валидация             | Infragistics | Angular, React, Web Components, Blazor |

CLI MCP сървърът и Theming MCP сървърът се стартират чрез `npx` и се свързват към всеки MCP-съвместим клиент чрез STDIO транспорт. Agent Skills са локални файлове поставени във вашия проект, които AI клиентът чете от диск.

## Agent Skills

Agent Skills са структурирани, разработчик-собствени пакети, които казват на AI кодирущите асистенти точно как да използват Ignite UI за конкретен framework. Пакетът Skill може да съдържа инструкционен файл `SKILL.md` със модели на компоненти, пътища на импортиране и потоци на решение; препратки към авторитетна Ignite UI документация; и ресурси като файлове на схеми или диаграми. Когато Skill е активна в AI клиента, агентът следва Skill вместо да разчита на общи обучителни данни - което може да референцира остарели API подписи или пътища на импортиране.

Ignite UI доставља посветени Skill пакети за Angular, React, Web Components и Blazor. Пакетът Skill е разработчик-собствен: редактирайте `SKILL.md` за съответствие с конвенциите на вашия екип, добавете модели, специфични за проект, референцирайте вашата вътрешна система за дизайн и версирайте пакета заедно с вашия codebase.

За пълни инструкции по настройка и свързване на IDE, вижте [Agent Skills](skills.md).

## CLI MCP сървър

Ignite UI CLI MCP сървърът (`igniteui-cli`) е MCP сървър, поддържан от Infragistics, който излага Ignite UI CLI scaffolding и документационни инструменти на активната AI агентна сесия. След свързване, AI асистентът може да създава Angular, React, Blazor или Web Components проекти, добавя и модифицира Ignite UI компоненти и отговара на документация и API въпроси - всички чрез естествени езикови подкани в чат сесията.

CLI MCP сървърът работи чрез `npx` без глобална инсталация:

```bash
npx -y igniteui-cli mcp
```

Използвайте `ai-config` за автоматично написание на MCP конфигурация за вашия AI клиент. Сървърът се свързва с VS Code с GitHub Copilot, Cursor, Claude Desktop, Claude Code, JetBrains AI Assistant и всеки друг MCP-съвместим клиент, който поддържа STDIO транспорт. Точният формат на конфигурация се различава по клиент - вижте [CLI MCP](cli-mcp.md) за пълното ръководство по настройка.

Той не генерира код автономно - излага инструменти на AI агента, който их призовава в отговор на подкани на разработчика.

## Theming MCP сървър

Ignite UI Theming MCP сървърът (`igniteui-theming`) е отделен MCP сървър, който излага Ignite UI Theming Engine като изпълнимо агентно контекст. Той покрива достъп до design tokens, определения на палета, генериране на CSS custom свойства и WCAG AA контрастна валидация. Той е архитектурно отделен от CLI MCP сървъра - може да бъде свързан независимо, за да даде на AI агента достъп до theming инструменти без излагане на scaffolding инструменти на проект.

Theming MCP сървърът се стартира чрез `npx`:

```bash
npx -y igniteui-theming igniteui-theming-mcp
```

Theming MCP сървърът поддържа Angular, React, Web Components и Blazor. Той се актуализира с всяко Ignite UI издание, така че агентите винаги работят срещу текущата token повърхност.

За детайли на конфигурация, вижте [Theming MCP](theming-mcp.md).

## Поддържани AI клиенти

CLI MCP сървърът и Theming MCP сървърът работят с всеки редактор или AI клиент, който поддържа MCP със STDIO транспорт.

| Клиент                      | Метод на конфигурация                                       |
| --------------------------- | ---------------------------------------------------------- |
| VS Code с GitHub Copilot | `.vscode/mcp.json`                                             |
| Cursor                      | `.cursor/mcp.json`                                             |
| Claude Desktop (macOS)      | `~/Library/Application Support/Claude/claude_desktop_config.json`                                             |
| Claude Desktop (Windows)    | `%APPDATA%\Claude\claude_desktop_config.json`                                             |
| Claude Code                 | `.mcp.json` или Claude Code MCP CLI команда          |
| JetBrains AI Assistant      | \*\*Tools → AI Assistant → Model Context Protocol (MCP)\*\* |

Agent Skills са съвместими с GitHub Copilot чрез `.github/copilot-instructions.md`, Cursor чрез `.cursorrules` или `.cursor/rules/`, Windsurf чрез `.windsurfrules` и JetBrains AI Assistant чрез настройки на подкана на ниво проект.

### Бърза настройка

Командата `ai-config` конфигурира MCP сървърите, копира файлове на навици, специфични за framework, в директорията с навици на всеки агент и настройва инструкционни файлове. Използвайте `--assistants` за избор на които кодирущи асистенти получават MCP конфигурация и `--agents` за избор на които агенти получават skill файлове. Ако не са предоставени параметри, командата влиза в интерактивен режим, подканвайки ви да изберете асистенти и агенти.

**Използвайки Angular Schematics:**

```bash
ng generate @igniteui/angular-schematics:ai-config
```

Това също регистрира `@angular/cli` MCP сървърът заедно със Ignite UI сървърите за вашите избрани клиенти.

**Използвайки Ignite UI CLI:**

```bash
npx igniteui-cli ai-config
```

Ако имате Ignite UI CLI инсталиран глобално, използвайте по-кратката форма:

```bash
ig ai-config
```

<DocsAside type="info">

Формите `npx igniteui-cli` и `ig` не регистрират `@angular/cli` MCP сървъра. Използвайте Angular Schematics командата по-горе, ако желаете всички три сървъра конфигурирани в една стъпка.

</DocsAside>

<DocsAside type="info">

Командата изисква Ignite UI пакети да бъдат инсталирани в проекта (`npm install`). Ако не са намерени skill файлове, убедете се, че вашите пакети са актуализирани.

</DocsAside>

### Стъпка 1 - Зареждане на Agent Skills

Копирайте Ignite UI Skill пакета за вашия framework във вашия проектна агентна открита пътека.

Пакетът Skill идва с библиотеката в `node_modules/igniteui-{framework}/skills/`.

Свържете го към вашия IDE, използвайки постоянна настройка за вашия клиент.

Вижте [Agent Skills](skills.md) за пълната настройка.

### Стъпка 2 - Свързване на CLI MCP сървър

Добавете запис за `igniteui-cli` MCP сървър към конфигурационния файл за вашия AI клиент. Използвайте JSON структурата, която съответства на вашия клиент:

**VS Code (`.vscode/mcp.json`):**

```json
{
  "servers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    }
  }
}
```

**Cursor, Claude Desktop, Claude Code, JetBrains и други MCP клиенти:**

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    }
  }
}
```

За пълното ръководство по настройка, включително VS Code, GitHub, Cursor, Claude Desktop, Claude Code, JetBrains и други MCP-съвместими клиенти, вижте [CLI MCP](cli-mcp.md).

### Стъпка 3 - Свързване на Theming MCP сървър (по избор)

Добавете запис за `igniteui-theming` към същия MCP конфигурационен файл, заедно с `igniteui-cli`:

**VS Code (`.vscode/mcp.json`):**

```json
{
  "servers": {
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

**Cursor, Claude Desktop, Claude Code, JetBrains и други MCP клиенти:**

```json
{
  "mcpServers": {
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

За детайли на конфигурация и theming работни процеси, вижте [Theming MCP](theming-mcp.md).
