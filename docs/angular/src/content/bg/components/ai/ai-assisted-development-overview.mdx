# AI-Подпомогнато разработване с Ignite UI

Ignite UI for Angular осигурява пълен AI инструментариум - Agent Skills, Ignite UI CLI MCP сървърът, Ignite UI Theming MCP сървърът и MAKER MCP сървърът - което закрепва AI кодиращите асистенти в верни API компоненти, пътища за импортиране и дизайнерски токени. Agent Skills са пакети с инструкции, собствени на разработчика, които определят как AI агентите използват Ignite UI в определен проект. CLI MCP сървърът (`igniteui-cli`) излага Ignite UI CLI scaffolding, управление на компоненти и инструменти за документация на активния AI агентски сеанс чрез Model Context Protocol. Theming MCP сървърът (`igniteui-theming`) излага Ignite UI Theming Engine като контекст, който се може запитвам от агент. MAKER MCP (`@igniteui/maker-mcp`) е многоагентен AI оркестрационен MCP сървър от Infragistics, който разлага сложни задачи в валидирани, изпълними планове на стъпки, използвайки консенсус-базиран алгоритъм за гласуване в няколко AI агента. Skills, CLI MCP и Theming MCP - и трите се конфигурират с един команден ред: `npx igniteui-cli ai-config`

MCP сървърите и Agent Skills служат за различни цели и имат различни предварителни условия:

| Компонент             | Какво предоставя                                             | Изисква ли инсталиран Ignite UI? |
| -------------------- | -------------------------------------------------------------- | ----------------------------- |
| Ignite UI MCP server | Запитвания в документация, API справка, инструменти за scaffolding | Не                            |
| Theming MCP server   | Дизайнерски токени, инструменти за палета, WCAG валидация на контраст | Не                            |
| Agent Skills         | Пакети с инструкции на ниво проект за правилно използване на компоненти | Да                           |

Можете да започнете да оценявате AI помощта на Ignite UI само със MCP сървърите - Ignite UI не е необходимо да бъде инсталиран в проекта ви. Agent Skills стават налични щом инсталирате пакетите на Ignite UI.

CLI MCP сървърът изисква STDIO транспорт; HTTP-базирани MCP клиенти не се поддържат. Agent Skills и CLI MCP сървърът не модифицират файлове на проект автономно - те излагат инструменти и инструкции на активния AI агент, който действа въз основа на подкани на разработчика.

## Конфигурирайте AI инструментариума

Изпълнете тази команда от корена на съществуващия Angular, React, Blazor или Web Components проект. Тя конфигурира MCP сървъри, копира файлове със умения, специфични за рамката, в директориум за умения на всеки агент, и настройва файлове с инструкции. Използвайте `--assistants` за да изберете кои кодиращи асистенти получават MCP конфигурация и `--agents` за да изберете кои агенти получават файлове със умения. Ако не са предоставени параметри, командата влиза в интерактивен режим и ви предлага да изберете асистенти и агенти. Съществуващите файлове се актуализират само ако съдържанието им се е променило.

```bash
npx igniteui-cli ai-config
```

<DocsAside type="info">
Без фиксирана версия, `npx` може да изтегли по-ста версия на CLI, която не разпознава подкомандата `ai-config` и вместо това ще стартира интерактивен подканване за създаване на проект, което ще създаде нов проект вътре в съществуващия ви. Уверете се, че сте инсталирали CLI версия 16.x.
</DocsAside>

Ако `ai-config` не може да открие рамката от файловете на проекта ви, ви подсказва да изберете една - така че командата работи дори от проект, където още не е инсталиран пакет на Ignite UI.

След като командата завърши, стартирайте MCP сървърите в AI клиента ви. Сървърите са конфигурирани, но не са още работещи - клиентът трябва да стартира всеки сървър преди неговите инструменти да бъдат налични на агента.

### Какво да очаквате

Ако Ignite UI **не е** инсталиран в проекта:

<DocsAside type="info">
Ignite UI MCP сървъри конфигурирани за вашите избрани клиенти
Нямат намерени AI файлове със умения. Уверете се, че пакетите са инсталирани (`npm install`) и вашите Ignite UI пакети са актуални.
</DocsAside>

MCP сървърите са готови за използване. Умествата ще бъдат добавени автоматично следващия път, когато изпълните `ai-config` след инсталиране на Ignite UI.

Ако Ignite UI **е** инсталиран в проекта:

<DocsAside type="info">
Ignite UI MCP сървъри конфигурирани за вашите избрани клиенти
Умения на Agent копирани в директориите за умения на вашите избрани агенти
</DocsAside>

Както MCP сървърите, така и Умествата са конфигурирани.

### Стартирайте сървърите

**VS Code с GitHub Copilot:**

Отворете `.vscode/mcp.json`. VS Code показва вътрешен бутон **Start** над всеки запис за сървър. Щракнете **Start** и за `igniteui` и за `igniteui-theming`. След като бъдат стартирани, VS Code показва наличния брой инструменти до всеки сървър (например, *"13 tools | 1 prompt"*). Алтернативно, изпълнете **MCP: List Servers** от Command Palette (`Ctrl+Shift+P` / `Cmd+Shift+P`), изберете всеки сървър и изберете **Start**.

**Cursor:**

Отворете **Settings → MCP**, намерете записите `igniteui` и `igniteui-theming` и включете всеки един. Cursor стартира всеки сървър веднага и показва брой инструменти.

**Claude Code:**

Изпълнете `claude mcp list` за да потвърдите, че двата сървъра са регистрирани. Claude Code стартира MCP сървъри автоматично, когато техните инструменти се използват за първи път - не е необходим ръчен стъпка за стартиране.

**JetBrains AI Assistant:**

Отворете **Settings → Tools → AI Assistant → Model Context Protocol (MCP)**. Щракнете върху иконата play до всеки запис за сървър за да го стартирате.

**Claude Desktop:**

Затворете и преоткрийте Claude Desktop. Сървърите се стартират автоматично при стартиране.

### Инсталирайте Ignite UI и добавете Умества по-късно

Ако изпълните `ai-config` без инсталиран Ignite UI и искате да добавите Умества, инсталирайте пакета на Ignite UI за вашата рамка и изпълнете отново командата:

```bash
npm install igniteui-angular
npx igniteui-cli@latest ai-config
```

Командата открива, че Умествата са сега налични и ги копира. Файловете с MCP конфигурация за избраните клиенти остават непроменени (вече са актуални).

## AI инструментариумът на един поглед

AI инструментариумът на Ignite UI се състои от три независимо използваеми слоя. Всеки слой може да бъде включен отделно; те са проектирани да работят заедно.

| Слой                               | Какво предоставя                                                                                            | Собственик        | Рамки                             |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------ | -------------------------------------- |
| Agent Skills                        | Пакети с инструкции, собствени на разработчика: пътища за импортиране, шаблони на компоненти, потоци на решения, конвенции на проект | Разработчик    | Angular, React, Web Components, Blazor |
| CLI MCP server (`igniteui-cli`)     | Scaffolding на проект, управление на компоненти, документация и API запитвания чрез MCP                            | Infragistics | Angular, React, Web Components, Blazor |
| Theming MCP server (`igniteui-theming`) | Дизайнерски токени, определения на палета, генериране на CSS персонални свойства, WCAG AA валидация на контраст             | Infragistics | Angular, React, Web Components, Blazor |

CLI MCP сървърът и Theming MCP сървърът и двата се стартират чрез `npx` и се свързват към всеки MCP-съвместим клиент чрез STDIO транспорт. Agent Skills са локални файлове, поставени в проекта ви, които AI клиентът чита от диска.

## Agent Skills

Agent Skills са структурирани, собствени на разработчика пакети, които казват на AI кодиращите асистенти точно как да използват Ignite UI за определена рамка. Пакет със Умение може да съдържа файл с инструкции `SKILL.md` със шаблони на компоненти, пътища за импортиране и потоци на решения; препратки към авторитетна документация на Ignite UI; и активи като файлове със схеми или диаграми. Когато Умение е активно в AI клиента, агентът следва Умението вместо да разчита на общи тренировъчни данни - които могат да référenciat остарели подписи на API или пътища за импортиране.

Ignite UI осигурява посветени пакети със Умения за Angular, React, Web Components и Blazor. Пакетът със Умение е собственост на разработчика: редактирайте `SKILL.md` за да съответства на конвенциите на екипа ви, добавете шаблони, специфични за проект, направете препратка към вътрешната дизайнерска система и версионирайте пакета заедно с кодовата база.

За пълни инструкции за настройка и IDE свързване, вижте [Agent Skills](skills.md).

## CLI MCP Server

Ignite UI CLI MCP сървърът (`igniteui-cli`) е MCP сървър, поддържан от Infragistics, който излага Ignite UI CLI scaffolding и инструменти за документация на активния AI агентски сеанс. След като бъде свързан, AI асистентът може да създава Angular, React, Blazor или Web Components проекти, да добавя и модифицира компоненти на Ignite UI и да отговаря на въпроси относно документацията и API - всичко това чрез естествени езикови подкани в сеанса на чата.

CLI MCP сървърът работи чрез `npx` без глобална инсталация:

```bash
npx -y igniteui-cli mcp
```

Използвайте `ai-config` за да напишете MCP конфигурацията за AI клиента ви автоматично. Сървърът се свързва с VS Code с GitHub Copilot, Cursor, Claude Desktop, Claude Code, JetBrains AI Assistant и всеки друг MCP-съвместим клиент, който поддържа STDIO транспорт. Точният формат на конфигурацията се различава по клиент - вижте [CLI MCP](cli-mcp.md) за пълното ръководство за настройка.

Той не генерира код автономно - той излага инструменти на AI агента, който ги призовава в отговор на подкани на разработчика.

## Theming MCP Server

Ignite UI Theming MCP сървърът (`igniteui-theming`) е отделен MCP сървър, който излага Ignite UI Theming Engine като контекст, който се може запитвам от агент. Той покрива достъпа до дизайнерски токени, определения на палета, генериране на CSS персонални свойства и WCAG AA валидация на контраст. Той е архитектурно отделен от CLI MCP сървърът - може да бъде свързан независимо за да даде на AI агента достъп до инструменти за оформление без излагане на инструменти за scaffolding на проект.

Theming MCP сървърът се стартира чрез `npx`:

```bash
npx -y igniteui-theming igniteui-theming-mcp
```

Theming MCP сървърът поддържа Angular, React, Web Components и Blazor. Той се актуализира с всяко издание на Ignite UI, така че агентите винаги работят срещу текущата повърхност на токени.

За детали на конфигурацията, вижте [Theming MCP](theming-mcp.md).

## Поддържани AI клиенти

CLI MCP сървърът и Theming MCP сървърът работят с всеки редактор или AI клиент, който поддържа MCP със STDIO транспорт.

| Клиент                      | Метод на конфигурация                                       |
| --------------------------- | ---------------------------------------------------------- |
| VS Code с GitHub Copilot | `.vscode/mcp.json`                                             |
| Cursor                      | `.cursor/mcp.json`                                             |
| Claude Desktop (macOS)      | `~/Library/Application Support/Claude/claude_desktop_config.json`                                             |
| Claude Desktop (Windows)    | `%APPDATA%\Claude\claude_desktop_config.json`                                             |
| Claude Code                 | `.mcp.json` или Claude Code MCP CLI командата          |
| JetBrains AI Assistant      | \*\*Tools → AI Assistant → Model Context Protocol (MCP)\*- |

Agent Skills са съвместими с GitHub Copilot чрез `.github/copilot-instructions.md`, Cursor чрез `.cursorrules` или `.cursor/rules/`, Windsurf чрез `.windsurfrules` и JetBrains AI Assistant чрез настройки на подкани на ниво проект.

### Бързо настройване

Командата `ai-config` конфигурира MCP сървъри, копира файлове със умения, специфични за рамката, в директориум за умения на всеки агент, и настройва файлове с инструкции. Използвайте `--assistants` за да изберете кои кодиращи асистенти получават MCP конфигурация и `--agents` за да изберете кои агенти получават файлове със умения. Ако не са предоставени параметри, командата влиза в интерактивен режим и ви предлага да изберете асистенти и агенти.

**Използвайки Angular Schematics:**

```bash
ng generate @igniteui/angular-schematics:ai-config
```

Това също така регистрира `@angular/cli` MCP сървърът заедно със сървърите на Ignite UI за избраните клиенти.

**Използвайки Ignite UI CLI:**

```bash
npx igniteui-cli ai-config
```

Ако имате инсталиран Ignite UI CLI глобално, използвайте по-краткото формуляр:

```bash
ig ai-config
```

<DocsAside type="info">

Формулярите `npx igniteui-cli` и `ig` не регистрират `@angular/cli` MCP сървър. Използвайте командата Angular Schematics по-горе, ако искате всички три сървъра да бъдат конфигурирани в един стъпка.

</DocsAside>

<DocsAside type="info">

Командата изисква пакетите на Ignite UI да бъдат инсталирани в проекта ви (`npm install`). Ако не са намерени файлове със умения, уверете се, че пакетите ви са актуални.

</DocsAside>

### Стъпка 1 - Заредете Agent Skills

Копирайте пакета със Умение на Ignite UI за вашата рамка в пътя за открояване на агент на проекта ви.

Пакетът със Умение се доставя с библиотеката в `node_modules/igniteui-{framework}/skills/`.

Свържете го към IDE ви, използвайки постоянната настройка за клиента ви.

Вижте [Agent Skills](skills.md) за пълната настройка.

### Стъпка 2 - Свържете CLI MCP Server

Добавете записа `igniteui-cli` MCP сървър към файла на конфигурация за AI клиента ви. Използвайте JSON структурата, която съответства на клиента ви:

**VS Code (`.vscode/mcp.json`):**

```json
{
  "servers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    }
  }
}
```

**Cursor, Claude Desktop, Claude Code, JetBrains и други MCP клиенти:**

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    }
  }
}
```

За пълното ръководство за настройка, включително VS Code, GitHub, Cursor, Claude Desktop, Claude Code, JetBrains и други MCP-съвместими клиенти, вижте [CLI MCP](cli-mcp.md).

### Стъпка 3 - Свържете Theming MCP Server (незадължително)

Добавете записа `igniteui-theming` към същия файл на MCP конфигурация, заедно с `igniteui-cli`:

**VS Code (`.vscode/mcp.json`):**

```json
{
  "servers": {
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

**Cursor, Claude Desktop, Claude Code, JetBrains и други MCP клиенти:**

```json
{
  "mcpServers": {
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

За детали на конфигурацията и работни потоци на оформление, вижте [Theming MCP](theming-mcp.md).
