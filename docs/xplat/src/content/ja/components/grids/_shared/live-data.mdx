# {Platform} {ComponentTitle} ライブ データ更新

{Platform} {ComponentTitle} の {ProductName} ライブ データ更新機能は、グリッド内に表示されるデータのリアルタイムまたはほぼリアルタイムの更新を有効にするために使用されます。これは、株式市場トラッカー、ライブ スポーツ スコア、IoT (モノのインターネット) ダッシュボードなど、データが絶えず変化するアプリで非常に役立ちます。<ApiLink type="{ComponentName}" /> は、ユーザー操作への応答性を維持しながら、1 秒あたり数千件の更新を処理できます。

## {Platform} ライブ データ更新の例

以下のサンプルは、すべてのレコードが 1 秒間に複数回更新される場合の {ComponentTitle} のパフォーマンスを示しています。UI コントロールを使用して、読み込むレコード数と更新頻度を選択します。
同じデータを [Column Chart](../../charts/types/column-chart.md) に入力して、Ignite UI for Angular の強力なチャート機能を体験してください。`Chart` ボタンは、選択された行の地域別カテゴリ価格データを表示し、`Chart` 列ボタンは現在の行について同じ内容を表示します。

## Dock Manager と igxGrid コンポーネントによるライブ データ フィード

このデモの目的は、[SignalR](https://dotnet.microsoft.com/apps/aspnet/signalr) ハブ バックエンドを使用したリアルタイム データ ストリームを備えた金融画面ボードを紹介することです。
ご覧のとおり、igxGrid コンポーネントはサーバーからの高頻度更新を容易に処理します。SignalR を使用する ASP.NET Core アプリケーションのコードは、この [公開 GitHub リポジトリ](https://github.com/IgniteUI/finjs-web-api) にあります。

<Sample src="/{ComponentSample}/finjs-dock-manager" height={700} alt="{Platform} {ComponentTitle} Live data Update Example with a service" />

### ハブ接続を開始する

signal-r.service は、公開されている管理可能なパラメーターである **frequency**、**volume**、および **live-update state toggle** (Start/Stop) の接続と更新を処理します。

```ts
this.hubConnection = new signalR.HubConnectionBuilder()
        .configureLogging(signalR.LogLevel.Trace)
        .withUrl('https://www.infragistics.com/angular-apis/webapi/streamHub')
        .build();
    this.hubConnection
        .start()
        .then(() => {
            this.hasRemoteConnection = true;
            this.registerSignalEvents();
            this.broadcastParams(interval, volume, live, updateAll);
        })
        .catch(() => {});
```

指定された頻度に基づいて、サーバーから合計 30 件の新しい更新がグリッドによって受信されます。変更を処理する 3 つの列 (Price、Change、Change in percent) に、特定の cellStyle クラスが適用されます。

### 更新頻度とデータ量

左側のアクション パネルを使用して、データ フィードの頻度と要求するデータ量を管理できます。すべてのグリッドは同じデータ ソースを使用します。他のアクション要素を使用して、**データ フィードの停止**、**アプリケーション テーマ**の変更、または igxGrid を持つ **DockSlot コンテナーの動的な追加**を自由に行えます。

特定の頻度で新しいデータ セットを要求するために、`updateParameters` メソッドを使用します。このメソッドは、SignalR の [ストリーム ハブ実装](https://github.com/IgniteUI/finjs-web-api/blob/master/WebAPI/Models/StreamHub.cs#L18) の一部です。

```ts
this.hubConnection.invoke('updateparameters', frequency, volume, live, updateAll)
    .then(() => console.log('requestLiveData', volume))
    .catch(err => {
        console.error(err);
    });
```

### DockSlot と Grid コンポーネントを動的に作成する

[ComponentFactoryResolver](https://angular.io/api/core/ComponentFactoryResolver) を使用することで、DockSlot と Grid コンポーネントをその場で作成できます。

### DockManager コンポーネント

[Dock Manager](../../layouts/dock-manager.md) WebComponent を活用し、ドッキングされたパネルまたはフローティング パネルを使用して独自の Web ビューを構築します。新しいフローティング パネルを追加するには、右側のアクション ペインを開き、'Add floating pane' ボタンをクリックします。新しいペインを目的の場所にドラッグ アンド ドロップします。
