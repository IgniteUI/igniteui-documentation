# {Platform} {ComponentTitle} 選択の概要

{Platform} {ComponentTitle} の {ProductName} 選択機能を使用すると、シンプルなマウス操作でデータを簡単に操作および処理できます。使用可能な選択モードは 3 つあります。

- 行選択
- セル選択
- 列選択

<ApiLink type="{ComponentName}" member="rowSelection" /> プロパティを使用すると、次を指定できます。

- なし
- 単一
- 複数選択

## {Platform} {ComponentTitle} 選択オプション

<ComponentBlock for="Grid, HierarchicalGrid">

{ProductName} <ApiLink type="{ComponentName}" /> コンポーネントは、[行選択](row-selection.md)、[セル選択](cell-selection.md)、[列選択](column-selection.md) の 3 つの異なる選択モードを提供します。既定では、<ApiLink type="{ComponentName}" /> で有効になっているのは**複数セル選択**モードのみです。選択モードを変更または有効にするには、<ApiLink type="{ComponentName}" member="rowSelection" />、<ApiLink type="{ComponentName}" member="cellSelection" />、または <ApiLink type="Column" member="selectable" /> プロパティを使用できます。

</ComponentBlock>

### {Platform} {ComponentTitle} 行選択

<ApiLink type="{ComponentName}" member="rowSelection" /> プロパティを使用すると、次のオプションを指定できます。

- `None` - <ApiLink type="{ComponentName}" /> の行選択が無効になります。
- `Single` - <ApiLink type="{ComponentName}" /> 内で 1 行のみ選択できます。
- `Multiple` - 行セレクターを使用するか、<kbd>コントロール</kbd> + <kbd>クリック</kbd> のようなキーの組み合わせを使用するか、セルにフォーカスがある状態で <kbd>スペース キー</kbd> を押すことで、複数行選択を使用できます。

<ComponentBlock for="TreeGrid, HierarchicalGrid">

- `MultipleCascade` - これはカスケード選択のモードであり、ユーザーが操作によって選択したレコードの下にあるツリー内のすべての子が選択されます。このモードでは、親の選択状態はその子の選択状態に完全に依存します。

</ComponentBlock>

> 詳細については、[行選択トピック](row-selection.md)を参照してください。

### {Platform} {ComponentTitle} セル選択

<ApiLink type="{ComponentName}" member="cellSelection" /> プロパティを使用すると、次のオプションを指定できます。

- `None` - <ApiLink type="{ComponentName}" /> のセル選択が無効になります。
- `Single` - <ApiLink type="{ComponentName}" /> 内で 1 つのセルのみ選択できます。
- `Multiple` - 現在、これは <ApiLink type="{ComponentName}" /> における選択の既定の状態です。左マウス ボタンをクリックしたままセル上をドラッグすることで、複数セル選択を使用できます。

<ComponentBlock for="Grid, TreeGrid, HierarchicalGrid">

> 詳細については、[セル選択トピック](cell-selection.md)を参照してください。

</ComponentBlock>

### {Platform} {ComponentTitle} 列選択

<ApiLink type="Column" member="selectable" /> プロパティを使用すると、各 <ApiLink type="Column" /> に対して次のオプションを指定できます。このプロパティが true または false に設定されている場合、対応する列選択がそれぞれ有効または無効になります。

これにより、次の 3 つのバリエーションがあります。

- 単一選択 - 列セル上で <kbd>マウスクリック</kbd>。
- 複数列選択 - 列セル上で <kbd>コントロール</kbd> + <kbd>マウスクリック</kbd> を押したままにします。
- 範囲列選択 - <kbd>シフト</kbd> + <kbd>マウスクリック</kbd> を押したままにすると、その間のすべてが選択されます。

<ComponentBlock for="Grid, TreeGrid, HierarchicalGrid">

> 詳細については、[列選択トピック](column-selection.md)を参照してください。

</ComponentBlock>

## {Platform} {ComponentTitle} コンテキスト メニュー

<ApiLink type="{ComponentName}" member="contextMenu" /> イベントを使用すると、<ApiLink type="{ComponentName}" /> での作業を容易にするカスタム コンテキスト メニューを追加できます。グリッドの本体を**右クリック**すると、イベントはトリガーされたセルを発行します。**コンテキスト メニュー**は、発行されたセルを使用して動作します。

**複数セル選択**がある場合は、選択されたセルが複数セル選択の範囲内にあるかどうかを確認するロジックを配置します。範囲内にある場合は、選択されたセルの値も発行します。

基本的に、メイン関数は次のようになります。

<PlatformBlock for="Angular">

```typescript
public rightClick(eventArgs: any) {
     // Prevent the default behavior of the right click
    eventArgs.event.preventDefault();
    this.multiCellArgs = {};
    // If we have multi-cell selection, check if selected cell is within the ranges
    if (this.multiCellSelection) {
        const node = eventArgs.cell.selectionNode;
        const isCellWithinRange = this.grid1.getSelectedRanges().some(range => {
            if (node.column >= range.columnStart &&
                node.column <= range.columnEnd &&
                node.row >= range.rowStart &&
                node.row <= range.rowEnd) {
                return true;
            }
            return false;
        })
        // If the cell is within a multi-cell selection range, bind all the selected cells data
        if (isCellWithinRange) {
            this.multiCellArgs = { data: this.multiCellSelection.data };
        }
    }
    // Set the position of the context menu
    this.contextmenuX = eventArgs.event.clientX;
    this.contextmenuY = eventArgs.event.clientY;
    this.clickedCell = eventArgs.cell;
    // Enable the context menu
    this.contextmenu = true;
}
```

コンテキスト メニューには、次の機能があります。

- 選択したセルの*値*をコピーします。
- 選択したセルの *dataRow* をコピーします。
- 選択したセルが**複数セル選択範囲**内にある場合、すべての*選択データ*をコピーします。

public copySelectedCellData(event) {
    const selectedData = { [this.cell.column.field]: this.cell.value };
    this.copyData(JSON.stringify({ [this.cell.column.field]: this.cell.value }));
    this.onCellValueCopy.emit({ data: selectedData });
}

public copyRowData(event) {
    const selectedData = this.cell.row.data ;
    this.copyData(JSON.stringify(this.cell.row.data));
    this.onCellValueCopy.emit({ data: selectedData });
}

public copySelectedCells(event) {
    const selectedData = this.selectedCells.data;
    this.copyData(JSON.stringify(selectedData));
    this.onCellValueCopy.emit({ data: selectedData });
}

    public copySelectedRowData() {

        const selectedData = this.clickedCell.value;
        this.copyData(selectedData);
        const selectedDataArea = document.getElementById('selectedArea');
        selectedDataArea.innerText = JSON.stringify(selectedData);
        this.toggleContextMenu();
    }


    public copySelectedData() {
        const selectedData = this.grid.getSelectedData();
        this.copyData(selectedData);
        const selectedDataArea = document.getElementById('selectedArea');
        selectedDataArea.innerText = JSON.stringify(selectedData);

        this.toggleContextMenu();
    }

    private copyData(data: any[]) {
        const tempElement = document.createElement('input');
        document.body.appendChild(tempElement);
        tempElement.setAttribute('id', 'temp_id');
        (document.getElementById('temp_id') as HTMLInputElement).value = JSON.stringify(data);
        tempElement.select();
        document.execCommand('copy');
        document.body.removeChild(tempElement);
    }

## 既知の問題と制限事項

<PlatformBlock for="Angular">

- IE11 で選択を有効にした <ApiLink type="{ComponentName}" /> を使用するには、{Platform} アプリケーションの polyfill.ts で配列 polyfill を明示的にインポートする必要があります。IE11 はバージョン 13.0.0 以降サポートされなくなりました。

```typescript
import "core-js/es7/array";
```

</PlatformBlock>

グリッドに <ApiLink type="{ComponentName}" member="primaryKey" /> が設定されておらず、リモート データ シナリオが有効になっている場合 (ページング、並べ替え、フィルタリング、スクロールが、グリッドに表示するデータを取得するためにリモート サーバーへの要求をトリガーする場合)、データ要求が完了した後、行は次の状態を失います。

- 行選択
- 行の展開/折りたたみ
- 行編集
- 行のピン固定
