# {Platform} {ComponentTitle} セル結合

<PlatformBlock for="WebComponents, React">

Ignite UI for {Platform} の {ComponentTitle} は、同じ値を持つ隣接する 2 つ以上のセルを 1 つの大きなセルに結合するセル結合機能を提供します。結合は列内で垂直方向に適用され、重複する値を減らすことで可読性を向上させます。この機能は、既定で一致するデータ値に基づいてセルを結合するように、またはカスタム条件を適用して結合するように構成できます。

## セル結合の有効化と使用

グリッドでのセル結合は、次の 2 つのレベルで制御されます。

- グリッド レベルの結合モード – 結合が適用されるタイミングを決定します。
- 列レベルの結合切り替え – セルを結合できる列を決定します。

### グリッド結合モード

グリッドは、<ApiLink type="GridCellMergeMode" /> 列挙型の値を受け入れる <ApiLink type="{ComponentName}" member="cellMergeMode" /> プロパティを公開します。

- `always` - ソート状態に関係なく、結合条件を満たす隣接セルを結合します。
- `onSort` - 列がソートされている場合にのみ隣接セルを結合します **(既定値)**。

<PlatformBlock for="React">

```tsx
<{ComponentSelector} data={data} cellMergeMode={cellMergeMode} >
    ...
</{ComponentSelector}>
```

</PlatformBlock>

```tsx
const cellMergeMode: GridCellMergeMode = 'always';
```

</PlatformBlock>

```html
    ...
</{ComponentSelector}>
```

</PlatformBlock>

```razor
    ...
```

- **OrderID** 列は、隣接する重複値を結合します。
- **ShipperName** 列は、結合せずに通常どおりレンダリングされます。

## カスタム結合条件

組み込みの `always` モードと `onSort` モードに加えて、グリッドでは <ApiLink type="{ComponentName}" member="mergeStrategy" /> プロパティを通じてセルを結合するためのカスタム条件を定義できます。このストラテジは、セルの比較方法と結合範囲の計算方法の両方を制御します。

### 結合ストラテジ クラス

カスタム結合ストラテジは、<ApiLink type="GridMergeStrategy" /> クラスを実装する必要があります。

<PlatformBlock for="React">

```ts
export declare class IgrGridMergeStrategy {
    merge: (
        data: any[],
        field: string,
        comparer: (prevRecord: any, currentRecord: any, field: string) => boolean,
        result: any[],
        activeRowIndex?: number,
        grid?: GridType
    ) => any[];

    comparer: (prevRecord: any, record: any, field: string) => boolean;
}
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```ts
export declare class IgcGridMergeStrategy {
    merge: (
        data: any[],
        field: string,
        comparer: (prevRecord: any, currentRecord: any, field: string) => boolean,
        result: any[],
        activeRowIndex?: number,
        grid?: GridType
    ) => any[];

    comparer: (prevRecord: any, record: any, field: string) => boolean;
}

```

</PlatformBlock>

- `merge` - 結合されたセルを生成する方法を定義します。
- `comparer` - 隣接する 2 つのレコードを結合するかどうかを決定する条件を定義します。

<ComponentBlock for="Grid, HierarchicalGrid">

### 既定のストラテジの拡張

動作の一部のみをカスタマイズしたい場合 (たとえば、比較ロジック)、組み込みの <ApiLink type="DefaultMergeStrategy" /> を拡張し、関連するメソッドをオーバーライドできます。

<PlatformBlock for="React">

```ts
export class MyCustomStrategy extends IgrDefaultMergeStrategy {
    /* Merge only cells within their respective projects */
    public override comparer(prevRecord: any, record: any, field: string): boolean {
        const a = prevRecord[field];
        const b = record[field];
        const projA = prevRecord['ProjectName'];
        const projB = record['ProjectName'];
        return a === b && projA === projB;
    }
}
```

</PlatformBlock>

<PlatformBlock for="WebComponents">


<ComponentBlock for="TreeGrid">

`IgxTreeGrid` は、`IGridMergeStrategy` インターフェイスを実装する 2 つの組み込みストラテジ、<ApiLink type="DefaultTreeGridMergeStrategy" /> と <ApiLink type="ByLevelTreeGridMergeStrategy" /> を提供します。<ApiLink type="DefaultTreeGridMergeStrategy" /> は、階層レベルに関係なく、同じ値を持つすべてのセルを結合します。これに対して、<ApiLink type="ByLevelTreeGridMergeStrategy" /> は、同じ値を持ち、同じレベルに配置されている場合にのみセルを結合するため、レベルが結合の必須条件になります。

## 機能の統合

結合セルの特定の動作により、グリッドの他のいくつかの機能とどのように連携するかを正確に把握しておく必要があります。

<ComponentBlock for="Grid">

- **展開/折りたたみ**: 機能 (マスター詳細、グループ化など) が非データ行を生成する場合、セル結合は中断され、グループは分割されます。

</ComponentBlock>

- **Excel エクスポート**: 結合セルは、Excel にエクスポートされたときも結合されたままになります。
- **列の固定**: 列が固定されてもセルは結合されたままで、固定領域に表示されます。
- **行の固定**: セルは、それらが含まれる領域内でのみ結合されます。つまり、固定行のセルは他の固定行のセルとのみ結合され、固定されていない行のセルは固定されていない行のセルとのみ結合されます。
- **更新/編集**: アクティブ化により結合シーケンスが分断されるため、編集モードになるのは単一のセルのみです。
- **行選択**: 選択された行が結合セルと交差する場合、関連するすべての結合セルを選択の一部としてマークする必要があります。
- **ナビゲーション/アクティブ化**: セルがアクティブになると、同じ行内のすべての結合セルは単一セルになります。つまり、それらの結合シーケンスは分断されます。これには、キーボード ナビゲーションによるアクティブ化も含まれます。

<DocsAside type="info">

結合セルがクリックされると、結合シーケンス内で最も近いセルがアクティブになります。

</DocsAside>

## 制限事項

| 既知の制限事項                                                   | 説明                                                                                                                        |
| ------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| セル結合は、複数行レイアウトとの組み合わせではサポートされません。 | 両方とも複雑なレイアウトにまたがるため、組み合わせても意味がありません。このような無効な構成が検出された場合は、警告がスローされます。 |
