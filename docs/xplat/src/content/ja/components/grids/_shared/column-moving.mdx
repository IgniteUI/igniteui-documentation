# {ComponentTitle} 列の並べ替えと移動

{ProductName} の {Platform} {ComponentTitle} 列移動機能を使用すると、列をすばやく簡単に並べ替えることができます。これは、列移動 API を使用するか、マウスまたはタッチ ジェスチャでヘッダーを別の位置にドラッグ アンド ドロップすることで実行できます。{Platform} {ComponentTitle} では、固定列と固定されていない列、および [複数列ヘッダー](multi-column-headers.md) に対しても列移動を有効にできます。

<DocsAside type="info">

列と列グループ間の並べ替えは、それらが階層内で同じレベルにあり、両方が同じグループに属している場合にのみ許可されます。列/列グループがトップ レベルの列である場合、それらの間で移動が許可されます。

</DocsAside>

<DocsAside type="info">

列ヘッダーがテンプレート化されていて、列移動が有効になっている場合、または対応する列がグループ化可能な場合、テンプレート化された要素には **draggable** 属性を **false** に設定する必要があります。

</DocsAside>

<PlatformBlock for="Angular">

これにより、要素によって発行される任意のイベントにハンドラーをアタッチできます。そうしない場合、イベントは `igxDrag` ディレクティブによって消費されます。

</PlatformBlock>

<DocsAside type="info">

固定領域が許可される最大幅（<ApiLink type="{ComponentName}" /> の合計幅の 80%）を超える場合、ドロップ操作が禁止されており、固定できないことをエンド ユーザーに通知する視覚的な手がかりが表示されます。これは、固定領域に列をドロップできないことを意味します。

</DocsAside>

<PlatformBlock for="Angular">

```html
<ng-template igxHeader>
    <igx-icon [attr.draggable]="false" (click)="onClick()"></igx-icon>
</ng-template>
```

</PlatformBlock>

```razor

```

</PlatformBlock>

## API

ドラッグ アンド ドロップ機能に加えて、列移動機能では、列の移動/列の並べ替えをプログラムで行うための API メソッドも提供されます。

<ApiLink type="GridBaseDirective" member="moveColumn" /> - 列を別の列（ターゲット）の前または後に移動します。最初のパラメーターは移動する列で、2 番目のパラメーターはターゲット列です。また、列をターゲット列の前または後のどちらに配置するかを決定する、任意の 3 番目のパラメーター `Position`（<ApiLink type="DropPosition" /> 値を表す）も受け入れます。

<PlatformBlock for="React, Angular, WebComponents">

```typescript
// Move the ID column after the Name column
const idColumn = grid.getColumnByName("ID");
const nameColumn = grid.getColumnByName("Name");

grid.moveColumn(idColumn, nameColumn, DropPosition.AfterDropTarget);

```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
    public async void HandleClick()
    {
        IgbColumn Col1 = await this.grid.GetColumnByVisibleIndexAsync(0);
        IgbColumn Col2 = await this.grid.GetColumnByVisibleIndexAsync(1);
        this.Grid.MoveColumn(Col1,Col2, DropPosition.AfterDropTarget);
    }
```

</PlatformBlock>

<ApiLink type="Column" member="move" label="Move" /> - 列を指定された表示インデックスに移動します。渡されたインデックス パラメーターが無効な場合（負の値である、または列数を超える場合）、または列がこのインデックスへの移動を許可されていない場合（別のグループ内にある場合）、操作は実行されません。

<PlatformBlock for="WebComponents">

```typescript
// Move the ID column at 3rd position.
const idColumn = grid.getColumnByName("ID");
idColumn.move(3);
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
    public async void HandleClick()
    {
        IgbColumn Col1 = await this.grid.GetColumnByVisibleIndexAsync(0);
        this.Col1.Move(3);
    }
```

</PlatformBlock>

列移動機能を使用する場合、操作が成功すると <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingEnd" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingEnd" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingEnd" /></PlatformBlock> イベントが発行されることに注意してください。また、ドラッグ アンド ドロップ機能と比較して、列移動機能を使用する場合は <ApiLink type="{ComponentName}" member="moving" /> プロパティを true に設定する必要がないことにも注意してください。

## イベント

列移動に関連するイベントがいくつかあり、列のドラッグ アンド ドロップ操作にフックする手段を提供します。これらは <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingStart" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingStart" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingStart" /></PlatformBlock>、`ColumnMoving`、および <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingEnd" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingEnd" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingEnd" /></PlatformBlock> です。

<ApiLink type="{ComponentName}" /> の <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingEnd" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingEnd" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingEnd" /></PlatformBlock> イベントをサブスクライブして、列が新しい位置にドロップされたときにカスタム ロジックを実装できます。たとえば、次のコード スニペットでは、**Category** 列を **Change On Year(%)** 列の後にドロップすることをキャンセルできます。

## スタイル設定

<ApiLink type="{ComponentName}" /> 列移動ヘッダーのスタイル設定を開始するには、すべてのテーマ関数とコンポーネント ミックスインが含まれている `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

最もシンプルな方法として、<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張し、`$ghost-header-background`、`$ghost-header-text-color`、および `$ghost-header-icon-color` パラメーターを受け入れる新しいテーマを作成します。

```scss
// Define dark theme for the column moving
$dark-grid-column-moving-theme: grid-theme(
    $ghost-header-text-color: #F4D45C,
    $ghost-header-background: #575757,
    $ghost-header-icon-color: #f4bb5c
);
```

最後の手順は、それぞれのテーマとともにコンポーネント ミックスインを**含める**ことです。

```scss
@include grid($dark-grid-column-moving-theme);
```

<DocsAside type="info">

コンポーネントの [**ビュー カプセル化**](/components/themes/sass/component-themes.html#view-encapsulation) 戦略によっては、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要がある場合があります。

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($dark-grid-column-moving-theme);
    }
}
```

### カラー パレットの定義

先ほどのようにカラー値をハードコードする代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、カラーに関してより高い柔軟性を実現できます。

**igx-palette** は、渡されたプライマリ カラーとセカンダリ カラーに基づいてカラー パレットを生成します。

```scss
$yellow-color: #F4D45C;
$black-color: #575757;

$dark-palette: palette($primary: $yellow-color, $secondary: $black-color);
```

その後、<ApiLink type="function-igx-color" label="**igx-color**" kind="sass" module="index.html" /> を使用すると、パレットからカラーを簡単に取得できます。

```scss
$dark-grid-column-moving-theme: grid-theme(
    $palette: $dark-palette,
    $ghost-header-text-color: color($dark-palette, "primary", 400),
    $ghost-header-background: color($dark-palette, "secondary", 200),
    $ghost-header-icon-color: color($dark-palette, "primary", 500)
);
```

<DocsAside type="info">

color と palette は、カラーの生成と取得に強力な関数です。使用方法の詳細なガイダンスについては、[パレット](/components/themes/palettes.html) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[スキーマ](/components/themes/sass/schemas.html) の利点を生かした堅牢で柔軟な構造を構築できます。**スキーマ**はテーマのレシピです。

各コンポーネントに提供されている 2 つの事前定義スキーマのいずれかを拡張します。この場合は <ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" /> です。

```scss
// Extending the dark grid schema
$dark-grid-column-moving-schema: extend($_light-grid,
        (
            ghost-header-text-color:(
               color: ("primary", 400)
            ),
            ghost-header-background:(
               color: ("secondary", 200)
            ),
            ghost-header-icon-color:(
               color: ("primary", 500)
            )
        )
);
```

カスタム スキーマを適用するには、グローバルのいずれか（<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> または <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />）を**拡張**する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを示し、その後、それを対応するコンポーネント テーマに追加します。

```scss
// Extending the global dark-schema
$custom-light-schema: extend($light-schema,(
    igx-grid: $dark-grid-column-moving-schema,
));

// Defining dark-grid-theme with the global dark schema
$dark-grid-column-moving-theme: grid-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);
```

上記で示したのと同じ方法でテーマを含めることを忘れないでください。
