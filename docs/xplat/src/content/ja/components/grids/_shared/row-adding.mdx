# {Platform} {ComponentTitle} 行の追加

<ComponentBlock for="Grid, HierarchicalGrid">

{Platform} {ComponentTitle} の {ProductName} 行追加機能により、ユーザーは別のフォームやページに移動することなく、新しいデータ レコードを入力して送信できます。<ApiLink type="{ComponentName}" /> により、ユーザーはインライン行追加と CRUD 操作用の強力な API を通じてデータを操作できます。
編集アクションを有効にした <ApiLink type="ActionStrip" /> コンポーネントをグリッドのテンプレートに追加します。その後、行にホバーして、提供されたボタンを使用します。最後に <kbd>代替</kbd> + <kbd>+</kbd> を押して、行追加 UI を表示します。

</ComponentBlock>

<ComponentBlock for="TreeGrid">

<ApiLink type="{ComponentName}" /> は、インライン行追加と {Platform} CRUD 操作用の強力な API を通じてデータ操作を行う便利な方法を提供します。編集アクションを有効にした <ApiLink type="ActionStrip" /> コンポーネントをグリッドのテンプレートに追加し、行にホバーして提供されたボタンを使用します。<kbd>代替</kbd> + <kbd>+</kbd> を押して行追加 UI を表示するか、<kbd>代替</kbd> + <kbd>シフト</kbd> + <kbd>+</kbd> を押して選択した行の子を追加する UI を表示します。

</ComponentBlock>

## プログラムによる行追加の開始

<ApiLink type="{ComponentName}" /> では、2 つの異なるパブリック メソッドを使用して、プログラムにより行追加 UI を表示できます。1 つは UI を表示する対象となる行の下を指定するために行 ID を受け取るメソッドで、もう 1 つはインデックスで動作するメソッドです。これらのメソッドを使用すると、現在のデータ ビュー内の任意の場所に UI を表示できます。ページを変更することや、たとえばフィルターで除外されている行を指定することはサポートされていません。

<ComponentBlock for="Grid, HierarchicalGrid">

<ApiLink type="{ComponentName}" member="beginAddRowById" /> を使用するには、操作のコンテキストとして使用する行をその `RowID` (PK) で指定する必要があります。その後、このメソッドは、エンド ユーザーが指定された行の行追加アクション ストリップ ボタンをクリックした場合と同じように機能し、その行の下に UI を表示します。最初のパラメーターに `null` を渡すことで、グリッドの最初の行として UI を表示することもできます。

<PlatformBlock for="WebComponents">

```typescript
this.grid.beginAddRowById('ALFKI');  // Spawns the add row UI under the row with PK 'ALFKI'
this.grid.beginAddRowById(null);     // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="React">

```typescript
gridRef.current.beginAddRowById('ALFKI');  // Spawns the add row UI under the row with PK 'ALFKI'
gridRef.current.beginAddRowById(null);     // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
@code {
    await this.grid.BeginAddRowByIdAsync('ALFKI', false);  // Spawns the add row UI under the row with PK 'ALFKI'
    await this.grid.BeginAddRowByIdAsync(null, false);     // Spawns the add row UI as the first record
}
```

</PlatformBlock>

<ApiLink type="{ComponentName}" member="beginAddRowByIndex" /> メソッドも同様に機能しますが、UI を表示するインデックスを指定する必要があります。許可される値は、0 からデータ ビューのサイズ - 1 までです。

<PlatformBlock for="WebComponents">

```typescript

this.grid.beginAddRowByIndex(0);    // Spawns the add row UI as the first record
```

</PlatformBlock>

</PlatformBlock>

## 配置

- 行追加 UI の既定の位置は、エンド ユーザーが行追加ボタンをクリックした行の下です。

- <ApiLink type="{ComponentName}" /> は、行追加 UI 全体が表示されるように自動的にスクロールします。

- 行追加 UI のオーバーレイは、スクロール中もその位置を維持します。

## 動作

行追加 UI は、エンド ユーザーに一貫した編集体験を提供するように設計されているため、行編集 UI と同じ動作をします。詳細については、[{ComponentTitle} 行編集](row-editing.md) トピックを参照してください。

行追加 UI を通じて新しい行が追加された後、その位置や表示状態は <ApiLink type="{ComponentName}" /> の並べ替え、フィルタリング、グループ化の状態によって決まります。これらの状態がいずれも適用されていない <ApiLink type="{ComponentName}" /> では、最後のレコードとして表示されます。表示範囲外にある場合、エンド ユーザーが <ApiLink type="{ComponentName}" /> をその位置までスクロールするために使用できるボタンを含むスナックバーが短時間表示されます。

## キーボード ナビゲーション

- <kbd>代替</kbd> + <kbd>+</kbd> - 行を追加するための編集モードに入ります

<ComponentBlock for="TreeGrid">

- <kbd>代替</kbd> + <kbd>シフト</kbd> + <kbd>+</kbd> - 子を追加するための編集モードに入ります

</ComponentBlock>

- <kbd>エスケープ</kbd> は、変更を送信せずに行追加モードを終了します

- <kbd>タブ</kbd> は、行内の編集可能なセルから次の編集可能なセルへ、また最も右側の編集可能なセルから CANCEL ボタンおよび DONE ボタンへフォーカスを移動します。DONE ボタンからのナビゲーションでは、現在編集中の行内の最も左側の編集可能なセルに移動します。

## 機能の統合

- <ApiLink type="{ComponentName}" /> のデータ ビューが変更されると、行追加操作はすべて停止します。エンド ユーザーによって行われた変更は送信されます。データ ビューを変更する操作には、並べ替え、グループ化、フィルタリング、ページングなどが含まれますが、これらに限定されません。

- 集計は、行追加操作の完了後に更新されます。同じことは、並べ替え、フィルタリングなど、データ ビューに依存する他の機能にも当てはまります。

<ComponentBlock for="HierarchicalGrid">

- <ApiLink type="{ComponentName}" /> の UI を表示する際、エンド ユーザーが行追加ボタンをクリックした行で現在展開されている子レイアウトは折りたたまれます。

</ComponentBlock>

## リモート シナリオ

ほとんどのリモート データ シナリオでは、主キーの割り当ては作成サーバー要求で行われます。この場合、クライアントに追加されたレコードは、サーバーのデータベースに保存されるまで最終的な主キー値を持ちません。その場合、<ApiLink type="{ComponentName}" /> でこの更新を処理する推奨方法は次のとおりです。

- <ApiLink type="{ComponentName}" /> がトランザクションを使用しない場合。

  作成要求が正常に完了し、追加されたレコード データが返されたら、ローカル データ レコード インスタンス内のそのレコードの ID を置き換えることができます。

- <ApiLink type="{ComponentName}" /> がトランザクションを使用する場合。

  作成要求または一括更新要求が正常に完了し、追加されたレコード インスタンス (データベースで生成された ID を含む) が返されたら、関連する ADD トランザクションを `Clear` API メソッドを使用してトランザクション ログからクリアする必要があります。これは、ローカル トランザクションには生成された ID フィールドがあり、データベースで作成されたものと異なる可能性があるため、クリアする必要があるからです。その後、応答で渡されたレコードをローカル データ インスタンスに追加できます。

これにより、リモートで生成された ID が常にローカル データに反映され、後続の更新/削除操作が正しいレコード ID を対象にすることが保証されます。

</PlatformBlock>
