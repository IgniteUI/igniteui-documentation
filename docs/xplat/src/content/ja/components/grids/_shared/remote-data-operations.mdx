## リモート仮想化

<ApiLink type="{ComponentName}" /> は、内部で使用する `ForOf` ディレクティブに実装されている動作を公開し、リモート サービスからデータ チャンクが要求されるシナリオをサポートします。

この機能を利用するには、受け取った引数に基づいて適切な要求を行うために <ApiLink type="{ComponentName}" member="dataPreLoad" /> 出力をサブスクライブし、サービスから取得した該当情報を使用してパブリック <ApiLink type="{ComponentName}" /> プロパティ <ApiLink type="{ComponentName}" member="totalItemCount" /> を設定する必要があります。

## 無限スクロール

エンドポイントからチャンク単位でデータを取得する必要があるシナリオでよく使用される設計が、いわゆる無限スクロールです。データ グリッドでは、エンドユーザーが一番下までスクロールすることをトリガーとして、読み込まれたデータが継続的に増加することが特徴です。次の段落では、利用可能な API を使用して <ApiLink type="{ComponentName}" /> で無限スクロールを簡単に実現する方法を説明します。

無限スクロールを実装するには、データをチャンク単位で取得する必要があります。すでに取得されたデータはローカルに保存し、チャンクの長さとチャンクの数を決定する必要があります。また、グリッド内の最後に表示されているデータ行インデックスも追跡する必要があります。このように、<ApiLink type="ForOfState" member="chunkSize" /> および <ApiLink type="ForOfState" member="chunkSize" /> プロパティを使用して、ユーザーが上にスクロールしているため取得済みのデータを表示する必要があるのか、下にスクロールしているためエンドポイントからさらにデータを取得する必要があるのかを判断できます。

最初に行うことは、データの最初のチャンクを取得することです。<ApiLink type="{ComponentName}" member="totalItemCount" /> プロパティを設定することは重要です。これにより、グリッドがスクロールバーのサイズを正しく設定できるようになります。

```typescript
public ngAfterViewInit() {

        if (request.data) {
            this.grid.totalItemCount = this.page * this.pageSize;
            this.grid.data = this._remoteService.getCachedData({startIndex: 0, chunkSize: 10});
            this.totalItems = request.data['@odata.count'];
```

```typescript
public handlePreLoad() {
    const isLastChunk = this.grid.totalItemCount ===
                        this.grid.virtualizationState.startIndex + this.grid.virtualizationState.chunkSize;
    // when last chunk reached load another page of data
    if (isLastChunk) {
        if (this.totalPageCount === this.page) {
            this.grid.data = this._remoteService.getCachedData(this.grid.virtualizationState);
            return;
        }
        this.page++;
        this.grid.isLoading = true;
        this._remoteService.loadDataForPage(this.page, this.pageSize, (request) => {
            if (request.data) {
                this.grid.totalItemCount = Math.min(this.page * this.pageSize, this.totalItems);
                this.grid.data = this._remoteService.getCachedData(this.grid.virtualizationState);
                this.grid.isLoading = false;
            }
        });
    } else {
        this.grid.data = this._remoteService.getCachedData(this.grid.virtualizationState);
    }
}
```

```razor
            int chunkSize = (int)e.Detail.ChunkSize;
```

## リモート ソート/フィルタリング

リモート ソートとフィルタリングを提供するには、受け取った引数に基づいて適切な要求を行うために、<ApiLink type="{ComponentName}" member="dataPreLoad" />、<ApiLink type="{ComponentName}" member="sortingExpressionsChange" />、<ApiLink type="{ComponentName}" member="filteringExpressionsTreeChange" /> 出力をサブスクライブし、サービスから取得した該当情報を使用してパブリック <ApiLink type="{ComponentName}" /> プロパティ <ApiLink type="{ComponentName}" member="totalItemCount" /> を設定する必要があります。

また、**rxjs** の `debounceTime` 関数も利用します。この関数は、ソース Observable から別のソース発行がないまま特定の時間が経過した後にのみ、ソース Observable から値を発行します。これにより、ユーザーが中断しないまま指定された時間が経過した場合にのみ、リモート操作がトリガーされます。

```typescript
const DEBOUNCE_TIME = 300;
public ngAfterViewInit() {
    this.grid.dataPreLoad.pipe(
        debounceTime(DEBOUNCE_TIME),
        takeUntil(this.destroy$)
    ).subscribe(() => {
        this.processData();
    });

    this.grid.filteringExpressionsTreeChange.pipe(
        debounceTime(DEBOUNCE_TIME),
        takeUntil(this.destroy$)
    ).subscribe(() => {
        this.processData(true);
    });

    this.grid.sortingExpressionsChange.pipe(
        debounceTime(DEBOUNCE_TIME),
        takeUntil(this.destroy$)
    ).subscribe(() => {
        this.processData();
    });
}
```

## 一意の列値ストラテジ

Excel スタイル フィルタリング ダイアログ内のリスト項目は、対応する列の一意の値を表します。<ApiLink type="{ComponentName}" /> は既定で、データ ソースに基づいてこれらの値を生成します。リモート フィルタリングの場合、グリッド データにはサーバーからのすべてのデータが含まれていません。一意の値を手動で提供し、必要に応じて読み込むために、<ApiLink type="{ComponentName}" /> の <ApiLink type="{ComponentName}" member="uniqueColumnValuesStrategy" label="UniqueColumnValuesStrategy" /> 入力を利用できます。この入力は実際には、3 つの引数を提供するメソッドです。

- <ApiLink type="Column" />  - 対応する列インスタンス。
- <ApiLink type="FilteringExpressionsTree" /> - 対応する列に基づいて縮小されたフィルタリング式ツリー。
- `Done` - サーバーから取得されたときに、新しく生成された列値を指定して呼び出す必要があるコールバック。

開発者は、<ApiLink type="Column" /> および <ApiLink type="FilteringExpressionsTree" /> 引数によって提供される情報に基づいて、必要な一意の列値を手動で生成し、その後 `Done` コールバックを呼び出すことができます。

<DocsAside type="info">

`UniqueColumnValuesStrategy` 入力が提供されている場合、Excel スタイル フィルタリングでの既定の一意値生成プロセスは使用されません。

</DocsAside>

## リモート ページング

<ComponentBlock for="Grid">

ページング機能はリモート データで動作できます。これを示すために、まずデータ取得を担当するサービスを宣言しましょう。ページ数を計算するために、すべてのデータ項目の数が必要になります。このロジックをサービスに追加します。

<PlatformBlock for="Angular">

```typescript
@Injectable()
export class RemotePagingService {
    public remoteData: BehaviorSubject<any[]>;
    public dataLenght: BehaviorSubject<number> = new BehaviorSubject(0);
    public url = 'https://www.igniteui.com/api/products';

    constructor(private http: HttpClient) {
        this.remoteData = new BehaviorSubject([]) as any;
    }

    public getData(index?: number, perPage?: number): any {
        let qS = '';

        if (perPage) {
            qS = `?$skip=${index}&$top=${perPage}&$count=true`;
        }

        this.http
            .get(`${this.url + qS}`).pipe(
                map((data: any) => data)
            ).subscribe((data) => this.remoteData.next(data));
    }

    public getDataLength(): any {
        return this.http.get(this.url).pipe(
            map((data: any) => data.length)
        );
    }
}

```

```ts
export class RemotePagingService {
    public static CUSTOMERS_URL = `https://data-northwind.indigo.design/Customers/GetCustomersWithPage`;
    constructor() {}

    public static getDataWithPaging(pageIndex?: number, pageSize?: number) {
        return fetch(RemotePagingService.buildUrl(RemotePagingService.CUSTOMERS_URL, pageIndex, pageSize))
        .then((result) => result.json())
        .catch((error) => console.error(error.message));
    }

    private static buildUrl(baseUrl: string, pageIndex?: number, pageSize?: number) {
        let qS = "";
        if (baseUrl) {
```

```typescript
    public dataLenght: BehaviorSubject<number> = new BehaviorSubject(0);

    constructor(private http: HttpClient) {
        this.remoteData = new BehaviorSubject([]) as any;

    public getData(index?: number, perPage?: number): any {
        let qS = '';

        if (perPage) {
            qS = `?$skip=${index}&$top=${perPage}&$count=true`;
        }

        this.http
            .get(`${this.url + qS}`).pipe(
                map((data: any) => data)
            ).subscribe((data) => this.remoteData.next(data));
    }

    public getDataLength(): any {
        return this.http.get(this.url).pipe(
            map((data: any) => data.length)
        );
    }
```

### 既定のテンプレートを使用したリモート ページング

既定のページング テンプレートを使用する場合は、Paginator の <ApiLink type="GridBaseDirective" member="totalRecords" label="TotalRecords" /> プロパティを設定する必要があります。そうすることで初めて、グリッドはリモート レコードの合計数に基づいて総ページ数を計算できるようになります。リモート ページネーションを実行する場合、Paginator は現在のページのデータのみを Grid に渡すため、グリッドは提供されたデータ ソースをページネーションしようとしません。そのため、Grid の <ApiLink type="GridBaseDirective" member="pagingMode" label="PagingMode" /> プロパティを <ApiLink type="GridPagingMode" member="Remote" label="GridPagingMode.Remote" /> に設定する必要があります。また、リモート サービスからデータを取得するには、`PagingDone` または `PerPageChange` イベントのいずれかをサブスクライブする必要があります。どちらのイベントを使用するかはユース ケースによって異なります。

以下に、独自の `next` および `previous` ページ アクションを実装するために定義したメソッドを示します。

```typescript
@ViewChild('grid1', { static: true }) public grid1: IgxGridComponent;

public ngAfterViewInit() {
    this.grid1.isLoading = true;
    this.remoteService.getData(0, this.perPage);
}

public nextPage() {
    this.firstPage = false;
    this.page++;
    const skip = this.page * this.perPage;
    const top = this.perPage;
    this.remoteService.getData(skip, top);
    if (this.page + 1 >= this.totalPages) {
        this.lastPage = true;
    }
    this.setNumberOfPagingItems(this.page, this.totalPages);
}

public previousPage() {
    this.lastPage = false;
    this.page--;
    const skip = this.page * this.perPage;
    const top = this.perPage;
    this.remoteService.getData(skip, top);
    if (this.page <= 0) {
        this.firstPage = true;
    }
    this.setNumberOfPagingItems(this.page, this.totalPages);
}

public paginate(page: number, recalculate = false) {
    this.page = page;
    const skip = this.page * this.perPage;
    const top = this.perPage;
    if (recalculate) {
        this.totalPages = Math.ceil(this.totalCount / this.perPage);
    }
    this.setNumberOfPagingItems(this.page, this.totalPages);
    this.remoteService.getData(skip, top);
    this.buttonDeselection(this.page, this.totalPages);
}
```

### バッチ編集を使用したリモート ページング

ここまでの例で、リモート データを使用して <ApiLink type="{ComponentName}" /> を設定する方法を明確にしました。次に、[バッチ編集のトピック/ガイド](batch-editing.md) に従って、グリッドでバッチ編集を有効にすることに焦点を当てます。

サンプルを続ける前に、現在のユース ケースを明確にしておくとよいでしょう。ページネーションがサーバー上で行われる場合、グリッドには現在のページのデータのみが含まれます。新しい行を追加すると、新しく追加された行（バッチ編集によるもの）は、グリッドが保持している現在のデータと連結されます。したがって、サーバーが特定のページに対してデータを返さない場合、グリッドのデータ ソースは新しく追加された行のみで構成され、グリッドは定義されたページネーション設定（page、perPage）に基づいてそれらをページネーションします。

```typescript
public ngOnInit() {
    this._dataLengthSubscriber = this.remoteService.getDataLength().subscribe((data) => {
        this.totalCount = data;
        this._recordOnServer = data;
        this._totalPagesOnServer = Math.floor(this.totalCount / this.perPage);
        this.grid1.isLoading = false;
    });
}
```

このユース ケースを適切に処理するには、いくつかのカスタム ロジックを実装する必要があります。

まず、サーバー上にあるレコードの総数を把握する必要があります。それに基づいて、サーバー上のデータ ページの総数を計算し、その値に基づいてカスタム ページネーション ロジックを実装します。

```typescript
public paginate(page: number) {
    this.grid1.endEdit(true);
    if (page > this._totalPagesOnServer) {
        if (this.page !== this._totalPagesOnServer) {
            const skipEl = this._totalPagesOnServer * this.perPage;
            this.remoteService.getData(skipEl, this.perPage);
        }
        this.page = page - this._totalPagesOnServer;
        this.page = page;
        return;
    } else {
        this.page = 0;
    }
    this.page = page;
    const skip = this.page * this.perPage;
    this.remoteService.getData(skip, this.perPage);
}
```
