# {Platform} {ComponentTitle} フィルタリング

{Platform} {ComponentTitle} の {ProductName} フィルタリングは、特定の条件または基準に基づいてデータを選択的に表示または非表示にできる機能です。<ApiLink type="{ComponentName}" /> コンポーネントが豊富なフィルタリング API とすべてのフィルタリング機能を提供する、バインドされたデータ コンテナーがあります。ここで利用可能なフィルタリングの種類は 3 つです。

- クイック フィルタリング
- [Excel スタイル フィルタリング](excel-style-filtering.md)
- [高度なフィルタリング](advanced-filtering.md)

## セットアップ

フィルタリングを有効にするかどうか、および使用するフィルタリング モードを指定するために、<ApiLink type="{ComponentName}" /> は次のプロパティを公開します - <ApiLink type="{ComponentName}" member="allowFiltering" />、<ApiLink type="{ComponentName}" member="allowAdvancedFiltering" />、<ApiLink type="{ComponentName}" member="filterMode" />、および <ApiLink type="Column" member="filterable" />。

プロパティ <ApiLink type="{ComponentName}" member="allowFiltering" /> では、次のオプションを指定できます。

- **false** - 対応するグリッドのフィルタリングは無効になります。これがデフォルト値です。
- **true** - 対応するグリッドのフィルタリングは有効になります。

プロパティ <ApiLink type="{ComponentName}" member="allowAdvancedFiltering" /> では、次のオプションを指定できます。

- **false** - 対応するグリッドの高度なフィルタリングは無効になります。これがデフォルト値です。
- **true** - 対応するグリッドの高度なフィルタリングは有効になります。

プロパティ <ApiLink type="{ComponentName}" member="filterMode" /> では、次のオプションを指定できます。

- **QuickFilter** - シンプルなフィルタリング UI。これがデフォルト値です。
- **ExcelStyleFilter** - Excel のようなフィルタリング UI。

プロパティ <ApiLink type="Column" member="filterable" /> では、次のオプションを指定できます。

- **true** - 対応する列のフィルタリングは有効になります。これがデフォルト値です。
- **false** - 対応する列のフィルタリングは無効になります。

## 操作

特定の列のフィルター行を開くには、そのヘッダーの下にある「フィルター」チップをクリックします。条件を追加するには、入力の左側にあるドロップダウンを使用してフィルター オペランドを選択し、値を入力します。**number** および **date** 列では既定で「Equals」が選択され、**string** では「Contains」、**boolean** では「All」が選択されます。「Enter」を押すと条件が確定され、別の条件を追加できるようになります。「condition」チップの間にはドロップダウンがあり、それらの間の論理演算子を決定します。既定では「AND」が選択されます。条件を削除するにはチップの「X」ボタンをクリックします。編集するにはチップを選択すると、入力にチップのデータが入力されます。フィルター行が開いている間は、フィルタリング可能な任意の列のヘッダーをクリックして選択し、その列のフィルター条件を追加できるようにできます。

列にいくつかのフィルタリング条件が適用され、フィルター行が閉じている場合は、チップの閉じるボタンをクリックして条件を削除するか、いずれかのチップを選択してフィルター行を開くことができます。すべての条件を表示する十分なスペースがない場合、フィルター アイコンが表示され、さらにいくつ条件があるかを示すバッジが付きます。これをクリックしてフィルター行を開くこともできます。

## 使用方法

標準のフィルタリング条件と同様に、既定のフィルタリング ストラテジがすぐに使用できる状態で提供されており、開発者はそれを独自の実装に置き換えることができます。さらに、独自のカスタム フィルタリング条件を簡単に組み込む方法も提供されています。<ApiLink type="{ComponentName}" /> は現在、シンプルなフィルタリング UI だけでなく、より複雑なフィルタリング オプションも提供しています。列に設定された <ApiLink type="Column" member="dataType" /> に応じて、正しい **フィルタリング操作** のセットがフィルター UI ドロップダウン内に読み込まれます。さらに、<ApiLink type="FilteringExpression" member="ignoreCase" label="IgnoreCase" /> と初期 <ApiLink type="BooleanFilteringOperand" member="condition" label="Condition" /> プロパティを設定できます。

<ApiLink type="{ComponentName}" /> コンポーネントのフィルタリング機能は、<ApiLink type="{ComponentName}" member="allowFiltering" /> 入力を **true** に設定することで有効になります。既定の <ApiLink type="{ComponentName}" member="filterMode" /> は `QuickFilter` であり、実行時に変更することは**できません**。特定の列でこの機能を無効にするには、<ApiLink type="Column" member="filterable" /> 入力を **false** に設定します。

<ApiLink type="{ComponentName}" /> API を使用して、任意の列または列の組み合わせをフィルタリングできます。<ApiLink type="{ComponentName}" /> は、このタスクのためにいくつかのメソッドを公開しています - <ApiLink type="Filter" />、`FilterGlobal`、および `ClearFilter`。

- <ApiLink type="Filter" /> - 単一の列または列の組み合わせをフィルタリングします。

公開されているフィルタリング オペランド クラスは 5 つあります。

- <ApiLink type="FilteringOperand" />: これは基本のフィルタリング オペランドで、カスタム フィルタリング条件を定義する際に継承できます。
- <ApiLink type="BooleanFilteringOperand" /> は、**boolean** 型のすべての既定のフィルタリング条件を定義します。
- <ApiLink type="NumberFilteringOperand" /> は、**numeric** 型のすべての既定のフィルタリング条件を定義します。
- <ApiLink type="StringFilteringOperand" /> は、**string** 型のすべての既定のフィルタリング条件を定義します。
- <ApiLink type="DateFilteringOperand" /> は、**date** 型のすべての既定のフィルタリング条件を定義します。

```typescript
// Single column filtering

// Filter the `ProductName` column for values which `contains` the `myproduct` substring, ignoring case
this.grid.filter('ProductName', 'myproduct', IgcStringFilteringOperand.instance().condition('contains'), true);
```

必須パラメーターは、列フィールド キーとフィルタリング語句のみです。条件と大文字小文字の区別は、指定されていない場合、列プロパティから推論されます。複数フィルタリングの場合、このメソッドはフィルタリング式の配列を受け取ります。

## カスタム フィルタリング オペランド

フィルタリング オペランドを追加、削除、または変更することで、フィルタリング メニューをカスタマイズできます。既定では、フィルタリング メニューには列のデータ型 (<ApiLink type="BooleanFilteringOperand" />、<ApiLink type="DateFilteringOperand" />、<ApiLink type="NumberFilteringOperand" />、および <ApiLink type="StringFilteringOperand" />) に基づく特定のオペランドが含まれています。これらのクラスまたはそれらの基本クラス <ApiLink type="FilteringOperand" /> を拡張して、フィルタリング メニュー項目の動作を変更できます。

以下のサンプルでは、「Product Name」列と「Discontinued」列のフィルター メニューを確認してください。「Discontinued」列フィルターでは、オペランドの数を All、True、False に制限しています。「Product Name」列フィルターでは、Contains および Does Not Contain オペランドのロジックを変更して大文字小文字を区別する検索を実行し、さらに Empty および Not Empty オペランドも追加しています。

これを行うには、<ApiLink type="StringFilteringOperand" /> と <ApiLink type="BooleanFilteringOperand" /> を拡張し、操作とそのロジックを変更し、列の `filters` 入力を新しいオペランドに設定します。

<PlatformBlock for="Angular">

## スタイル設定

フィルタリング行のスタイル設定を開始するには、すべてのテーマ関数とコンポーネント mixin が存在する `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

最もシンプルなアプローチに従い、`grid-theme` を拡張し、`$filtering-row-text-color`、`$filtering-row-background`、`$filtering-header-text-color`、および `$filtering-header-background` パラメーターを受け取る新しいテーマを作成します。

```scss
$custom-grid: grid-theme(
    $filtering-row-text-color: #292826,
    $filtering-row-background: #FFCD0F,
    $filtering-header-text-color: #292826,
    $filtering-header-background: #FFCD0F
);
```

ご覧のとおり、`grid-theme` はフィルタリング行と、フィルタリングされている対応する列ヘッダーの色のみを制御します。フィルタリング行内には、入力グループ、チップ、ボタンなど、明らかにさらに多くのコンポーネントがあります。それらをスタイル設定するには、それぞれに個別のテーマを作成する必要があるため、新しい入力グループ テーマと新しいボタン テーマを作成します。

```scss
$dark-input-group: input-group-theme(
    $box-background: #FFCD0F,
    $idle-text-color: #292826,
    $focused-text-color: #292826,
    $filled-text-color: #292826
);

$dark-button: button-theme(
    $flat-background: #FFCD0F,
    $flat-text-color: #292826,
    $flat-hover-background: #292826,
    $flat-hover-text-color: #FFCD0F
);
```

この例では、入力グループとボタンの一部のパラメーターのみを変更しましたが、`input-group-theme` と `button-theme` は、それぞれのスタイル設定を制御するためのはるかに多くのパラメーターを提供します。

最後の手順は、コンポーネント mixin を、それぞれのテーマとともに**インクルード**することです。入力のプレースホルダーの color プロパティも設定します。

```scss
@include grid($custom-grid);
.igx-grid__filtering-row {
    @include button($dark-button);
    @include input-group($dark-input-group);

    .igx-input-group__input::placeholder {
        color: #FFCD0F;
    }
}
```

<DocsAside type="info">

**igx-button** と **igx-input-group** の mixin を `.igx-grid__filtering-row` 内にスコープ設定しているため、フィルタリング行のボタンとその入力グループのみがスタイル設定されます。そうしないと、グリッド内の他のボタンや入力グループにも影響します。

</DocsAside>

 <DocsAside type="info">

コンポーネントが `Emulated` ViewEncapsulation を使用している場合、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

 </DocsAside>

```scss
:host {
     ::ng-deep {
        @include grid($custom-grid);
        .igx-grid__filtering-row {
            @include button($dark-button);
            @include input-group($dark-input-group);

            .igx-input-group__input::placeholder {
                color: #FFCD0F;
            }
        }
    }
}
```

### カラー パレットの定義

先ほどのように色の値をハードコードする代わりに、`igx-palette` 関数と `igx-color` 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します。

```scss
$yellow-color: #FFCD0F;
$black-color: #292826;

$dark-palette: palette($primary: $black-color, $secondary: $yellow-color);
```

そして `igx-color` を使用すると、パレットから色を簡単に取得できます。

```scss
$custom-grid: grid-theme(
    $filtering-row-text-color: color($dark-palette, "primary", 400),
    $filtering-row-background: color($dark-palette, "secondary", 400),
    $filtering-header-text-color: color($dark-palette, "primary", 400),
    $filtering-header-background: color($dark-palette, "secondary", 400)
);

$dark-input-group: input-group-theme(
    $box-background: color($dark-palette, "secondary", 400),
    $idle-text-color: color($dark-palette, "primary", 400),
    $focused-text-color: color($dark-palette, "primary", 400),
    $filled-text-color: color($dark-palette, "primary", 400)
);

$dark-button: button-theme(
    $flat-background: color($dark-palette, "secondary", 400),
    $flat-text-color: color($dark-palette, "primary", 400),
    $flat-hover-background: color($dark-palette, "primary", 400),
    $flat-hover-text-color: color($dark-palette, "secondary", 400)
);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色を生成および取得するための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/sass/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[スキーマ](../themes/sass/schemas.md)の恩恵を受ける堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

すべてのコンポーネントに対して提供される 2 つの定義済みスキーマのいずれかを拡張します。この場合は、`light-grid`、`light-input-group`、および `light-button` スキーマです。

```scss
// Extending the light grid schema
$custom-grid-schema: extend($_light-grid,
    (
        filtering-row-text-color:(
           color: ("primary", 400)
        ),
        filtering-row-background:(
           color: ("secondary", 400)
        ),
        filtering-header-text-color:(
           color: ("primary", 400)
        ),
        filtering-header-background:(
           color: ("secondary", 400)
        )
    )
);

// Extending the light input group schema
$custom-input-group-schema: extend($_light-input-group,
    (
        box-background:(
           color: ("secondary", 400)
        ),
        idle-text-color:(
           color: ("primary", 400)
        ),
        focused-text-color:(
           color: ("primary", 400)
        ),
        filled-text-color:(
           color: ("primary", 400)
        )
    )
);

// Extending the light button schema
$custom-button-schema: extend($_light-button,
    (
        flat-background:(
           color: ("secondary", 400)
        ),
        flat-text-color:(
           color: ("primary", 400)
        ),
        flat-hover-background:(
           color: ("primary", 400)
        ),
        flat-hover-text-color:(
           color: ("secondary", 400)
        )
    )
);
```

カスタム スキーマを適用するには、グローバルのいずれか (`light` または `dark`) を**拡張**する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを指定するもので、その後それを対応するコンポーネント テーマに追加します。

```scss
// Extending the global light-schema
$custom-light-schema: extend($light-schema,(
    igx-grid: $custom-grid-schema,
    igx-input-group: $custom-input-group-schema,
    igx-button: $custom-button-schema
));

// Defining grid-theme with the global light schema
$custom-grid: grid-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);

// Defining button-theme with the global light schema
$custom-button: button-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);

// Defining input-group-theme with the global light schema
$custom-input-group: input-group-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);
```

上記で示したのと同じ方法でテーマをインクルードすることを忘れないでください。

### 6.1.0 の重大な変更

- <ApiLink type="{ComponentName}" /> `filteringExpressions` プロパティは削除されました。代わりに <ApiLink type="FilteringExpressionsTree" /> を使用してください。
- `filter_multiple` メソッドは削除されました。代わりに <ApiLink type="Filter" /> メソッドと <ApiLink type="FilteringExpressionsTree" /> プロパティを使用してください。
- <ApiLink type="Filter" /> メソッドには新しいシグネチャがあります。現在は次のパラメーターを受け取ります。
  - <ApiLink type="Column" member="name" label="Name" /> - フィルタリングする列の名前。
  - `Value` - フィルタリングに使用する値。
  - `ConditionOrExpressionTree` (オプション) - このパラメーターは <ApiLink type="FilteringOperation" /> または <ApiLink type="FilteringExpressionsTree" /> 型のオブジェクトを受け取ります。単純なフィルタリングのみが必要な場合は、フィルタリング操作を引数として渡すことができます。高度なフィルタリングの場合は、複雑なフィルタリング ロジックを含む式ツリーを引数として渡すことができます。
  - <ApiLink type="FilteringExpression" member="ignoreCase" label="IgnoreCase" /> (オプション) - フィルタリングで大文字小文字を区別するかどうか。
- `FilteringDone` イベントは現在、フィルタリングされた列のフィルタリング状態を含む <ApiLink type="FilteringExpressionsTree" /> 型のパラメーターを 1 つだけ持ちます。
- フィルタリング オペランド: <ApiLink type="FilteringExpression" /> 条件プロパティは、フィルタリング条件メソッドへの直接参照ではなくなり、代わりに <ApiLink type="FilteringOperation" /> への参照になりました。
- <ApiLink type="ColumnComponent" /> は現在、<ApiLink type="FilteringOperand" /> クラス参照を受け取る <ApiLink type="Column" member="filters" label="Filters" /> プロパティを公開しています。
- カスタム フィルターは、<ApiLink type="FilteringOperand" /> の <ApiLink type="BooleanFilteringOperand" member="operations" label="Operations" /> プロパティに <ApiLink type="FilteringOperation" /> 型の操作を設定することで、{ComponentTitle} 列に提供できます。
