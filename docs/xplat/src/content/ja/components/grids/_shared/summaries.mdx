# {Platform} {ComponentTitle} 集計

{Platform} {ComponentTitle} の {ProductName} 集計機能は、列ごとのレベルでグループ フッターとして機能します。{Platform} {ComponentName} 集計は、列内のデータ型に応じた事前定義済みの既定の集計項目セット、または <ApiLink type="{ComponentName}" /> でカスタム テンプレートを実装することによって、ユーザーが列情報を別のコンテナーで確認できる強力な機能です。

## {Platform} {ComponentTitle} 集計の概要例

<Sample src="/{ComponentSample}/data-summary-options" height={650} alt="{Platform} {ComponentTitle} data summary options" />

<DocsAside type="info">

フィルタリングが適用されていない限り、列の集計は **すべての列値の関数**です。フィルタリングが適用されている場合、列の集計は **フィルタリングされた結果値の関数**になります。

</DocsAside>

<ApiLink type="{ComponentName}" /> 集計は、{ProductName} で列ごとのレベルでも有効にできます。つまり、必要な列に対してのみ有効化できます。<ApiLink type="{ComponentName}" /> 集計は、列内のデータ型に応じて、事前定義済みの既定の集計セットを提供するため、時間を節約できます。

`string` および `boolean` <ApiLink type="Column" member="dataType" /> の場合、次の関数を使用できます。

- カウント

`number`、`currency`、`percent` データ型の場合、次の関数を使用できます。

- カウント
- 最小
- 最大
- 平均
- 合計

`date` データ型の場合、次の関数を使用できます。

- カウント
- 最も早い
- 最も遅い

使用可能なすべての列データ型は、公式の [列型トピック](column-types.md#default-template) で確認できます。

<ApiLink type="{ComponentName}" /> 集計は、<ApiLink type="ColumnState" member="hasSummary" /> プロパティを **true** に設定することで列ごとに有効化されます。また、各列の集計は列データ型に従って解決されることに注意することも重要です。<ApiLink type="{ComponentName}" /> では既定の列データ型は `string` であるため、`number` または `date` 固有の集計が必要な場合は、<ApiLink type="Column" member="dataType" /> プロパティを `number` または `date` として指定する必要があります。集計値は、グリッドの <ApiLink type="GridBaseDirective" member="locale" label="Locale" /> と列の <ApiLink type="Column" member="pipeArgs" /> に従ってローカライズされて表示されることに注意してください。

## カスタム {ComponentTitle} 集計

これらの関数が要件を満たさない場合は、特定の列に対してカスタム集計を提供できます。

<PlatformBlock for="WebComponents">

これを実現するには、列データ型と要件に応じて、基本クラス <ApiLink type="SummaryOperand" />、<ApiLink type="NumberSummaryOperand" />、または <ApiLink type="DateSummaryOperand" /> のいずれかをオーバーライドする必要があります。この方法で、既存の関数を再定義したり、新しい関数を追加したりできます。<ApiLink type="SummaryOperand" /> クラスは、<ApiLink type="SummaryOperand" member="count" label="Count" /> メソッドに対する既定の実装のみを提供します。<ApiLink type="NumberSummaryOperand" /> は <ApiLink type="SummaryOperand" /> を拡張し、<ApiLink type="NumberSummaryOperand" member="min" label="Min" />、<ApiLink type="NumberSummaryOperand" member="max" label="Max" />、<ApiLink type="NumberSummaryOperand" member="sum" label="Sum" />、<ApiLink type="NumberSummaryOperand" member="average" label="Average" /> の実装を提供します。<ApiLink type="DateSummaryOperand" /> は <ApiLink type="SummaryOperand" /> を拡張し、さらに <ApiLink type="DateSummaryOperand" member="earliest" label="Earliest" /> と <ApiLink type="DateSummaryOperand" member="latest" label="Latest" /> を提供します。

</PlatformBlock>

<PlatformBlock for="Angular">


## 集計の書式設定

既定では、組み込みの集計オペランドによって生成される集計結果は、グリッドの <ApiLink type="GridBaseDirective" member="locale" label="Locale" /> と列の <ApiLink type="Column" member="pipeArgs" /> に従ってローカライズおよび書式設定されます。カスタム オペランドを使用する場合、<ApiLink type="GridBaseDirective" member="locale" label="Locale" /> と <ApiLink type="Column" member="pipeArgs" /> は適用されません。集計結果の既定の外観を変更したい場合は、<ApiLink type="Column" member="summaryFormatter" label="SummaryFormatter" /> プロパティを使用して書式設定できます。

```typescript
public dateSummaryFormat(summary: IgxSummaryResult, summaryOperand: IgxSummaryOperand): string {
    const result = summary.summaryResult;
    if (summaryOperand instanceof IgxDateSummaryOperand && summary.key !== 'count'
        && result !== null && result !== undefined) {
        const pipe = new DatePipe('en-US');
        return pipe.transform(result,'MMM YYYY');
    }
    return result;
}
```


## グループ化での集計

列でグループ化している場合、<ApiLink type="{ComponentName}" /> を使用すると、<ApiLink type="{ComponentName}" member="summaryCalculationMode" label="SummaryCalculationMode" /> プロパティと <ApiLink type="{ComponentName}" member="summaryPosition" label="SummaryPosition" /> プロパティを使用して、集計の位置と計算モードを変更できます。これら 2 つのプロパティに加えて、<ApiLink type="{ComponentName}" /> は <ApiLink type="{ComponentName}" member="showSummaryOnCollapse" label="ShowSummaryOnCollapse" /> プロパティを公開しており、参照先のグループ行が折りたたまれたときに集計行を表示したままにするかどうかを決定できます。

<ApiLink type="{ComponentName}" member="summaryCalculationMode" label="SummaryCalculationMode" /> プロパティで使用可能な値は次のとおりです。

- <ApiLink type="GridSummaryCalculationMode" label="RootLevelOnly" /> - 集計はルート レベルに対してのみ計算されます。
- <ApiLink type="GridSummaryCalculationMode" label="ChildLevelsOnly" /> - 集計は子レベルに対してのみ計算されます。
- <ApiLink type="GridSummaryCalculationMode" label="RootAndChildLevels" /> - 集計はルート レベルと子レベルの両方に対して計算されます。これが既定値です。

<ApiLink type="{ComponentName}" member="summaryPosition" label="SummaryPosition" /> プロパティで使用可能な値は次のとおりです。

- <ApiLink type="GridSummaryPosition" label="Top" /> - 集計行はグループ化行の子の前に表示されます。
- <ApiLink type="GridSummaryPosition" label="Bottom" /> - 集計行はグループ化行の子の後に表示されます。これが既定値です。

<ApiLink type="{ComponentName}" member="showSummaryOnCollapse" label="ShowSummaryOnCollapse" /> プロパティは boolean です。既定値は **false** に設定されており、グループ行が折りたたまれると集計行が非表示になることを意味します。このプロパティが **true** に設定されている場合、グループ行が折りたたまれても集計行は表示されたままになります。

<DocsAside type="info">

<ApiLink type="{ComponentName}" member="summaryPosition" label="SummaryPosition" /> プロパティは、子レベルの集計にのみ適用されます。ルート レベルの集計は、常に <ApiLink type="{ComponentName}" /> の下部に固定表示されます。

</DocsAside>

## スタイル設定

ソート動作のスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが含まれる `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

最も簡単な方法として、<ApiLink type="function-grid-summary-theme" label="grid-summary-theme" kind="sass" module="index.html" /> を拡張し、`$background-color`、`$focus-background-color`、`$label-color`、`$result-color`、`$pinned-border-width`、`$pinned-border-style`、`$pinned-border-color` パラメーターを受け取る新しいテーマを作成します。

```scss
$custom-theme: grid-summary-theme(
    $background-color: #e0f3ff,
    $focus-background-color: rgba( #94d1f7, .3 ),
    $label-color: #e41c77,
    $result-color: black,
    $pinned-border-width: 2px,
    $pinned-border-style: dotted,
    $pinned-border-color: #e41c77
);
```

最後のステップは、コンポーネント ミックスインを **include** することです。

```scss
@include grid-summary($custom-theme);
```

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合は、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

</DocsAside>

```scss
:host {
   ::ng-deep {
       @include grid-summary($custom-theme);
   }
}
```

### カラー パレットの定義

先ほどのように色の値をハードコーディングする代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します。

```scss
$blue-color: #7793b1;
$green-color: #00ff2d;

$my-custom-palette: palette($primary: $blue-color, $secondary: $green-color);
```

その後、<ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> を使用してパレットから色を簡単に取得できます。

```scss
$custom-theme: grid-summary-theme(
    $background-color: color($my-custom-palette, "primary", 700),
    $focus-background-color: color($my-custom-palette, "primary", 800),
    $label-color: color($my-custom-palette, "secondary", 500),
    $result-color: color($my-custom-palette, "grays", 900),
    $pinned-border-width: 2px,
    $pinned-border-style: dotted,
    $pinned-border-color: color($my-custom-palette, "secondary", 500)
);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色の生成と取得のための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[**スキーマ**](../themes/sass/schemas.md) の利点を活かした堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

すべてのコンポーネントに提供されている 2 つの事前定義済みスキーマのうち、この場合は <ApiLink type="variable-_light-grid-summary" label="_light-grid-summary" kind="sass" module="index.html" /> を拡張します。

```scss
// Extending the light grid summary schema
$my-summary-schema: extend($_light-grid-summary,
    (
        background-color: (igx-color: ('primary', 700)),
        focus-background-color: (igx-color: ('primary', 800)),
        label-color: (igx-color: ('secondary', 500)),
        result-color: (igx-color: ('grays', 900)),
        pinned-border-width: 2px,
        pinned-border-style: dotted,
        pinned-border-color: (igx-color: ('secondary', 500))
    )
);
```

カスタム スキーマを適用するには、グローバルのいずれか（<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> または <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />）を **extend** する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを指し示し、その後、それを対応するコンポーネント テーマに追加します。

```scss
// Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid-summary: $my-summary-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: grid-summary-theme(
    $palette: $my-custom-palette,
    $schema: $my-custom-schema
);
```

上で示したのと同じ方法でテーマを含めることを忘れないでください。
