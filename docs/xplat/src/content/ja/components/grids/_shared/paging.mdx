# {Platform} {ComponentTitle} ページネーション概要

{Platform} {ComponentTitle} の {ProductName} ページネーション機能は、大規模なデータ セットを、類似したコンテンツを持つ一連のページに分割するために使用されます。React グリッドのページネーションは、ユーザー エクスペリエンスとデータ操作を向上させます。<ApiLink type="{ComponentName}" /> ページネーションは、列の追加と同様に、<ApiLink type="Paginator" /> タグを定義してグリッド ツリーに投影される別のコンポーネントを介して構成できます。他の {Platform} テーブルと同様に、{Platform} {ComponentTitle} のページネーションでは、カスタム ページ用のテンプレートがサポートされています。

## {Platform} でのページネーションのスタイル設定

Paginator のスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが含まれている `index` ファイルをインポートする必要があります:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

最も単純なアプローチに従い、`PaginatorTheme` を拡張し、`$text-color`、`$background-color`、および `$border-color` パラメーターを受け取る新しいテーマを作成します。

```scss
$dark-paginator: paginator-theme(
    $text-color: #F4D45C,
    $background-color: #575757,
    $border-color: #292826
);
```

ご覧のように、`PaginatorTheme` はページング コンテナーの色のみを制御し、ページャー UI のボタンには影響しません。これらのボタンのスタイルを設定するために、新しいボタン テーマを作成しましょう:

```scss
$dark-button: button-theme(
    $icon-color: #FFCD0F,
    $icon-hover-color: #292826,
    $icon-hover-background: #FFCD0F,
    $icon-focus-color: #292826,
    $icon-focus-background: #FFCD0F,
    $disabled-color: #16130C
);
```

この例では、アイコンの色と背景、およびボタンの無効状態の色のみを変更しましたが、`ButtonTheme` はボタン スタイルを制御するためのさらに多くのパラメーターを提供します。

最後の手順は、各コンポーネント ミックスインをそれぞれのテーマとともに **include** することです:

```scss
@include grid-paginator($dark-grid-paginator);
.igx-grid-paginator__pager {
    @include button($dark-button);
}
```

<DocsAside type="info">

Paginator ボタンのみにスタイルが適用されるように、<ApiLink pkg="core" type="Button" /> ミックスインを `.igx-paginator__pager` 内にスコープします。そうしないと、グリッド内の他のボタンにも影響します。

</DocsAside>

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合は、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります:

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include paginator($dark-paginator);
        .igx-paginator__pager {
            @include button($dark-button);
        }
    }
}
```

### カラー パレットの定義

先ほどのように色の値をハードコードする代わりに、`Palette` 関数と <ApiLink pkg="core" type="Color" /> 関数を使用することで、色に関してより高い柔軟性を実現できます。

`Palette` は、渡されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します:

```scss
$yellow-color: #F9D342;
$black-color: #292826;

$dark-palette: palette($primary: $black-color, $secondary: $yellow-color);
```

その後、<ApiLink pkg="core" type="Color" /> を使用してパレットから簡単に色を取得できます。

```scss
$dark-paginator: paginator-theme(
    $palette: $dark-palette,
    $text-color: color($dark-palette, "secondary", 400),
    $background-color: color($dark-palette, "primary", 200),
    $border-color: color($dark-palette, "primary", 500)
);

$dark-button: button-theme(
    $palette: $dark-palette,
    $icon-color: color($dark-palette, "secondary", 700),
    $icon-hover-color: color($dark-palette, "primary", 500),
    $icon-hover-background: color($dark-palette, "secondary", 500),
    $icon-focus-color: color($dark-palette, "primary", 500),
    $icon-focus-background: color($dark-palette, "secondary", 500),
    $disabled-color: color($dark-palette, "primary", 700)
);
```

<DocsAside type="info">

<ApiLink pkg="core" type="Color" /> と `Palette` は、色を生成および取得するための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/sass/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[スキーマ](../themes/sass/schemas.md) の利点を活かした堅牢で柔軟な構造を構築できます。スキーマはテーマのレシピです。

すべてのコンポーネントに対して提供されている 2 つの定義済みスキーマのいずれかを拡張します。この場合は、`DarkPagination` スキーマと `DarkButton` スキーマです:

```scss
// Extending the dark paginator schema
$dark-paginator-schema: extend($_dark-pagination,
        (
            text-color:(
               color: ("secondary", 400)
            ),
            background-color:(
               color: ("primary", 200)
            ),
            border-color:(
               color: ("primary", 500)
            )
        )
);
// Extending the dark button schema
$dark-button-schema: extend($_dark-button,
        (
            icon-color:(
               color: ("secondary", 700)
            ),
            icon-hover-color:(
               color: ("primary", 500)
            ),
            icon-hover-background:(
               color: ("secondary", 500)
            ),
            icon-focus-color:(
               color: ("primary", 500)
            ),
            icon-focus-background:(
               color: ("secondary", 500)
            ),
            disabled-color:(
               color: ("primary", 700)
            )
        )
);
```

カスタム スキーマを適用するには、グローバルの `Light` または `Dark` のいずれかを **extend** する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを示すもので、その後、対応するコンポーネント テーマに追加します:

```scss
// Extending the global dark-schema
$custom-dark-schema: extend($dark-schema,(
    igx-paginator: $dark-paginator-schema,
    igx-button: $dark-button-schema
));

// Definingpaginator-theme with the global dark schema
$dark-paginator: paginator-theme(
  $palette: $dark-palette,
  $schema: $custom-dark-schema
);

// Defining button-theme with the global dark schema
$dark-button: button-theme(
  $palette: $dark-palette,
  $schema: $custom-dark-schema
);
```

上記で示したのと同じ方法でテーマを含めることを忘れないでください。
