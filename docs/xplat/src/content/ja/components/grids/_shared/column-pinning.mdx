# {Platform} {ComponentTitle} 列のピン固定

{ProductName} の列のピン固定機能は、{Platform} {ComponentTitle} で開発者が特定の列を目的の順序で固定できるようにし、ユーザーが `{ComponentName}` を水平方向にスクロールしている場合でも常に表示されるようにします。列のピン固定には統合された UI があり、{Platform} {ComponentTitle} ツールバーからアクセスできます。さらに、開発者は列のピン状態を変更するカスタム ユーザー インターフェイスを構築する柔軟性も備えています。

## 列のピン固定 API

列のピン固定は、<ApiLink type="Column" /> の <ApiLink type="Column" member="pinned" /> プロパティを通じて制御されます。ピン固定された列は、既定では <ApiLink type="{ComponentName}" /> の左側に描画され、<ApiLink type="{ComponentName}" /> 本体内のピン固定されていない列を水平方向にスクロールしても固定されたままになります。

## ピン固定位置

<ApiLink type="Column" member="pinningPosition" /> 構成オプションを使用して、列のピン固定位置を変更できます。これにより、列の位置を Start または End のいずれかに設定できます。
End に設定すると、列はグリッドの末尾、ピン固定されていない列の後に描画されます。ピン固定されていない列は水平方向にスクロールできますが、ピン固定された列は右側に固定されたままになります。

## カスタム列ピン固定 UI

カスタム UI を定義し、関連する API を使用して列のピン状態を変更できます。

ツールバーの代わりに、エンド ユーザーがクリックして特定の列のピン状態を変更できるピン アイコンを列ヘッダーに定義したいとします。

これは、カスタム アイコンを使用して列のヘッダー テンプレートを作成することで実現できます。

// In JavaScript
igRegisterScript("WebGridPinHeaderTemplate", (ctx) => {
    var html = window.igTemplating.html;
    window.toggleColumnPin = function toggleColumnPin(field) {
        var grid = document.getElementsByTagName["igc-grid"](0);
        var col = grid.getColumnByName(field);
        col.pinned = !col.pinned;
        grid.markForCheck();
    }
    return html`<div>
    <span style="float:left">${ctx.column.field}</span>
    <span style="float:right" @pointerdown="${() => toggleColumnPin(ctx.column.field)}">📌</span>
</div>`;

## スタイル設定

<ApiLink type="{ComponentName}" /> は、[{ProductName} テーマ ライブラリ](../themes/styles.md)を通じたスタイル設定を可能にします。グリッドの <ApiLink type="function-grid-theme" label="theme" kind="sass" module="index.html" /> は、グリッドのすべての機能をカスタマイズできるさまざまなプロパティを公開します。

以下の手順では、グリッドのピン固定のスタイルをカスタマイズする手順を説明します。

### グローバル テーマのインポート

ピン固定機能のカスタマイズを開始するには、すべてのスタイル設定関数とミックスインが含まれる `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

### カスタム テーマの定義

次に、<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張し、ピン固定機能を目的どおりにカスタマイズするために必要なパラメーターを受け取る新しいテーマを作成します。

```scss
$custom-theme: grid-theme(
    /* Pinning properties that affect styling */
    $pinned-border-width: 5px,
    $pinned-border-style: double,
    $pinned-border-color: #FFCD0F,
    $cell-active-border-color: #FFCD0F
    /* add other features properties here... */
);
```

### カスタム カラー パレットの定義

上記で説明した方法では、色の値はハードコードされていました。代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用して、より高い柔軟性を実現できます。
`igx-palette` は、指定されたプライマリ カラーとセカンダリ カラーに基づいてカラー パレットを生成します。

```scss
$primary-color: #292826;
$secondary-color: #ffcd0f;

$custom-palette: palette(
 $primary: $primary-color,
 $secondary: $secondary-color
);
```

カスタム パレットが生成された後、`igx-color` 関数を使用して、プライマリ カラーとセカンダリ カラーのさまざまなバリエーションを取得できます。

```scss
$custom-theme: grid-theme(
    $pinned-border-width: 5px,
    $pinned-border-style: double,
    $pinned-border-color: color($custom-palette, "secondary", 500),
    $cell-active-border-color: color($custom-palette, "secondary", 500)
);
```

`$custom-theme` には前のセクションのものと同じプロパティが含まれていますが、今回は色がハードコードされていません。代わりに、カスタム `igx-palette` が使用され、指定されたカラー バリアントにより、そのプライマリ カラーとセカンダリ カラーから色が取得されています。

### カスタム スキーマの定義

さらに進めて、[**スキーマ**](../themes/sass/schemas.md)のすべての利点を備えた柔軟な構造を構築できます。**スキーマ**はテーマのレシピです。
各コンポーネントに提供されている 2 つの定義済みスキーマのいずれかを拡張します。この場合は、`$_light_grid` を使用します。

```scss
$custom-grid-schema: extend($_light-grid,(
    pinned-border-width: 5px,
    pinned-border-style: double,
    pinned-border-color: color: ("secondary", 500),
    cell-active-border-color: color: ("secondary", 500)
));
```

カスタム スキーマを適用するには、`light` または `dark` のグローバルのいずれかを拡張する必要があります。全体のプロセスは、実際にはコンポーネントにカスタム スキーマを指定し、その後それを対応するコンポーネント テーマに追加することです。

```scss
$my-custom-schema: extend($light-schema, (
    igx-grid: $custom-grid-schema
));
$custom-theme: grid-theme(
    $palette: $custom-palette,
    $schema: $my-custom-schema
);
```

### カスタム テーマの適用

テーマを適用する最も簡単な方法は、グローバル スタイル ファイルで `sass` `@include` ステートメントを使用することです。

```scss
@include grid($custom-theme);
```

### スコープ付きコンポーネント テーマ

カスタム テーマが特定のコンポーネントのみに影響するようにするには、グローバル スタイル ファイルで定義したすべてのスタイルをカスタム コンポーネントのスタイル ファイルに移動できます（`index` ファイルのインポートを含む）。

この方法では、Angular の [ViewEncapsulation](https://angular.io/api/core/Component#encapsulation) により、スタイルはカスタム コンポーネントのみに適用されます。

 <DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、グリッドにスタイルを適用するには、`::ng-deep` を使用してこのカプセル化を透過する必要があります。

 </DocsAside>

 <DocsAside type="info">

スタイルがコンポーネント外部の要素に影響しないように、ステートメントを `:host` セレクター内にラップします。

 </DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-theme);
    }
}
```
