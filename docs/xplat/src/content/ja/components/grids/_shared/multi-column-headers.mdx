# {Platform} {ComponentTitle} 複数列ヘッダーの概要

{Platform} {ComponentTitle} の {ProductName} 複数列ヘッダー機能を使用すると、列を共通の複数ヘッダーの下に配置してグループ化できます。<ApiLink type="{ComponentName}" /> 内の各複数列ヘッダー グループは、他のグループまたは列の組み合わせを表すことができます。この機能は、水平スクロールが煩雑になる可能性のある大規模なデータセットを扱う場合に特に便利です。

## {Platform} {ComponentTitle} 複数列ヘッダーの例

<Sample src="/{ComponentSample}/multi-column-headers-overview" height={550} alt="{Platform} {ComponentTitle} Multi Column Headers Overview Example" />

すべての <ApiLink type="ColumnGroup" /> は、[移動](column-moving.md)、[ピン固定](column-pinning.md)、および[非表示](column-hiding.md)をサポートします。

<DocsAside type="info">

列と列グループのセットがある場合、ピン固定は最上位の列親に対してのみ機能します。より具体的には、ネストされた列グループまたは列ごとのピン固定は許可されていません。<br />
列と列グループ間の移動は、それらが階層内で同じレベルにあり、両方が同じ `group` 内にある場合にのみ許可されます。<br />
`columns/column-groups` が現在の `group` にラップされていない場合、つまりそれらが**最上位**の `columns` である場合は、表示されている列全体の間で移動が許可されます。

</DocsAside>

<PlatformBlock for="Angular">

グリッドの各列グループは個別にテンプレート化できます。列グループには、`igxHeader` ディレクティブで装飾された `ng-template` タグが必要です。
`ng-template` には、コンテキストとして列グループ オブジェクトが提供されます。

```html

</igx-column-group>
```

```html
<ng-template #columnGroupHeaderTemplate let-columnGroup>

</igx-column-group>
```

## スタイル設定

並べ替え動作のスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが定義されている `index` ファイルをインポートする必要があります。

```scss
@import '~igniteui-angular/lib/core/styles/themes/index';
```

最もシンプルなアプローチに従い、<ApiLink type="function-igx-grid-theme" label="igx-grid-theme" kind="sass" module="index.html" /> を拡張し、`$header-background`、`$header-text-color`、`$header-border-width`、`$header-border-style`、および `$header-border-color` パラメーターを受け取る新しいテーマを作成します。

```scss
$custom-theme: igx-grid-theme(
    $header-background: #e0f3ff,
    $header-text-color: #e41c77,
    $header-border-width: 1px,
    $header-border-style: solid,
    $header-border-color: rgba(0, 0, 0, 0.08)
);
```

最後の手順は、コンポーネント ミックスインを**含める**ことです。

```scss
@include igx-grid($custom-theme);
```

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合は、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

</DocsAside>

```scss
:host {
   ::ng-deep {
       @include igx-grid($custom-theme);
   }
}
```

### カラー パレットの定義

先ほどのように色の値をハードコーディングする代わりに、`igx-palette` および `igx-color` 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ カラーとセカンダリ カラーに基づいてカラー パレットを生成します。

```scss
$light-blue-color: #e0f3ff;
$deep-pink-color: #e41c77;

$custom-palette: igx-palette($primary: $light-blue-color, $deep-pink-color);
```

そして、`igx-color` を使用すると、パレットから色を簡単に取得できます。

```scss
$custom-theme: igx-grid-theme(
    $header-background: igx-color($custom-palette, "primary", 500),
    $header-text-color: igx-color($custom-palette, "secondary", 500),
    $header-border-width: 1px,
    $header-border-style: solid,
    $header-border-color: igx-color($custom-palette, "grays", 200)
);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色を生成および取得するための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/palette.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[スキーマ](../themes/schemas.md) の利点を活かした堅牢で柔軟な構造を構築できます。スキーマはテーマのレシピです。

各コンポーネントに提供されている 2 つの定義済みスキーマのいずれかを拡張します。この場合は `_light-grid` です。

```scss
// Extending the light grid schema
$custom-grid-schema: extend($_light-grid,
    (
        header-background: (igx-color:('primary', 500)),
        header-text-color: (igx-color:('secondary', 500)),
        header-border-width: 1px,
        header-border-style: solid,
        header-border-color: (igx-color:('grays', 200))
    )
);
```

カスタム スキーマを適用するには、グローバルの `light` または `dark` のいずれかを**拡張**する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを指し示すもので、その後それを対応するコンポーネント テーマに追加します。

```scss
Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid: $custom-grid-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: igx-grid-theme(
  $palette: $custom-palette,
  $schema: $my-custom-schema
);
```

上記で示したのと同じ方法でテーマを含めることを忘れないでください。
