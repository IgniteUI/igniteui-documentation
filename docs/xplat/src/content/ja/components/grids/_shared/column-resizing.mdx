# {Platform}  {ComponentTitle} 列サイズ変更の概要

{Platform} {ComponentTitle} の {ProductName} 列サイズ変更機能を使用すると、ユーザーは <ApiLink type="{ComponentName}" /> の列の幅を簡単に調整できます。既定では、ドラッグによるサイズ変更操作が有効な間、一時的なサイズ変更インジケーターが表示されます。ピクセル/パーセンテージでの列サイズ変更、列サイズ変更の制限、ダブルクリックによる列の自動サイズ調整、初期化時の列の自動サイズ調整など、いくつかのサイズ変更オプションを利用できます。

<DocsAside type="info">

ピクセルで設定された列とパーセンテージで設定された列では、サイズ変更の動作にわずかな違いがあります。

</DocsAside>

**ピクセル**

ピクセル単位の幅を持つ列のサイズ変更は、マウス移動の水平方向の量を列のサイズに直接加算または減算することで動作します。

**パーセンテージ**

パーセンテージ単位の幅を持つ列のサイズ変更では、マウス移動の水平方向の量（ピクセル単位）が、グリッド幅に対するおおよそのパーセンテージ量に変換されます。列はレスポンシブなままで、今後グリッドのサイズが変更されても列にも反映されます。

## 列サイズ変更の制限

許可される最小および最大の列幅も構成できます。これは、<ApiLink type="Column" /> の <ApiLink type="Column" member="minWidth" /> および <ApiLink type="ColumnState" member="maxWidth" /> 入力を使用して行います。この場合、サイズ変更インジケーターのドラッグ操作は、<ApiLink type="Column" member="minWidth" /> と <ApiLink type="ColumnState" member="maxWidth" /> で定義された境界の外側に列のサイズを変更できないことをユーザーに通知するように制限されます。

## ダブルクリックによる列の自動サイズ調整

各列は、ヘッダーの右側をダブルクリックすることで**自動サイズ調整**できます。列はヘッダー自体を含め、現在表示されているセル値のうち最も長い値に合わせてサイズ調整されます。この動作は既定で有効になっており、追加の構成は不要です。ただし、その列に <ApiLink type="ColumnState" member="maxWidth" /> が設定されていて、新しい幅がその <ApiLink type="ColumnState" member="maxWidth" /> 値を超える場合、列は自動サイズ調整されません。この場合、列は事前設定された <ApiLink type="ColumnState" member="maxWidth" /> 値に従ってサイズ調整されます。

<ApiLink type="Column" /> で公開されている <ApiLink type="Column" member="autosize" /> メソッドを使用して、列を動的に自動サイズ調整することもできます。

<ComponentBlock for="Grid, TreeGrid">

<PlatformBlock for="Angular">

## スタイル設定

<ApiLink type="{ComponentName}" /> 列サイズ変更線のスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが含まれているインデックス ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

これを実現する最も簡単な方法は、<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張し、多くのパラメーターと `$resize-line-color` パラメーターを受け入れる新しいテーマを作成することです。

```scss
$custom-grid-theme: grid-theme(
    $resize-line-color: #0288D1
);
```

 <DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合は、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

 </DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-grid-theme);
    }
}
```

### カラー パレットの定義

カラー値をハードコーディングする代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、指定されたプライマリ カラーとセカンダリ カラーに基づいてカラー パレットを生成します。

```scss
$primary-color: #0288D1;
$secondary-color: #BDBDBD;

$custom-theme-palette: palette($primary: $primary-color, $secondary: $secondary-color);
```

その後、<ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> を使用すると、パレットから色を簡単に取得できます。

```scss
$custom-grid-theme: grid-theme(
    $palette: $custom-theme-palette,
    $resize-line-color: color($custom-theme-palette, 'secondary', 500)
);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色を生成および取得するための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/sass/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[**スキーマ**](../themes/sass/schemas.md) の利点を得られる堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

各コンポーネントに提供されている定義済みスキーマを拡張します。この場合は <ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" /> スキーマです。

```scss
// Extending the light grid schema
$light-grid-schema: extend($_light-grid,
    (
        resize-line-color: (
           color: ('secondary', 500)
            ),
        header-background: (
           color: ("primary", 100)
            ),
        header-text-color: (
           color: ("primary", 600)
            )
    )
);
```

カスタム スキーマを適用するには、グローバルの 1 つ（<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> または <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />）を**拡張**する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを指定し、その後それを対応するコンポーネント テーマに追加することを意味します。

```scss
// Extending the global light-schema
$custom-light-grid-schema: extend($light-schema,(
    igx-grid: $light-grid-schema
));

// Specifying the palette and schema of the custom grid theme
$custom-grid-theme: grid-theme(
    $palette: $custom-theme-palette,
    $schema: $custom-light-grid-schema
);
```

上記で示したのと同じ方法でテーマを含めることを忘れないでください。
