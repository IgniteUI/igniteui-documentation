# {Platform} {ComponentTitle} 列選択の概要

{ProductName} の {Platform} {ComponentTitle} 列選択機能は、列全体を 1 回のクリックで選択してハイライトできる、シンプルで Excel のような方法を提供します。これは <ApiLink type="{ComponentName}" member="columnSelection" /> 入力で有効にできます。豊富な API により、この機能では選択状態の簡単な操作、選択された部分からのデータ抽出、データ分析操作、および可視化が可能です。

## 基本的な使用方法

列選択機能は、<ApiLink type="{ComponentName}" member="columnSelection" /> 値を受け取る <ApiLink type="GridSelectionMode" /> 入力で有効にできます。

## 操作

デフォルトの選択モードは `None` です。`Single` または `Multiple` に設定すると、表示されているすべての列が <ApiLink type="Column" member="selectable" /> になります。つまり、列を選択するには、その列をクリックするだけで、<ApiLink type="Column" member="selected" /> としてマークされます。列が選択できない場合、ホバー時にヘッダーへ選択スタイルは適用されません。

<DocsAside type="info">

[複数列ヘッダー](multi-column-headers.md) 機能は、<ApiLink type="{ComponentName}" member="selectable" /> 入力には反映されません。子の少なくとも 1 つで選択動作が有効になっている場合、<ApiLink type="ColumnGroupComponent" /> は <ApiLink type="{ComponentName}" member="selectable" /> になります。さらに、すべての <ApiLink type="{ComponentName}" member="selectable" /> 子孫が <ApiLink type="{ComponentName}" member="selected" /> である場合、コンポーネントは <ApiLink type="{ComponentName}" member="selected" /> としてマークされます。

</DocsAside>

## キーボードの組み合わせ

<DocsAside type="info">

キーボードの組み合わせは、グリッドの <ApiLink type="{ComponentName}" member="columnSelection" /> 入力が `multiple` に設定されている場合にのみ使用できます。

</DocsAside>

**列選択** 機能のキーボード ナビゲーションには 2 つのシナリオがあります。

- 複数列選択 - 各 **選択可能な** ヘッダー セルで <kbd>コントロール</kbd> + <kbd>クリック</kbd> を押したままにします。
- 範囲列選択 - <kbd>シフト</kbd> + <kbd>クリック</kbd> を押したままにすると、その間にあるすべての **選択可能な** 列が選択されます。

## API 操作

**API** は、**表示されていない** 列に関して、対応する **setter** を設定することで各 **非表示** 列を <ApiLink type="Column" member="selected" /> としてマークできるような追加機能を提供します。

<DocsAside type="info">

上記の記述は <ApiLink type="ColumnGroupComponent" /> にも適用されますが、<ApiLink type="{ComponentName}" member="selected" /> プロパティが変更されると、その子孫の状態が変更される点が異なります。

</DocsAside>

API 操作に関する詳細情報は、[API リファレンス](#api-references) セクションを参照してください。

## スタイル設定

定義済みテーマに加えて、利用可能な [CSS プロパティ](../theming-grid.md) の一部を設定することで、グリッドをさらにカスタマイズできます。
一部の色を変更する場合は、まずグリッドに `class` を設定する必要があります。

<PlatformBlock for="WebComponents">

```html
<{ComponentSelector} class="grid"></{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<{ComponentSelector} className="grid"></{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<{ComponentSelector} class="grid"></{ComponentSelector}>
```

</PlatformBlock>

次に、関連する CSS プロパティをこのクラスに設定します。

```css
.grid {
    --ig-grid-row-selected-background: #0062A3;
    --ig-grid-row-selected-text-color: #ecaa53;
    --ig-grid-row-selected-hover-background: #0062A3;
    --ig-grid-header-selected-text-color: #ecaa53;
    --ig-grid-header-selected-background: #0062A3;
    --ig-grid-row-selected-hover-text-color: #ecaa53;
    --ig-grid-row-selected-hover-background: #0062A3;
}
```

## スタイル設定

スタイル設定オプションを詳しく見る前に、コア モジュールとすべてのコンポーネント ミックスインをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

<ComponentBlock for="Grid">

> [行選択](row-selection.md) と [列選択](column-selection.md) は個別に操作できないことに注意してください。これらは同じ `variables` に依存しています。

</ComponentBlock>

それでは、**選択** と **ホバー** のスタイルを変更します。<br/>

最もシンプルな方法として、カスタム **テーマ** を定義します。

<ComponentBlock for="TreeGrid">

```scss
$custom-grid-theme: grid-theme(
    $row-selected-background: #011627,
    $row-selected-text-color: #ECAA53,
    $row-selected-hover-background: #011627,
    $header-selected-text-color: #ECAA53,
    $header-selected-background: #011627,
    $expand-icon-color: #ECAA53,
    $expand-icon-hover-color: #B64B80
);
```

</ComponentBlock>

<ComponentBlock for="Grid, HierarchicalGrid">

```scss
$custom-grid-theme: grid-theme(
    $row-selected-background: #011627,
    $row-selected-text-color: #ECAA53,
    $row-selected-hover-background: #011627,
    $header-selected-text-color: #ECAA53,
    $header-selected-background: #011627
);
```

</ComponentBlock>

<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> は複数のパラメーターを受け取りますが、選択されたすべての列の外観変更を担うのは次の 5 つです。

- **$row-selected-background** - 選択された部分の背景を設定します。
- **$row-selected-text-color** - 選択された部分のテキスト色を設定します。
- **$row-selected-hover-background** - ホバーされたセルまたはセルのまとまりの色を設定します。
- **$header-selected-text-color** - 選択された列ヘッダーのテキスト色を設定します。
- **$header-selected-background** - 選択された列ヘッダーの背景色を設定します。

### CSS 変数の使用

最後の手順は、カスタム `{ComponentSelector}` テーマを含めることです。

```scss
@include css-vars($custom-grid-theme)
```

### ベース テーマのオーバーライド

Internet Explorer 11 向けにコンポーネントのスタイルを設定するには、CSS 変数がサポートされていないため、別の方法を使用する必要があります。

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。カスタム テーマが他のコンポーネントに漏れないようにするため、`::ng-deep` の前に `:host` セレクターを含めていることを確認してください。

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-column-selection-theme);
    }
}
```
