# {Platform} {ComponentTitle} 行の固定

{Platform} {ComponentTitle} の {ProductName} 行の固定機能を使用すると、1 つまたは複数の行をグリッドの上部または下部に固定できます。行の固定により、エンドユーザーは特定の順序で行を固定し、<ApiLink type="{ComponentName}" /> を垂直方向にスクロールしても常に表示される特別な領域にそれらを複製できます。{Platform} {ComponentTitle} には組み込みの行固定 UI があり、{ComponentTitle} のコンテキストで <ApiLink type="ActionStrip" /> コンポーネントを初期化することで有効になります。さらに、カスタム UI を定義し、行固定 API を介して行の固定状態を変更できます。

## 行固定 API

行の固定は、`Row` の <ApiLink type="RowDirective" member="pinned" /> 入力を通じて制御されます。固定された行は既定で <ApiLink type="{ComponentName}" /> の上部に描画され、<ApiLink type="{ComponentName}" /> 本体内の固定されていない行を垂直方向にスクロールしても固定されたままになります。

<PlatformBlock for="Angular, WebComponents">

```typescript

```

</PlatformBlock>

<PlatformBlock for="React">

```tsx

```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp

```

</PlatformBlock>

行 ID は、グリッドの <ApiLink type="{ComponentName}" member="primaryKey" /> によって定義される主キー値、またはレコード インスタンス自体であることに注意してください。どちらのメソッドも、それぞれの操作が成功したかどうかを示すブール値を返します。通常、失敗する理由は、行がすでに目的の状態にあるためです。

行は、最後に固定された行の下に固定されます。固定された行の順序は、<ApiLink type="{ComponentName}" member="rowPinning" /> イベントをサブスクライブし、イベント引数の <ApiLink type="PinRowEventArgs" /> プロパティを目的の位置インデックスに変更することで変更できます。

## 固定位置

<ApiLink type="{ComponentName}" member="pinning" /> 構成オプションを使用して、行の固定位置を変更できます。これにより、固定領域の位置を Top または Bottom に設定できます。
Bottom に設定すると、固定された行は固定されていない行の後、グリッドの下部に描画されます。固定されていない行は垂直方向にスクロールできますが、固定された行は下部に固定されたままになります。

### 行のドラッグを使用する

行を固定された行と固定されていない行の間で直接ドラッグ アンド ドロップして、その固定状態を変更できるようにしたいとします。
これは、行ドラッグ機能を有効にし、ドロップ時に API を介して行を固定または固定解除することで実現できます。

まず、`igxDrop` ディレクティブを使用してグリッドをドロップ領域としてマークし、`rowDraggable` オプションを介して行ドラッグ機能を有効にする必要があります。

```html
<igx-grid [data]="data" [autoGenerate]="true" [rowDraggable]="true"
    [primaryKey]="'ID'" igxDrop (dropped)="onDropAllowed($event)">
</igx-grid>
```

次に、`dropped` イベントを使用して、並べ替えおよび固定/固定解除のロジックを処理できます。

```typescript
public onDropAllowed(args) {
    const event = args.originalEvent;
    let currRowPinnedIndex;
    const currRowIndex = this.getCurrentRowIndex(this.grid.rowList.toArray(),
        { x: event.clientX, y: event.clientY });
    if (currRowIndex === -1) { return; }

    const currRowID = this.getCurrentRowID(this.grid.rowList.toArray(),
        { x: event.clientX, y: event.clientY });

    const currentRow = this.grid.rowList.toArray().find((r) => r.rowID === currRowID);
    if (currentRow.pinned) {
        currRowPinnedIndex = this.grid.pinnedRows.indexOf(this.grid.pinnedRows.find((r) => r.rowID === currRowID));
    }
    // remove the row that was dragged and place it onto its new location
    this.grid.deleteRow(args.dragData.key);
    this.data.splice(currRowIndex, 0, args.dragData.data);
    if (currentRow.pinned && !args.dragData.pinned) {
        this.grid.pinRow(args.dragData.key, currRowPinnedIndex);
    } else if (!currentRow.pinned && args.dragData.pinned) {
        this.grid.unpinRow(args.dragData.key);
    } else if (currentRow.pinned && args.dragData.pinned) {
        this.grid.unpinRow(args.dragData.key);
        this.grid.pinRow(args.dragData.key, currRowPinnedIndex);
    }
}
```

これにより、行を並べ替えたり、固定された行コレクションと固定されていない行コレクションの間で移動したりできるようになります。

## 行固定の制限

- データ ソースに存在するレコードのみ固定できます。
- 行の固定状態は Excel にエクスポートされません。行の固定が適用されていないものとしてグリッドがエクスポートされます。
- グリッドのスクロール可能領域にある固定行のコピーは、固定行が存在する場合に他のグリッド機能がその機能を実現するための仕組みに不可欠な部分であるため、その作成を無効にしたり削除したりすることはできません。
- 行選択は完全に行 ID で動作するため、固定された行を選択するとそのコピーも選択されます (逆も同様です)。さらに、固定領域内での範囲選択 (例: <kbd>シフト</kbd> + クリックの使用) は、スクロール可能領域内で行の範囲を選択する場合と同じように動作します。結果の選択には、現在固定されていない場合でも、その間にあるすべての行が含まれます。API を介して選択された行を取得すると、選択された各レコードの単一のインスタンスのみが返されます。

<PlatformBlock for="Angular">

- 固定された行がグリッドの固定領域と固定されていない領域の両方に表示されるよう内部的に保存される仕組みのため、グリッド内のレコードがリモート エンドポイントからオンデマンドで取得される場合 (リモート仮想化)、行の固定はサポートされません。
- グリッドに <ApiLink type="{ComponentName}" member="primaryKey" /> が設定されておらず、リモート データ シナリオが有効になっている場合 (ページング、並べ替え、フィルタリング、スクロールによってリモート サーバーへの要求がトリガーされ、グリッドに表示するデータが取得される場合)、データ要求の完了後に行は次の状態を失います。
  - 行選択
  - 行の展開/折りたたみ
  - 行編集
  - 行の固定

</PlatformBlock>

## スタイル設定

<ApiLink type="{ComponentName}" /> は、[{ProductName} テーマ ライブラリ](../themes/styles.md)を通じたスタイル設定を可能にします。{ComponentTitle} の <ApiLink type="function-grid-theme" label="theme" kind="sass" module="index.html" /> は、{ComponentTitle} のすべての機能をカスタマイズできる多様なプロパティを公開しています。

以下では、{ComponentTitle} の行固定スタイルをカスタマイズする手順を説明します。

### スタイル ライブラリのインポート

行固定機能のカスタマイズを開始するには、すべてのスタイル関数とミックスインが配置されている `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

### テーマの定義

次に、<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張し、行固定機能を必要に応じてカスタマイズするために必要なパラメーターを受け取る新しいテーマを作成します。

```scss
$custom-grid-theme: grid-theme(
    /* Pinning properties that affect styling */
    $pinned-border-width: 5px,
    $pinned-border-style: double,
    $pinned-border-color: #FFCD0F,
    $cell-active-border-color: #FFCD0F
    /* add other features properties here... */
);
```

### CSS 変数の使用

最後の手順は、カスタム グリッド テーマを渡すことです。

```scss
@include css-vars($custom-grid-theme);
```

### ミックスインの使用

Internet Explorer 11 向けにコンポーネントのスタイルを設定するには、CSS 変数がサポートされていないため、別のアプローチを使用する必要があります。

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。ただし、カスタム テーマが他のコンポーネントに漏れないようにするため、`::ng-deep` の前に必ず `:host` セレクターを含めてください。

```scss
:host {
    ::ng-deep {
        // Pass the custom grid theme to the `igx-grid` mixin
        @include grid($custom-grid-theme);
    }
}
```
