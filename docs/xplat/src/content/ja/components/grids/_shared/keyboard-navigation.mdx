# {Platform} {ComponentTitle} キーボード ナビゲーション

{Platform} {ComponentTitle} の {ProductName} キーボード ナビゲーション機能は、ユーザーに豊富なキーボード操作を提供します。<ApiLink type="{ComponentName}" /> のアクセシビリティを向上させ、ユーザーが内部のあらゆる種類の要素 (セル、行、列ヘッダー、ツールバー、フッターなど) を移動できるようにします。この機能はデフォルトで有効になっており、開発者は既定の動作を簡単にオーバーライドできます。

<ApiLink type="{ComponentName}" /> のタブ移動は、ナビゲーションが W3C アクセシビリティ標準に準拠し、使いやすくなるように削減されています。

現在、<ApiLink type="{ComponentName}" /> には次のタブ ストップがあります。

- **GroupBy またはツールバー領域** (有効な場合)。
- **{ComponentTitle} ヘッダー**。
- **{ComponentTitle} 本体**。
- **列集計** (有効な場合)。
- **{ComponentTitle} ページネーター** (有効な場合)。

<DocsAside type="info">

この変更により、<ApiLink type="{ComponentName}" /> では <kbd>タブ</kbd> および <kbd>シフト</kbd> + <kbd>タブ</kbd> によるセル間の移動はサポートされなくなりました。
<kbd>タブ</kbd> キーを押すと、次の順序でタブ ストップを移動するようになりました: **GroupBy** / **ツールバー** -> **ヘッダー** -> **本体** -> **集計** -> **フッター / ページネーター**。

</DocsAside>

<DocsAside type="info">

テンプレートを使用して <ApiLink type="{ComponentName}" /> 本体に **フォーカス可能な** 要素を公開すると、既定の
ブラウザー動作が抑制されないため、キーボード ナビゲーションに **副作用** が発生する可能性があります。これを適切に防止または変更することは、開発者の責任です。

</DocsAside>

## ヘッダー ナビゲーション

<ApiLink type="{ComponentName}" /> ヘッダーで完全な **キーボード ナビゲーション** サポートが導入されました。列ヘッダーは矢印キーで簡単に移動できます。さらに、列に対して **フィルタリング**、**並べ替え**、**グループ化** などのアクションをトリガーするキーの組み合わせが多数あります。
<ApiLink type="{ComponentName}" /> ヘッダー コンテナーにフォーカスがある場合、次のキーの組み合わせを使用できます。

### キーの組み合わせ

- <kbd>↑</kbd> ヘッダー内で 1 セル上に移動します (ループなし)。Multi-row Layout (MRL) または Multi-column Headers (MCH) が定義されている場合にのみ使用できます。
- <kbd>↓</kbd> ヘッダー内で 1 セル下に移動します (折り返しなし)。Multi-row Layout (MRL) または Multi-column Headers (MCH) が定義されている場合にのみ使用できます。
- <kbd>←</kbd> 1 セル左に移動します (ループなし)。
- <kbd>→</kbd> 1 セル右に移動します (行間の折り返しなし)。
- <kbd>コントロール</kbd> + <kbd>←</kbd> 行の最も左のセルに移動します。MRL または MCH が有効な場合は、同じレベルの最も左のセルに移動します。
- <kbd>ホーム</kbd> 行の最も左のセルに移動します。MRL または MCH が有効な場合は、同じレベルの最も左のセルに移動します。
- <kbd>コントロール</kbd> + <kbd>→</kbd> 行の最も右のセルに移動します。MRL または MCH が有効な場合は、同じレベルの最も右のセルに移動します。
- <kbd>終了</kbd> 行の最も右のセルに移動します。MRL または MCH が有効な場合は、同じレベルの最も右のセルに移動します。
- <kbd>代替</kbd> + <kbd>エル</kbd> Advanced Filtering が有効な場合、Advanced Filtering ダイアログを開きます。
- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>エル</kbd> 列がフィルター可能な場合、Excel Style Filter ダイアログまたは既定の (行) フィルターを開きます。
- <kbd>コントロール</kbd> + <kbd>↑</kbd> アクティブな列ヘッダーを ASC 順で並べ替えます。列がすでに ASC で並べ替えられている場合、並べ替え状態はクリアされます。
- <kbd>コントロール</kbd> + <kbd>↓</kbd> アクティブな列ヘッダーを DSC 順で並べ替えます。列がすでに DSC で並べ替えられている場合、並べ替え状態はクリアされます。
- <kbd>スペース</kbd> 列を選択します。列がすでに選択されている場合、選択はクリアされます。

<ComponentBlock for="Grid">

- <kbd>シフト</kbd> + <kbd>代替</kbd> + <kbd>←</kbd> 列がグループ化可能としてマークされている場合、その列をグループ化します。

</ComponentBlock>

- <kbd>シフト</kbd> + <kbd>代替</kbd> + <kbd>→</kbd> 列がグループ化可能としてマークされている場合、その列のグループ化を解除します。
- <kbd>代替</kbd> + <kbd>←</kbd> または <kbd>代替</kbd> + <kbd>↑</kbd> ヘッダーがまだ折りたたまれていない場合、列グループ ヘッダーを折りたたみます。
- <kbd>代替</kbd> + <kbd>→</kbd> または <kbd>代替</kbd> + <kbd>↓</kbd> ヘッダーがまだ展開されていない場合、列グループ ヘッダーを展開します。

## カスタム キーボード ナビゲーション
