# {Platform} {ComponentTitle} 列の非表示

<PlatformBlock for="Angular">

{ProductName} <ApiLink type="{ComponentName}" /> は、ユーザーがユーザー インターフェイスから直接、または {Platform} コンポーネントを使用して列の非表示を実行できる `ColumnHidingDirective` を備えた <ApiLink type="ColumnActionsComponent" /> を提供します。

</PlatformBlock>

{ProductName} には組み込みの列非表示 UI があり、{Platform} {ComponentTitle} ツールバーを通じて使用し、列の表示状態を変更できます。開発者は必要に応じて、ページ内の任意の場所に列非表示 UI を定義できます。{Platform} {ComponentTitle} 列非表示機能は、グリッドのサイズを小さくし、冗長なフィールドをタブ移動する必要をなくしたい場合に特に便利です。

## ツールバーの列非表示 UI

組み込みの列非表示 UI は、<ApiLink type="{ComponentName}" /> のツールバー内の <ApiLink pkg="core" type="Dropdown" /> の中に配置されます。この正確なドロップダウンを使用して、列非表示 UI の表示/非表示を切り替えることができます。

このために必要なのは、<ApiLink type="{ComponentName}" /> 内で <ApiLink type="GridToolbarActions" /> と <ApiLink type="GridToolbarHiding" /> の両方を設定することだけです。

<ComponentBlock for="Grid">

<PlatformBlock for="Angular">

```html
<igx-grid [data]="localdata">
    <igx-grid-toolbar>
        <igx-grid-toolbar-actions>
            <igx-grid-toolbar-hiding></igx-grid-toolbar-hiding>
        </igx-grid-toolbar-actions>
    </igx-grid-toolbar>
</igx-grid>
```

</PlatformBlock>

<ApiLink type="{ComponentName}" /> は、ツールバーの列非表示 UI を使用する際に便利なプロパティをいくつか提供します。

<ApiLink type="GridToolbarHiding" member="title" label="Title" /> プロパティを使用することで、ツールバーのドロップダウン ボタン内に表示されるタイトルを設定します。

## スタイル設定

列アクション コンポーネントのスタイル設定を開始するには、すべてのテーマ関数とコンポーネント mixin が含まれているインデックス ファイルをインポートする必要があります。

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

最も簡単な方法を使用して、`column-actions-theme` を拡張し、`$title-color` パラメーターと `$background-color` パラメーターを受け取る新しいテーマを作成します。

```scss
$custom-column-actions-theme: column-actions-theme(
    $background-color: steelblue,
    $title-color: gold
);
```

ご覧のとおり、`column-actions-theme` は列アクション コンテナーの色のみを制御し、その中のボタン、チェックボックス、入力グループには影響しません。ボタンもスタイル設定したい場合は、新しいボタン テーマを作成します。

```scss
$custom-button: button-theme($flat-text-color: gold, $disabled-color: black);
```

この例では、フラット ボタンのテキスト色とボタンの無効状態の色のみを変更しましたが、`button-theme` はボタン スタイルを制御するためのさらに多くのパラメーターを提供しています。

最後の手順は、コンポーネント mixin を、それぞれ対応するテーマとともに **含める** ことです。

```scss
@include column-actions($custom-column-actions-theme);
.igx-column-actions {
    @include button($custom-button);
}
```

> **注**
> **igx-button** mixin を `.igx-column-actions` 内にスコープ設定しているため、列非表示ボタンのみがスタイル設定されます。そうしないと、グリッド内の他のボタンにも影響します。

> **注**
> コンポーネントが `Emulated`ViewEncapsulation を使用している場合、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

```scss
:host {
    ::ng-deep {
        @include column-actions($custom-column-actions-theme);
        .igx-column-actions {
            @include button($custom-button);
        }
    }
}
```

### カラー パレットの定義

先ほどのように色の値をハードコーディングする代わりに、`igx-palette` 関数と `igx-color` 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します。

```scss
$yellow-color: gold;
$blue-color: steelblue;

$custom-palette: palette($primary: $blue-color, $secondary: $yellow-color);
```

そして、`igx-color` を使用すると、パレットから簡単に色を取得できます。

```scss
$custom-column-actions-theme: column-actions-theme(
    $palette: $custom-palette,
    $title-color: color($custom-palette, "secondary", 400),
    $background-color: color($custom-palette, "primary", 200)
);

$custom-button: button-theme(
    $palette: $custom-palette,
    $flat-text-color: color($custom-palette, "secondary", 400),
    $disabled-color: black
);
```

> **注**
> `igx-color` と `igx-palette` は、色を生成および取得するための強力な関数です。これらの使用方法に関する詳細なガイダンスについては、\`パレット]\(themes/sass/palettes.md) トピックを参照してください。

### スキーマの使用

テーマ エンジンをさらに活用すると、[**スキーマ**](themes/sass/schemas.md) のメリットを得られる堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

```scss
// Extending the dark column actions schema
$custom-column-actions-schema: extend($_dark-column-actions,
    (
        title-color:(
           color: ("secondary", 400)
        ),
        background-color:(
           color: ("primary", 200)
        )
    )
);
// Extending the dark button schema
$custom-button-schema: extend($_dark-button,
    (
        flat-text-color:(
           color: ("secondary", 500)
        ),
        disabled-color:(
           color: ("primary", 700)
        )
    )
);
```

カスタム スキーマを適用するには、グローバルの `light` または `dark` のいずれかを **拡張** する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを示しており、その後、それを対応するコンポーネント テーマに追加します。

```scss
// Extending the global dark-schema
$custom-dark-schema: extend($dark-schema,(
    igx-column-actions: $custom-column-actions-schema,
    igx-button: $custom-button-schema
));

// Defining column-actions-theme with the global dark schema
$custom-column-actions-theme: column-actions-theme(
  $palette: $custom-palette,
  $schema: $custom-dark-schema
);

// Defining button-theme with the global dark schema
$custom-button: button-theme(
  $palette: $custom-palette,
  $schema: $custom-dark-schema
);
```

上記で示したのと同じ方法で、テーマを含めることを忘れないでください。
