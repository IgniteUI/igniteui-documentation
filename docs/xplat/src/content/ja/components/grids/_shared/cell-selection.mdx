# {Platform} {ComponentTitle} セル選択

{Platform} {ComponentTitle} の {ProductName} セル選択は、豊富なデータ選択機能を有効にし、<ApiLink type="{ComponentName}" /> コンポーネントで強力な API を提供します。{Platform} {ComponentTitle} は、次の 3 つの選択モードをサポートします。

- {ComponentTitle} 複数セル選択
- {ComponentTitle} 単一選択
- {ComponentTitle} 選択なし

<ComponentBlock for="HierarchicalGrid">

<ApiLink type="{ComponentName}" /> では、グリッド レベルでセル選択モードを指定できます。たとえば、親グリッドでは複数セル選択を有効にし、子グリッドではセル選択モードを単一または無効にできます。

</ComponentBlock>

これらの各オプションについて詳しく見ていきましょう。

## 選択タイプ

### {ComponentTitle} 複数セル選択

<ComponentBlock for="HierarchicalGrid">

これは、親グリッドと子グリッドの両方における既定のセル選択モードです。セル選択は一度に 1 つのグリッドに対してのみ実行でき、グリッドをまたいだ範囲選択や、複数のグリッドでセルを選択した状態にすることはできない点に注意してください。範囲選択に関連する各キーの組み合わせとマウス ドラッグ機能は、同じグリッド内でのみ使用できます。

</ComponentBlock>

セルを選択する方法:

- **マウス ドラッグ** - セルの矩形データ選択が実行されます。
- <kbd>コントロール</kbd> キー押下 + **マウス ドラッグ** - 複数範囲の選択が実行されます。既存の他のセル選択は保持されます。
- <kbd>シフト</kbd> キーを使用した即時の複数セル選択。単一セルを選択し、<kbd>シフト</kbd> キーを押しながら別の単一セルを選択します。2 つのセル間のセル範囲が選択されます。<kbd>シフト</kbd> キーを押したまま別の 2 つ目のセルを選択した場合、最初に選択したセル位置 (開始点) に基づいてセル選択範囲が更新される点に注意してください。
- <kbd>シフト</kbd> キーを押しながら <kbd>矢印</kbd> キーを使用したキーボードによる複数セル選択。フォーカスされたセルに基づいて複数セル選択範囲が作成されます。
- <kbd>シフト</kbd> キーを押しながら <kbd>コントロール</kbd> + <kbd>↑</kbd> <kbd>↓</kbd> <kbd>←</kbd> <kbd>→</kbd>  キー、および <kbd>コントロール</kbd> + <kbd>ホーム</kbd>/<kbd>終了</kbd> を使用したキーボードによる複数セル選択。フォーカスされたセルに基づいて複数セル選択範囲が作成されます。
- <kbd>コントロール</kbd> キーを押しながら **左マウス** ボタンでクリックすると、選択済みセル コレクションに単一セル範囲が追加されます。
- マウスでクリックしてドラッグすることで、連続した複数セル選択を使用できます。

### {ComponentTitle} 単一選択

<ApiLink type="{ComponentName}" member="cellSelection" /> を **single** に設定すると、グリッド内で一度に 1 つのセルのみを選択できます。また、**マウス ドラッグ** モードは機能せず、セルを選択する代わりに既定のテキスト選択が行われます。

> 単一セルが選択されると、**選択モード** が **single** であるか **multiple** であるかに関係なく、<ApiLink type="Column" member="selected" /> イベントが発行されます。複数セル選択モードでセル範囲を選択すると、<ApiLink type="{ComponentName}" member="rangeSelected" /> イベントが発行されます。

### {ComponentTitle} 選択なし

セル選択を無効にする場合は、<ApiLink type="{ComponentName}" member="cellSelection" /> を **none** に設定するだけです。このモードでは、セルをクリックしたりキーボードで移動しようとしたりしても、セルは**選択されず**、**アクティブ化スタイル**のみが適用されます。このスタイルは、スクロールしたりページ上の他の要素をクリックしたりすると失われます。選択を定義する唯一の方法は、以下で説明する API メソッドを使用することです。

## キーボード ナビゲーションの操作

### Shift キーが押されている間

- <kbd>シフト</kbd> + <kbd>↑</kbd> で、上のセルを現在の選択に追加します。
- <kbd>シフト</kbd> + <kbd>↓</kbd> で、下のセルを現在の選択に追加します。
- <kbd>シフト</kbd> + <kbd>←</kbd> で、左のセルを現在の選択に追加します。
- <kbd>シフト</kbd> + <kbd>→</kbd> で、右のセルを現在の選択に追加します。

### Ctrl + Shift キーが押されている間

- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>↑</kbd> で、列内のフォーカスされたセルより上にあるすべてのセルを選択します。
- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>↓</kbd> で、列内のフォーカスされたセルより下にあるすべてのセルを選択します。
- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>←</kbd> で、行の先頭までのすべてのセルを選択します。
- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>→</kbd> で、行の末尾までのすべてのセルを選択します。
- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>ホーム</kbd> で、フォーカスされたセルからグリッド内の最初のセルまでのすべてのセルを選択します。
- <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>終了</kbd> で、フォーカスされたセルからグリッド内の最後のセルまでのすべてのセルを選択します。

<DocsAside type="info">

連続スクロールは、Grid の本体内でのみ可能です。

</DocsAside>

<PlatformBlock for="Angular, WebComponents, React">

<ApiLink type="{ComponentName}" member="getSelectedData" /> は、選択に応じた形式で選択されたデータの配列を返します。以下に例を示します。

- 3 つの異なる単一セルが選択されている場合:

```typescript
expectedData = [
    { CompanyName: 'Infragistics' },
    { Name: 'Michael Langdon' },
    { ParentID: 147 }
];
```

- 1 つの列から 3 つのセルが選択されている場合:

```typescript
expectedData = [
    { Address: 'Obere Str. 57'},
    { Address: 'Avda. de la Constitución 2222'},
    { Address: 'Mataderos 2312'}

```

- 1 つの行と 3 つの列から、マウス ドラッグで 3 つのセルが選択されている場合:

```typescript
expectedData = [
    { Address: 'Avda. de la Constitución 2222', City: 'México D.F.', ContactTitle: 'Owner' }
];
```

- 2 つの行と 3 つの列から、マウス ドラッグで 3 つのセルが選択されている場合:

```typescript
expectedData = [
    { ContactTitle: 'Sales Agent', Address: 'Cerrito 333', City: 'Buenos Aires'},
    { ContactTitle: 'Marketing Manager', Address: 'Sierras de Granada 9993', City: 'México D.F.'}
];
```

- 2 つの異なる範囲が選択されている場合:

```typescript
expectedData = [
    { ContactName: 'Martín Sommer', ContactTitle: 'Owner'},
```

```typescript
expectedData = [
    { ContactName: 'Diego Roel', ContactTitle: 'Accounting Manager', Address: 'C/ Moralzarzal, 86'},
```

## 機能の統合

複数セル選択はインデックス ベースです (DOM 要素の選択)。

- <ApiLink type="{ComponentName}" member="sortingExpressions" label="Sorting" /> - 並べ替えが実行されても、選択はクリアされません。昇順または降順に並べ替えている間、現在選択されているセルはそのまま保持されます。
- <ApiLink type="{ComponentName}" member="pagingMode" label="Paging" /> - ページング時に、選択されたセルはクリアされます。選択はページ間で保持されません。
- <ApiLink type="{ComponentName}" member="allowFiltering" label="Filtering" /> - フィルタリングが実行されても、選択はクリアされません。フィルタリングがクリアされると、最初に選択されたセルが戻されます。
- <ApiLink type="Column" member="resizable" label="Resizing" /> - 列のサイズ変更時に、選択されたセルはクリアされません。
- <ApiLink type="Column" member="hidden" label="Hiding" /> - 選択されたセルはクリアされません。列が非表示の場合、次に表示されている列のセルが選択されます。
- <ApiLink type="{ComponentName}" member="pinning" /> - 選択されたセルはクリアされません。非表示の場合と同じです。
- <ApiLink type="{ComponentName}" member="groupingExpressions" label="GroupBy" /> - 列グループ化時に、選択されたセルはクリアされません。

## スタイル設定ガイドライン

テーマ エンジンは、**選択されたセルの範囲**のスタイル設定を可能にするプロパティを公開しています。

### テーマのインポート

選択のスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが含まれている `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

### 色の定義

完了したら、<ApiLink type="function-igx-contrast-color" label="igx-contrast-color" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用できます。これらを使用して、選択範囲に使用したい色を定義します。

```scss
    $text-color:contrast-color($default-palette, 'primary', 900);
    $background-color: color($default-palette, "primary", 900);
    $border-yellow: #f2c43c;
```

### カスタム テーマの作成

次に、`text-color`、`background-color`、および `border-yellow` 変数をそれぞれ `$cell-selected-text-color`、`$cell-selected-background`、および `$cell-active-border-color` として渡して <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張する新しいテーマを作成します。

```scss
$custom-grid-theme: grid-theme(
    $cell-selected-text-color: $text-color,
    $cell-active-border-color: $border-yellow,
    $cell-selected-background: $background-color
);
```

### テーマの適用

その後、コンポーネントのスタイル (アプリのスタイルでも可) にミックスインを含めるだけで、@@igSelector は既定のテーマではなく、新しく作成したテーマを使用します。

```scss
    @include grid($custom-grid-theme);
```

 <DocsAside type="info">

コンポーネントが [Emulated ViewEncapsulation](../themes/styles.md#view-encapsulation) を使用している場合、`::ng-deep` を使用してこのカプセル化を貫通する必要があります。
アプリケーション内の他のグリッドに影響を与えないように、`:host` セレクターの下にスタイルをスコープします。

 </DocsAside>

```scss
   :host {
       ::ng-deep {
           @include grid($custom-grid-theme);
       }
   }
```

カスタム テーマを適用すると、選択されたグリッド セルは選択した色で強調表示されます。
