# {Platform} {ComponentTitle} 行選択

{Platform} {ComponentTitle} の {ProductName} 行選択機能により、ユーザーはデータの単一行または複数行を対話的に選択、強調表示、または選択解除できます。<ApiLink type="{ComponentName}" /> では、いくつかの選択モードを使用できます。

- 選択なし
- 複数選択
- 単一選択

## セットアップ

<ApiLink type="{ComponentName}" /> で行選択を設定するには、<ApiLink type="{ComponentName}" member="rowSelection" /> プロパティを設定するだけです。このプロパティは <ApiLink type="GridSelectionMode" /> 値を受け取ります。

<ApiLink type="GridSelectionMode" /> は次のモードを公開します。

- **なし**
- **単一**
- **複数**

<ComponentBlock for="TreeGrid">

- **複数カスケード**

</ComponentBlock>

以下では、それぞれについて詳しく見ていきます。

### 選択なし

<ApiLink type="{ComponentName}" /> では、既定で行選択が無効になっています (<ApiLink type="{ComponentName}" member="rowSelection" /> は None)。そのため、<ApiLink type="{ComponentName}" /> UI との操作によって行を選択または選択解除することは**できません**。これらの操作を完了する唯一の方法は、提供されている API メソッドを使用することです。

### 単一選択

単一行選択は簡単に設定できます。必要なことは、<ApiLink type="{ComponentName}" member="rowSelection" /> を `Single` プロパティに設定することだけです。これにより、**グリッド内で 1 行のみを選択**できるようになります。セルをクリックするか、行のセルにフォーカスしているときに <kbd>スペース</kbd> キーを押すことで行を選択できます。もちろん、行セレクター フィールドをクリックして行を選択することもできます。行が選択または選択解除されると、<ApiLink type="{ComponentName}" member="rowSelectionChanging" /> イベントが発行されます。

```html

          (rowSelectionChanging)="handleRowSelection($event)" [allowFiltering]="true">
</{ComponentSelector}>
```

</PlatformBlock>

</{ComponentSelector}>

### 複数選択

<ApiLink type="{ComponentName}" /> で複数行選択を有効にするには、<ApiLink type="{ComponentName}" member="rowSelection" /> プロパティを `Multiple` に設定するだけです。これにより、各行および <ApiLink type="{ComponentName}" /> ヘッダーで行セレクター フィールドが有効になります。行セレクターを使用すると、ユーザーは複数の行を選択でき、選択状態はスクロール、ページング、および並べ替えやフィルタリングなどのデータ操作を通じて保持されます。セルをクリックするか、セルにフォーカスがあるときに <kbd>スペース</kbd> キーを押すことでも行を選択できます。1 行を選択した状態で <kbd>シフト</kbd> キーを押しながら別の行をクリックすると、行の範囲全体が選択されます。この選択モードでは、単一の行をクリックすると、以前に選択されていた行は選択解除されます。<kbd>コントロール</kbd> キーを押しながら**クリック**すると、その行の選択状態が切り替わり、以前の選択は保持されます。

### カスケード選択

<ApiLink type="{ComponentName}" /> でカスケード行選択を有効にするには、<ApiLink type="{ComponentName}" member="rowSelection" /> プロパティを `MultipleCascade` に設定するだけです。これにより、各行および <ApiLink type="{ComponentName}" /> ヘッダーで行セレクター フィールドが有効になります。行セレクターを使用すると、ユーザーは複数の行を選択でき、下位ツリー内のすべての子も選択されます。選択状態は、スクロール、ページング、および並べ替えやフィルタリングなどのデータ操作を通じて保持されます。セルをクリックするか、セルにフォーカスがあるときに <kbd>スペース</kbd> キーを押すことでも行を選択できます。1 行を選択した状態で <kbd>シフト</kbd> キーを押しながら別の行を**クリック**すると、親レコードの選択によって、選択範囲内に含まれていない場合でも、そのすべての子が選択されます。この選択モードでは、単一の行を**クリック**すると、以前に選択されていた行は選択解除されます。<kbd>コントロール</kbd> キーを押しながら**クリック**すると、その行とその子の選択状態が切り替わり、以前の選択は保持されます。

- 行選択は <ApiLink type="{ComponentName}" member="rowSelectionChanging" /> イベントをトリガーします。このイベントは、**新しい選択**、**古い選択**、古い選択から**追加**および**削除**された行に関する情報を提供します。また、このイベントは**キャンセル可能**であるため、選択を防止できます。
- 行選択が有効な場合、行セレクターが表示されます。ただし、表示したくない場合は、<ApiLink type="GridBaseDirective" member="hideRowSelectors" label="HideRowSelectors" /> を **true** に設定できます。
- 実行時に行選択モードを切り替えると、以前の行選択状態はクリアされます。

### 行選択イベント

行選択に変更があると、<ApiLink type="{ComponentName}" member="rowSelectionChanging" /> イベントが発行されます。<ApiLink type="{ComponentName}" member="rowSelectionChanging" /> は次の引数を公開します。

- `OldSelection`  - 行選択の以前の状態を含む行 ID の配列。
- `NewSelection` - 行選択の新しい状態に一致する行 ID の配列。
- `Added` - 現在選択に追加されている行 ID の配列。
- `Removed` - 古い選択状態に従って現在削除されている行 ID の配列。
- `Event` - 行選択の変更をトリガーした元のイベント。
- `Cancel` - 行選択の変更を防止できます。

<ComponentBlock for="HierarchicalGrid">

### すべての行を選択

<ApiLink type="{ComponentName}" /> が提供するもう 1 つの便利な API メソッドは <ApiLink type="{ComponentName}" member="selectAllRows" label="SelectAllRows" /> です。既定では、このメソッドはすべてのデータ行を選択しますが、フィルタリングが適用されている場合は、フィルター条件に一致する行のみを選択します。**false** パラメーターを指定してメソッドを呼び出すと、`SelectAllRows(false)` はフィルタリングが適用されている場合でも、常にグリッド内のすべてのデータを選択します。

> **注** <ApiLink type="GridBaseDirective" member="selectAllRows" label="SelectAllRows" /> は削除された行を選択しないことに注意してください。

### すべての行を選択解除

<ApiLink type="{ComponentName}" /> は <ApiLink type="{ComponentName}" member="deselectAllRows" label="DeselectAllRows" /> メソッドを提供します。このメソッドは既定で、すべてのデータ行を選択解除しますが、フィルタリングが適用されている場合は、フィルター条件に一致する行のみを選択解除します。**false** パラメーターを指定してメソッドを呼び出すと、`DeselectAllRows(false)` はフィルタリングが適用されている場合でも、常にすべての行選択状態をクリアします。

### 行セレクター テンプレート

<ApiLink type="{ComponentName}" /> では、ヘッダーおよび行セレクターをテンプレート化でき、さまざまなシナリオで役立つ機能を提供するそれらのコンテキストにもアクセスできます。

既定では、<ApiLink type="{ComponentName}" /> は行セレクターの親コンテナーまたは行自体での**すべての行選択操作を処理**し、テンプレートには状態の視覚化のみを残します。基本機能のオーバーライドは、通常、[RowSelectionChanging イベント](#row-selection-event)を使用して行う必要があります。基本機能をオーバーライドする <ApiLink pkg="core" type="Checkbox" member="click" label="Click" /> ハンドラーを持つカスタム テンプレートを実装する場合は、正しい行状態を維持するために、イベントの伝播を停止する必要があります。

#### 行テンプレート

カスタム行セレクター テンプレートを作成するには、`{ComponentSelector}` 内で <ApiLink type="GridBaseDirective" member="rowSelectorTemplate" label="RowSelectorTemplate" /> プロパティを使用できます。テンプレートからは、暗黙的に提供されるコンテキスト変数にアクセスでき、そのプロパティから行の状態に関する情報を取得できます。

<ApiLink type="Column" member="selected" /> プロパティは現在の行が選択されているかどうかを示し、<ApiLink type="RowSelectorTemplateDetails" member="index" label="Index" /> プロパティは行インデックスにアクセスするために使用できます。

<PlatformBlock for="Angular">

```html
<ng-template igxRowSelector let-rowContext>
    {{ rowContext.index }}
    <igx-checkbox
        [checked]="rowContext.selected"
        [readonly]="true"
    ></igx-checkbox>
</ng-template>
```

</PlatformBlock>

### ヘッダー テンプレート

カスタム ヘッダー セレクター テンプレートを作成するには、<ApiLink type="{ComponentName}" /> 内で <ApiLink type="GridBaseDirective" member="headSelectorTemplate" label="HeadSelectorTemplate" /> プロパティを使用できます。テンプレートからは、暗黙的に提供されるコンテキスト変数にアクセスでき、そのプロパティからヘッダーの状態に関する情報を取得できます。

<ApiLink type="HeadSelectorTemplateDetails" member="selectedCount" label="SelectedCount" /> プロパティは現在選択されている行数を示し、<ApiLink type="HeadSelectorTemplateDetails" member="totalCount" label="TotalCount" /> は <ApiLink type="{ComponentName}" /> 内に合計で何行あるかを示します。

constructor() {
    const grid = document.getElementById('grid') as {ComponentName}Component;
    grid.data = this.data;

public headSelectorTemplate = (ctx: IgcHeadSelectorTemplateContext) => {
    const implicit: any = ctx.implicit;
    if (implicit.selectedCount > 0 && implicit.selectedCount === implicit.totalCount) {
            return html`<igc-checkbox checked></igc-checkbox>`;
        } else if (implicit.selectedCount > 0 && implicit.selectedCount !== implicit.totalCount) {

}
