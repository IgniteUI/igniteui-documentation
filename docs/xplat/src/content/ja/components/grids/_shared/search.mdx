# {Platform} {ComponentTitle} 検索フィルター

{Platform} {ComponentTitle} の {ProductName} 検索フィルター機能により、データのコレクション内で値を検索できます。この機能を簡単に設定できるようになっており、検索入力ボックス、ボタン、キーボード ナビゲーション、その他の便利な機能を使用して実装でき、ユーザー エクスペリエンスをさらに向上させます。ブラウザーにはネイティブのコンテンツ検索機能がありますが、ほとんどの場合、<ApiLink type="{ComponentName}" /> は表示領域外の列と行を仮想化します。このような場合、仮想化されたセルは DOM の一部ではないため、ブラウザーのネイティブ検索ではそれらのセル内のデータを検索できません。{Platform} Material のテーブルベースのグリッドを拡張し、<ApiLink type="{ComponentName}" /> の**仮想化されたコンテンツ**を検索できる**検索 API** を追加しました。

### {Platform} 検索ボックス入力

<PlatformBlock for="WebComponents">

次に、検索入力を作成しましょう。入力要素を取得することで、その現在の値を取得できます。これにより、<ApiLink type="{ComponentName}" /> の `FindNext` メソッドと `FindPrev` メソッドを使用して、<ApiLink type="{ComponentName}" member="findNext" /> のすべての出現箇所を強調表示し、次または前の出現箇所へスクロールできます (呼び出したメソッドによって異なります)。

</PlatformBlock>

<PlatformBlock for="Angular">

次に、検索入力を作成しましょう。新しく作成した入力に <ApiLink type="{ComponentName}" member="findNext" /> を ngModel としてバインドし、ngModelChange イベントをサブスクライブすることで、ユーザーによる <ApiLink type="{ComponentName}" member="findNext" /> のすべての変更を検出できます。これにより、<ApiLink type="{ComponentName}" /> の `FindNext` メソッドと `FindPrev` メソッドを使用して、<ApiLink type="{ComponentName}" member="findNext" /> のすべての出現箇所を強調表示し、次または前の出現箇所へスクロールできます (呼び出したメソッドによって異なります)。

</PlatformBlock>

<PlatformBlock for="Blazor">

次に、検索入力を作成しましょう。新しく作成した入力の `Value` プロパティに <ApiLink type="{ComponentName}" member="findNext" /> をバインドし、`ValueChanging` イベントをサブスクライブすることで、ユーザーによる <ApiLink type="{ComponentName}" member="findNext" /> のすべての変更を検出できます。これにより、<ApiLink type="{ComponentName}" /> の `FindNext` メソッドと `FindPrev` メソッドを使用して、<ApiLink type="{ComponentName}" member="findNext" /> のすべての出現箇所を強調表示し、次または前の出現箇所へスクロールできます (呼び出したメソッドによって異なります)。

</PlatformBlock>

<PlatformBlock for="React">

次に、検索入力を作成しましょう。新しく作成した入力の `value` プロパティに `searchText` をバインドし、`inputOccured` イベントをサブスクライブすることで、ユーザーによる `searchText` のすべての変更を検出できます。これにより、<ApiLink type="{ComponentName}" /> の `findNext` メソッドと `findPrev` メソッドを使用して、`searchText` のすべての出現箇所を強調表示し、次または前の出現箇所へスクロールできます (呼び出したメソッドによって異なります)。

</PlatformBlock>

`FindNext` メソッドと `FindPrev` メソッドはいずれも 3 つの引数を持ちます。

- `Text`: **string** (検索するテキスト)
- (省略可能) <PlatformBlock for="Angular"><ApiLink type="ISearchInfo" member="caseSensitive" /></PlatformBlock><PlatformBlock for="React,WebComponents"><ApiLink type="SearchInfo" member="caseSensitive" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="SearchInfo" member="caseSensitive" /></PlatformBlock>: **boolean** (検索で大文字と小文字を区別するかどうか。既定値は false)
- (省略可能) <PlatformBlock for="Angular"><ApiLink type="ISearchInfo" member="exactMatch" /></PlatformBlock><PlatformBlock for="React,WebComponents"><ApiLink type="SearchInfo" member="exactMatch" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="SearchInfo" member="exactMatch" /></PlatformBlock>: **boolean** (検索を完全一致で行うかどうか。既定値は false)

完全一致で検索する場合、検索 API は大文字と小文字の区別も考慮したうえで、<ApiLink type="{ComponentName}" member="findNext" /> と完全に一致するセル値のみを結果として強調表示します。たとえば、文字列「*software*」と「*Software*」は、大文字と小文字の区別を無視すると完全一致です。

上記のメソッドは **number** 値 (<ApiLink type="{ComponentName}" /> に指定した文字列が含まれる回数) を返します。

### 結果数の表示

現在の出現箇所の位置と、結果の合計数も表示しましょう。これは、グリッドの <ApiLink type="{ComponentName}" member="lastSearchInfo" /> プロパティを使用して実行できます。このプロパティは、**find** メソッドを使用すると自動的に更新されます。

### キーボード検索の追加

キーボードの矢印キーと <kbd>入力</kbd> キーを使用して、ユーザーが結果間を移動できるようにすることもできます。これを実現するには、検索入力の **keydown** イベントを処理し、`PreventDefault` メソッドで入力の既定のキャレット移動を防止して、ユーザーが押したキーに応じて `FindNext`/`FindPrev` メソッドを呼び出します。

<PlatformBlock for="Angular">

```html
<input #search1 id="search1" placeholder="Search" [(ngModel)]="searchText" (ngModelChange)="@@igObjectRef.findNext(searchText, caseSensitive, exactMatch)"
       (keydown)="searchKeyDown($event)" />
```

```typescript
public searchKeyDown(ev) {
    if (ev.key === 'Enter') {
        ev.preventDefault();
        this.@@igObjectRef.findNext(this.searchText, this.caseSensitive, this.exactMatch);
    } else if (ev.key === 'ArrowUp' || ev.key === 'ArrowLeft') {
        ev.preventDefault();
        this.@@igObjectRef.findPrev(this.searchText, this.caseSensitive, this.exactMatch);
    }
}
```

</PlatformBlock>

### 永続性

<ApiLink type="{ComponentName}" /> をフィルターおよび並べ替えしたい場合、またはレコードを追加および削除したい場合はどうでしょうか。このような操作の後でも、現在の検索の強調表示は自動的に更新され、<ApiLink type="{ComponentName}" member="findNext" /> と一致するすべてのテキスト上で保持されます。さらに、検索はページングでも機能し、<ApiLink type="{ComponentName}" /> の `PerPage` プロパティの変更を通じて強調表示が保持されます。

## 既知の制限事項

| 制限事項                         | 説明                                                                                                                                                                                                                                                                                       |
| ---------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| テンプレートを持つセルでの検索 | 検索機能の強調表示は、既定のセル テンプレートでのみ機能します。カスタム セル テンプレートを持つ列がある場合、強調表示は機能しないため、列フォーマッターなどの代替方法を使用するか、列の <ApiLink type="Column" member="searchable" label="Searchable" /> プロパティを false に設定する必要があります。 |
| リモート仮想化              | リモート仮想化を使用している場合、検索は正しく機能しません                                                                                                                                                                                                                                |
| 切り捨てられたテキストを持つセル            | セル内のテキストが大きすぎて収まらず、検索対象のテキストが省略記号で切り捨てられている場合でも、そのセルまでスクロールし、一致数に含めますが、何も強調表示されません                                                                                     |
