# {Platform} {ComponentTitle} における Excel フィルタリング

{Platform} {ComponentTitle} は、Excel に似たフィルタリング UI を提供する Excel スタイルのフィルタリング機能を公開します。これにより、大規模なデータセットを扱うプロセスが簡素化されます。主な目的は、関連性の低いエントリを除外しながら、最も関連性の高いデータをフィルターできるようにすることです。

## 操作

特定の列のフィルター メニューを開くには、ヘッダー内の {Platform} フィルター アイコンをクリックします。さらに、選択したヘッダーで <kbd>コントロール</kbd> + <kbd>シフト</kbd> + <kbd>エル</kbd> の組み合わせを使用できます。列がフィルタリング機能に加えて、並べ替え、固定、移動、選択、または非表示にできる場合、有効になっている機能のボタンが表示されます。

フィルターが適用されていない場合、リスト内のすべての項目が選択されます。リストの上にある入力からフィルターできます。データをフィルターするには、リスト内の項目を選択/選択解除し、適用ボタンをクリックするか、<kbd>入力</kbd> を押します。リスト項目を通じて適用されるフィルタリングでは、`equals` 演算子を使用したフィルター式が作成され、式間の論理演算子は `OR` になります。

検索ボックスに何かを入力してフィルターを適用すると、検索条件に一致する項目のみが選択されます。ただし、現在フィルターされている項目に項目を追加したい場合は、**現在の選択をフィルターに追加** オプションを選択する必要があります。

フィルターをクリアする場合は、**すべて選択** オプションをオンにしてから、適用ボタンをクリックします。

異なる式でフィルターを適用するには、**テキスト フィルター** をクリックします。これにより、特定の列で使用可能なすべてのフィルター演算子を含むサブメニューが開きます。そのいずれかを選択すると、カスタム フィルター ダイアログが開き、異なるフィルター演算子と論理演算子を使用して、必要な数だけ式を追加できます。フィルターをクリアできるクリア ボタンもあります。

## テンプレート

<PlatformBlock for="Angular">

列機能を無効にせずに Excel スタイル フィルター メニューをさらにカスタマイズしたい場合は、カスタム テンプレートを使用できます。Excel スタイル フィルター メニューには、テンプレート化のために 2 つのディレクティブが用意されています。

- `IgxExcelStyleColumnOperationsTemplateDirective` - 並べ替え、固定など、すべての列操作を含む領域を再テンプレート化します。
- `IgxExcelStyleFilterOperationsTemplateDirective` - すべてのフィルター固有の操作を含む領域を再テンプレート化します。

これらの領域の一方だけ、または両方を再テンプレート化できます。これらのディレクティブ内には任意のカスタム コンテンツを配置でき、組み込みの Excel スタイル フィルタリング コンポーネントも使用できます。

次のコードは、`excel-style-header`、`excel-style-sorting`、`excel-style-search` コンポーネントを使用してカスタム Excel スタイル フィルター メニューを定義する方法を示しています。

</PlatformBlock>

## 外部アウトレット

{ComponentTitle} の [z-index](https://developer.mozilla.org/en-US/docs/Web/CSS/z-index) は、DOM 内の各グリッドに対して個別のスタッキング コンテキストを作成します。これにより、グリッドのすべての子孫要素が、互いに重なり合うことなく意図どおりにレンダリングされます。
ただし、グリッドの外側に出る要素 (例: Excel スタイル フィルター) は、同じ `z-index` を持つ外部要素 (例: 2 つのグリッドが上下に並んでいる場合) と競合し、不正なレンダリングにつながります。この問題の解決策は、`Outlet` プロパティを外部アウトレット ディレクティブに設定することです。これにより、オーバーレイ要素が常に最前面に表示されます。

## スタイル設定

Excel スタイル フィルタリング ダイアログのスタイル設定を開始するには、すべてのテーマ関数とコンポーネント ミックスインが含まれている `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Excel スタイル フィルタリング ダイアログは、`filtering-row-background` パラメーターを使用して、グリッドのテーマから背景色を取得します。背景を変更するには、カスタム テーマを作成する必要があります。

```scss
$custom-grid: grid-theme(
    $filtering-row-background: #FFCD0F
);
```

Excel に似たフィルタリング ダイアログ内には、ボタン、チェックボックス、リスト、さらにはドロップダウンなど、他にも多くのコンポーネントがあります。それらのスタイルを設定するには、それぞれに個別のテーマを作成する必要があります。

```scss
$dark-button: button-theme(
    $background: #FFCD0F,
    $foreground: #292826,
    $hover-background: #292826,
    $hover-foreground: #FFCD0F
);

$dark-input-group: input-group-theme(
    $box-background: #FFCD0F,
    $idle-text-color: #292826,
    $focused-text-color: #292826,
    $filled-text-color: #292826
);

$custom-list: list-theme(
    $background: #FFCD0F
);

$custom-checkbox: checkbox-theme(
    $empty-color: #292826,
    $fill-color: #292826,
    $tick-color: #FFCD0F,
    $label-color: #292826
);

$custom-drop-down: drop-down-theme(
    $background-color: #FFCD0F,
    $item-text-color: #292826,
    $hover-item-background: #292826,
    $hover-item-text-color: #FFCD0F
);
```

この例では、一覧にあるコンポーネントの一部のパラメーターのみを変更しましたが、<ApiLink type="function-button-theme" label="button-theme" kind="sass" module="index.html" />、<ApiLink type="function-checkbox-theme" label="checkbox-theme" kind="sass" module="index.html" />、<ApiLink type="function-drop-down-theme" label="drop-down-theme" kind="sass" module="index.html" />、<ApiLink type="function-input-group-theme" label="input-group-theme" kind="sass" module="index.html" />、<ApiLink type="function-list-theme" label="list-theme" kind="sass" module="index.html" /> テーマは、それぞれのスタイル設定を制御するためのより多くのパラメーターを提供します。

最後の手順は、それぞれのテーマを使用してコンポーネント ミックスインを **含める** ことです。また、入力のプレースホルダーに対して color プロパティを設定します。

```scss
@include drop-down($custom-drop-down);
@include grid($custom-grid);
.igx-excel-filter, .igx-excel-filter__secondary {
    @include button($dark-button);
    @include input-group($dark-input-group);
    @include list($custom-list);
    @include checkbox($custom-checkbox);
    .igx-input-group__input::placeholder {
        color: #FFCD0F;
    }
}
```

<DocsAside type="info">

ほとんどのコンポーネントのミックスインを `.igx-excel-filter` と `.igx-excel-filter__secondary` の範囲内にスコープすることで、これらのカスタム テーマが Excel スタイル フィルタリング ダイアログおよびそのすべてのサブダイアログ内にネストされたコンポーネントにのみ影響するようにしています。そうしないと、他のボタン、チェックボックス、入力グループ、リストにも影響します。

</DocsAside>

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include drop-down($custom-drop-down);
        @include grid($custom-grid);
        .igx-excel-filter, .igx-excel-filter__secondary {
            @include button($dark-button);
            @include input-group($dark-input-group);
            @include list($custom-list);
            @include checkbox($custom-checkbox);
            .igx-input-group__input::placeholder {
                color: #FFCD0F;
            }
        }
    }
}
```

### カラー パレットの定義

先ほどのように色の値をハードコーディングする代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します。

```scss
$yellow-color: #FFCD0F;
$black-color: #292826;

$dark-palette: palette($primary: $black-color, $secondary: $yellow-color);
```

そして、<ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> を使用すると、パレットから色を簡単に取得できます。

```scss
$custom-grid: grid-theme(
    $filtering-row-background: #FFCD0F
);

$dark-button: button-theme(
    $background: color($dark-palette, "secondary", 400),
    $foreground: color($dark-palette, "primary", 400),
    $hover-background: color($dark-palette, "primary", 400),
    $hover-foreground: color($dark-palette, "secondary", 400)
);

$dark-input-group: input-group-theme(
    $box-background: color($dark-palette, "secondary", 400),
    $idle-text-color: color($dark-palette, "primary", 400),
    $focused-text-color: color($dark-palette, "primary", 400),
    $filled-text-color: color($dark-palette, "primary", 400)
);

$custom-list: list-theme(
    $background: color($dark-palette, "secondary", 400)
);

$custom-checkbox: checkbox-theme(
    $empty-color: color($dark-palette, "primary", 400),
    $fill-color: color($dark-palette, "primary", 400),
    $tick-color: color($dark-palette, "secondary", 400),
    $label-color: color($dark-palette, "primary", 400)
);

$custom-drop-down:drop-down-theme(
    $background-color: color($dark-palette, "secondary", 400),
    $item-text-color: color($dark-palette, "primary", 400),
    $hover-item-background: color($dark-palette, "primary", 400),
    $hover-item-text-color: color($dark-palette, "secondary", 400)
);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色の生成と取得のための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/sass/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[**スキーマ**](../themes/sass/schemas.md) の利点を活かした堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

すべてのコンポーネントに提供されている 2 つの定義済みスキーマのいずれかを拡張します。この場合は、<ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" />、<ApiLink type="variable-_light-input-group" label="light-input-group" kind="sass" module="index.html" />、<ApiLink type="variable-_light-button" label="light-button" kind="sass" module="index.html" />、<ApiLink type="variable-_light-list" label="light-list" kind="sass" module="index.html" />、<ApiLink type="variable-_light-checkbox" label="light-checkbox" kind="sass" module="index.html" />、<ApiLink type="variable-_light-drop-down" label="light-drop-down" kind="sass" module="index.html" /> スキーマです。

```scss
$custom-grid-schema: extend($_light-grid,
    (
        filtering-row-background:(
           color: ("secondary", 400)
        )
    )
);

$custom-button-schema: extend($_light-button,
    (
        flat-background:(
           color: ("secondary", 400)
        ),
        flat-text-color:(
           color: ("primary", 400)
        ),
        flat-hover-background:(
           color: ("primary", 400)
        ),
        flat-hover-text-color:(
           color: ("secondary", 400)
        ),

        raised-background:(
           color: ("secondary", 400)
        ),
        raised-text-color:(
           color: ("primary", 400)
        ),
        raised-hover-background:(
           color: ("primary", 400)
        ),
        raised-hover-text-color:(
           color: ("secondary", 400)
        )
    )
);

$custom-input-group-schema: extend($_light-input-group,
    (
        box-background:(
           color: ("secondary", 400)
        ),
        idle-text-color:(
           color: ("primary", 400)
        ),
        focused-text-color:(
           color: ("primary", 400)
        ),
        filled-text-color:(
           color: ("primary", 400)
        )
    )
);

$custom-list-schema: extend($_light-list,
    (
        background:(
           color: ("secondary", 400)
        )
    )
);

$custom-checkbox-schema: extend($_light-checkbox,
    (
        empty-color:(
           color: ("primary", 400)
        ),
        fill-color:(
           color: ("primary", 400)
        ),
        tick-color:(
           color: ("secondary", 400)
        ),
        label-color:(
           color: ("primary", 400)
        )
    )
);

$custom-drop-down-schema: extend($_light-drop-down,
    (
        background-color:(
           color: ("secondary", 400)
        ),
        item-text-color:(
           color: ("primary", 400)
        ),
        hover-item-background:(
           color: ("primary", 400)
        ),
        hover-item-text-color:(
           color: ("secondary", 400)
        )
    )
);
```

カスタム スキーマを適用するには、グローバルの 1 つ (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> または <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />) を **拡張** する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを示すもので、その後、それを対応するコンポーネント テーマに追加します。

```scss
$custom-light-schema: extend($light-schema,(
   grid: $custom-grid-schema,
   button: $custom-button-schema,
   input-group: $custom-input-group-schema,
   list: $custom-list-schema,
   checkbox: $custom-checkbox-schema,
   drop-down: $custom-drop-down-schema
));

$custom-grid: grid-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-button: button-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-input-group: input-group-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-list: list-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-checkbox: checkbox-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-drop-down: drop-down-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);
```

上記で示したのと同じ方法でテーマを含めることを忘れないでください。
