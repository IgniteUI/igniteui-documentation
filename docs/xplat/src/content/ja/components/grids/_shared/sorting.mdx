# {Platform} {ComponentTitle} ソート

{Platform} {ComponentTitle} の {ProductName} データ ソート機能は列ごとに有効化されます。つまり、<ApiLink type="{ComponentName}" /> にはソート可能な列とソート不可の列を混在させることができます。{Platform} ソート操作を実行すると、指定した条件に基づいてレコードの表示順序を変更できます。

<PlatformBlock for="Angular">

<DocsAside type="info">

これまでは、グループ化/ソートは相互に連動して機能していました。バージョン 13.2 では、グループ化とソートを切り離す新しい動作が導入されました。たとえば、グループ化をクリアしてもグリッド内のソート式はクリアされず、その逆も同様です。ただし、列がソートされ、かつグループ化されている場合は、グループ化式が優先されます。

</DocsAside>

</PlatformBlock>

## スタイル設定

ソート動作のスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが含まれている `index` ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

最も簡単な方法として、<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張し、`$sorted-header-icon-color` および `sortable-header-icon-hover-color` パラメーターを受け取る新しいテーマを作成します。

```scss
$custom-theme: grid-theme(
    $sorted-header-icon-color: #ffb06a,
    $sortable-header-icon-hover-color: black
);
```

最後の手順は、コンポーネント ミックスインを **含める** ことです。

```scss
 @include grid($custom-theme);
```

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合は、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

</DocsAside>

```scss
:host {
   ::ng-deep {
       @include grid($custom-theme);
   }
}
```

### カラー パレットの定義

先ほどのようにカラー値をハードコーディングする代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、色に関するより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ カラーとセカンダリ カラーに基づいてカラー パレットを生成します。

```scss
$black-color: black;
$orange-color: #ffb06a;

$custom-palette: palette($primary: $black-color, $secondary: $orange-color);
```

そして <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> を使用すると、パレットから簡単に色を取得できます。

```scss
$custom-theme: grid-theme(
    $sorted-header-icon-color: color($custom-palette, "secondary", 500),
    $sortable-header-icon-hover-color: color($custom-palette, "primary", 500)
);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色の生成と取得のための強力な関数です。使用方法の詳細なガイダンスについては、[パレット](../themes/sass/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[**スキーマ**](../themes/sass/schemas.md) を利用した堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

各コンポーネントに提供されている 2 つの定義済みスキーマのいずれかを拡張します。この場合は <ApiLink type="variable-_light-grid" label="_light-grid" kind="sass" module="index.html" /> です。

```scss
// Extending the light grid schema
$custom-grid-schema: extend($_light-grid,
    (
        sorted-header-icon-color: (igx-color:('secondary', 500)),
        sortable-header-icon-hover-color: (igx-color:('primary', 500))
    )
);
```

カスタム スキーマを適用するには、グローバル (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> または <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />) のいずれかを **拡張** する必要があります。これは基本的にカスタム スキーマを持つコンポーネントを指し示すもので、その後、それぞれのコンポーネント テーマに追加します。

```scss
// Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid: $custom-grid-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: grid-theme(
  $palette: $custom-palette,
  $schema: $my-custom-schema
);
```

上記で示したのと同じ方法でテーマを含めることを忘れないでください。
