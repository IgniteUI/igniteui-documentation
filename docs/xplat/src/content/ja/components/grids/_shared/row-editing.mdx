# {Platform} {ComponentTitle} 行編集

{Platform} {ComponentTitle} の {ProductName} 行編集機能を使用すると、`{ComponentName}` 内でデータを直接編集できます。この便利なデータ操作方法に加えて、完全な CRUD 操作用の強力な API も用意されています。行をクリックして **Enter キー**を押すことで、グリッドの行編集を実行できます。変更する必要がある行をマウスでダブルクリックする方法もあります。

## {Platform} {ComponentTitle} 行編集の例

次のサンプルは、`{ComponentName}` で行編集を有効にする方法を示しています。セル値を変更した後、同じ行の別のセルをクリックするか移動しても、**完了**ボタンで確定するか、**キャンセル**ボタンで破棄するまで、行の値は更新されません。

<Sample src="/{ComponentSample}/row-editing-options" height={550} alt="{Platform} {ComponentTitle} Editing Rows Example" />

<DocsAside type="info">

行が編集モードの場合、別の行のセルをクリックすると「完了」ボタンが押された場合と同様に動作し、前の行で行われたすべての変更が送信されます。新しくフォーカスされたセルが編集可能な場合、新しい行も編集モードになります。ただし、そのセルが編集不可の場合は、前の行だけが編集モードを終了します。

</DocsAside>

## 配置

- オーバーレイの既定位置は、編集モードになっている行の下です。

- 行の下にスペースがない場合、オーバーレイは行の上に表示されます。

- 上または下に一度表示されると、オーバーレイが閉じられるまで、スクロール中もその位置を維持します。

## 動作

- 行が編集モードの場合、同じ行のセルがクリックされると編集が継続されます。

- 「完了」ボタンをクリックすると行編集が終了し、データ ソース、または利用可能な場合はトランザクションに変更が送信されます。さらに、行は編集モードを終了します。

- 「キャンセル」ボタンをクリックすると、行内の現在のすべての変更が元に戻され、行は編集モードを終了します。

- 行が編集モードの場合、別の行のセルをクリックすると現在の行編集が終了し、新しい行の変更が送信されます（「完了」ボタンをクリックした場合と同じ動作）。フォーカスを受け取る新しいセルが編集可能な場合、新しい行も編集モードになります。一方、そのセルが編集不可の場合は、前の行だけが編集モードを終了します。

- 行が編集モードで、`{ComponentName}` がスクロールされて行が表示領域の外に出た場合でも、その行は編集モードのままです。`{ComponentName}` がスクロールされて行が再び表示されるようになっても、その行は編集モードのままです。`{ComponentName}` の外側をクリックした場合も、セルは編集モードのままです。

- **並べ替え**、**フィルタリング**、**検索**、**非表示**操作を実行すると、行内の現在のすべての変更が元に戻され、行は編集モードを終了します。

- **ページング**、**サイズ変更**、**固定**、**移動**操作を実行すると、編集モードを終了し、最新の値が送信されます。

- 変更された各セルには、行編集が完了するまで編集済みスタイルが適用されます。これは、`{ComponentName}` にトランザクションが提供されていない場合の動作です。トランザクションが利用可能な場合、すべての変更がコミットされるまでセルの編集済みスタイルが適用されます。

## キーボード ナビゲーション

- <kbd>入力</kbd> と <kbd>F2</kbd> は行編集モードに入ります。

- <kbd>エスケープ</kbd> は行編集モードを終了し、行が編集モードの間に行われたセルの変更を送信しません。

- <kbd>タブ</kbd> は、行内の編集可能なセル間でフォーカスを次へ移動し、最も右の編集可能なセルからキャンセルおよび完了ボタンへ移動します。完了ボタンからのナビゲーションは、現在編集中の行内の最も左の編集可能なセルへ移動します。

## 機能の統合

- データを変更する操作はすべて行編集操作を終了し、現在の行の変更を送信します。これには、並べ替え、グループ化およびフィルタリング条件の変更、ページングなどの操作が含まれます。

- 行編集が完了すると集計が更新されます。並べ替え、フィルタリングなどの他の機能についても同様です。

<ComponentBlock for="Grid">

- グループ化された行を展開または折りたたんでも、現在の行の編集は終了しません。

</ComponentBlock>

## スタイル設定

[{ProductName} テーマ ライブラリ](themes/index.md)を使用すると、行編集オーバーレイを大幅に変更できます。

行編集オーバーレイは複合要素であり、その UI はいくつかの他のコンポーネントで構成されています。

- コンテンツをレンダリングするための [igx-banner](banner.md)
- 既定のテンプレートでは [igx-button](button.md) がレンダリングされます（`Done` ボタンと `Cancel` ボタン用）。

以下の例では、これら 2 つのコンポーネントのスタイル設定オプション（[ボタンのスタイル設定](button.md#styling)および[バナーのスタイル設定](../banner.md#styling)）を使用して、`{ComponentName}` の行編集エクスペリエンスをカスタマイズします。

また、現在のセルのエディターと背景にスタイルを設定して、より目立たせます。セルのスタイル設定について詳しくは、この[トピック](cell-editing.md#styling)を参照してください。

### テーマのインポート

行編集バナーにスタイルを設定する最も簡単な方法は、`app` のグローバル スタイル ファイル（通常は `styles.scss`）にスタイルを定義することです。
最初に行う必要があるのは、`themes/index` ファイルをインポートすることです。これにより、{ProductName} Sass フレームワークの強力なツールすべてにアクセスできます。

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

テーマ ファイルをインポートしたら、カスタム テーマを作成できます。

#### テーマの定義

これで、行編集の背景に影響を与えるカスタム <ApiLink type="function-banner-theme" label="banner theme" kind="sass" module="index.html" /> を定義し、定義済みパレットの 1 つである `$purple-palette` を使用できます。

```scss
    $my-light-gray: #e3e3e3;
    $my-banner-palette: $purple-palette;

    $banner-theme: banner-theme(
        $banner-background: $my-light-gray,
        $banner-message-color: color($my-banner-palette, "secondary", 600)
    );
```

ここでは、色を生成するために、`my-banner-palette` を（テーマ ライブラリによって公開されている）`igx-color` と組み合わせて使用しています。

### テーマのインクルード

あとは、Sass の `@include` ステートメントでテーマを適用するだけです。新しく定義した `$banner-theme` を **igx-banner ミックスイン**に渡します。

```scss
@include banner($banner-theme);
```

これにより、カスタム バナー テーマが行編集オーバーレイに適用されます。ただし、これらはグローバル スタイル ファイルで定義したため、これらのスタイルはアプリケーション内の**すべての**バナーにも適用されます。

### コンポーネント スタイル

行編集オーバーレイは他の多くのコンポーネントのテーマを使用するため、グローバル スタイルでスタイルを設定すると、アプリケーションの他の部分（バナー、ボタンなど）に影響する可能性があります。それを防ぐ最適な方法は、バナー テーマのスコープを限定することです。`{ComponentName}` を含むコンポーネント内で、（[テーマのインポート](#import-theme)を含む）スタイルを定義できます。

<DocsAside type="info">

コンポーネントが [Emulated](themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、グリッドにスタイルを設定するには、`::ng-deep` を使用してこのカプセル化を貫通する必要があります。
スタイルがコンポーネント外の要素に影響しないように、このステートメントを `:host` セレクター内にラップします。

</DocsAside>

```scss
// custom.component.scss

:host {
    ::ng-deep {
        @include banner($banner-theme);
    }
}
```

上記の構文により、カスタム バナー テーマがグリッドの行編集オーバーレイに適切に適用されます。

### カスタム テンプレート

行編集オーバーレイをさらにカスタマイズするには、カスタム テンプレートを渡して、`Done` ボタンと `Cancel` ボタンを個別にスタイル設定できます。

```html
<!-- in component.html -->
<{ComponentSelector}>
    <ng-template igxRowEditActions let-endRowEdit>
        <div class="custom-buttons">
            <button igxButton="icon" class="custom-button" igxRowEditTabStop (click)="endRowEdit(false)">
                <igx-icon>clear</igx-icon>
            </button>
            <button igxButton="icon" class="custom-button" igxRowEditTabStop (click)="endRowEdit(true)">
                <igx-icon>check</igx-icon>
            </button>
        </div>
    </ng-template>
</{ComponentSelector}>
```

カスタム ボタンを定義した後、<ApiLink type="function-button-theme" label="button-theme" kind="sass" module="index.html" /> を使用してそれらにスタイルを設定できます。`igx-button` のスタイル設定について詳しくは、この[トピック](../button.md#styling)を参照してください。`Done` と `Cancel` 用のカスタム テーマを作成できます。

```scss
// custom.component.scss
// ...

$button-theme: button-theme(
  $palette: $purple-palette
);

.custom-buttons {
    @include button($button-theme);
}
```

`@include` ステートメントを `.custom-buttons` 内にスコープ設定し、`Done` ボタンと `Cancel` ボタンにのみ適用されるようにします。

## 既知の問題と制限

- グリッドに <ApiLink type="{ComponentName}" member="primaryKey" /> が設定されておらず、リモート データ シナリオが有効になっている場合（ページング、並べ替え、フィルタリング、スクロールによって、グリッドに表示するデータを取得するためのリクエストがリモート サーバーに送信される場合）、データ リクエストの完了後に行は次の状態を失います。

- 行選択

- 行の展開/折りたたみ

- 行編集

- 行固定
