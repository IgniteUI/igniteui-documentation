# {Platform} {ComponentTitle} エクスポート

<PlatformBlock for="Blazor, React, WebComponents">

{ProductName} {ComponentTitle} は、グリッド ツールバー エクスポーター コンポーネントを通じてデータ エクスポート機能を提供します。表示されているデータを Excel、CSV、または PDF 形式にエクスポートできます。Excel エクスポートでは、フィルタリングや並べ替えなどの機能をサポートする MS Excel テーブル形式が使用されます。エクスポートを有効にするには、グリッドのツールバー内に <ApiLink type="GridToolbarExporter" /> を配置します。既定では、すべてのエクスポート形式が有効です。

## {ComponentTitle} データのエクスポート

IgniteUI Excel および PDF エクスポーターの使用を開始するには、まず app.module.ts ファイルで <ApiLink type="ExcelExporterService" /> および/または <ApiLink type="PdfExporterService" /> をインポートし、サービスを `providers` 配列に追加します。

```ts
// app.module.ts
import { ExcelExporterService, PdfExporterService } from 'igniteui-{Platform}';

@NgModule({
  providers: [ ExcelExporterService, PdfExporterService ]
})

export class AppModule {}
```

<DocsAside type="info">

v12.2.1 以降では、エクスポーター サービスはルートで提供されるため、AppModule providers で宣言する必要はなくなりました。

</DocsAside>

エクスポートを開始するには、コンポーネントのテンプレート内のボタンのハンドラーを使用できます。

```html
<{ComponentTitle} #{ComponentTitle} [data]="localData" [autoGenerate]="true"></{ComponentTitle}>
<button (click)="exportButtonHandler()">Export {ComponentTitle} to Excel</button>
<button (click)="exportPdfButtonHandler()">Export {ComponentTitle} to PDF</button>
```

<ApiLink type="ExcelExporterService" /> または <ApiLink type="PdfExporterService" /> 型のコンストラクター引数を定義することで、エクスポーター サービスにアクセスできます。{Platform} フレームワークがサービスのインスタンスを提供します。データを Excel または PDF にエクスポートするには、`Export` メソッドを呼び出し、最初の引数として {ComponentTitle} コンポーネントを渡します。

次のコードは、コンポーネント ファイルでエクスポート処理を実行します。

```ts
// component.ts
import { ExcelExporterService, ExcelExporterOptions, PdfExporterService, PdfExporterOptions } from 'igniteui-{Platform}';
import { {ComponentName} } from 'igniteui-{Platform}';

@ViewChild('{ComponentName}') public {ComponentName}: {ComponentName};

constructor(private excelExportService: ExcelExporterService, private pdfExportService: PdfExporterService) {
}

public exportButtonHandler() {
  this.excelExportService.export(this.{ComponentName}, new ExcelExporterOptions('ExportedDataFile'));
}

public exportPdfButtonHandler() {
  this.pdfExportService.export(this.{ComponentName}, new PdfExporterOptions('ExportedDataFile'));
}
```

すべてが正常に完了すると、{ComponentTitle} コンポーネントと 2 つのエクスポート ボタンが表示されます。対応するボタンを押すと、エクスポート処理がトリガーされ、ブラウザーは <ApiLink type="{ComponentName}" /> コンポーネントのデータを Excel または PDF 形式で含む "ExportedDataFile.xlsx" または "ExportedDataFile.pdf" という名前のファイルをダウンロードします。

## すべてのデータのエクスポート

**ページング** などのリモート操作を使用する場合、グリッドはデータ セット全体に一度にアクセスできないことがあります。このような場合は、[Excel エクスポート サービス](../exporter-excel.md)を使用し、可能であればデータ コレクション全体を渡すことをお勧めします。例:

```ts
public exportButtonHandler() {
  this.excelExportService.exportData(this.localData, new ExcelExporterOptions('ExportedDataFile'));
}
```

リモート データの PDF ダウンロードを提供する場合は、まず完全なデータ セットを取得してから `export` を呼び出し、ユーザーが期待する表示内容をドキュメントに反映させることを検討してください。

</PlatformBlock>

## 固定された列ヘッダーを持つグリッドのエクスポート

既定では、Excel エクスポーター サービスは、スクロール可能な（固定されていない）列ヘッダーを持つグリッドをエクスポートします。多くのシナリオでは、エクスポートされた Excel ファイルの上部にすべてのヘッダーを固定し、ユーザーがレコードをスクロールしても常に表示されるようにしたい場合があります。これを実現するには、`ExporterOption` <PlatformBlock for="Angular"><ApiLink type="IGridToolbarExportEventArgs" member="freezeHeaders" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="ExporterOptionsBase" member="freezeHeaders" /></PlatformBlock><PlatformBlock for="WebComponents"><ApiLink type="ExporterOptionsBase" member="freezeHeaders" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="GridToolbarExportEventArgs" member="freezeHeaders" /></PlatformBlock> を `true` に設定します。

<DocsAside type="info">

PDF エクスポートでは、ドキュメントの上部に列ヘッダー行が自動的に含まれるため、読者がファイルを開いたり印刷したりしたときに同じコンテキストを維持できます。

</DocsAside>

```ts
public exportButtonHandler() {
    const exporterOptions = new ExcelExporterOptions('ExportedDataFile');
    exporterOptions.freezeHeaders = true;
    this.excelExportService.export(this.grid, exporterOptions);
}
```

## エクスポートされるコンテンツのカスタマイズ

上記の例では、Excel エクスポーター サービスは利用可能なすべてのデータをエクスポートします。行または列全体のエクスポートをスキップしたい場合があります。これを実現するには、各列および各行に対してそれぞれ発生する <ApiLink type="{ComponentName}" member="columnExporting" /> イベントおよび/または <ApiLink type="{ComponentName}" member="rowExporting" /> イベントをサブスクライブし、イベント引数オブジェクトの `cancel` プロパティを `true` に設定してイベントをキャンセルします。

次の例では、ヘッダーが "Age" で、インデックスが 1 の場合に、その列をエクスポートから除外します。

```ts
// component.ts

this.excelExportService.columnExporting.subscribe((args: IColumnExportingEventArgs) => {
  if (args.header == 'Age' && args.columnIndex == 1) {
      args.cancel = true;
  }
});
this.excelExportService.export(this.{ComponentTitle}, new ExcelExporterOptions('ExportedDataFile'));
```

<ApiLink type="{ComponentName}" /> コンポーネントからデータをエクスポートする場合、エクスポート処理では行フィルタリングや列の非表示などの機能が考慮され、<ApiLink type="{ComponentName}" /> に表示されているデータのみがエクスポートされます。<ApiLink type="ExcelExporterOptions" /> または <ApiLink type="PdfExporterOptions" /> オブジェクトのプロパティを設定することで、フィルター処理された行または非表示の列を含めるようにエクスポーター サービスを構成できます。

</PlatformBlock>

| 最大ワークシート サイズ | Excel でサポートされる最大ワークシート サイズは、1,048,576 行 x 16,384 列です。                                                                                     |
| セルのスタイル設定 | Excel エクスポーター サービスは、セル コンポーネントに適用されたカスタム スタイルのエクスポートをサポートしていません。このようなシナリオでは、[Excel ライブラリ](../../excel-library.md)の使用をお勧めします。 |
