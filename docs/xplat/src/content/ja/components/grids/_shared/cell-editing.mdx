# {Platform} {ComponentTitle} セル編集

{Platform} {ComponentTitle} の {ProductName} セル編集は、{Platform} {ComponentTitle} コンポーネント内の個々のセルの内容に対して優れたデータ操作機能を提供し、React CRUD 操作用の強力な API を備えています。これは、スプレッドシート、データ テーブル、データ グリッドなどのアプリにおける基本機能であり、ユーザーは特定のセル内でデータを追加、編集、または更新できます。
既定では、{ProductName} のグリッドはセル編集で使用されます。また、**既定のセル編集テンプレート**により、列のデータ型 Top of Form に基づいて異なるエディターが使用されます。

さらに、データ更新アクション用に独自のカスタム テンプレートを定義し、変更をコミットおよび破棄する既定の動作をオーバーライドできます。

## セル編集

### UI による編集

編集可能なセルにフォーカスがある場合、次のいずれかの方法で特定のセルの編集モードに入ることができます。

- ダブルクリック時。
- シングルクリック時 - 以前に選択したセルが編集モードで、現在選択しているセルが編集可能な場合にのみ、シングルクリックで編集モードに入ります。以前に選択したセルが編集モードでなかった場合、シングルクリックでは編集モードに入らずにセルを選択します。
- <kbd>入力</kbd> キー押下時。
- <kbd>F2</kbd> キー押下時。

次のいずれかの方法で、変更を**コミットせずに**編集モードを終了できます。

- <kbd>エスケープ</kbd> キー押下時。
- **ソート**、**フィルタリング**、**検索**、**非表示**操作を実行した場合。

次のいずれかの方法で、編集モードを終了して変更を**コミット**できます。

- <kbd>入力</kbd> キー押下時。
- <kbd>F2</kbd> キー押下時。
- <kbd>タブ</kbd> キー押下時。
- 別のセルをシングルクリックした場合 - <ApiLink type="{ComponentName}" /> 内の別のセルをクリックすると、変更が送信されます。
- ページング、サイズ変更、ピン固定、移動などの操作では編集モードが終了し、変更が送信されます。

<DocsAside type="info">

垂直方向または水平方向にスクロールした場合、または <ApiLink type="{ComponentName}" /> の外側をクリックした場合でも、セルは編集モードのままです。これはセル編集と行編集の両方に有効です。

</DocsAside>

### {ComponentTitle} Excel スタイル編集

Excel スタイル編集を使用すると、ユーザーは Excel を使用する場合と同じようにセル間を移動し、非常に素早く編集できます。

このカスタム機能の実装は、<ApiLink type="{ComponentName}" /> のイベントを利用することで行えます。まずグリッドの keydown イベントにフックし、そこから 2 つの機能を実装できます。

<PlatformBlock for="React">

```tsx
const gridRef = useRef<IgrGrid>();
useEffect(() => {
    gridRef.current.addEventListener("keydown", handleKeyDown);
    return () => {
        gridRef.current.removeEventListener("keydown", handleKeyDown);
    };
}, []);
<IgrGrid ref={gridRef} autoGenerate={false} data={nwindData} primaryKey="ProductID">
</IgrGrid>
```

<DocsAside type="info">

React の合成 onKeyDown イベントではなく、ネイティブ ブラウザーの keydown イベントを使用しています。セルが編集モードに入り、ENTER キーを押して次の行へ移動すると、グリッドの編集機能がセル値を更新し、編集モードを閉じます。その結果、編集に使用される input 要素が DOM から削除されます。React のイベント システム最適化により、その時点で要素が React ツリーに存在しなくなるため、onKeyDown 合成イベントはグリッドまでバブリングしません。したがって、期待される動作を確実にするには、ネイティブ イベント リスナーを使用する必要があります。

</DocsAside>

</PlatformBlock>

public keydownHandler(event) {
  const key = event.keyCode;
  const grid = this.grid;
  const activeElem = grid.navigation.activeNode;

  if ((key >= 48 && key <= 57) ||
      (key >= 65 && key <= 90) ||
      (key >= 97 && key <= 122)) {
        // Number or Alphabet upper case or Alphabet lower case
        const columnName = grid.getColumnByVisibleIndex(activeElem.column).field;
        const cell = grid.getCellByColumn(activeElem.row, columnName);
        if (cell && !grid.crudService.cellInEditMode) {
            grid.crudService.enterEditMode(cell);
            cell.editValue = event.key;
        }
    }
}

function handleKeyDown(event: KeyBoardEvent) {
    const code = event.code;
    const grid = event.currentTarget as IgrGrid;
    const activeElem = grid.selectedCells[0];

    if ((event.code >= "Digit0" && event.code <= "Digit9") || (event.code >= "KeyA" && event.code <= "KeyZ")
        || (event.code >= "Numpad0" && event.code <= "Numpad9" && event.code !== "Enter" && event.code !== "NumpadEnter")) {
        if (activeElem && !activeElem.editMode) {
            activeElem.editMode = true;
            activeElem.editValue = event.key;
        } else if (activeElem && activeElem.editMode) {
            event.preventDefault();
            activeElem.editValue = activeElem.editValue + event.key;
        }
    }
}

### スタイル ライブラリのインポート

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

これで、{ProductName} テーマ エンジンによって公開されているすべての関数を使用できます。

### パレットの定義

index ファイルを正しくインポートしたら、使用できるカスタム パレットを作成します。好みの 2 つの色を定義し、それらを使用して [igx-palette](../themes/palettes.md) でパレットを作成しましょう。

```scss
$white: #fff;
$blue: #4567bb;

$color-palette: palette($primary: $white, $secondary: $blue);
```

### テーマの定義

これで、パレットを使用してテーマを定義できます。セルは <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> によってスタイル設定されるため、これを使用して <ApiLink type="{ComponentName}" /> のテーマを生成できます。

```scss
$custom-grid-theme: grid-theme(
    $cell-editing-background: $blue,
    $cell-edited-value-color: $white,
    $cell-active-border-color: $white,
    $edit-mode-color: color($color-palette, "secondary", 200)
);
```

### テーマの適用

テーマを適用する最も簡単な方法は、グローバル スタイル ファイルで `sass` `@include` ステートメントを使用することです。

```scss
@include grid($custom-grid-theme);
```

この方法では、テーマはアプリケーション内の**すべての**グリッドに適用されます。このカスタム スタイル設定を特定のコンポーネントにのみ適用したい場合は、テーマのスコープを設定する必要があります。

### スコープ付きコンポーネント テーマ

カスタム テーマが特定のコンポーネントにのみ影響するようにするには、先ほど定義したすべてのスタイルをグローバル スタイル ファイルからカスタム コンポーネントのスタイル ファイルに移動できます（`index` ファイルの [import](#importing-style-library) を含む）。

この方法では、{Platform} の [ViewEncapsulation](https://angular.io/api/core/Component#encapsulation) により、スタイルはカスタム コンポーネントにのみ適用されます。

 <DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、グリッドにスタイルを設定するために `::ng-deep` を使用してこのカプセル化を貫通する必要があります。

 </DocsAside>

 <DocsAside type="info">

スタイルがコンポーネント外の要素に影響しないように、ステートメントを `:host` セレクター内にラップします。

 </DocsAside>

```scss
:host {
    ::ng-deep {
            @include grid($custom-grid-theme);
        }
    }
}
```
