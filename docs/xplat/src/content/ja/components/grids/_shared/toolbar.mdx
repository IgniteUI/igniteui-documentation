# {Platform} {ComponentTitle} ツールバー

{ProductName} ツールバーは、{Platform} {ComponentTitle} における UI 操作用のコンテナーです。{Platform} ツールバーは {Platform} コンポーネントの上部、つまり <ApiLink type="{ComponentName}" /> に配置され、その水平方向のサイズに一致します。ツールバー コンテナーには、任意のカスタム コンテンツ、または事前定義された UI コントロールのセットを配置できます。{Platform} {ComponentTitle} のデフォルト セットには、次が含まれます。

- 列の非表示
- 列の固定
- Excel エクスポート
- 高度なフィルタリング

ツールバーと事前定義された UI コンポーネントは {Platform} イベントをサポートし、開発者向けの API を公開します。

## 子グリッドを持つツールバー

<PlatformBlock for="Angular">

IgxHierarchicalGrid の子グリッドの実装方法と DI スコープの動作には特定の制限があるため、子グリッドのスコープでツールバーを使用する場合には注意点があります。
`igx-row-island` タグ内でツールバー コンポーネントを定義する場合は、必ず IgxGridToolbar を使用し、提供されたグリッド インスタンスを入力プロパティとしてツールバー自体に渡してください。
これにより、テンプレートのスコープ内で常に正しいグリッド インスタンスを使用できます。

```html
<igx-hierarchical-grid>
    <!--...-->
    <igx-row-island>
        <!--
            You can name the binding from igxGridToolbar however you want. Just make sure to use
            it inside the template if you need to access the grid instance.
        -->
        <igx-grid-toolbar [grid]="gridRef" *igxGridToolbar="let gridRef">
            <igx-grid-toolbar-title>Child toolbar {{ gridRef.parentIsland.level }}</igx-grid-toolbar-title>
        </igx-grid-toolbar>
    </igx-row-island>
    <!--...-->
</igx-hierarchical-grid>
```

</PlatformBlock>

<PlatformBlock for="WebComponents, React">

## 機能

ツールバーは、グリッド全体に影響するロジックやインタラクションを分離するのに非常に適しています。

上記のように、グリッドからのデータの制御、列の非表示、列の固定、高度なフィルタリング、およびエクスポートのためのデフォルト コンポーネントを提供するように構成できます。

これらの機能は、{ProductName} スイートのカード コンポーネントに似たパターンに従うことで、互いに独立して有効にできます。

### アクション

ツールバー インタラクション コンポーネントを含む任意の内容を指定できます。

constructor() {
    var pinTool = this.pinTool = document.getElementById('pinTool') as IgcGridToolbarPinningComponent;

### データのエクスポート

<ComponentBlock for="HierarchicalGrid">

<DocsAside type="info">

<ApiLink type="{ComponentName}" /> または階層内の任意の子グリッドをエクスポートする場合、エクスポートされるデータは、それぞれのグリッドに属する行のフラットなコレクションになります
（子グリッドはエクスポートされたデータに含まれません）。

</DocsAside>

</ComponentBlock>

他のツールバー アクションと同様に、エクスポートは標準で <ApiLink type="GridToolbarExporter" /> を通じて提供されます。

<PlatformBlock for="Angular">

エクスポート コンポーネントは、対象データ形式 <ApiLink type="ExcelExporterService" /> および `CSVExporterService` に対応するサービスを使用します。つまり、対応するサービスが依存性注入チェーンを通じて提供されていない場合、コンポーネントは何もエクスポートできません。

{Platform} の DI について復習が必要な場合は、[公式ガイド](https://{Platform}.io/guide/dependency-injection)を確認してください。以下は、アプリケーションですべてのエクスポート サービスを有効にする方法を示すサンプル スニペットです。

```typescript
// app.module.ts

import { IgxExcelExporterService, IgxCsvExporterService } from 'igniteui-{Platform}';

@NgModule({
    //...
    providers: [IgxExcelExporterService, IgxCsvExporterService ]
})
export class AppModule { ... }
```

<DocsAside type="info">

v12.2.1 以降では、エクスポーター サービスは root で提供されるため、AppModule の providers で宣言する必要はなくなりました。

</DocsAside>

</PlatformBlock>

## スタイル設定

ツールバーのスタイル設定を始めるには、すべてのテーマ関数とコンポーネント ミックスインが含まれている index ファイルをインポートする必要があります。

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

まず、新しいパレットを作成します。

```scss
$my-dark-palette: palette(
    $primary: #2466ff,
    $secondary: #FFCD0F,
    $surface: #2a2b2f,
    $grays: #fff,
);

$my-dark-color: color($my-dark-palette, 'surface');
```

次に、<ApiLink type="function-grid-toolbar-theme" label="grid-toolbar-theme" kind="sass" module="index.html" /> を拡張する新しいテーマを作成し、`$background-color` および `$title-text-color` パラメーターを変更します。

```scss
$dark-grid-toolbar-theme: grid-toolbar-theme(
    $palette: $my-dark-palette,
    $background-color: $my-dark-color,
    $title-text-color: color($my-dark-palette, 'secondary'),
    $dropdown-background: $my-dark-color,
);
```

ツールバーの列アクション メニューにテーマを適用するには、`column-actions-theme` コンポーネントのテーマを変更する必要があります。

```scss
$dark-column-actions-theme: column-actions-theme(
    $palette: $my-dark-palette,
    $title-color: color($my-dark-palette, 'secondary'),
    $background-color: color($my-dark-palette, 'surface')
);
```

列アクションでは他のコンポーネント（igx-button、igx-checkbox、igx-input-group）を使用しているため、新しいツールバー テーマに合わせてそれらのテーマを変更する必要があります。

```scss
$dark-button-theme: button-theme(
    $palette: $my-dark-palette,
    $outlined-background: color($my-dark-palette, 'secondary'),
    $outlined-hover-background: color($my-dark-palette, 'grays', 100),
    $outlined-hover-text-color: color($my-dark-palette, 'secondary')
);

$dark-checkbox-theme: checkbox-theme(
    $palette: $my-dark-palette,
    $tick-color: $my-dark-color,
);

$dark-input-group-theme: input-group-theme(
    $palette: $my-dark-palette
);
```

最後の手順は、新しく作成したテーマを **include** することです。

```scss
:host {
    @include grid-toolbar($dark-grid-toolbar-theme);
    @include column-actions($dark-column-actions-theme);
    @include checkbox($dark-checkbox-theme);
    @include input-group($dark-input-group-theme);
    @include button($dark-button-theme);
}
```

<DocsAside type="info">

`$legacy-support` が `false(default)` に設定されている場合は、次のようにコンポーネントの css 変数を含めます。

</DocsAside>

```scss
:host {
    @include css-vars($dark-grid-toolbar-theme);
    @include css-vars($dark-column-actions-theme);
    @include css-vars($dark-checkbox-theme);
    @include css-vars($dark-input-group-theme);
    @include css-vars($dark-button-theme);
}
```

<DocsAside type="info">

コンポーネントが [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation を使用している場合、`::ng-deep` を使用してこのカプセル化を `penetrate` する必要があります。

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid-toolbar($dark-grid-toolbar-theme);
        @include column-actions($dark-column-actions-theme);
        @include checkbox($dark-checkbox-theme);
        @include input-group($dark-input-group-theme);
        @include button($dark-button-theme);
    }
}
```
