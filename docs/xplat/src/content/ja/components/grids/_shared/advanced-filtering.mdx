# {Platform} {ComponentTitle} 高度なフィルタリング

{Platform} {ComponentTitle} の {ProductName} 高度なフィルタリングでは、<ApiLink type="{ComponentName}" /> 内のすべての列にわたるフィルタリング条件を持つさまざまなグループを作成できるダイアログが提供され、データを操作できます。

## {Platform} {ComponentTitle} 高度なフィルタリングの例

<Sample src="/{ComponentSample}/advanced-filtering-options" height={530} alt="{Platform} {ComponentTitle} Advanced Filtering Example" />

## インタラクション

高度なフィルタリング ダイアログを開くには、グリッド ツールバーの **Advanced Filtering** ボタンをクリックします。高度なフィルターが適用されていない場合は、**AND** または **OR** で結合されたフィルタリング条件のグループを作成することから始めます。その後、フィルタリング条件またはサブグループを追加できます。

フィルタリング条件を追加するには、<ApiLink type="Column" member="filterable" /> 列のいずれか、列 <ApiLink type="Column" member="dataType" /> に基づく演算子、および演算子が単項でない場合は値を選択する必要があります。条件が確定されると、条件情報を含むチップが表示されます。チップにホバーするかクリックすると、そのチップを変更したり、その直後に別の条件またはグループを追加したりするオプションが表示されます。

複数のフィルタリング条件チップを選択すると、グループを作成するオプションまたはフィルターを削除するオプションを含むコンテキスト メニューが表示されます。選択した条件でグループを作成することを選択した場合、新しく作成されたグループは、選択された条件のうち最上位の条件が配置されていた場所に表示されます。

グループを選択するには、その垂直線をクリックすることもできます。この線は、結合条件 **AND** または **OR** に基づいて色付けされています。単一のグループが選択されている場合、そのフィルタリング ロジックを変更する、グループ化を解除する、または削除するオプションを含むコンテキスト メニューが表示されます。

フィルタリング条件とグループの作成が完了してデータをフィルターするには、**Apply** ボタンをクリックします。高度なフィルターを変更したものの変更を保持したくない場合は、**Cancel** ボタンをクリックします。**Clear Filter** ボタンをクリックして高度なフィルターをクリアすることもできます。

        searchVal: 'a',
        ignoreCase: true

    const subTree = new FilteringExpressionsTree(FilteringLogic.Or);

        ignoreCase: true

        searchVal: 'c',

## スタイル設定

高度なフィルタリング ダイアログのスタイル設定を開始するには、すべてのテーマ関数とコンポーネント mixin が含まれている **index** ファイルをインポートする必要があります。

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

高度なフィルタリング ダイアログは、**filtering-row-background** パラメーターを使用して、グリッドのテーマから背景色を取得します。背景を変更するには、カスタム テーマを作成する必要があります。

```scss
$custom-grid: grid-theme(
    $filtering-row-background: #FFCD0F
);
```

高度なフィルタリング ダイアログ内には、ボタン、チップ、ドロップダウン、入力などの他のコンポーネントがあるため、それぞれに個別のテーマを作成する必要があります。

```scss
$custom-button: button-theme(
    $disabled-color: gray,

);

$custom-button-group: button-group-theme(
    $item-background:  #292826,

);

$custom-input-group: input-group-theme(
    $box-background: #4a4a4a,

);

$custom-chip: chip-theme(
    $background: #FFCD0F,

);

$custom-drop-down: drop-down-theme(
    $background-color: #292826,

);
```

この例では、一覧表示されたコンポーネントのパラメーターの一部のみを変更しましたが、<ApiLink type="function-button-theme" label="button-theme" kind="sass" module="index.html" />、<ApiLink type="function-button-group-theme" label="button-group-theme" kind="sass" module="index.html" />、<ApiLink type="function-chip-theme" label="chip-theme" kind="sass" module="index.html" />、<ApiLink type="function-drop-down-theme" label="drop-down-theme" kind="sass" module="index.html" />、<ApiLink type="function-input-group-theme" label="input-group-theme" kind="sass" module="index.html" /> テーマには、それぞれのスタイル設定を制御するためのさらに多くのパラメーターが用意されています。

最後の手順は、コンポーネント mixin をそれぞれ対応するテーマで **インクルード** することです。また、高度なフィルタリング ダイアログ内の他の要素にもいくつかのスタイルを追加します。

```scss
@include grid($custom-grid);
igx-advanced-filtering-dialog {
    @include button($custom-button);
    @include button-group($custom-button-group);
    @include input-group($custom-input-group);
    @include chip($custom-chip);
    @include drop-down($custom-drop-down);
    .igx-filter-empty__title {
        color: #FFCD0F
    }
    .igx-advanced-filter__header {
        color: #FFCD0F
    }
    .igx-filter-tree__expression-actions igx-icon {
        color: #FFCD0F
    }
    .igx-filter-tree__expression-actions igx-icon:hover {
        color: #ffe482
    }
    .igx-filter-tree__expression-actions igx-icon:focus {
        color: #ffe482
    }
    .igx-filter-contextual-menu {
        border: 1px solid #FFCD0F
    }
    .igx-filter-contextual-menu__close-btn {
        position: absolute !important;
        background: #292826 !important;
        border-color: #FFCD0F !important;
    }
    .igx-input-group__input::placeholder {
        color: gray;
    }
}
```

<DocsAside type="info">

ほとんどのコンポーネントの mixin は `igx-advanced-filtering-dialog` 内にスコープ設定しているため、これらのカスタム テーマは高度なフィルタリング ダイアログ内にネストされたコンポーネントのみに影響します。そうしないと、アプリケーション内の他のボタン、チップ、入力、およびドロップダウンにも影響します。

</DocsAside>

### カラー パレットの定義

先ほどのように色の値をハードコーディングする代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、色に関してより高い柔軟性を実現できます。

`igx-palette` は、渡されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します。

```scss
$yellow-color: #FFCD0F;
$black-color: #292826;
$dark-palette: palette($primary: $yellow-color, $secondary: $black-color);
```

そして <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> を使用すると、パレットから色を簡単に取得できます。

```scss
$custom-grid: grid-theme(
    $filtering-row-background: color($dark-palette, "secondary", 400)
);

$custom-button: button-theme(
    $disabled-color: color($dark-palette, "secondary", 100),

);

$custom-button-group: button-group-theme(
    $item-background: color($dark-palette, "secondary", 400),

);

$custom-input-group: input-group-theme(
    $box-background: color($dark-palette, "secondary", 200),

);

$custom-chip: chip-theme(
    $background: color($dark-palette, "primary", 400),

);

$custom-drop-down: drop-down-theme(
    $background-color: color($dark-palette, "secondary", 400),

);
```

<DocsAside type="info">

`igx-color` と `igx-palette` は、色の生成と取得のための強力な関数です。これらの使用方法に関する詳細なガイダンスについては、[パレット](../themes/sass/palettes.md) トピックを参照してください。

</DocsAside>

### スキーマの使用

テーマ エンジンをさらに活用すると、[**スキーマ**](../themes/sass/schemas.md) の利点を活かした堅牢で柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。

各コンポーネントに用意されている 2 つの定義済みスキーマのうち 1 つを拡張します。この場合は、<ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" />、<ApiLink type="variable-_light-button" label="light-button" kind="sass" module="index.html" />、<ApiLink type="variable-_light-button-group" label="light-button-group" kind="sass" module="index.html" />、<ApiLink type="variable-_light-chip" label="light-chip" kind="sass" module="index.html" />、<ApiLink type="variable-_light-input-group" label="light-input-group" kind="sass" module="index.html" />、および <ApiLink type="variable-_light-drop-down" label="light-drop-down" kind="sass" module="index.html" /> スキーマです。

```scss
$grid-dark-palette: palette($primary: #11bd7b, $secondary: #e32057, $info: $black-color);

$custom-grid-schema: extend($_light-grid,
    (
        filtering-row-background:(
           color: ("info")
        )
    )
);

$custom-button-schema: extend($_light-button,
    (
        disabled-color:(
           color: ("secondary", 100)
        ),

    )
);

$custom-button-group-schema: extend($_light-button-group,
    (
        item-background:(
           color: ("secondary", 400)
        ),

    )
);

$custom-input-group-schema: extend($_light-input-group,
    (
        box-background:(
           color: ("secondary", 200)
        ),

    )
);

$custom-chip-schema: extend($_light-chip,
    (
        background:(
           color: ("primary", 400)
        ),

    )
);

$custom-drop-down-schema: extend($_light-drop-down,
    (
        background-color:(
           color: ("secondary", 400)
        ),

    )
);
```

カスタム スキーマを適用するには、グローバルの 1 つ（<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> または <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />）を **拡張** する必要があります。これは基本的に、カスタム スキーマを持つコンポーネントを指し示すもので、その後、それを対応するコンポーネント テーマに追加します。

```scss
$custom-light-schema: extend($light-schema,(
    igx-grid: $custom-grid-schema,
    igx-button: $custom-button-schema,
    igx-button-group: $custom-button-group-schema,
    igx-input-group: $custom-input-group-schema,
    igx-chip: $custom-chip-schema,
    igx-drop-down: $custom-drop-down-schema
));

$custom-grid: grid-theme(
    $palette: $grid-dark-palette,
    $schema: $custom-light-schema
);

$custom-button: button-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-button-group: button-group-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-input-group: input-group-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-chip: chip-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-drop-down: drop-down-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);
```

上記で示したのと同じ方法で、テーマをインクルードすることを忘れないでください。

## スタイル設定

定義済みテーマに加えて、利用可能な [CSS プロパティ](../theming-grid.md) の一部を設定することで、グリッドをさらにカスタマイズできます。
一部の色を変更したい場合は、まずグリッドにクラスを設定する必要があります。

<PlatformBlock for="WebComponents">

```html
<{ComponentSelector} class="grid"></{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<{ComponentSelector} className="grid"></{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<{ComponentSelector} class="grid"></{ComponentSelector}>
```

</PlatformBlock>

次に、このクラスに関連する CSS プロパティを設定します。

```css
.grid {
    --ig-query-builder-header-foreground: #512da8;
    --ig-query-builder-color-expression-group-and: #eb0000;
    --ig-query-builder-color-expression-group-or: #0000f3;
    --ig-query-builder-subquery-header-background: var(--ig-gray-300);
    --ig-query-builder-subquery-border-color: var(--ig-warn-500);
    --ig-query-builder-subquery-border-radius: 4px;
}
```
