# {Platform} {ComponentTitle} 状態の永続化

<PlatformBlock for="Angular">

{Platform} {ComponentTitle} の {ProductName} 状態の永続化により、開発者はグリッドの状態を簡単に保存および復元できます。<ApiLink type="GridState" /> が {Platform} <ApiLink type="{ComponentName}" /> に適用されると、開発者が任意のシナリオで状態の永続化を実現するために使用できる <ApiLink type="GridState" member="getState" /> および <ApiLink type="{ComponentName}" member="setState" /> メソッドが公開されます。

## 使用方法

<PlatformBlock for="Angular">

<ApiLink type="GridState" member="getState" /> メソッドは、グリッドの状態をシリアル化された JSON 文字列で返すため、開発者はそれを取得して任意のデータ ストレージ (データベース、クラウド、ブラウザーの localStorage など) に保存できます。このメソッドは最初の省略可能なパラメーター `Serialize` を受け取り、これにより <ApiLink type="GridState" member="getState" /> が <ApiLink type="IGridState" /> オブジェクトを返すか、シリアル化された JSON 文字列を返すかが決まります。

開発者は、機能名、または機能名の配列を第 2 引数として渡すことで、特定の機能の状態のみ、または複数の機能の状態のみを取得することを選択できます。

</PlatformBlock>

使いやすい単一ポイント API により、わずか数行のコードで完全な状態永続化機能を実現できます。**以下のコードをコピーして貼り付けてください** - ユーザーが現在のページを離れるたびに、グリッドの状態がブラウザーの `LocalStorage` オブジェクトに保存されます。ユーザーがメイン ページに戻るたびに、グリッドの状態が復元されます。必要なデータを取得するために、複雑な高度なフィルタリング式や並べ替え式を毎回構成する必要はもうありません。一度だけ実行し、あとは以下のコードがユーザーのために処理します。

<PlatformBlock for="Angular">

```typescript
@ViewChild(IgxGridStateDirective, { static: true })
public state!: IgxGridStateDirective;

public ngOnInit() {
    this.router.events.pipe(take(1)).subscribe((event: NavigationStart) => {
        this.saveGridState();
    });
}

public ngAfterViewInit() {
    this.restoreGridState();
}

public saveGridState() {
    const state = this.state.getState() as string;
    window.sessionStorage.setItem('grid1-state', state);
}

public restoreGridState() {
    const state = window.sessionStorage.getItem('grid1-state');
    this.state.setState(state);
}

```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```typescript
constructor() {
    window.addEventListener("load", () => { this.restoreGridState(); });
    window.addEventListener("beforeunload", () => { this.saveGridState(); });
}

// Using methods that work with IgcGridStateInfo object.
public saveGridState() {
    const state = this.gridState.getState();
    window.localStorage.setItem('grid-state', JSON.stringify(state));
}

public restoreGridState() {
    const state = window.localStorage.getItem('grid-state');
    if (state) {
        this.gridState.applyState(JSON.parse(state));
    }
}

// Or using string alternative methods.
public saveGridStateString() {
    const state = this.gridState.getStateAsString();
    window.localStorage.setItem('grid-state', state);
}

public restoreGridStateString() {
    const state = window.localStorage.getItem('grid-state');
    if (state) {
        this.gridState.applyStateFromString(state);
    }
}
```

</PlatformBlock>

## 列の復元

<ApiLink type="GridState" /> は、デフォルトでは列テンプレート、列フォーマッターなどを永続化しません ([制限事項](state-persistence.md#limitations) を参照)。これらのいずれの復元も、アプリケーション レベルのコードで実現できます。テンプレート化された列に対してこれを行う方法を示します。

1 - テンプレート参照変数 (以下の例では `#activeTemplate`) を定義し、`ColumnInit` イベントにイベント ハンドラーを割り当てます。
