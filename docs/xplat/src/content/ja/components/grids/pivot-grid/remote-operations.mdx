# {Platform} ピボット グリッドのリモート操作

ピボット データがリモート サービスで既にグループ化および集計されており、クライアントで追加処理を行う必要がないシナリオでは、ピボット グリッドをカスタムの空ストラテジを使用するように構成できます。これにより、クライアントでのデータ処理をスキップし、データをそのまま直接表示できます。

<PlatformBlock for="Angular">

```typescript
public pivotConfigHierarchy: IPivotConfiguration = {
    columnStrategy: NoopPivotDimensionsStrategy.instance(),
    rowStrategy: NoopPivotDimensionsStrategy.instance(),
}
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```typescript
public pivotConfigHierarchy: IgcPivotConfiguration = {
    columnStrategy: IgcNoopPivotDimensionsStrategy.instance(),
    rowStrategy: IgcNoopPivotDimensionsStrategy.instance(),
}
```

</PlatformBlock>

ユーザーは、既に集計済みのデータをピボット グリッドに渡すことで、特定のシナリオを実現できます。
データの見た目に関する要件と、ピボット ビュー内の階層に関するいくつかの仕様があります。たとえば、**rows** ディメンションで階層を宣言するには、次のようにします。

```typescript
rows: [
    {
        memberName: 'AllProducts',
        memberFunction: () => 'All Products',
        enabled: true,
        childLevel: {
            memberName: 'ProductCategory',
            enabled: true
        }
    }
]
```

集計済みデータの例は次のようになります。

```typescript
public aggregatedData = [
    {
        ProductCategory: 'All', AllProducts: 'All Products', All: 1000, 'All-Bulgaria': 774, 'All-USA': 829, 'All-Uruguay': 524, AllProducts_records: [
            { ProductCategory: 'Clothing', 'All-Bulgaria': 774, 'All-USA': 296, 'All-Uruguay': 456 },
            { ProductCategory: 'Bikes', 'All-Uruguay': 68 },
            { ProductCategory: 'Accessories', 'All-USA': 293 },
            { ProductCategory: 'Components', 'All-USA': 240 }
        ]
    }
];
```

Pivot グリッドは、ピボット計算を行うために使用するオブジェクト キー フィールドを提供します。

- <ApiLink type="PivotKeys" member="children" label="Children" /> - 階層構築用の子を格納するフィールドです。グループ化された値と、その値に基づくすべての pivotGridRecords とのマップを表します。階層の作成中に何らかの処理を行う必要がある、非常に特殊なシナリオで利用できます。一般的な使用ではこれを変更する必要はありません。
- <ApiLink type="PivotKeys" member="records" label="Records" /> - 元のデータ レコードへの参照を格納するフィールドです。上記の例では **AllProducts\_records** として確認できます。このプロパティと同じ名前のフィールドをデータに設定しないでください。データ レコードに **records** プロパティがある場合は、<ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" /> を使用して別の一意の値を指定できます。
- <ApiLink type="PivotKeys" member="aggregations" label="Aggregations" /> - 集計値を格納するフィールドです。階層の作成中に適用されるものであり、一般的なシナリオでは変更しないでください。
- <ApiLink type="PivotKeys" member="level" label="Level" /> - 階層に基づくディメンション レベルを格納するフィールドです。このプロパティと同じ名前のフィールドをデータに設定しないでください。データ レコードに **level** プロパティがある場合は、<ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" /> を使用して別の一意の値を指定できます。
- <ApiLink type="PivotKeys" member="columnDimensionSeparator" label="ColumnDimensionSeparator" /> - 一意の列フィールド値を生成するときに使用される区切り文字です。上記の例の **All-Bulgaria** に含まれるダッシュ (**-**) です。
- <ApiLink type="PivotKeys" member="rowDimensionSeparator" label="RowDimensionSeparator" /> - 一意の行フィールド値を生成するときに使用される区切り文字です。上記の例の **AllProducts\_records** に含まれるアンダースコア (**\_**) です。<ApiLink type="PivotKeys" member="records" label="Records" /> および <ApiLink type="PivotKeys" member="level" label="Level" /> フィールドを作成するときに使用されます。

これらはすべて、<ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" /> の一部である <ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> プロパティに格納されており、既定のピボット キーを変更するために使用できます。
これらの既定値は次のとおりです。

```typescript
export const DEFAULT_PIVOT_KEYS = {
    aggregations: 'aggregations', records: 'records', children: 'children', level: 'level',
    rowDimensionSeparator: '_', columnDimensionSeparator: '-'
};
```

<ApiLink type="NoopPivotDimensionsStrategy" /> および <ApiLink type="PivotConfiguration" member="columnStrategy" label="ColumnStrategy" /> に <ApiLink type="PivotConfiguration" member="rowStrategy" label="RowStrategy" /> を設定すると、データ パイプによって行われるデータのグループ化と集計はスキップされますが、ピボット グリッドでは、期待どおりにピボット ビューをレンダリングするために、行、列、値、フィルターの宣言が引き続き必要です。

<PlatformBlock for="Angular">

```typescript
public pivotConfig: IPivotConfiguration = {
    rows: [
        {
            memberName: 'AllProducts',
            memberFunction: () => 'All Products',
            enabled: true,
            childLevel: {
                memberName: 'ProductCategory',
                enabled: true
            }
        }
    ],
    columns: [
        {
            memberName: 'All',
            enabled: true,
            childLevel: {
                memberName: 'Country',
                enabled: true
            }
        }
    ],
    values: [
        {
            member: 'UnitsSold',
            aggregate: {
                aggregator: IgxPivotNumericAggregate.sum,
                key: 'sum',
                label: 'Sum'
            },
            enabled: true
        },
    ]
}
```

</PlatformBlock>

データが構成と一致していることが重要です。最良の結果を得るには、集計済みデータに追加のフィールドを含めず、提供されたデータのフィールドを行または列として未宣言のままにしないでください。<ApiLink type="PivotGrid" /> コンポーネントは <ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> に基づいてデータを構築するため、構成と集計済みデータが適切に一致していることが期待されます。

並べ替えやフィルタリングなど、その他のリモート データ操作についても、関連する空ストラテジである <ApiLink type="GridBaseDirective" member="filterStrategy" label="FilterStrategy" />、<ApiLink type="GridBaseDirective" member="sortStrategy" label="SortStrategy" /> を設定することで、データ処理をスキップできます。

<PlatformBlock for="Angular">

```html
<{PivotGridSelector} [filterStrategy]="noopFilterStrategy" [sortStrategy]="noopSortStrategy">
</{PivotGridSelector}>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<{PivotGridSelector} filter-strategy="noopFilterStrategy" sort-strategy="noopSortStrategy">
</{PivotGridSelector}>
```

</PlatformBlock>

```typescript
public noopFilterStrategy = NoopFilteringStrategy.instance();
public noopSortStrategy = NoopSortingStrategy.instance();
```
