# {Platform} {PivotGridTitle} 機能

ピボット グリッド コンポーネントとフラット グリッド コンポーネントは共通の基底から継承しているため、いくつかの機能を共有しています。

<DocsAside type="info">

一部の機能はピボット テーブルのコンテキストでは意味のある動作を持たないため、<ApiLink type="PivotGrid" /> では有効にできません。これには以下が含まれます。

- CRUD 操作
- グループ化
- 行/列の固定
- 集計
- ページング

</DocsAside>

{PivotGridTitle} コンポーネントには、以下で説明するように、ディメンションに関連する追加の機能があります。

## ディメンションのフィルタリング

すべてのディメンション（フィルター、行、列）は、チップ UI または API を介してフィルタリングできます。この機能は組み込まれており、デフォルトで有効になっています。

<DocsAside type="info">

フィルタリング ディメンションを使用すると、ピボット ビューの一部ではないデータ値を除外できます。

</DocsAside>

フィルタリング UI は、ディメンション チップのフィルター アイコンから開くことができ、一意のディメンション値に対して Excel 形式のフィルタリングを行えます。

<DocsAside type="info">

すべてのフィルタリング チップを表示する十分なスペースがない場合、{PivotGridTitle} は収まりきらなかったものをドロップダウンに表示します。エンドユーザーはそこでアクセスして操作できます。

</DocsAside>

ディメンションは、<ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> のディメンション構成で、ディメンションの `filter` プロパティを使用して初期状態でフィルタリングすることもできます。
たとえば、関連するフィルター条件を持つ新しい <ApiLink type="GridBaseDirective" member="filteringExpressionsTree" label="FilteringExpressionsTree" /> に設定できます。

```typescript
public filterExpTree = new FilteringExpressionsTree(FilteringLogic.And);

constructor() {
    this.filterExpTree.filteringOperands = [
        {
            condition: IgxStringFilteringOperand.instance().condition('equals'),
            fieldName: 'SellerName',
            searchVal: 'Stanley'
        }
    ];
}

public pivotConfigHierarchy: IPivotConfiguration = {
    filters: [
        {
            memberName: 'SellerName',
            enabled: true,
            filter: this.filterExpTree
        }
    ]
}
```

</PlatformBlock>

## ディメンションの並べ替え

`rows` または `columns` のディメンション値は、関連するチップまたは API を介して並べ替えることができます。この機能は組み込まれており、デフォルトで有効になっています。

関連するチップをクリックするとディメンションが並べ替えられ、その結果、ディメンション値が昇順/降順に並べ替えられます。

並べ替えは、ディメンション定義の `sortDirection` プロパティを介して初期状態で適用することもできます。

<PlatformBlock for="Angular, WebComponents">

```typescript
public pivotConfigHierarchy: IPivotConfiguration = {
    rows: [
        {
            memberName: 'SellerName',
            enabled: true,
            sortDirection: SortingDirection.Asc
        }
    ]
}
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
    var pivotConfiguration = new IgbPivotConfiguration();
    var rowDimension = new IgbPivotDimension() {
        MemberName = "SellerName",
        Enabled = true,
        SortDirection = SortingDirection.Asc
    };
    pivotConfiguration.Rows.Add(rowDimension);
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
const dimension: IgrPivotDimension = {
    memberName: "SellerName",
    enabled: true,
    sortDirection: SortingDirection.Asc
};
```

</PlatformBlock>

## ディメンションのサイズ変更

行ディメンションは、列のサイズ変更と同様に、セルの右端にあるサイズ変更インジケーターを介してサイズ変更できます。
サイズ変更インジケーターをダブルクリックするか、関連する API（<ApiLink type="PivotGrid" member="autoSizeRowDimension" label="AutoSizeRowDimension" />）を使用して、自動サイズ調整することもできます。

ディメンション定義で利用可能な <ApiLink type="PivotGrid" member="width" label="Width" /> プロパティを使用して、初期状態で異なるサイズを設定することもできます。

<PlatformBlock for="Angular, WebComponents">

```typescript
public pivotConfigHierarchy: IPivotConfiguration = {
    rows: [
        {
            memberName: 'Country',
            enabled: true,
            width: '400px'
        }
    ]
}
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
    var pivotConfiguration = new IgbPivotConfiguration();
    var rowDimension = new IgbPivotDimension() {
        MemberName = "SellerName",
        Enabled = true,
        Width = "400px"
    };
    pivotConfiguration.Rows.Add(rowDimension);
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
const igrPivotDimension2: IgrPivotDimension = {
    memberName: "SellerName",
    enabled: true,
    width = "400px"
};
```

</PlatformBlock>

## ディメンションの選択

{PivotGridTitle} は、基本グリッドと同様に有効化される単一選択をサポートしています。例:

<PlatformBlock for="Angular">

```html
<{PivotGridSelector} #grid1 [rowSelection]="'single'" [data]="data" [pivotConfiguration]="pivotConfigHierarchy">
</{PivotGridSelector}>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<{PivotGridSelector} id="grid" row-selection="single">
</{PivotGridSelector}>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<{PivotGridSelector} PivotConfiguration="PivotConfiguration"
        Data="PivotSalesData"
        Name="grid"
        RowSelection=GridSelectionMode.Single
        @ref="grid">
</{PivotGridSelector}>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrPivotGrid data={pivotData} pivotConfiguration={pivotConfiguration} rowSelection="single">
</IgrPivotGrid>
```

</PlatformBlock>

複数の行または列ディメンションがあり、複数の行/列にまたがるグループが作成される場合、選択は選択したグループに属するすべてのセルに適用されます。

## スーパー コンパクト モード

<ApiLink type="PivotGrid" /> コンポーネントには <ApiLink type="PivotGrid" member="superCompactMode" label="SuperCompactMode" /> 入力があります。これは、一度に多数のセルを画面上に表示する必要がある場合に適しています。有効にすると、このオプションは {PivotGridTitle} の `--ig-size` CSS 変数を無視します。<ApiLink type="PivotGrid" member="superCompactMode" label="SuperCompactMode" /> を有効にすると、--ig-size オプションを持たない各子コンポーネント（small など）の `<ApiLink pkg="core" type="Chip" />` も `<ApiLink type="PivotGrid" member="superCompactMode" label="SuperCompactMode" />` に設定されます。

## 追加の集計列

`column` ディメンションが階層を定義している場合、{PivotGridTitle} はグループ内のすべての列の集計を累積する追加の集計/合計列をレンダリングします。グループが折りたたまれている場合は、集計列のみが残ります。また、グループが展開されている場合は、追加の集計列がグループの末尾に表示されます。

## インタラクション

### キーボード ナビゲーション

`{PivotGridSelector}` のキーボード ナビゲーションは、`{GridSelector}` のものと同様に動作します。{PivotGridTitle} は、`rows`、`columns`、`values` の 3 つの領域に分割されます。`rows` と `columns` の領域はナビゲーションの目的ではヘッダーと見なされ、`values` の領域は本体です。
キーボードの矢印キーを使用すると、現在の領域内でのみアクティブ要素を移動できます。

### ディメンションのドラッグ & ドロップ

ディメンションはチップで表され、ドラッグ & ドロップできます。
すべてのチップは、ドラッグ & ドロップによってそれぞれの領域内で順序を変更できます。
`rows`、`column`、`filter`（ディメンション チップ）のチップは、これらの領域のいずれからでも、他の任意の領域の任意の場所へ移動できます。
これらの領域のチップを `values` 領域へ移動することはできず、`values` 領域のチップをディメンション領域へ移動することもできません。

<DocsAside type="info">

{PivotGridTitle} のチップを Pivot Data Selector へ移動することはできず、Pivot Data Selector の項目を {PivotGridTitle} へ移動することもできません。

</DocsAside>
