# {Platform} ピボット グリッドの概要

{Platform} ピボット グリッドは、大量の多次元データを集計し、クロス集計形式で表現するために使用されます。データの集計は、簡単かつ迅速に並べ替え、グループ化、またはフィルター処理できます。このようなデータには、合計、平均、その他の統計情報を含めることができます。エンドユーザーは、必要に応じてドラッグ アンド ドロップ操作によりピボット テーブル レイアウトを変更できます。

{Platform} ピボット グリッドは、データをピボット テーブルで表示し、ユーザーが提供されたデータ セットに対して複雑な分析を実行できるようにします。この高度なピボット グリッド コントロールは、大量のデータを整理、要約、フィルター処理するために使用され、その後クロス テーブル形式で表示されます。{Platform} ピボット グリッドの主な機能は、行ディメンション、列ディメンション、集計、およびフィルターです。

<ApiLink type="PivotGrid" /> は、ユーザーが多次元ピボット テーブル構造でデータを構成して表示できるようにします。
行と列は個別のデータ グループを表し、データ セルの値は集計を表します。これにより、単純なフラット データ セットに基づく複雑なデータ分析が可能になります。<ApiLink type="PivotGrid" /> は、さまざまなディメンションと値の簡単な構成に加え、フィルター処理や並べ替えなど、それらに対する追加のデータ操作を提供する機能豊富なピボット テーブルです。

これは、**rows**、**columns**、**values** の 3 つの主要なディメンションによって定義されます。**rows** と **columns** は、グリッドの行と列に表示されるグループ化された構造を定義します。**values** は、グループの関連値を計算して表示するために使用される集計フィールドと集計を定義します。

### ディメンションの構成

各基本ディメンション構成には、提供された **data** のフィールドと一致する <ApiLink type="PivotDimension" member="memberName" label="MemberName" /> が必要です。

複数の兄弟ディメンションを定義でき、関連する行または列ディメンション領域に、より複雑なネストされたグループを作成します。

ディメンションは、対応するチップを使用してドラッグ アンド ドロップにより並べ替えたり、ある領域から別の領域へ移動したりできます。

### 値の構成

値の構成には、提供された **data** のフィールドと一致する **member** が必要です。または、より複雑なカスタム シナリオ向けにカスタム **aggregator** 関数を定義できます。標準で、データ フィールドのデータ型に応じて使用できる 4 つの定義済み集計があります。

- `PivotNumericAggregate` - 数値フィールド用。
  次の集計関数が含まれます: `SUM`、`AVG`、`MIN`、`MAX`、`COUNT`。
- `PivotDateAggregate` - 日付フィールド用。
  次の集計関数が含まれます: `LATEST`、`EARLIEST`、`COUNT`。
- `PivotTimeAggregate` - 時刻フィールド用。
  次の集計関数が含まれます: `LATEST`、`EARLIEST`、`COUNT`。
- `PivotAggregate` - その他のデータ型用。これは基本集計です。
  次の集計関数が含まれます: `COUNT`。

現在の集計関数は、値チップのドロップダウンを使用してランタイムに変更できます。既定では、フィールドのデータ型に基づいて使用可能な集計のリストが表示されます。集計のカスタム リストは、<ApiLink type="PivotValue" member="aggregateList" label="AggregateList" /> プロパティを介して設定することもできます。例:

    return data.map((x:any) => x.UnitPrice * x.UnitsSold).reduce((a:number, b:number) => Math.min(a, b));

### Enable プロパティ

<ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> は、<ApiLink type="PivotGrid" /> コンポーネントの現在の状態を記述するインターフェイスです。これにより、開発者はデータのフィールドを **rows**、**columns**、**filters**、または **values** として宣言できます。この構成では、これらの各要素を個別に有効または無効にできます。有効な要素のみがピボット グリッドの現在の状態に含まれます。<ApiLink type="PivotDataSelector" /> コンポーネントは同じ構成を利用し、有効および無効のすべての要素のリストを表示します。それぞれに対して、適切な状態のチェックボックスがあります。エンドユーザーは、これらのチェックボックスを使用してさまざまな要素を切り替えることで、ピボットの状態を簡単に調整できます。
`Enable` プロパティは、指定された <ApiLink type="PivotDimension" /> または <ApiLink type="PivotValue" /> がアクティブで、ピボット グリッドによってレンダリングされるピボット ビューに参加するかどうかを制御します。

### 構成の自動生成

<ApiLink type="PivotGrid" member="autoGenerateConfig" label="AutoGenerateConfig" /> プロパティは、データ ソース フィールドに基づいてディメンションと値を自動的に生成します。

- 数値フィールド:
  - `<ApiLink type="PivotValue" />` aggregator を使用して PivotNumericAggregate.sum として作成されます。
  - values コレクションに追加され、既定で有効になります。

- 非数値フィールド:
  - <ApiLink type="PivotDimension" /> として作成されます。
  - 既定で無効になります。
  - columns コレクションに追加されます。

- 日付フィールド (最初の `date` フィールドのみが有効になり、その他の `date` フィールドには非数値フィールドのルールが適用されます):
  - <ApiLink type="PivotDateDimension" /> として作成されます
  - 既定で有効になります
  - rows コレクションに追加されます。

この機能により、開発者はディメンションと値を手動で指定することなく、ピボット ビューを迅速に作成できます。ピボット グリッドの横にピボット セレクターを配置すると、ユーザーは必要に応じてディメンションと値を有効にしたり並べ替えたりできます。

### ピボット値計算キー

ピボット グリッドは、ピボット計算を行うために使用するオブジェクト キー フィールドをカスタマイズできます。
それらがどのように使用されるかの詳細なビューは、以下のサンプル データで確認できます。ここでは、すでに集計された値を確認できます。

```json
[
    {
        ProductCategory: 'All', AllProducts: 'All Products', All: 1000, 'All-Bulgaria': 774, 'All-USA': 829, 'All-Uruguay': 524,
        AllProducts_records: [
            { ProductCategory: 'Clothing', 'All-Bulgaria': 774, 'All-USA': 296, 'All-Uruguay': 456 },
            { ProductCategory: 'Bikes', 'All-Uruguay': 68 },
            { ProductCategory: 'Accessories', 'All-USA': 293 },
            { ProductCategory: 'Components', 'All-USA': 240 }
        ]
    }
];
```

これらはすべて、<ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> の一部である **pivotKeys** プロパティに格納され、既定のピボット キーを変更するために使用できます。

- **children** - 階層構築のために子を格納するフィールドです。グループ化された値と、その値に基づくすべての pivotGridRecords からのマップを表します。階層の作成中に何かを行う必要がある非常に特定のシナリオで利用できます。一般的な使用ではこれを変更する必要はありません。
- **records** - 元のデータ レコードへの参照を格納するフィールドです。上記の例で確認できます - **AllProducts\_records**。このプロパティと同じ名前のフィールドをデータに設定しないでください。データ レコードに **records** プロパティがある場合は、**pivotKeys** を使用して、別の一意の値を指定できます。
- **aggregations** - 集計値を格納するフィールドです。階層の作成中に適用され、一般的なシナリオでは変更しないでください。
- **level** - 階層に基づくディメンション レベルを格納するフィールドです。このプロパティと同じ名前のフィールドをデータに設定しないでください。データ レコードに **level** プロパティがある場合は、**pivotKeys** を使用して、別の一意の値を指定できます。
- **columnDimensionSeparator** - 一意の列フィールド値を生成するときに使用される区切り文字です。例の値 **All-Bulgaria** のダッシュ (**-**) です。
- **rowDimensionSeparator** - 一意の行フィールド値を生成するときに使用される区切り文字です。**records** および **level** フィールドの作成時に使用されます。

既定値は次のとおりです。

<PlatformBlock for="Angular, WebComponents">

```typescript
{
    aggregations: 'aggregations',
    records: 'records',
    children: 'children',
    level: 'level',
    rowDimensionSeparator: '_',
    columnDimensionSeparator: '-'
};
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
@code {
    {
        aggregations: 'aggregations',
        records: 'records',
        children: 'children',
        level: 'level',
        rowDimensionSeparator: '_',
        columnDimensionSeparator: '-'
    };
}
```

</PlatformBlock>

<DocsAside type="info">

既定のキーを含むデータ フィールド値がある場合は、現在使用していない他の記号に一致する区切り文字へ必ず変更してください。そうしないと、集計値の計算および表示で予期しない動作が発生する可能性があります。

</DocsAside>

<PlatformBlock for="Blazor">

Blazor で <ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" /> をオーバーライドする場合、現在は新しい PivotKeys オブジェクトを割り当てると既定のものが完全に置き換えられるため、他のすべてのキーを定義する必要があります。

```razor
@code {
    var pivotConfiguration = new IgbPivotConfiguration();
    pivotConfiguration.PivotKeys = new IgbPivotKeys()
    {
        Aggregations = "aggregations",
        Records = "records",
        Children = "children",
        Level = "level",
        RowDimensionSeparator = "_",
        ColumnDimensionSeparator = "^"
    };
}
```

</PlatformBlock>

## 既知の問題と制限

| 制限                                                                              | 説明                                                                                                                                                                          |
| --------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 列を宣言的に設定することはサポートされていません。                                         | ピボット グリッドは <ApiLink type="PivotGrid" member="columns" label="Columns" /> 構成に基づいて列を生成するため、基本グリッドのように宣言的に設定することはサポートされていません。そのような列は無視されます。 |
| ディメンション/値に重複する <ApiLink type="PivotDimension" member="memberName" label="MemberName" /> または <ApiLink type="PivotValue" member="member" label="Member" /> プロパティ値を設定する。 | これらのプロパティは、各ディメンション/値ごとに一意である必要があります。重複すると、最終結果からデータが失われる可能性があります。                                                            |
| 行選択は **Single** モードでのみサポートされます。                                     | 複数選択は現在サポートされていません。                                                                                                                                       |

<PlatformBlock for="Angular">

\| ディメンション メンバーのマージでは大文字と小文字が区別されます| ピボット グリッドはグループを作成し、同じ (大文字と小文字を区別する) 値をマージします。ただし、ディメンションは <ApiLink type="PivotDimension" member="memberFunction" label="MemberFunction" /> を提供しており、そこで変更できます。<ApiLink type="PivotDimension" member="memberFunction" label="MemberFunction" /> の結果が比較され、表示値として使用されます。|

</PlatformBlock>
