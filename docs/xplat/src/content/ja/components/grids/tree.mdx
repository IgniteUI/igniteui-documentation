# {Platform} ツリーの概要

{ProductName} ツリーは、TreeView コンポーネントとも呼ばれ、ツリー状の UI 内で展開可能なデータ構造を視覚化する高性能コントロールであり、子項目に対してオンデマンド読み込みを適用できます。{ProductName} ツリーは、ノードの展開と折りたたみ、ネストされたアプリ ナビゲーションなどの機能も提供します。{ProductName} ツリー ノードは、手動で生成することも、バインドされたデータ ソースから生成することもできます。

エンドユーザーにとって、これは異なるアプリ ページ間を簡単に移動し、選択、チェックボックス、テキスト、アイコン、画像などを使用できることを意味します。

{ProductName} ツリー コンポーネントを使用すると、ユーザーは親子関係を維持しながら階層データをツリー ビュー構造で表現できるほか、対応するデータ モデルなしで静的なツリー ビュー構造を定義できます。その主な目的は、エンドユーザーが階層データ構造内で視覚化およびナビゲーションできるようにすることです。<ApiLink pkg="core" type="Tree" /> コンポーネントは、オンデマンド読み込み機能、項目のアクティブ化、組み込みチェックボックスによる項目の複数選択およびカスケード選択、組み込みキーボード ナビゲーションなども提供します。

### 項目の操作

<ApiLink pkg="core" type="TreeItem" /> は展開または折りたたむことができます:

- 項目の展開インジケーターをクリックする (既定の動作)。
- <ApiLink pkg="core" type="Tree" /> <ApiLink pkg="core" type="Tree" member="toggleNodeOnClick" label="ToggleNodeOnClick" /> プロパティが `true` に設定されている場合に、項目をクリックする。

<PlatformBlock for="WebComponents">

```html
<igc-tree toggle-node-on-click="true">
    <igc-tree-item label="North America">
        <igc-tree-item label="United States"></igc-tree-item>
        <igc-tree-item label="Canada"></igc-tree-item>
        <igc-tree-item label="Mexico"></igc-tree-item>
    </igc-tree-item>
    <igc-tree-item label="South America">
        <igc-tree-item label="Brazil"></igc-tree-item>
    </igc-tree-item>
</igc-tree>
```

</PlatformBlock>

既定では、複数の項目を同時に展開できます。この動作を変更して、一度に 1 つのブランチのみを展開できるようにするには、<ApiLink pkg="core" type="Tree" member="singleBranchExpand" label="SingleBranchExpand" /> プロパティを有効にできます。これにより、項目が展開されると、同じレベルで既に展開されている他のすべてのブランチが折りたたまれます。

<PlatformBlock for="WebComponents">

```html
<igc-tree single-branch-expand="true">
    <igc-tree-item label="North America">
        <igc-tree-item label="United States"></igc-tree-item>
        <igc-tree-item label="Canada"></igc-tree-item>
        <igc-tree-item label="Mexico"></igc-tree-item>
    </igc-tree-item>
    <igc-tree-item label="South America">
        <igc-tree-item label="Brazil"></igc-tree-item>
    </igc-tree-item>
</igc-tree>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

さらに、<ApiLink pkg="core" type="Tree" /> は、項目操作のために次の API メソッドを提供します:

- <ApiLink type="Tree" member="Expand" label="Tree.Expand" /> - すべての項目を展開します。項目配列が渡された場合は、指定された項目のみを展開します。
- <ApiLink type="Tree" member="Collapse" label="Tree.Collapse" /> - すべての項目を折りたたみます。項目配列が渡された場合は、指定された項目のみを折りたたみます。
- <ApiLink type="Tree" member="Select" label="Tree.Select" /> - すべての項目を選択します。項目配列が渡された場合は、指定された項目のみを選択します。`selection` イベントは発行しません。
- <ApiLink type="Tree" member="Deselect" label="Tree.Deselect" /> - すべての項目の選択を解除します。項目配列が渡された場合は、指定された項目のみの選択を解除します。`selection` イベントは発行しません。

</PlatformBlock>

## {Platform} ツリーの選択

{ProductName} ツリー コンポーネントで項目選択を設定するには、その <ApiLink pkg="core" type="Tree" member="selection" label="Selection" /> プロパティを設定するだけです。このプロパティは、**None**、**Multiple**、**Cascade** の 3 つのモードを受け入れます。以下では、それぞれについて詳しく見ていきます。

### None

<ApiLink pkg="core" type="Tree" /> では、既定で項目選択が無効になっています。ユーザーは UI 操作を通じて項目を選択または選択解除できませんが、これらの操作は提供されている API メソッドを通じて引き続き実行できます。

### Multiple

<ApiLink pkg="core" type="Tree" /> で複数項目選択を有効にするには、<ApiLink pkg="core" type="Tree" member="selection" label="Selection" /> プロパティを **multiple** に設定するだけです。これにより、すべての項目にチェックボックスが描画されます。各項目には、選択済みまたは未選択の 2 つの状態があります。このモードは複数選択をサポートします。

<PlatformBlock for="WebComponents">

```html
<igc-tree selection="multiple">
</igc-tree>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbTree Selection=TreeSelection.Multiple>
</IgbTree>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrTree selection="multiple" />
```

</PlatformBlock>

### Cascade

<ApiLink pkg="core" type="Tree" /> でカスケード項目選択を有効にするには、selection プロパティを **cascade** に設定するだけです。これにより、すべての項目にチェックボックスが描画されます。

<PlatformBlock for="WebComponents">

```html
<igc-tree selection="Cascade">
</igc-tree>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbTree Selection=TreeSelection.Cascade>
</IgbTree>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrTree selection="cascade" />
```

</PlatformBlock>

このモードでは、親の選択状態は子の選択状態に完全に依存します。親に選択済みの子と未選択の子が混在している場合、そのチェックボックスは不確定状態になります。

## キーボード ナビゲーション

<ApiLink pkg="core" type="Tree" /> のキーボード ナビゲーションは、ユーザーに多様なキーボード操作を提供します。この機能は既定で有効になっており、ユーザーは項目間を移動できます。

<ApiLink pkg="core" type="Tree" /> ナビゲーションは W3C アクセシビリティ標準に準拠しており、使いやすくなっています。

**キーの組み合わせ**

- <kbd>↓</kbd> - 次の表示項目に移動します。項目をアクティブとしてマークします。最後の項目にいる場合は何もしません。
- <kbd>CTRL</kbd> + <kbd>↓</kbd> - 次の表示項目に移動します。最後の項目にいる場合は何もしません。
- <kbd>↑</kbd> - 前の表示項目に移動します。項目をアクティブとしてマークします。最初の項目にいる場合は何もしません。
- <kbd>CTRL</kbd> + <kbd>↑</kbd> - 前の表示項目に移動します。最初の項目にいる場合は何もしません。
- <kbd>←</kbd> - 展開された親項目上では、それを折りたたみます。項目が折りたたまれている場合、または子を持たない場合は、その親項目に移動します。親項目がない場合は何もしません。
- <kbd>→</kbd> - 展開された親項目上では、その項目の最初の子に移動します。折りたたまれた親項目上では、それを展開します。項目に子がない場合は何もしません。
- <kbd>HOME</kbd> - 最初の項目に移動します。
- <kbd>END</kbd> - 最後の表示項目に移動します。
- <kbd>TAB</kbd> - ツリーの外側にある、ページ上の次のフォーカス可能な要素に移動します。
- <kbd>SHIFT</kbd> + <kbd>TAB</kbd> - ツリーの外側にある、ページ上の前のフォーカス可能な要素に移動します。
- <kbd>SPACE</kbd> - 現在の項目の選択を切り替えます。ノードをアクティブとしてマークします。
- <kbd>SHIFT</kbd> + <kbd>SPACE</kbd> - 選択が有効な場合、<kbd>SHIFT</kbd> を押したまま Space を押した項目とアクティブな項目の間にあるすべての項目の選択を切り替えます。
- <kbd>ENTER</kbd> - フォーカスされている項目をアクティブ化します。項目にリンクが含まれている場合、そのリンクを開きます。
- <kbd>\*</kbd> - 項目と、同じレベルにあるすべての兄弟項目を展開します。

選択が有効な場合、エンドユーザーによる項目の選択は、描画されたチェックボックスを通じてのみ許可されます。どちらの選択タイプも複数選択を許可するため、次のマウスおよびキーボード操作を使用できます:

- <kbd>Click</kbd> - 項目のチェックボックス上で実行された場合、選択が有効であれば項目の選択を切り替えます。それ以外の場合は項目にフォーカスします
- <kbd>SHIFT</kbd> + <kbd>Click</kbd> - 項目のチェックボックス上で実行された場合、選択が有効であれば、<kbd>SHIFT</kbd> を押したままクリックした項目とアクティブな項目の間にあるすべての項目の選択を切り替えます

## {Platform} ツリーのオンデマンド読み込み

{ProductName} ツリーは、ユーザーができるだけ早く表示できるように、サーバーから取得するデータ量を最小限に抑える必要がある形で描画できます。この動的データ読み込みアプローチでは、ユーザーが項目を展開した後にのみ、その特定の親項目の子が取得されます。この仕組みはオンデマンド読み込みとも呼ばれ、任意のリモート データで動作するように簡単に構成できます。

<Sample src="/grids/tree/load-on-demand" height={400} alt="{Platform} Tree Load On Demand Example" />

ユーザーが展開アイコンをクリックすると、それは読み込みインジケーターに置き換えられます。loading プロパティが false に解決されると、読み込みインジケーターが消え、子が読み込まれます。

`loadingIndicator` スロットを使用して、読み込み領域にカスタム スロット コンテンツを提供できます。このようなスロットが定義されていない場合は、<ApiLink pkg="core" type="CircularProgress" /> が使用されます。

### 仮想化によるオンデマンド読み込み

{ProductName} ツリーで多数の子をオンデマンドで読み込むと、ツリー項目は設計上宣言的に定義されるため、パフォーマンスに悪影響を与える可能性があります。次のデモでは、[@lit-labs/virtualizer](https://github.com/lit/lit/tree/main/packages/labs/virtualizer) ライブラリを使用して、子ツリー項目を仮想化されたコンテナー内に描画する方法を示します。その結果、子の表示されている部分のみが DOM に描画されるため、パフォーマンスが向上します。

## スタイル設定

以下に示す公開されている CSS パーツの一部を使用して、<ApiLink pkg="core" type="TreeItem" /> の外観を変更できます:

| パーツ名       | 説明                                              |
| --------------- | -------------------------------------------------------- |
| `wrapper` | ツリー項目のラッパー。                           |
| `selected` | 選択状態を示します。`wrapper` に適用されます。    |
| `focused` | フォーカス状態を示します。`wrapper` に適用されます。     |
| `active` | アクティブ状態を示します。`wrapper` に適用されます。   |
| `indicator` | ツリー項目の展開インジケーター。                   |
| `label` | ツリー項目のコンテンツ。                                   |
| `text` | ツリー項目に表示されるテキスト。                            |
| `select` | 選択が有効な場合のツリー項目のチェックボックス。 |

これらの CSS パーツを使用すると、<ApiLink pkg="core" type="Tree" /> コンポーネントの外観を次のようにカスタマイズできます:

```css
igc-tree-item {
    --background-active: var(--ig-secondary-500);
    --foreground-active: var(--ig-secondary-500-contrast);
}
```
