# {Platform} Tree の概要

{ProductName} Tree は、TreeView コンポーネントとも呼ばれる高性能コントロールで、ツリー状の UI 内で展開可能なデータ構造を視覚化し、子アイテムにオンデマンド読み込みを適用できます。{ProductName} Tree は、ノードの展開と折りたたみ、ネストされたアプリ ナビゲーションなどの機能も提供します。{ProductName} Tree ノードは手動で生成することも、バインドされたデータ ソースから生成することもできます。

エンドユーザーは、さまざまなアプリ ページ間を簡単に移動し、選択、チェックボックス、テキスト、アイコン、画像などを使用できます。

{ProductName} Tree コンポーネントを使用すると、ユーザーは親子関係を維持しながら階層データをツリー ビュー構造で表現できるほか、対応するデータ モデルなしで静的なツリー ビュー構造を定義できます。その主な目的は、エンドユーザーが階層データ構造内で視覚化およびナビゲートできるようにすることです。<ApiLink pkg="core" type="Tree" /> コンポーネントは、オンデマンド読み込み機能、アイテムのアクティブ化、組み込みチェックボックスによるアイテムの複数選択およびカスケード選択、組み込みキーボード ナビゲーションなども提供します。

### アイテムの操作

<ApiLink pkg="core" type="TreeItem" /> は展開または折りたたみできます。

- アイテムの展開インジケーターをクリックする（既定の動作）。
- <ApiLink pkg="core" type="Tree" /> <ApiLink pkg="core" type="Tree" member="toggleNodeOnClick" label="ToggleNodeOnClick" /> プロパティが `true` に設定されている場合、アイテムをクリックする。

<PlatformBlock for="WebComponents">

```html
<igc-tree toggle-node-on-click="true">
    <igc-tree-item label="North America">
        <igc-tree-item label="United States"></igc-tree-item>
        <igc-tree-item label="Canada"></igc-tree-item>
        <igc-tree-item label="Mexico"></igc-tree-item>
    </igc-tree-item>
    <igc-tree-item label="South America">
        <igc-tree-item label="Brazil"></igc-tree-item>
    </igc-tree-item>
</igc-tree>
```

</PlatformBlock>

既定では、複数のアイテムを同時に展開できます。この動作を変更し、一度に 1 つのブランチのみを展開できるようにするには、<ApiLink pkg="core" type="Tree" member="singleBranchExpand" label="SingleBranchExpand" /> プロパティを有効にします。これにより、アイテムが展開されると、同じレベルで既に展開されている他のすべてのブランチが折りたたまれます。

```html
    </igc-tree-item>

```

</PlatformBlock>

<PlatformBlock for="WebComponents">

さらに、<ApiLink pkg="core" type="Tree" /> はアイテム操作のために次の API メソッドを提供します。

- <ApiLink type="Tree" member="Expand" label="Tree.Expand" /> - すべてのアイテムを展開します。アイテム配列が渡された場合、指定されたアイテムのみを展開します。
- <ApiLink type="Tree" member="Collapse" label="Tree.Collapse" /> - すべてのアイテムを折りたたみます。アイテム配列が渡された場合、指定されたアイテムのみを折りたたみます。
- <ApiLink type="Tree" member="Select" label="Tree.Select" /> - すべてのアイテムを選択します。アイテム配列が渡された場合、指定されたアイテムのみを選択します。`selection` イベントは発行しません。
- <ApiLink type="Tree" member="Deselect" label="Tree.Deselect" /> - すべてのアイテムの選択を解除します。アイテム配列が渡された場合、指定されたアイテムのみの選択を解除します。`selection` イベントは発行しません。

</PlatformBlock>

## {Platform} Tree の選択

{ProductName} Tree コンポーネントでアイテム選択を設定するには、その <ApiLink pkg="core" type="Tree" member="selection" label="Selection" /> プロパティを設定するだけです。このプロパティは、**None**、**Multiple**、**Cascade** の 3 つのモードを受け入れます。以下では、それぞれについて詳しく見ていきます。

### None

<ApiLink pkg="core" type="Tree" /> では、既定でアイテム選択が無効になっています。ユーザーは UI 操作を通じてアイテムを選択または選択解除できませんが、提供されている API メソッドを通じてこれらの操作を実行できます。

### Multiple

<ApiLink pkg="core" type="Tree" /> で複数アイテム選択を有効にするには、<ApiLink pkg="core" type="Tree" member="selection" label="Selection" /> プロパティを **multiple** に設定するだけです。これにより、各アイテムにチェックボックスが描画されます。各アイテムには、選択済みまたは未選択の 2 つの状態があります。このモードは複数選択をサポートします。

<PlatformBlock for="WebComponents">

```html
<igc-tree selection="multiple">
</igc-tree>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbTree Selection=TreeSelection.Multiple>
</IgbTree>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrTree selection="multiple" />
```

</PlatformBlock>

### Cascade

<ApiLink pkg="core" type="Tree" /> でカスケード アイテム選択を有効にするには、selection プロパティを **cascade** に設定するだけです。これにより、各アイテムにチェックボックスが描画されます。

<PlatformBlock for="WebComponents">

```html
<igc-tree selection="Cascade">
</igc-tree>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbTree Selection=TreeSelection.Cascade>
</IgbTree>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrTree selection="cascade" />
```

</PlatformBlock>

このモードでは、親の選択状態はその子の選択状態に完全に依存します。親に選択済みの子と未選択の子が混在している場合、そのチェックボックスは不確定状態になります。

## キーボード ナビゲーション

<ApiLink pkg="core" type="Tree" /> のキーボード ナビゲーションは、ユーザーに多様なキーボード操作を提供します。この機能は既定で有効になっており、ユーザーがアイテム間を移動できます。

<ApiLink pkg="core" type="Tree" /> ナビゲーションは W3C アクセシビリティ標準に準拠しており、便利に使用できます。

**キーの組み合わせ**

- <kbd>↓</kbd> - 次の表示アイテムに移動します。アイテムをアクティブとしてマークします。最後のアイテム上にある場合は何もしません。
- <kbd>コントロール</kbd> + <kbd>↓</kbd> - 次の表示アイテムに移動します。最後のアイテム上にある場合は何もしません。
- <kbd>↑</kbd> - 前の表示アイテムに移動します。アイテムをアクティブとしてマークします。最初のアイテム上にある場合は何もしません。
- <kbd>コントロール</kbd> + <kbd>↑</kbd> - 前の表示アイテムに移動します。最初のアイテム上にある場合は何もしません。
- <kbd>←</kbd> - 展開された親アイテム上では、それを折りたたみます。アイテムが折りたたまれている、または子を持たない場合は、その親アイテムに移動します。親アイテムがない場合は何もしません。
- <kbd>→</kbd> - 展開された親アイテム上では、そのアイテムの最初の子に移動します。折りたたまれた親アイテム上では、それを展開します。アイテムに子がない場合は何もしません。
- <kbd>ホーム</kbd> - 最初のアイテムに移動します。
- <kbd>終了</kbd> - 最後の表示アイテムに移動します。
- <kbd>タブ</kbd> - ツリーの外にある、ページ上の次のフォーカス可能な要素に移動します。
- <kbd>シフト</kbd> + <kbd>タブ</kbd> - ツリーの外にある、ページ上の前のフォーカス可能な要素に移動します。
- <kbd>スペース</kbd> - 現在のアイテムの選択を切り替えます。ノードをアクティブとしてマークします。
- <kbd>シフト</kbd> + <kbd>スペース</kbd> - 選択が有効な場合、<kbd>シフト</kbd> を押しながら Space を押したアイテムとアクティブなアイテムの間にあるすべてのアイテムの選択を切り替えます。
- <kbd>入力</kbd> - フォーカスされているアイテムをアクティブ化します。アイテム内にリンクがある場合、そのリンクを開きます。
- <kbd>\*</kbd> - アイテムと同じレベルにあるすべての兄弟アイテムを展開します。

選択が有効な場合、エンドユーザーによるアイテム選択は描画されたチェックボックスを通じてのみ許可されます。どちらの選択タイプも複数選択を許可するため、次のマウスおよびキーボード操作を使用できます。

- <kbd>クリック</kbd> - アイテムのチェックボックス上で実行した場合、選択が有効であればアイテムの選択を切り替えます。それ以外の場合は、アイテムにフォーカスします
- <kbd>シフト</kbd> + <kbd>クリック</kbd> - アイテムのチェックボックス上で実行した場合、選択が有効であれば、<kbd>シフト</kbd> を押しながらクリックしたアイテムとアクティブなアイテムの間にあるすべてのアイテムの選択を切り替えます

## {Platform} Tree のオンデマンド読み込み

{ProductName} Tree は、ユーザーができるだけ早く表示できるよう、サーバーから取得するデータ量を最小限に抑える方法で描画できます。この動的データ読み込みアプローチでは、ユーザーがアイテムを展開した後にのみ、その特定の親アイテムの子が取得されます。この仕組みはオンデマンド読み込みとも呼ばれ、任意のリモート データで動作するように簡単に構成できます。

<Sample src="/grids/tree/load-on-demand" height={400} alt="{Platform} Tree Load On Demand Example" />

ユーザーが展開アイコンをクリックすると、読み込みインジケーターに置き換わります。loading プロパティが false に解決されると、読み込みインジケーターが消え、子が読み込まれます。

`loadingIndicator` スロットを使用して、読み込み領域にカスタム スロット コンテンツを指定できます。そのようなスロットが定義されていない場合は、<ApiLink pkg="core" type="CircularProgress" /> が使用されます。

### 仮想化を使用したオンデマンド読み込み

{ProductName} Tree でより多くの子をオンデマンドで読み込むと、ツリー アイテムは設計上宣言的に定義されるため、パフォーマンスに悪影響を及ぼす可能性があります。次のデモでは、[@lit-labs/virtualizer](https://github.com/lit/lit/tree/main/packages/labs/virtualizer) ライブラリを使用して、子ツリー アイテムを仮想化コンテナー内に描画する方法を示します。その結果、子の表示されている部分のみが DOM に描画されるため、パフォーマンスが向上します。

## スタイル設定

以下に示す公開済み CSS パーツの一部を使用して、<ApiLink pkg="core" type="TreeItem" /> の外観を変更できます。

| パーツ名        | 説明                                                     |
| --------------- | -------------------------------------------------------- |
| `wrapper` | ツリー アイテムのラッパー。                             |
| `selected` | 選択状態を示します。`wrapper` に適用されます。    |
| `focused` | フォーカス状態を示します。`wrapper` に適用されます。 |
| `active` | アクティブ状態を示します。`wrapper` に適用されます。 |
| `indicator` | ツリー アイテムの展開インジケーター。                   |
| `label` | ツリー アイテムのコンテンツ。                           |
| `text` | ツリー アイテムの表示テキスト。                         |
| `select` | 選択が有効な場合のツリー アイテムのチェックボックス。   |

これらの CSS パーツを使用して、<ApiLink pkg="core" type="Tree" /> コンポーネントの外観を次のようにカスタマイズできます。

```css
igc-tree-item {
    --background-active: var(--ig-secondary-500);
    --foreground-active: var(--ig-secondary-500-contrast);
}
```
