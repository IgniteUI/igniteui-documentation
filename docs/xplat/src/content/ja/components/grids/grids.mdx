# 最速の {Platform} データ グリッド

<PlatformBlock for="React">

{Platform} データ グリッドまたはデータ テーブルは、表形式のデータを一連の行と列で表示するために使用されるコンポーネントです。テーブルとも呼ばれるデータ グリッドは、Microsoft Excel などの一般的なソフトウェアによってデスクトップの世界でよく知られています。グリッドは長い間デスクトップ プラットフォームで利用可能でしたが、{Platform} UI などの Web アプリ UI の一部となったのはごく最近のことです。最新のグリッドとテーブルは複雑になる場合があり、データバインディング、編集、Excel のようなフィルタリング、カスタム並べ替え、グループ化、行の並べ替え、行と列の固定、行の集計、Excel、CSV、pdf 形式へのエクスポートなど、さまざまな機能を含むことがあります。

</PlatformBlock>

<PlatformBlock for="WebComponents">

{Platform} データ テーブルとも呼ばれる {Platform} データ グリッドは、データを一連の行と列としてバインドして表示できるデータ可視化コンポーネントです。Microsoft Excel など、デスクトップ プラットフォームではかなり長い間さまざまな形で利用されてきましたが、Web アプリケーションのユーザー インターフェイスの世界では比較的新しいものです。これらのグリッドとテーブルは一般に非常に複雑にすることができ、セルと行の選択、フィルタリング、編集、行と列の非表示、列集計、検索、型固有の列でデータを表示する機能などを備えています。

</PlatformBlock>

<PlatformBlock for="Blazor">

{Platform} データ グリッドとしてより一般的に知られる {Platform} データ テーブルは、バインドされたデータを一連の行と列として可視化できるコンポーネントです。これらのデータ グリッドは、WPF や Windows Forms などのデスクトップ関連プラットフォームではかなり長い間利用できましたが、Web アプリケーション開発で利用できるようになったのは比較的最近です。最新の Web アプリケーションでは、データバインディング、セルと行の選択、セルと行の編集およびエラー検証、列集計、対話型の Outlook スタイル グループ化、Microsoft Excel スタイルのフィルタリング、数百万件のレコードを読み込める可能性を持つ仮想化された行と列などの機能を備えたテーブルとデータ グリッドが使用されます。

</PlatformBlock>

<PlatformBlock for="WebComponents">

## Web Components とは

Web Components は、既存の Web 標準に従う Web ベースの API ライブラリであり、Web アプリケーションやページで使用する再利用可能なカスタム HTML オブジェクトとタグを作成できます。これらのコンポーネントは、JavaScript ライブラリを使用する任意の最新 Web アプリケーション、または HTML を利用する任意のフレームワーク内で使用できます。

</PlatformBlock>

## {Platform} データ グリッドを使用する理由

{Platform} データ グリッドは、大量のデータを保存し、すばやく並べ替える必要があるユース ケースに不可欠です。これには、大量かつ高速なデータを頻繁に使用する金融サービス、銀行、保険などの業界が含まれます。これらの企業の成功は、これらのデータ グリッドの機能、性能、パフォーマンスに左右されます。たとえば、株式に関する判断をマイクロ秒単位で行う必要がある場合、データ グリッドが遅延時間やちらつきなしで動作することが不可欠です。

### データ仮想化とパフォーマンス

<span class="h3">データ仮想化とパフォーマンス</span>

<PlatformBlock for="React">

データ グリッドの列レベルおよび行レベルの仮想化により、{Platform} グリッドで無制限の行と列をシームレスにスクロールできます。ローカルまたはリモートのデータ ソースをサポートしているため、データがどこにあっても最高のパフォーマンスが得られます。ユーザーは、エンタープライズ レベルの速度で Excel のようなスクロールを体験できます。遅延、画面のちらつき、視覚的な遅れがなく、パフォーマンスを損なうことなく最高のユーザー エクスペリエンス (UX) を提供します。

</PlatformBlock>

<PlatformBlock for="WebComponents">

{Platform} Data Grid は、高速で滑らかなパフォーマンスを維持しながら、無限の数のレコードをプロットできる能力を備えています。これは、表示範囲内にあるセルと行だけをメモリ フットプリントに残す、組み込みの強力な行および列の仮想化技術によって実現されます。エンド ユーザーは、遅延、視覚的な遅れ、画面のちらつきの痕跡が一切ない、Microsoft Excel のようなスクロールによって、可能な限り最高のユーザー エクスペリエンスを得られます。

</PlatformBlock>

<PlatformBlock for="Blazor">

{ProductName} Data Grid コンポーネントには、グリッドの行とセルに対する組み込みの仮想化が備わっています。つまり、データ ソースがローカルであるかリモートであるかに関係なく、特定の時点で表示範囲内にある行とセルだけがメモリに保持されます。これにより、数百万件のレコードをグリッドにプロットする場合でも、視覚的な遅れや画面のちらつきのない、シームレスな垂直および水平の Excel のようなスクロールで、可能な限り最高のパフォーマンスを実現します。

</PlatformBlock>
