# {Platform} グリッドの選択ベースのデータ集計

以下に示すサンプルでは、複数選択をカスタム集計関数と組み合わせて使用し、グリッドのフッターに選択された値に基づく集計を表示する方法を確認できます。

## トピックの概要

選択ベースの集計機能を実現するには、`Grid Selection` 機能を `Grid Summaries` と組み合わせて使用できます。
集計では、列のデータ型と要件に応じて、基本クラスのいずれかである <ApiLink type="SummaryOperand" />、<ApiLink type="NumberSummaryOperand" />.html)、または <ApiLink type="DateSummaryOperand" /> を拡張することで、基本的な集計機能をカスタマイズできます。

## 選択

選択されたグリッド範囲のデータの操作を開始するには、グリッドの選択変更を通知するイベントをサブスクライブする必要があります。これは、`selected` イベントと `rangeSelected` イベントをサブスクライブすることで実行できます。選択機能は単一セルの選択とセル範囲の選択を区別するため、両方にバインドする必要があります。

イベントのサブスクリプション ロジックでは、グリッドの `getSelectedData`.html#getSelectedData) 関数を使用して選択されたデータを抽出し、その選択データをカスタム集計オペランドに渡すことができます。

## 集計

カスタム集計クラス内では、グリッド内のデータ型を区別する必要があります。たとえば、以下のシナリオでは、カスタム集計に適したデータ型を持つ列が 4 つあります。これらは、単価、在庫数、廃止ステータス、注文日です。
<ApiLink type="SummaryOperand" /> の派生クラスの `operate` メソッドでは、データ型に基づいてデータをさまざまなカテゴリに分類することから開始して、データを処理します。

```typescript
const numberData = data.filter(rec => typeof rec === "number");
const boolData = data.filter(rec => typeof rec === "boolean");
const dates = data.filter(rec => isDate(rec));
```

<DocsAside type="info">

なお、`isDate` はカスタム関数であることに注意してください。

</DocsAside>

データ型を適切にグループ化した後は、集計自体に進むことができます。そのために、<ApiLink type="NumberSummaryOperand" /> と <ApiLink type="DateSummaryOperand" /> のすでに公開されているメソッドを使用できます。
その後、集計されたデータを同じ配列に入れる必要があり、その配列がテンプレートに返されます。
データの視覚化にはグリッドのフッターを使用するとよいでしょう。`custom-summaries` クラスと組み合わせることで、集計の自然な外観を得られます。
