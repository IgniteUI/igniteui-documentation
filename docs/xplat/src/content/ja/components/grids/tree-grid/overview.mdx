# {Platform} ツリー グリッドの概要と構成

{Platform} ツリー グリッドは、データ グリッド (テーブル) の機能とツリー ビューを組み合わせた UI コンポーネントであり、階層データを表形式で簡単に表示できます。通常のグリッドとは異なり、ツリー グリッドでは行を展開および折りたたみでき、親行の下にネストされた子行を表示できるため、ファイル エクスプローラー、組織図、プロジェクト タスク、製品カテゴリなどの構造化データを表す場合に役立ちます。

{ProductName} ツリー グリッドは、階層データまたはフラット データを簡単に表示および操作するために使用されます。わずかなコードでデータをすばやくバインドしたり、さまざまなイベントを使用して異なる動作をカスタマイズしたりできます。このコンポーネントは、データ選択、Excel スタイルのフィルタリング、ソート、ページング、テンプレート化、列移動などの豊富な機能を提供します。Material Table ベースの UI ツリー グリッドにより、表形式データの表示はこれまでになく簡単で美しくなりました。

## {Platform} ツリー グリッドの例

この例では、ユーザーが階層データまたはフラット データをどのように操作できるかを確認できます。フィルタリングとソートのオプション、ピン固定と非表示、行選択、Excel および CSV へのエクスポートが含まれています。

## 使用方法

ツリー グリッドはグリッドと多くの機能を共有していますが、データを階層的に表示する機能も追加されています。
これを実現するために、ツリー グリッドでは、各データ オブジェクトに対して [子コレクション](overview.md#child-collection) を使用する方法、または各データ オブジェクトに対して [主キーと外部キー](overview.md#primary-and-foreign-keys) を使用する方法という、データ オブジェクト間の関係を定義するいくつかの方法が提供されています。

### ツリー セル

ツリー グリッドの階層を構築するためにどのオプション (子コレクションまたは主キーと外部キー) を使用するかに関係なく、ツリー グリッドの行は 2 種類のセルで構成されます。

- `GridCell` - 値を含む通常のセル。
- `TreeGridCell` - 値、展開/折りたたみインジケーター、およびセルの行のレベルに基づくインデント div 要素を含むツリー セル。行コンポーネントのレベルには、内部の `level` の `treeRow` プロパティを通じてアクセスできます。

<DocsAside type="info">

各行に含めることができるツリー セルは 1 つだけですが、通常のセルは複数含めることも、まったく含めないこともできます。

</DocsAside>

### 初期展開の深さ

初期状態では、ツリー グリッドはすべてのノード レベルを展開して表示します。この動作は <ApiLink type="TreeGrid" member="expansionDepth" label="ExpansionDepth" /> プロパティを使用して構成できます。既定では、その値は **Infinity** で、すべてのノード レベルが展開されることを意味します。このプロパティに数値を設定することで、初期展開の深さを制御できます。たとえば、**0** はルート レベルのノードのみを表示し、**1** はルート レベルのノードとその子ノードを表示する、といった具合です。

### 子コレクション

子コレクション オプションを使用する場合、各データ オブジェクトには、親データ オブジェクトと同じ型の項目が格納された子コレクションが含まれます。この方法では、ツリー グリッド内の各レコードが、その子のいずれかへの直接参照を持つことになります。この場合、元のデータ ソースを含むツリー グリッドの data プロパティは、階層的に定義されたコレクションになります。

## 永続化と統合

ツリー グリッド セルのインデントは、フィルタリング、ソート、ページングなど、他のツリー グリッド機能全体で保持されます。

- 列に `Sorting` が適用されると、データ行はレベルごとにソートされます。つまり、ルート レベルの行は、それぞれの子とは独立してソートされます。それぞれの子コレクションも同様に個別にソートされ、以降も同じです。
- 最初の列 (<ApiLink type="Column" member="visibleIndex" label="VisibleIndex" /> が 0 の列) は常にツリー列です。
- 列のピン固定、列の非表示、列の移動などの操作後に <ApiLink type="Column" member="visibleIndex" label="VisibleIndex" /> が 0 になった列がツリー列になります。
- エクスポートされた Excel ワークシートは、ツリー グリッド自体でレコードがグループ化されているのと同じようにレコードをグループ化することで、階層を反映します。すべてのレコードの展開状態も保持され、反映されます。
- CSV にエクスポートする場合、レベルと展開状態は無視され、すべてのデータがフラットとしてエクスポートされます。

## {Platform} ツリー グリッドのスタイル設定構成

定義済みのテーマに加えて、利用可能な [CSS プロパティ](../theming-grid.md) の一部を設定することで、グリッドをさらにカスタマイズできます。

<igc-tree-grid class="tree-grid"></igc-tree-grid>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

<IgbTreeGrid Class="tree-grid"></IgbTreeGrid>
```

</PlatformBlock>

次に、そのクラスに関連する CSS プロパティを設定します。

```css
.tree-grid {
    --ig-grid-header-background: #494949;
    --ig-grid-header-text-color: #FFF;
    --ig-grid-expand-icon-color: #FFCD0F;
    --ig-grid-expand-icon-hover-color: #E0B710;
    --ig-grid-row-hover-background: #F8E495;
}
```
