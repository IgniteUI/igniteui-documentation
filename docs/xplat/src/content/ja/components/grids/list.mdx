# {Platform} リストの概要

{ProductName} リスト要素は、項目のグループを表示する場合に非常に便利です。テキスト項目のシンプルなリストを作成することも、さまざまなレイアウト要素の配列を含む、より複雑なリストを作成することもできます。<ApiLink pkg="core" type="List" /> コンポーネントは項目の行を表示し、1 つ以上のヘッダーもサポートします。各リスト項目は完全にテンプレート化可能で、有効な HTML やその他のコンポーネントをサポートします。

## {Platform} リストの例

次の例は、名前と電話番号のプロパティを持つ連絡先が入力されたリストを表しています。以下に示す <ApiLink pkg="core" type="List" /> コンポーネントでは、<ApiLink pkg="core" type="Avatar" /> 要素と <ApiLink pkg="core" type="Button" /> 要素を使用してユーザー エクスペリエンスを向上させ、アバター画像やテキスト送信および通話アクション用のボタンを設定する機能を示しています。

`start` スロットは、リスト項目の他のすべてのコンテンツの前に何らかのメディアを追加するために使用することを想定しています。対象要素、この場合は <ApiLink pkg="core" type="Avatar" /> コンポーネントには、既定の位置と間隔も提供されます。

`end` スロットは、スイッチ、ボタン、チェックボックスなどで表される、何らかのアクションまたはメタデータを持つリスト項目に使用することを想定しています。ここでは <ApiLink pkg="core" type="Button" /> コンポーネントを使用します。

  <igc-radio name="size" value="small" label-position="after">Small</igc-radio>
  <igc-radio name="size" value="medium" label-position="after">Medium</igc-radio>
  <igc-radio name="size" value="large" label-position="after" checked>Large</igc-radio>

```

@code {
  public SizableComponentSize ListSize { get; set; }

  public void OnRadioOptionClick(IgbComponentBoolValueChangedEventArgs e)
  {
    IgbRadio radio = e.Parent as IgbRadio;
    switch (radio.Value)
    {
      case "Small":
      {
        this.ListSize = SizableComponentSize.Small;
        break;
      }
      case "Medium":
      {
        this.ListSize = SizableComponentSize.Medium;
        break;
      }
      case "Large":
      {
        this.ListSize = SizableComponentSize.Large;
        break;
      }
    }
  }
}
```

### スロットに推奨される要素

コンテンツを `title` スロットおよび `subtitle` スロットに配置する場合は、見出し要素 (`<h1>`～`<h6>`) ではなく、`<span>` 要素を使用することをお勧めします。見出し要素には、フォント サイズ、行の高さ、余白などの組み込みスタイルがあり、コンポーネントが意図するタイポグラフィやレイアウトに干渉する可能性があります。`<span>` を使用すると、コンポーネントのスタイルをきれいに継承するニュートラルなコンテナーを提供できます。

最適な結果を得るには、メディア コンテンツ用に想定されている `start` スロットには [`<igc-avatar>`](../layouts/avatar.md) コンポーネントを使用することをお勧めします。アクション用に使用することを想定している `end` スロットには、表示したいコンテンツの種類に応じて、[`<igc-button>`](../inputs/button.md)、[`<igc-switch>`](../inputs/switch.md)、[`<igc-checkbox>`](../inputs/checkbox.md) などのコンポーネントを使用することをお勧めします。ただし、これら 2 つのスロットについては、表示したいコンテンツに適している限り、`start` スロットには `<img>`、`end` スロットには `<button>` または別のインタラクティブ要素など、プレーンな HTML 要素を使用することもできます。

## スタイル設定

<ApiLink pkg="core" type="List" /> は複数の CSS パーツを公開しており、そのスタイルを完全に制御できます。

| Name            | Description                              |
| --------------- | ---------------------------------------- |
| `start`  | 開始コンテナー。                     |
| `end` | 終了コンテナー。                       |
| `content` | ヘッダーおよびカスタム コンテンツ コンテナー。 |
| `header` | タイトルおよびサブタイトル コンテナー。        |
| `title` | タイトル コンテナー。                     |
| `subtitle` | サブタイトル コンテナー。                  |

```css
igc-list-header {
  font-size: 20px;
  font-weight: 700;
  color: var(--ig-primary-700);
}

igc-list-item::part(title) {
  font-size: 18px;
  color: var(--ig-primary-600);
}

igc-list-item::part(subtitle) {
  color: var(--ig-primary-300);
}
```
