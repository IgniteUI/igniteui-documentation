# 階層データ グリッドの概要と構成

{ProductName} 階層データ グリッドは、階層型の表形式データを表示および操作するために使用されます。ごく少ないコードでデータをすばやくバインドしたり、さまざまなイベントを使用して異なる動作をカスタマイズしたりできます。このコンポーネントは、データ選択、Excel スタイルのフィルタリング、並べ替え、ページング、テンプレート化、列の移動、列の固定、Excel および CSV へのエクスポートなど、豊富な機能セットを提供します。階層グリッドはフラット グリッド コンポーネントを基盤として構築されており、より詳細な情報が必要な場合に、親グリッドの行を展開または折りたたんで対応する子グリッドを表示できるようにすることで、その機能を拡張します。

## {Platform} 階層データ グリッドの使用

### データバインディング

**{HierarchicalGridSelector}** は {GridSelector} から派生しており、その機能の大部分を共有しています。主な違いは、複数レベルの階層を定義できる点です。これらは、**{HierarchicalGridSelector}** の定義内にある **{RowIslandSelector}** と呼ばれる別個のタグを通じて構成されます。**{RowIslandSelector}** コンポーネントは、特定のレベルにおける各子グリッドの構成を定義します。レベルごとに複数の行アイランドもサポートされています。
階層グリッドは、データへのバインド方法を 2 つサポートしています。

### 階層データの使用

### Load-On-Demand の使用

ほとんどのアプリケーションは、初期段階でできるだけ少ないデータを読み込むように設計されており、その結果、読み込み時間が短縮されます。このような場合、{HierarchicalGridSelector} は、ユーザーが作成したサービスがオンデマンドでデータを供給できるように構成できます。

<PlatformBlock for="Angular">

```html
<!-- hierarchicalGridSample.component.html -->

<igx-hierarchical-grid #hGrid [primaryKey]="'customerId'" [autoGenerate]="true" [height]="'600px'" [width]="'100%'">
    <igx-row-island [key]="'Orders'" [primaryKey]="'orderId'" [autoGenerate]="true"  (gridCreated)="gridCreated($event, 'Customers')">
        <igx-row-island [key]="'Details'" [primaryKey]="'productId'" [autoGenerate]="true" (gridCreated)="gridCreated($event, 'Orders')">
        </igx-row-island>
    </igx-row-island>
</igx-hierarchical-grid>
```

```typescript
//  hierarchicalGridSample.component.ts

@Component({...})
export class HierarchicalGridLoDSampleComponent implements AfterViewInit {
    @ViewChild("hGrid")
    public hGrid: IgxHierarchicalGridComponent;

    constructor(private remoteService: RemoteLoDService) { }

    public ngAfterViewInit() {
        this.hGrid.isLoading = true;
        this.remoteService.getData({ parentID: null, rootLevel: true, key: "Customers" }).subscribe((data) => {
            this.hGrid.isLoading = false;
            this.hGrid.data = data;
            this.hGrid.cdr.detectChanges();
        });
    }

    public gridCreated(event: IGridCreatedEventArgs, _parentKey: string) {
        const dataState = {
            key: event.owner.key,
            parentID: event.parentID,
            parentKey: _parentKey,
            rootLevel: false
        };
        event.grid.isLoading = true;
        this.remoteService.getData(dataState).subscribe(
            (data) => {
                event.grid.isLoading = false;
                event.grid.data = data;
                event.grid.cdr.detectChanges();
            }
        );
    }
}
```

```typescript
// remote-load-on-demand.service.ts

@Injectable()
export class RemoteLoDService {
    public url = `https://data-northwind.indigo.design/`;

    constructor(private http: HttpClient) { }

    public getData(dataState: any): Observable<any[]> {
        return this.http.get(this.buildUrl(dataState)).pipe(
            map((response) => response)
        );
    }

    public buildUrl(dataState) {
        let qS = "";
        if (dataState) {
            if (dataState.rootLevel) {
                qS += `${dataState.key}`;
            } else {
                qS += `${dataState.parentKey}/${dataState.parentID}/${dataState.key}`;
            }
        }
        return `${this.url}${qS}`;
    }
}
```

export class HierarchicalGridLoadOnDemand {
    constructor() {
        const hierarchicalGrid = document.getElementById("hGrid") as IgcHierarchicalGridComponent;
        const ordersRowIsland = document.getElementById("ordersRowIsland");
        const orderDetailsRowIsland = document.getElementById("orderDetailsRowIsland");

        ordersRowIsland.addEventListener("gridCreated", (event: any) => {

        orderDetailsRowIsland.addEventListener("gridCreated", (event: any) => {
            this.gridCreated(event, "Orders");

            hierarchicalGrid.isLoading = false;
            hierarchicalGrid.data = data;
            hierarchicalGrid.markForCheck();

        const context = event.detail;

            key: context.owner.childDataKey,
            parentID: context.parentID,

この UI はパフォーマンス上の理由から既定で無効になっており、大量のデータを持つグリッドや Load-On-Demand のグリッドで有効にすることは推奨されません。

## 機能

グリッドの機能は、{RowIslandSelector} マークアップを通じて有効化および構成できます。これらは、そのために作成されるすべてのグリッドに適用されます。行アイランド インスタンスを通じて実行時にオプションを変更すると、その行アイランドが生成した各グリッドに対して変更が適用されます。

次のグリッド機能はグリッド レベルごとに動作します。つまり、各グリッド インスタンスは、他のグリッドとは独立してそれらを管理します。

- 並べ替え
- フィルタリング
- ページング
- 複数列ヘッダー
- 非表示
- 固定
- 移動
- 集計
- 検索

選択とナビゲーションの機能は、{HierarchicalGridTitle} 全体に対してグローバルに動作します。

- 選択
  選択では、選択されたセルが 2 つの異なる子グリッドに同時に存在することは許可されません。
- ナビゲーション
  <kbd>↑</kbd> + <kbd>↓</kbd> でナビゲーションする場合、次/前の要素が子グリッドであれば、関連する子グリッド内でナビゲーションが続行され、関連するセルが選択されフォーカスされます。子セルが現在の表示ビューポートの外にある場合は、選択されたセルが常に表示されるようにビュー内へスクロールされます。

## スタイル設定

igxHierarchicalGrid は、[Ignite UI for Angular テーマ ライブラリ](../themes/sass/component-themes.md) を通じたスタイル設定を可能にします。グリッドの <ApiLink type="function-grid-theme" label="theme" kind="sass" module="index.html" /> は、グリッドのすべての機能をカスタマイズできる幅広いプロパティを公開しています。

以下の手順では、igxHierarchicalGrid のスタイル設定をカスタマイズする手順を説明します。

### グローバル テーマのインポート

階層グリッドのカスタマイズを開始するには、すべてのスタイル設定関数と mixin が配置されている `index` ファイルをインポートする必要があります。

```scss
@import '~igniteui-angular/lib/core/styles/themes/index'
```

### カスタム テーマの定義

次に、<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> を拡張し、階層グリッドを必要に応じてカスタマイズするために必要なパラメーターを受け取る新しいテーマを作成します。

 <DocsAside type="info">

階層グリッド専用の `sass` 関数はありません。

 </DocsAside>

```scss
$custom-theme: grid-theme(
  $cell-active-border-color: #ffcd0f,
  $cell-selected-background: #6f6f6f,
  $row-hover-background: #f8e495,
  $row-selected-background: #8d8d8d,
  $header-background: #494949,
  $header-text-color: #fff,
  $expand-icon-color: #ffcd0f,
  $expand-icon-hover-color: #e0b710,
  $resize-line-color: #ffcd0f,
  $row-highlight: #ffcd0f
);
```

### カスタム カラー パレットの定義

上記で説明したアプローチでは、色の値がハードコーディングされていました。代わりに、<ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> 関数と <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> 関数を使用することで、より高い柔軟性を実現できます。
`igx-palette` は、指定されたプライマリ色とセカンダリ色に基づいてカラー パレットを生成します。

```scss
$black-color: #494949;
$yellow-color: #FFCD0F;

$custom-palette: palette(
 $primary: $black-color,
 $secondary: $yellow-color
);
```

カスタム パレットが生成された後、`igx-color` 関数を使用して、プライマリ色とセカンダリ色のさまざまなバリエーションを取得できます。

```scss
$custom-theme: grid-theme(
    $cell-active-border-color: (igx-color($custom-palette, "secondary", 500)),
    $cell-selected-background: (igx-color($custom-palette, "primary", 300)),
    $row-hover-background: (igx-color($custom-palette, "secondary", 100)),
    $row-selected-background: (igx-color($custom-palette, "primary", 100)),
    $header-background: (igx-color($custom-palette, "primary", 500)),
    $header-text-color: (igx-contrast-color($custom-palette, "primary", 500)),
    $expand-icon-color: (igx-color($custom-palette, "secondary", 500)),
    $expand-icon-hover-color: (igx-color($custom-palette, "secondary", 600)),
    $resize-line-color: (igx-color($custom-palette, "secondary", 500)),
    $row-highlight: (igx-color($custom-palette, "secondary", 500))
);
```

### カスタム スキーマの定義

さらに進んで、[**スキーマ**](../themes/sass/schemas.md) のすべての利点を備えた柔軟な構造を構築できます。**スキーマ** はテーマのレシピです。
各コンポーネントに対して提供されている 2 つの定義済みスキーマのいずれかを拡張します。この場合は、`$_light_grid` を使用します。

```scss
$custom-grid-schema: extend($_light-grid,(
    cell-active-border-color: (igx-color:('secondary', 500)),
    cell-selected-background: (igx-color:('primary', 300)),
    row-hover-background: (igx-color:('secondary', 100)),
    row-selected-background: (igx-color:('primary', 100)),
    header-background: (igx-color:('primary', 500)),
    header-text-color: (igx-contrast-color:('primary', 500)),
    expand-icon-color: (igx-color:('secondary', 500)),
    expand-icon-hover-color: (igx-color:('secondary', 600)),
    resize-line-color: (igx-color:('secondary', 500)),
    row-highlight: (igx-color:('secondary', 500))
));
```

カスタム スキーマを適用するには、`light` または `dark` のグローバルを拡張する必要があります。実際のプロセス全体は、コンポーネントにカスタム スキーマを提供し、その後、対応するコンポーネント テーマに追加することです。

```scss
$my-custom-schema: extend($light-schema, (
    igx-grid: $custom-grid-schema
));

$custom-theme: grid-theme(
    $palette: $custom-palette,
    $schema: $my-custom-schema
);
```

### カスタム テーマの適用

テーマを適用する最も簡単な方法は、グローバル スタイル ファイル内で `sass` `@include` ステートメントを使用することです。

```scss
@include grid($custom-theme);
```

### スコープ付きコンポーネント テーマ

カスタム テーマが特定のコンポーネントにのみ影響するようにするには、グローバル スタイル ファイルで定義したすべてのスタイルを、カスタム コンポーネントのスタイル ファイルへ移動できます（`index` ファイルのインポートを含む）。

この方法では、Angular の [ViewEncapsulation](https://angular.io/api/core/Component#encapsulation) により、スタイルはカスタム コンポーネントにのみ適用されます。

 <DocsAside type="info">

コンポーネントが [`Emulated`](../themes/sass/component-themes.md#view-encapsulation) ViewEncapsulation を使用している場合、グリッドにスタイルを設定するには、`::ng-deep` を使用してこのカプセル化を貫通する必要があります。

 </DocsAside>

 <DocsAside type="info">

スタイルがコンポーネント外の要素に影響しないようにするには、ステートメントを `:host` セレクターの内側でラップします。

 </DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-theme);
    }
}
```
