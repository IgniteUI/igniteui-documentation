# 階層グリッドのオンデマンド読み込み

Ignite UI for {Platform} <ApiLink type="HierarchicalGrid" /> は、サーバーから取得するデータ量を最小限に要求することで高速なレンダリングを可能にし、ユーザーが結果をビューで確認し、表示されているデータをできるだけ早く操作できるようにします。最初はルート グリッドのデータのみが取得およびレンダリングされ、ユーザーが子グリッドを含む行を展開した後にのみ、その特定の子グリッドのデータを受け取ります。この仕組みはオンデマンド読み込みとも呼ばれ、任意のリモート データで動作するように簡単に構成できます。

このトピックでは、既に利用可能なリモート サービスと通信するリモート サービス プロバイダーを作成して、オンデマンド読み込みを構成する方法を示します。以下に動作するデモを示し、その後で手順を追って作成プロセスを説明します。

## リモート サービス プロバイダー

最初に、階層グリッドに必要なデータを取得できるように、サービス プロバイダーを準備します。

### 基本データの取得

<PlatformBlock for="Angular">

ブラウザーが提供する XMLHttpRequest インターフェイスを使用し、HTTP プロトコル経由でバックエンド サービスと通信します。これをより簡単に実現するために、簡略化されたクライアント HTTP API を提供する Angular の [`HttpClient`](https://angular.io/api/common/http/HttpClient) モジュールを使用します。そのため、データを取得するには、サービス内に次のような単純なメソッドが必要です。

```typescript
public getData(dataState): Observable<any[]> {
    return this.http.get(this.buildUrl(dataState)).pipe(
        map(response => response),
    );
}
```

ご覧のとおり、`this.http` は `HttpCLient` モジュールへの参照になり、`buildUrl()` は受け取ったデータに基づいて URL を生成するメソッドになります。これは非同期に実行されるため、レスポンスをマップして Observable を返します。これにより、後で購読し、アプリケーション内でさらに処理して、グリッドに渡すことができます。

</PlatformBlock>

<PlatformBlock for="WebComponents, React">

ブラウザーが提供する [`fetch()`](https://developer.mozilla.org/en-US/docs/Web/API/fetch) グローバル関数を使用し、HTTP プロトコル経由でバックエンド サービスと通信します。そのため、データを取得するには、サービス内に次のような単純なメソッドが必要です。

```ts
export async function getData(dataState: any): Promise<any> {
    const response = await fetch(buildUrl(dataState));
    const data = await response.json();
    return data;
}
```

ご覧のとおり、`buildUrl()` は受け取ったデータに基づいて URL を生成するメソッドになります。これは非同期に実行されるため、Promise を返します。これにより、後で購読し、アプリケーション内でさらに処理して、グリッドに渡すことができます。

</PlatformBlock>

### リクエスト URL の構築

次に、GET リクエスト用の URL をどのように構築するかを定義します。ここでは、メイン グリッドのデータだけでなく、その内部にある任意の子グリッドのデータも取得できるようにします。ルート レベルには、この [トピック](https://data-northwind.indigo.design/swagger/index.html) の `Customers` データを使用し、下位レベルには `Orders` と `Details` を使用します。モデルはアプリケーションごとに異なりますが、ここでは次のものを使用します。

<Image src={hgridDatabase} alt="hgrid-database" class="responsive-img" />

最初に必要なのは、目的のグリッドのデータをどこから取得するかを決定するためのテーブルの `key`、親行の主キー、およびその一意の ID です。

<PlatformBlock for="Angular">

これらすべてを `IDataState` というインターフェイスで定義します。例:

```typescript
export interface IDataState {
    key: string;
    parentID: any;
    parentKey: string;
    rootLevel: boolean;
}

//...
public buildUrl(dataState: IDataState): string {
    let qS = "";
    if (dataState) {
        if (dataState.rootLevel) {
            qS += `${dataState.key}`;
        } else {
            qS += `${dataState.parentKey}/${dataState.parentID}/${dataState.key}`;
        }
    }
    return `${this.url}${qS}`;
}
//...
```

```typescript
import { map } from 'rxjs/operators';
```

展開された任意の子にデータを設定する方法は少し異なります。行が初めて展開されると、その行に対して新しい子 <ApiLink type="HierarchicalGrid" /> がレンダリングされ、新しく作成されたグリッドの参照を取得してデータを設定する必要があります。そのため、各 <ApiLink type="RowIsland" /> コンポーネントは、その特定の行アイランドに対して新しい子グリッドが作成されたときに発生する `GridCreated` イベントを提供します。これを使用して、新しいグリッドに必要な参照を取得し、サービスからデータを要求して適用できます。

すべての行アイランドに対して 1 つのメソッドを使用できます。これは、ルート レベルかどうか、行アイランドのキー、親行の主キー、およびその一意の識別子という情報だけを必要とするようにサービスを構築したためです。これらの情報はすべて、イベント引数から直接、またはイベントをトリガーした行アイランドからアクセスできます。


```typescript

        this.hGrid.cdr.detectChanges();

}
```

### 読み込みインジケーターの設定

<ApiLink type="HierarchicalGrid" /> は、データがない間に <ApiLink type="GridBaseDirective" member="isLoading" label="IsLoading" /> プロパティを **true** に設定することで、読み込みインジケーターを表示できます。ルート グリッドに対して初期設定し、さらに新しい子グリッドを作成するときにも、そのデータが読み込まれるまで設定する必要があります。テンプレート内で常に **true** に設定することもできますが、サービスが空の配列を返した場合は、これを **false** に設定して読み込みインジケーターを非表示にし、グリッドにデータがないことを表示したい場合があります。

この場合、構成の最終バージョンは次のようになります。

<PlatformBlock for="Angular">

```typescript
import { AfterViewInit, Component, ViewChild } from "@angular/core";
import {
    IGridCreatedEventArgs,
    IgxHierarchicalGridComponent,
    IgxRowIslandComponent
} from "igniteui-angular";
import { RemoteLoDService } from "../services/remote-lod.service";

@Component({
    providers: [RemoteLoDService],
    selector: "app-hierarchical-grid-lod",
    styleUrls: ["./hierarchical-grid-lod.component.scss"],
    templateUrl: "./hierarchical-grid-lod.component.html"
})
export class HierarchicalGridLoDSampleComponent implements AfterViewInit {
    @ViewChild("hGrid")
    public hGrid: IgxHierarchicalGridComponent;

    constructor(private remoteService: RemoteLoDService) { }

    public ngAfterViewInit() {
        this.hGrid.isLoading = true;
        this.remoteService.getData({ parentID: null, rootLevel: true, key: "Customers" }).subscribe((data) => {
            this.hGrid.isLoading = false;
            this.hGrid.data = data;
            this.hGrid.cdr.detectChanges();
        });
    }

    public gridCreated(event: IGridCreatedEventArgs, _parentKey: string) {
        const dataState = {
            key: event.owner.key,
            parentID: event.parentID,
            parentKey: _parentKey,
            rootLevel: false
        };
        event.grid.isLoading = true;
        this.remoteService.getData(dataState).subscribe(
            (data) => {
                event.grid.isLoading = false;
                event.grid.data = data;
                event.grid.cdr.detectChanges();
            }
        );
    }
}
```
