# {Platform} Slider & Range Slider Overview

The {Platform} Slider & Range Slider components allow selection in a given range by moving the thumb along the track. The track can be defined as continuous or stepped and you can choose between single and range slider.

### Value

The main difference between the Slider and Range Slider components is that the Slider component has a single thumb, while the Range Slider component has two thumbs. The single thumb of the Slider component displays its <ApiLink pkg="core" type="Slider" member="value" label="Value" /> property. The two thumbs of the Range Slider component display its <ApiLink pkg="core" type="RangeSlider" member="lower" label="Lower" /> and <ApiLink pkg="core" type="RangeSlider" member="upper" label="Upper" /> value properties.

Both sliders emit two events when any of the values is changed. The <ApiLink pkg="core" type="Input" /> event is emitted whenever a value is changed using keyboard or drag interaction while the `Change` event is emitted when the value change is committed on drag end or keyboard interaction.


<Sample src="/inputs/slider/value" height={250} alt="{Platform} Slider Value Example" />




While dragging a slider thumb, it displays its value in a tooltip. You could hide this tooltip using the <ApiLink pkg="core" type="Slider" member="hideTooltip" label="HideTooltip" /> property.

You can use the <ApiLink pkg="core" type="Slider" member="disabled" label="Disabled" /> property of the sliders to disable their user interactions.

### Constraints

The track of the sliders has a minimum and maximum values which are configured using the <ApiLink pkg="core" type="Slider" member="min" label="Min" /> and <ApiLink pkg="core" type="Slider" member="max" label="Max" /> properties. Additionally, you can restrict the thumb dragging using the <ApiLink pkg="core" type="RangeSlider" member="lowerBound" label="LowerBound" /> and <ApiLink pkg="core" type="RangeSlider" member="upperBound" label="UpperBound" /> properties.

<Sample src="/inputs/slider/constraints" height={120} alt="{Platform} Slider Constraints Example" />




### Step

The <ApiLink pkg="core" type="Slider" member="step" label="Step" /> property specifies the granularity of the slider that the value must adhere to. By default, the slider track looks continuous. Setting the <ApiLink pkg="core" type="Slider" member="discreteTrack" label="DiscreteTrack" /> property of the slider to **true** will make it to display the steps on the track.

<Sample src="/inputs/slider/discrete" height={120} alt="{Platform} Slider Discrete Track Example" />




If the <ApiLink pkg="core" type="Slider" member="step" label="Step" /> property is set to `0`, no stepping is implied and any value in the slider range is allowed. In this case, the slider will look continuous even if <ApiLink pkg="core" type="Slider" member="discreteTrack" label="DiscreteTrack" /> is set to **true**.

### Tick Marks

The slider components could display tick marks and labels. The slider components support two types of tick marks: primary and secondary. In order to display the primary tick marks, you should set the <ApiLink pkg="core" type="Slider" member="primaryTicks" label="PrimaryTicks" /> property to a value greater than `1`. The number of primary ticks will be evenly distributed on the track. In order to display the secondary tick marks, you should set the <ApiLink pkg="core" type="Slider" member="secondaryTicks" label="SecondaryTicks" /> property to a value greater than `0`. The value of <ApiLink pkg="core" type="Slider" member="secondaryTicks" label="SecondaryTicks" /> specifies the number of secondary ticks between every two primary ticks.

<Sample src="/inputs/slider/ticks" height={150} alt="{Platform} Slider Tick Marks Example" />




Additionally, you could configure the orientation of the tick marks using the <ApiLink pkg="core" type="Slider" member="tickOrientation" label="TickOrientation" /> property. By default, the <ApiLink pkg="core" type="Slider" member="tickOrientation" label="TickOrientation" /> value is `end` which displays the ticks below the slider track. You could set it to `start` which displays them above the track and `mirror` which mirrors the ticks above and below the track.

By default, the tick marks display labels with their values. You could modify the rotation of the tick labels using the <ApiLink pkg="core" type="Slider" member="tickLabelRotation" label="SliderTickLabelRotation" /> property. Additionally, you could hide the labels of the primary and secondary ticks using the <ApiLink pkg="core" type="Slider" member="hidePrimaryLabels" label="HidePrimaryLabels" /> and <ApiLink pkg="core" type="Slider" member="hideSecondaryLabels" label="HideSecondaryLabels" /> properties.

<Sample src="/inputs/slider/tick-labels" height={150} alt="{Platform} Slider Tick Mark Labels Example" />




### Value Format

If you want to format the thumb and tick label values, the slider provides <ApiLink pkg="core" type="Slider" member="valueFormat" label="ValueFormat" />, <ApiLink pkg="core" type="Slider" member="valueFormatOptions" label="ValueFormatOptions" /> and <ApiLink pkg="core" type="Slider" member="locale" label="Locale" /> properties. The <ApiLink pkg="core" type="Slider" member="valueFormatOptions" label="ValueFormatOptions" /> allows you to specify the number of fraction and significant digits, style (decimal, currency, percent, unit), notation and others taking into account the specified <ApiLink pkg="core" type="Slider" member="locale" label="Locale" />. The <ApiLink pkg="core" type="Slider" member="valueFormat" label="ValueFormat" /> is a string which may contain the `{0}` identifier which will be replaced by the value with applied format options.

<Sample src="/inputs/slider/value-format" height={230} alt="{Platform} Slider Value Format Example" />




### Labels

In some cases you would want to format the values of the slider as string values i.e. map the values **[0, 1, 2]** to **['Low', 'Medium', 'High']**. For this scenario the slider allows you to define <ApiLink pkg="core" type="SliderLabel" /> elements inside it. The text content of the slider labels is going to be used for thumb and tick labels. Please note that when slider labels are provided, the <ApiLink pkg="core" type="Slider" member="min" label="Min" />, <ApiLink pkg="core" type="Slider" member="max" label="Max" /> and <ApiLink pkg="core" type="Slider" member="step" label="Step" /> properties are automatically calculated so that they do not allow values that do not map to the provided labels. In the case of 'Low', 'Medium' and 'High' labels, <ApiLink pkg="core" type="Slider" member="min" label="Min" /> is set to `0`, <ApiLink pkg="core" type="Slider" member="max" label="Max" /> is set to `2` and <ApiLink pkg="core" type="Slider" member="step" label="Step" /> is set to `1`.

## Styling

The <ApiLink pkg="core" type="Slider" /> component exposes CSS parts for almost all of its inner elements. The following table lists all of the exposed CSS parts:

|Name|Description|
|--|--|
| `base` | The base wrapper of the slider. |
| `ticks` | The ticks container. |
| `tick-group` | The tick group container. |
| `tick` | The tick element. |
| `tick-label` | The tick label element. |
| `tick-label-inner` | The inner element of the tick label. |
| `thumbs` | The thumbs container. |
| `thumb` | The thumb element. |
| `thumb-label` | The label container of the thumb tooltip. |
| `thumb-label-inner` | The label element of the thumb tooltip. |
| `track` | The track container. |
| `steps` | The track steps element. |
| `inactive` | The inactive element of the track. |
| `fill` | The filled part of the track. |
