# {Platform} {ComponentTitle} ソート

{Platform} {ComponentTitle} の {ProductName} データ ソート機能は列ごとのレベルで有効になっています。つまり、<ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> にはソート可能な列とソート不可能な列を混在させることができます。{Platform} でソートを実行すると、指定した条件に基づいてレコードの表示順序を変更できます。

<PlatformBlock for="Angular">
<DocsAside type="note" title="注">
これまで、グループ化 / ソートは互いに連携して機能していました。13.2 バージョンでは、gropuing を sorting から切り離す新しい動作が導入されています。たとえば、グループ化をクリアしても、グリッド内のソート式はクリアされません。その逆も同様です。それでも、列がソートおよびグループ化されている場合は、グループ化された式が優先されます。
</DocsAside>

</PlatformBlock>

## 初期のソート状態

<ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> でソート状態を初期設定するには、ソート式の配列を <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> の `SortingExpressions` プロパティに渡します。

<ComponentBlock for="Grid">
<PlatformBlock for="Angular">
```typescript
public ngOnInit() {
    this.grid.sortingExpressions = [
        { fieldName: 'ProductName', dir: SortingDirection.Asc, ignoreCase: true },
        { fieldName: 'Price', dir: SortingDirection.Desc }
    ];
}
```
</PlatformBlock>

<PlatformBlock for="Blazor">
```razor
@code {
    protected override void OnAfterRender(bool first)
    {
        if (first)

                {
                    FieldName = "Title",
                    Dir = SortingDirection.Asc

    }
}
```
</PlatformBlock>

```html

    <igx-icon>unfold_more</igx-icon>
</ng-template>
```

```razor
<{ComponentSelector} SortHeaderIconTemplate="SortDefaultTemplate"></{ComponentSelector}>
```
