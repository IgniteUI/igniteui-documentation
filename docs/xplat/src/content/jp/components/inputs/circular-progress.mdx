# {Platform} Circular Progress (円形プログレス) の概要

{ProductName} Circular Progress Indicator コンポーネントは、変更でアプリケーションの進行状況を表す視覚的なインジケーターです。丸形インジケーターは状態変更で外観を更新します。

### Gradient Progress

Customizing the progress bar in order to use a color gradient instead of a solid color could be done via the exposed `gradient` slot and <ApiLink pkg="core" type="CircularGradient" /> which defines the gradient stops.

<Sample src="/inputs/circular-progress-indicator/dynamic" height={200} alt="{Platform} Circular Progress Dynamic Example" />




<DocsAside type="info" title="情報">
For each <ApiLink pkg="core" type="CircularGradient" /> defined as gradient slot of {ProductName} <ApiLink pkg="core" type="CircularProgress" /> a [SVG stop](https://developer.mozilla.org/en-US/docs/Web/SVG/Element/stop) element would be created. The values passed as `color`, `offset` and `opacity` would be set as stop-color, offset and stop-opacity of the SVG element without further validations.
</DocsAside>

## Styling

The <ApiLink pkg="core" type="CircularProgress" /> component exposes CSS parts for almost all of its inner elements:

|Name|Description|
|--|--|
| `svg`               | The progress SVG element.                 |
| `gradient_start`     | The progress linear-gradient start color. |
| `gradient_end`       | The progress linear-gradient end color.   |
| `track`              | The progress ring's track area.           |
| `fill`               | The progress indicator area.              |
| `label`              | The progress label.                       |
| `value`              | The progress label value.                 |
| `indeterminate`      | The progress indeterminate state.         |
| `primary`            | The progress indicator primary state.     |
| `danger`             | The progress indicator error state.       |
| `warning`            | The progress indicator warning state.     |
| `info`               | The progress indicator info state.        |
| `success`            | The progress indicator success state.     |

Using this CSS parts we have almost full control over the Circular Progress styling.

```css

igc-circular-progress {
  margin: 20px;
  --diameter: 50px;
}

igc-circular-progress::part(gradient_end),
igc-circular-progress::part(gradient_start) {
  stop-color: var(--ig-success-200);
}

igc-circular-progress::part(track) {
  stroke: var(--ig-gray-400);
}

```
