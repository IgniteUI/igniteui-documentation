# Desarrollo asistido por IA con Ignite UI

{ProductName} proporciona una cadena de herramientas de IA completa: Habilidades de agente, el servidor MCP de Ignite UI CLI, el servidor MCP de temas de Ignite UI y el servidor MCP de MAKER, que fundamenta a los asistentes de codificación con IA en las API de componente, las rutas de importación y los tokens de diseño correctos. Las Habilidades de agente son paquetes de instrucciones propiedad del desarrollador que definen cómo los agentes de IA usan Ignite UI en un proyecto específico. El servidor MCP de CLI (`igniteui-cli`) expone herramientas de scaffolding de Ignite UI CLI, administración de componentes y documentación a la sesión activa del agente de IA mediante el Model Context Protocol. El servidor MCP de temas (`igniteui-theming`) expone el motor de temas de Ignite UI como contexto de agente consultable. El MCP de MAKER (`@igniteui/maker-mcp`) es un servidor MCP de orquestación de IA multiagente de Infragistics que descompone tareas complejas en planes de pasos validados y ejecutables mediante un algoritmo de votación basado en consenso entre varios agentes de IA. Habilidades, MCP de CLI y MCP de temas: los tres se configuran con un solo comando: `npx igniteui-cli ai-config`

Los servidores MCP y las Habilidades de agente tienen distintos propósitos y diferentes requisitos previos:

| Componente           | Qué proporciona                                                | ¿Requiere Ignite UI instalado? |
| -------------------- | -------------------------------------------------------------- | ------------------------------ |
| Servidor MCP de Ignite UI | Consultas de documentación, referencia de API, herramientas de scaffolding | No                             |
| Servidor MCP de temas | Tokens de diseño, herramientas de paleta, validación de contraste WCAG | No                             |
| Habilidades de agente | Paquetes de instrucciones de nivel de proyecto para el uso correcto de componentes | Sí                             |

Puede empezar a evaluar la asistencia de IA de Ignite UI solo con los servidores MCP; no es necesario que Ignite UI esté instalado en su proyecto. Las Habilidades de agente estarán disponibles una vez que instale los paquetes de Ignite UI.

El servidor MCP de CLI requiere transporte STDIO; los clientes MCP basados en HTTP no son compatibles. Las Habilidades de agente y el servidor MCP de CLI no modifican archivos de proyecto de forma autónoma: exponen herramientas e instrucciones al agente de IA activo, que actúa según las indicaciones del desarrollador.

## Configurar la cadena de herramientas de IA

Ejecute este comando desde la raíz de su proyecto existente de Angular, React, Blazor o Web Components. Configura servidores MCP, copia archivos de habilidades específicos del framework en el directorio de habilidades de cada agente y configura archivos de instrucciones. Use `--assistants` para elegir qué asistentes de codificación reciben la configuración MCP y `--agents` para elegir qué agentes reciben archivos de habilidades. Si no se proporcionan parámetros, el comando entra en modo interactivo y le solicita seleccionar asistentes y agentes. Los archivos existentes solo se actualizan si su contenido ha cambiado.

```bash
npx igniteui-cli ai-config
```

<DocsAside type="warning" title="Important">
Sin fijar una versión, `npx` puede obtener una versión anterior de CLI que no reconoce el subcomando `ai-config` y, en su lugar, iniciará un mensaje interactivo de creación de proyectos, generando la estructura de un nuevo proyecto dentro del existente. Asegúrese de haber instalado la versión 16.x de CLI.
</DocsAside>

<PlatformBlock for="Blazor">
<DocsAside type="info">
Para Blazor, `ai-config` detecta tu proyecto buscando un archivo `.csproj` o `.sln`. No es necesario que el paquete NuGet `IgniteUI.Blazor` esté instalado.
</DocsAside>
</PlatformBlock>

Si `ai-config` no puede detectar el framework a partir de los archivos de su proyecto, le solicita seleccionar uno; por lo tanto, el comando funciona incluso desde un proyecto donde todavía no hay ningún paquete de Ignite UI instalado.

Cuando finalice el comando, inicie los servidores MCP en su cliente de IA. Los servidores están configurados, pero aún no se están ejecutando; el cliente debe iniciar cada servidor antes de que sus herramientas estén disponibles para el agente.

### Iniciar los servidores

**VS Code con GitHub Copilot:**

Abra `.vscode/mcp.json`. VS Code muestra un botón **Iniciar** en línea encima de cada entrada de servidor. Haga clic en **Iniciar** para `igniteui` y `igniteui-theming`. Una vez iniciados, VS Code muestra el recuento de herramientas disponibles junto a cada servidor (por ejemplo, *"13 herramientas | 1 prompt"*). Como alternativa, ejecute **MCP: List Servers** desde la paleta de comandos (`Ctrl+Shift+P` / `Cmd+Shift+P`), seleccione cada servidor y elija **Iniciar**.

**Cursor:**

Abra **Configuración → MCP**, localice las entradas `igniteui` y `igniteui-theming`, y active cada una. Cursor inicia cada servidor de inmediato y muestra su recuento de herramientas.

**Claude Code:**

Ejecute `claude mcp list` para confirmar que ambos servidores están registrados. Claude Code inicia los servidores MCP automáticamente cuando sus herramientas se invocan por primera vez; no se requiere ningún paso de inicio manual.

**JetBrains AI Assistant:**

Abra **Configuración → Herramientas → AI Assistant → Model Context Protocol (MCP)**. Haga clic en el icono de reproducción junto a cada entrada de servidor para iniciarlo.

**Claude Desktop:**

Cierre y vuelva a iniciar Claude Desktop. Los servidores se inician automáticamente al abrirse.

### Instalar Ignite UI y agregar habilidades más tarde

Si ejecutó `ai-config` sin Ignite UI instalado y desea agregar habilidades, instale el paquete de Ignite UI para su framework y vuelva a ejecutar el comando:

<PlatformBlock for="Angular">

```bash
npm install igniteui-angular
npx igniteui-cli@latest ai-config
```

</PlatformBlock>

<PlatformBlock for="React">

```bash
npm install igniteui-react
npx igniteui-cli@latest ai-config
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```bash
npm install igniteui-webcomponents
npx igniteui-cli@latest ai-config
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```bash
npx igniteui-cli@latest ai-config
```

</PlatformBlock>

El comando detecta que las habilidades ya están disponibles y las copia. Los archivos de configuración MCP para sus clientes seleccionados permanecen sin cambios (ya están actualizados).

## La cadena de herramientas de IA de un vistazo

La cadena de herramientas de IA de Ignite UI consta de tres capas utilizables de forma independiente. Cada capa puede habilitarse por sí sola; están diseñadas para funcionar juntas.

| Capa                                | Qué proporciona                                                                                             | Propietario  | Frameworks                             |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------- | ------------ | -------------------------------------- |
| Habilidades de agente               | Paquetes de instrucciones propiedad del desarrollador: rutas de importación, patrones de componente, flujos de decisión, convenciones de proyecto | Desarrollador | Angular, React, Web Components, Blazor |
| Servidor MCP de CLI (`igniteui-cli`) | Scaffolding de proyectos, administración de componentes, consultas de documentación y API mediante MCP       | Infragistics | Angular, React, Web Components, Blazor |
| Servidor MCP de temas (`igniteui-theming`) | Tokens de diseño, definiciones de paleta, generación de propiedades personalizadas de CSS, validación de contraste WCAG AA | Infragistics | Angular, React, Web Components, Blazor |

El servidor MCP de CLI y el servidor MCP de temas se inician ambos mediante `npx` y se conectan a cualquier cliente compatible con MCP mediante transporte STDIO. Las Habilidades de agente son archivos locales colocados en su proyecto que el cliente de IA lee desde el disco.

## Habilidades de agente

Las Habilidades de agente son paquetes estructurados, propiedad del desarrollador, que indican a los asistentes de codificación con IA exactamente cómo usar Ignite UI para un framework específico. Un paquete de habilidades puede contener un archivo de instrucciones `SKILL.md` con patrones de componente, rutas de importación y flujos de decisión; referencias a la documentación oficial de Ignite UI; y recursos como archivos de esquema o diagramas. Cuando una habilidad está activa en el cliente de IA, el agente sigue la habilidad en lugar de depender de datos generales de entrenamiento, que pueden hacer referencia a firmas de API o rutas de importación obsoletas.

Ignite UI incluye paquetes de habilidades dedicados para Angular, React, Web Components y Blazor. El paquete de habilidades es propiedad del desarrollador: edite el `SKILL.md` para que coincida con las convenciones de su equipo, agregue patrones específicos del proyecto, haga referencia a su sistema de diseño interno y versione el paquete junto con su base de código.

Para obtener instrucciones completas de configuración e integración con IDE, consulte [Habilidades de agente](skills.md).

## Servidor MCP de CLI

El servidor MCP de Ignite UI CLI (`igniteui-cli`) es un servidor MCP mantenido por Infragistics que expone las herramientas de scaffolding y documentación de Ignite UI CLI a la sesión activa del agente de IA. Una vez conectado, el asistente de IA puede crear proyectos de Angular, React, Blazor o Web Components, agregar y modificar componentes de Ignite UI, y responder preguntas de documentación y API, todo mediante indicaciones en lenguaje natural en la sesión de chat.

El servidor MCP de CLI se ejecuta mediante `npx` sin una instalación global:

```bash
npx -y igniteui-cli mcp
```

Use `ai-config` para escribir automáticamente la configuración MCP para su cliente de IA. El servidor se conecta a VS Code con GitHub Copilot, Cursor, Claude Desktop, Claude Code, JetBrains AI Assistant y cualquier otro cliente compatible con MCP que admita transporte STDIO. El formato de configuración exacto difiere según el cliente; consulte [MCP de CLI](cli-mcp.md) para obtener la guía de configuración completa.

No genera código de forma autónoma: expone herramientas al agente de IA, que las invoca en respuesta a las indicaciones del desarrollador.

## Servidor MCP de temas

El servidor MCP de temas de Ignite UI (`igniteui-theming`) es un servidor MCP independiente que expone el motor de temas de Ignite UI como contexto de agente consultable. Cubre acceso a tokens de diseño, definiciones de paleta, generación de propiedades personalizadas de CSS y validación de contraste WCAG AA. Está separado arquitectónicamente del servidor MCP de CLI; puede conectarse de forma independiente para dar al agente de IA acceso a herramientas de temas sin exponer herramientas de scaffolding de proyectos.

El servidor MCP de temas se inicia mediante `npx`:

```bash
npx -y igniteui-theming igniteui-theming-mcp
```

El servidor MCP de temas es compatible con Angular, React, Web Components y Blazor. Se actualiza con cada versión de Ignite UI para que los agentes siempre trabajen con la superficie de tokens actual.

Para detalles de configuración, consulte [MCP de temas](theming-mcp.md).

## Clientes de IA compatibles

El servidor MCP de CLI y el servidor MCP de temas funcionan con cualquier editor o cliente de IA que admita MCP con transporte STDIO.

| Cliente                     | Método de configuración                                   |
| --------------------------- | --------------------------------------------------------- |
| VS Code con GitHub Copilot  | `.vscode/mcp.json`                                            |
| Cursor                      | `.cursor/mcp.json`                                            |
| Claude Desktop (macOS)      | `~/Library/Application Support/Claude/claude_desktop_config.json`                                            |
| Claude Desktop (Windows)    | `%APPDATA%\Claude\claude_desktop_config.json`                                            |
| Claude Code                 | `.mcp.json` o el comando de CLI de MCP de Claude Code  |
| JetBrains AI Assistant      | **Herramientas → AI Assistant → Model Context Protocol (MCP)** |

Las Habilidades de agente son compatibles con GitHub Copilot mediante `.github/copilot-instructions.md`, Cursor mediante `.cursorrules` o `.cursor/rules/`, Windsurf mediante `.windsurfrules` y JetBrains AI Assistant mediante configuraciones de prompt de nivel de proyecto.

### Configuración rápida

El comando `ai-config` configura servidores MCP, copia archivos de habilidades específicos del framework en el directorio de habilidades de cada agente y configura archivos de instrucciones. Use `--assistants` para elegir qué asistentes de codificación reciben la configuración MCP y `--agents` para elegir qué agentes reciben archivos de habilidades. Si no se proporcionan parámetros, el comando entra en modo interactivo y le solicita seleccionar asistentes y agentes.

<PlatformBlock for="Angular">
**Uso de Angular Schematics:**

```bash
ng generate @igniteui/angular-schematics:ai-config
```

Esto también registra el servidor MCP `@angular/cli` junto con los servidores de Ignite UI para sus clientes seleccionados. </PlatformBlock>

**Uso de Ignite UI CLI:**

```bash
npx igniteui-cli ai-config
```

Si tiene Ignite UI CLI instalado globalmente, use la forma más corta:

```bash
ig ai-config
```

<PlatformBlock for="Angular">
<DocsAside type="info">
Las formas `npx igniteui-cli` e `ig` no registran el servidor MCP de `@angular/cli`. Usa el comando Angular Schematics anterior si quieres configurar los tres servidores en un solo paso.
</DocsAside>
</PlatformBlock>

<PlatformBlock for="Angular, React, WebComponents">
<DocsAside type="info">
El comando requiere que los paquetes de Ignite UI estén instalados en su proyecto (`npm install`). Si no se encuentran archivos de habilidades, asegúrese de que sus paquetes estén actualizados.
</DocsAside>
</PlatformBlock>

<PlatformBlock for="Blazor">
<DocsAside type="info">
Para Blazor, `ai-config` detecta tu proyecto mediante `.csproj` o `.sln`. Si no se encuentra ningún archivo de proyecto, el comando te solicita que selecciones un marco.
</DocsAside>
</PlatformBlock>

### Paso 1: Cargar Habilidades de agente

Copie el paquete de habilidades de Ignite UI para su framework en la ruta de descubrimiento de agentes de su proyecto.

<PlatformBlock for="Angular, React, WebComponents">
El paquete Skill se incluye con la biblioteca en `node_modules/igniteui-{framework}/skills/`.
</PlatformBlock>

<PlatformBlock for="Blazor">
El paquete Skill para Blazor se copia automáticamente cuando `ai-config` detecta un archivo `.csproj` o `.sln` en la raíz del proyecto.
</PlatformBlock>

Conéctelo a su IDE mediante la configuración persistente para su cliente.

Consulte [Habilidades de agente](skills.md) para ver la configuración completa.

### Paso 2: Conectar el servidor MCP de CLI

Agregue la entrada del servidor MCP `igniteui-cli` al archivo de configuración de su cliente de IA. Use la estructura JSON que coincida con su cliente:

**VS Code (`.vscode/mcp.json`):**

```json
{
  "servers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    }
  }
}
```

**Cursor, Claude Desktop, Claude Code, JetBrains y otros clientes MCP:**

```json
{
  "mcpServers": {
    "igniteui-cli": {
      "command": "npx",
      "args": ["-y", "igniteui-cli", "mcp"]
    }
  }
}
```

Para obtener la guía de configuración completa, incluidos VS Code, GitHub, Cursor, Claude Desktop, Claude Code, JetBrains y otros clientes compatibles con MCP, consulte [MCP de CLI](cli-mcp.md).

### Paso 3: Conectar el servidor MCP de temas (opcional)

Agregue la entrada `igniteui-theming` al mismo archivo de configuración MCP, junto con `igniteui-cli`:

**VS Code (`.vscode/mcp.json`):**

```json
{
  "servers": {
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

**Cursor, Claude Desktop, Claude Code, JetBrains y otros clientes MCP:**

```json
{
  "mcpServers": {
    "igniteui-theming": {
      "command": "npx",
      "args": ["-y", "igniteui-theming", "igniteui-theming-mcp"]
    }
  }
}
```

Para detalles de configuración y flujos de trabajo de temas, consulte [MCP de temas](theming-mcp.md).
