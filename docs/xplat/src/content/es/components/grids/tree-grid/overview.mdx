# Descripción general y configuración de la cuadrícula de árbol {Platform}

La cuadrícula de árbol {Platform} es un componente de UI que combina la funcionalidad de una cuadrícula de datos (tabla) con una vista de árbol, lo que permite mostrar fácilmente datos jerárquicos en un formato tabular. A diferencia de una cuadrícula normal, una cuadrícula de árbol permite expandir y contraer filas, revelando filas secundarias anidadas bajo filas primarias, lo que la hace útil para representar datos estructurados como exploradores de archivos, organigramas, tareas de proyectos o categorías de productos.

La cuadrícula de árbol {ProductName} se utiliza para mostrar y manipular datos jerárquicos o planos con facilidad. Enlace rápidamente sus datos con muy poco código o utilice una variedad de eventos para personalizar distintos comportamientos. Este componente proporciona un amplio conjunto de características como selección de datos, filtrado al estilo Excel, ordenación, paginación, plantillas y movimiento de columnas. Mostrar datos tabulares nunca ha sido tan fácil y atractivo gracias a la cuadrícula de árbol de UI basada en Material Table.

## Uso

La cuadrícula de árbol comparte muchas características con la cuadrícula, pero también agrega la capacidad de mostrar sus datos de forma jerárquica.
Para lograr esto, la cuadrícula de árbol nos proporciona un par de formas de definir las relaciones entre nuestros objetos de datos: mediante una [colección secundaria](overview.md#child-collection) para cada objeto de datos o mediante [claves primarias y externas](overview.md#primary-and-foreign-keys) para cada objeto de datos.

### Celdas de árbol

Independientemente de la opción que se utilice para construir la jerarquía de la cuadrícula de árbol (colección secundaria o claves primarias y externas), las filas de la cuadrícula de árbol se construyen con dos tipos de celdas:

- `GridCell` - Celda ordinaria que contiene un valor.
- `TreeGridCell` - Celda de árbol que contiene un valor, un indicador de expandir/contraer y un elemento div de sangría, que se basa en el nivel de la fila de la celda. Se puede acceder al nivel de un componente de fila mediante la propiedad `level` de su `treeRow` interno.

<DocsAside type="info">

Cada fila solo puede tener una celda de árbol, pero puede tener varias celdas ordinarias (o ninguna).

</DocsAside>

### Profundidad de expansión inicial

Inicialmente, la cuadrícula de árbol expandirá todos los niveles de nodos y los mostrará. Este comportamiento se puede configurar mediante la propiedad <ApiLink type="TreeGrid" member="expansionDepth" label="ExpansionDepth" />. De forma predeterminada, su valor es **Infinity**, lo que significa que se expandirán todos los niveles de nodos. Puede controlar la profundidad de expansión inicial estableciendo esta propiedad en un valor numérico. Por ejemplo, **0** mostrará solo los nodos de nivel raíz, **1** mostrará los nodos de nivel raíz y sus nodos secundarios, y así sucesivamente.

### Colección secundaria

Cuando usamos la opción de colección secundaria, cada objeto de datos contiene una colección secundaria, que se rellena con elementos del mismo tipo que el objeto de datos principal. De esta manera, cada registro de la cuadrícula de árbol tendrá una referencia directa a cualquiera de sus elementos secundarios. En este caso, la propiedad de datos de nuestra cuadrícula de árbol que contiene el origen de datos original será una colección definida jerárquicamente.

### Claves primarias y externas

Cuando usamos la opción de **claves primarias y externas**, cada objeto de datos contiene una clave primaria y una clave externa. La clave primaria es el identificador único del objeto de datos actual y la clave externa es el identificador único de su elemento principal. En este caso, la propiedad `data` de nuestra cuadrícula de árbol que contiene el origen de datos original será una colección plana.

En los datos de ejemplo anteriores, todos los registros tienen un ID, un ParentID y algunas propiedades adicionales como Name, JobTitle y Age. Como se mencionó anteriormente, el ID de los registros debe ser único, ya que será nuestro <ApiLink type="GridBaseDirective" member="primaryKey" label="PrimaryKey" />. ## Persistencia e integración

La sangría de la celda de la cuadrícula de árbol persiste en otras características de la cuadrícula de árbol, como el filtrado, la ordenación y la paginación.

- Cuando se aplica `Sorting` en una columna, las filas de datos se ordenan por niveles. Esto significa que las filas de nivel raíz se ordenarán independientemente de sus respectivos elementos secundarios. Sus respectivas colecciones secundarias también se ordenarán de forma independiente, y así sucesivamente.
- La primera columna (la que tiene un <ApiLink type="Column" member="visibleIndex" label="VisibleIndex" /> de 0) siempre es la columna de árbol.
- La columna que termina con un <ApiLink type="Column" member="visibleIndex" label="VisibleIndex" /> de 0 después de operaciones como el anclaje de columnas, la ocultación de columnas y el movimiento de columnas se convierte en la columna de árbol.
- Las hojas de cálculo de Excel exportadas reflejan la jerarquía agrupando los registros tal como están agrupados en la propia cuadrícula de árbol. Todos los estados expandidos de los registros también se conservarán y reflejarán.
- Al exportar a CSV, los niveles y los estados expandidos se ignoran y todos los datos se exportan como planos.
