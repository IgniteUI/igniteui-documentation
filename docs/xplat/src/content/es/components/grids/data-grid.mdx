La propiedad <ApiLink type="Grid" member="id" label="Id" /> es un valor de cadena y es el identificador único de la cuadrícula, que se generará automáticamente si no se proporciona, mientras que <ApiLink type="Grid" member="data" label="data" /> enlaza la cuadrícula, en este caso a datos locales.

La propiedad <ApiLink type="GridBaseDirective" member="autoGenerate" label="AutoGenerate" /> indica a la cuadrícula que genere automáticamente los componentes <ApiLink type="Column" /> de la cuadrícula en función de los campos del origen de datos. También intentará deducir el tipo de datos adecuado para la columna si es posible. De lo contrario, el desarrollador debe definir explícitamente las columnas y la asignación a los campos del origen de datos.

## Definición de cuadrícula Bootstrap

{ProductName} incluye un potente sistema de diseño de cuadrícula bootstrap basado en flex. Se espera que cualquier aplicación moderna actual siga un enfoque de diseño web adaptable, lo que significa que puede ajustar con fluidez el diseño de los elementos HTML según el tamaño del dispositivo o simplemente al cambiar el tamaño del navegador. Un diseño de cuadrícula bootstrap fue el enfoque más utilizado en el pasado, pero un sistema de diseño basado en flex, como CSS grid, se ha vuelto más popular, ya que funciona en cualquier navegador. La directiva de diseño de {ProductName} permite flujo vertical y horizontal, incluido el ajuste de contenido/texto, la justificación y la alineación. La cuadrícula {ProductName} admite un diseño adaptable mediante CSS, lo que le brinda la máxima flexibilidad en cuanto al comportamiento de la cuadrícula al cambiar de tamaño.

</PlatformBlock>

## Cuadrícula {Platform} editable

Cada operación de edición de la cuadrícula incluye operaciones por lotes, lo que significa que la API le ofrece la opción de agrupar ediciones en una sola llamada al servidor, o puede realizar operaciones de edición/actualización de la cuadrícula a medida que se producen con las interacciones de la cuadrícula. Junto con una excelente experiencia de desarrollador como cuadrícula editable con operaciones CRUD, la cuadrícula incluye navegación con teclado similar a Excel. Se incluye la navegación de cuadrícula predeterminada común, además de la opción de invalidar cualquier opción de navegación para satisfacer las necesidades de sus clientes. Una cuadrícula editable con un excelente esquema de navegación es fundamental para cualquier aplicación moderna de línea de negocio; con la cuadrícula de Ignite UI lo hacemos fácil.

Siguiendo este tema, aprenderá más sobre [plantilla de celda](data-grid.md#cell-template) y [plantilla de edición de celda](data-grid.md#cell-editing-template) y edición.

<PlatformBlock for="Angular">

Al cambiar datos mediante la **plantilla de celda** usando `ngModel`, debe llamar a los métodos API adecuados para asegurarse de que el valor se actualice correctamente en la colección de datos subyacente de la cuadrícula Angular. En el fragmento anterior, la llamada `ngModelChange` pasa por la [API de edición](grid/cell-editing.md#editing-through-api) de la cuadrícula y atraviesa la canalización de edición de la cuadrícula, activando correctamente las [transacciones](grid/batch-editing.md) (si corresponde) y gestionando [resúmenes](grid/summaries.md), [selección](grid/selection.md), etc. Sin embargo, este `ngModelChange` se ejecutará cada vez que cambie el valor de la celda, no solo cuando el usuario haya terminado de editar, lo que da lugar a muchas más llamadas API.

> **Nota**:
> La cuadrícula expone un manejo predeterminado para tipos de columna numéricos, de cadena, de fecha y booleanos. Por ejemplo, la columna mostrará el icono `check` o `close`, en lugar de true/false de forma predeterminada, para el tipo de columna booleano.

<PlatformBlock for="Angular">

Si los datos de una celda están enlazados con `[(ngModel)]` y no se controla el cambio de valor, el nuevo valor **no** se actualizará correctamente en el origen de datos subyacente de la cuadrícula Angular. Al trabajar con edición de celdas con una plantilla personalizada, se recomienda encarecidamente usar la **plantilla de edición de celda** de la celda.

</PlatformBlock>

Cuando se implementa correctamente, la plantilla de edición de celda también garantiza que el <ApiLink type="CellType" member="editValue" label="EditValue" /> de la celda pase correctamente por el [ciclo de eventos de edición](grid/editing.md#event-arguments-and-sequence) de la cuadrícula.

### Formato de visualización personalizado

Hay parámetros opcionales para el formato:

- <ApiLink type="ColumnPipeArgs" member="format" label="Format" /> - determina qué partes de fecha/hora se muestran; el valor predeterminado es `'mediumDate'`, equivalente a **'MMM d, y'**
- <ApiLink type="ColumnPipeArgs" member="timezone" label="Timezone" /> - el desplazamiento de zona horaria para fechas. De forma predeterminada usa la zona horaria del sistema local del usuario final
- <ApiLink type="ColumnPipeArgs" member="digitsInfo" label="DigitsInfo" /> - objetos de representación decimal. Valor predeterminado **1.0-3**

Para permitir la personalización del formato de visualización mediante estos parámetros, se expone la entrada <ApiLink type="Column" member="pipeArgs" label="PipeArgs" />. Una buena práctica es separar toda la lógica relacionada con la obtención de datos en un servicio de datos independiente, así que vamos a crear un servicio que se encargará de obtener los datos del servidor.

Implementemos nuestro servicio en un archivo independiente

```typescript
// northwind.service.ts

import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs/Observable';
import { of } from 'rxjs/observable/of';
import { catchError, map } from 'rxjs/operators';
```

Estamos importando el decorador `Injectable`, que es un [ingrediente esencial](https://angular.io/guide/dependency-injection) en toda definición de servicio de {Platform}. `HttpClient` nos proporcionará la funcionalidad para comunicarnos con servicios backend. Devuelve un `Observable` de algún resultado al que nos suscribiremos en nuestro componente de cuadrícula.

**Nota**: Antes de Angular 5, `HttpClient` se encontraba en `@angular/http` y se llamaba `Http`.

Dado que recibiremos una respuesta JSON que contiene una matriz de registros, también podemos ayudarnos especificando qué tipo de datos esperamos que se devuelvan en el observable mediante la definición de una interfaz con la forma correcta. La comprobación de tipos siempre se recomienda y puede ahorrarle algunos dolores de cabeza más adelante.

```typescript
// northwind.service.ts

export interface NorthwindRecord {
    ProductID: number;
    ProductName: string;
    SupplierID: number;
    CategoryID: number;
    QuantityPerUnit: string;
    UnitPrice: number;
    UnitsInStock: number;
    UnitsOnOrder: number;
    ReorderLevel: number;
    Discontinued: boolean;
    CategoryName: string;
}
```

El servicio en sí es bastante simple y consta de un método: `FetchData`, que devolverá un `Observable<NorthwindRecord[]>`. En los casos en que la solicitud falle por cualquier motivo (servidor no disponible, error de red, etc.), `HttpClient` devolverá un error. Aprovecharemos el operador `CatchError`, que intercepta un Observable que falló y pasa el error a un controlador de errores. Nuestro controlador de errores registrará el error y devolverá un valor seguro.

```typescript
// northwind.service.ts

@Injectable()
export class NorthwindService {
    private url = 'http://services.odata.org/V4/Northwind/Northwind.svc/Alphabetical_list_of_products';

    constructor(private http: HttpClient) {}

    public fetchData(): Observable<NorthwindRecord[]> {
        return this.http
            .get(this.url)
            .pipe(
                map(response => response['value']),
                catchError(
                    this.errorHandler('Error loading Northwind data', [])
                )
            );
    }

    private errorHandler<T>(message: string, result: T) {
        return (error: any): Observable<any> => {
            console.error(`${message}: ${error.message}`);
            return of(result as T);
        };
    }
}
```

Asegúrese de importar tanto `HttpClientModule` como nuestro servicio en el módulo de la aplicación y registrar el servicio como proveedor.

```typescript
// app.module.ts

import { HttpClientModule } from '@angular/common/http';
import { NorthwindService } from './northwind.service';

@NgModule({
    imports: [
        HttpClientModule
    ],
    providers: [
        NorthwindService
    ]
})
export class AppModule {}
```

Después de implementar el servicio, lo inyectaremos en el constructor de nuestro componente y lo usaremos para recuperar los datos. El hook de ciclo de vida `ngOnInit` es un buen lugar para enviar la solicitud inicial.

**Nota**: En el código siguiente, quizá se pregunte por qué estamos estableciendo la propiedad *records* en una matriz vacía antes de suscribirnos al servicio. La solicitud Http es asíncrona y, hasta que se complete, la propiedad *records* será *undefined*, lo que provocará un error cuando la cuadrícula intente enlazarse a ella. Debe inicializarla con un valor predeterminado o usar un `BehaviorSubject`.

```typescript
// my.component.ts

@Component({

})
export class MyComponent implements OnInit {

    public records: NorthwindRecord[];

    constructor(private northwindService: NorthwindService) {}

    ngOnInit() {
        this.records = [];
        this.northwindService.fetchData().subscribe((records) => this.records = records);
    }
}
```

y en la plantilla del componente:

```html
    <igx-grid [data]="records">
        <igx-column field="ProductId"></igx-column>
        <!-- rest of the column definitions -->
    </igx-grid>
```

</PlatformBlock>

**Nota**: Por ahora, es mejor evitar la propiedad <ApiLink type="GridBaseDirective" member="autoGenerate" label="AutoGenerate" /> de la cuadrícula al enlazar a datos remotos. Asume que los datos están disponibles para inspeccionarlos y generar las columnas adecuadas. Esto normalmente no ocurre hasta que el servicio remoto responde, y la cuadrícula generará un error. Hacer que <ApiLink type="GridBaseDirective" member="autoGenerate" label="AutoGenerate" /> esté disponible al enlazar a un servicio remoto está en nuestra hoja de ruta para versiones futuras.

## Rendimiento (experimental)

El diseño de <ApiLink type="GridToolbar" member="grid" label="Grid" /> le permite aprovechar la característica Event Coalescing que Angular ha introducido. Esta característica permite mejorar el rendimiento en aproximadamente **20%** en términos de interacciones y capacidad de respuesta. Esta característica se puede habilitar a nivel de aplicación simplemente estableciendo las propiedades `ngZoneEventCoalescing` y `ngZoneRunCoalescing` en **true** en el método `bootstrapModule`:

```typescript
platformBrowserDynamic()
  .bootstrapModule(AppModule, { ngZoneEventCoalescing: true, ngZoneRunCoalescing: true })
  .catch(err => console.error(err));
```

Esta sigue siendo una característica experimental para `IgxGridComponent`. Esto significa que podría haber algunos comportamientos inesperados en la cuadrícula. En [**Angular**](https://angular.io/), la mayoría de los estilos reciben prefijos implícitamente gracias al plugin [Autoprefixer](https://www.npmjs.com/package/autoprefixer).

Sin embargo, para aplicar prefijos a **diseños de cuadrícula**, debe habilitar la **propiedad grid** de [Autoprefixer](https://www.npmjs.com/package/autoprefixer) con el comentario `autoprefixer grid:on`.

Para facilitar su trabajo, aplique el comentario en el archivo `src/styles.scss`.

```scss
// src/styles.scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
@include core();
/* autoprefixer grid:on */
@include theme($default-palette);
```

</PlatformBlock>

## Limitaciones conocidas

| Limitación                                                                                                                                                                                            | Descripción                                                                                                                                                                                                                                                                           |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Anchos de columna establecidos en `percentage` y `px`                                                                                                                                              | Actualmente no admitimos la mezcla de anchos de columna con `%` y `px`.                                                                                                                                                                                         |
| Al intentar filtrar una columna de tipo `number`                                                                                                                                                | Si se introduce un valor distinto de `number` en la entrada de filtrado, se devuelve `NaN` debido a una conversión incorrecta.                                                                                                                                                  |
| El `width` de la cuadrícula no depende de los anchos de columna                                                                                                                                             | El `width` de todas las columnas no determina la extensión de la propia cuadrícula. Está determinada por las dimensiones del contenedor principal o por el `width` definido de la cuadrícula.                                                                                                     |
| Cuadrícula anidada en contenedor principal                                                                                                                                                                       | Cuando el `width` de la cuadrícula no está establecido y se coloca en un contenedor principal con dimensiones definidas, la cuadrícula se extiende hasta este contenedor.                                                                                                                                              |
| Las columnas tienen un ancho de columna mínimo permitido. Según la variable CSS `--ig-size`, son los siguientes: <br/>"small": 56px <br/> "medium": 64px <br/> "large ": 80px | Si se establece un ancho menor que el mínimo permitido, no afectará a los elementos representados. Se representarán con el ancho mínimo permitido para el `--ig-size` correspondiente. Esto puede provocar un comportamiento inesperado con la virtualización horizontal y, por lo tanto, no se admite. |
| La altura de fila no se ve afectada por la altura de las celdas que no están representadas actualmente en la vista.                                                                                                            | Debido a la virtualización, una columna con una plantilla personalizada (que cambia la altura de la celda) que no está en la vista no afectará a la altura de fila. La altura de fila solo se verá afectada mientras la columna relacionada se desplace dentro de la vista.                                               |
