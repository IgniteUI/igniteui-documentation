# Descripción general de la cuadrícula dinámica {Platform}

La cuadrícula dinámica {Platform} se utiliza para resumir y representar datos multidimensionales voluminosos en un formato de tabulación cruzada. El resumen de datos se puede ordenar, agrupar o filtrar de forma fácil y rápida. Dichos datos pueden incluir sumas, promedios y otras estadísticas. Los usuarios finales pueden modificar el diseño de la tabla dinámica mediante operaciones de arrastrar y soltar, según sus necesidades.

La cuadrícula dinámica {Platform} presenta datos en una tabla dinámica y ayuda a los usuarios a realizar análisis complejos sobre el conjunto de datos proporcionado. Este sofisticado control de cuadrícula dinámica se utiliza para organizar, resumir y filtrar grandes volúmenes de datos que luego se muestran en un formato de tabla cruzada. Las características clave de una cuadrícula dinámica {Platform} son las dimensiones de fila, las dimensiones de columna, las agregaciones y los filtros.

<ApiLink type="PivotGrid" /> ofrece a los usuarios la capacidad de configurar y mostrar sus datos en una estructura de tabla dinámica multidimensional.
Las filas y columnas representan grupos de datos distintos, y los valores de las celdas de datos representan agregaciones. Esto permite realizar análisis de datos complejos basados en un conjunto de datos plano simple. <ApiLink type="PivotGrid" /> es una tabla dinámica con muchas funciones que proporciona una configuración sencilla de las diferentes dimensiones y valores, así como operaciones de datos adicionales sobre ellos, como filtrado y ordenación.

Se define mediante tres dimensiones principales: **rows**, **columns** y **values**. **rows** y **columns** definen la estructura agrupada que se muestra en las filas y columnas de la cuadrícula. **values** define los campos de agregación y la agregación que se utilizará para calcular y mostrar los valores relacionados de los grupos.

También se puede definir un filtro mediante la propiedad de configuración **filters**. Se puede utilizar para campos que no desea agregar como dimensión o valor, pero cuyos valores de miembro relacionados desea filtrar mediante la UI.

### Configuración de dimensiones

Cada configuración básica de dimensión requiere un <ApiLink type="PivotDimension" member="memberName" label="MemberName" /> que coincida con un campo de los **data** proporcionados.

Se pueden definir varias dimensiones hermanas, lo que crea un grupo anidado más complejo en el área de dimensión de fila o columna relacionada.

Las dimensiones se pueden reordenar o mover de un área a otra mediante sus chips correspondientes usando arrastrar y soltar.

Una dimensión también puede describir una jerarquía expandible mediante la propiedad <ApiLink type="PivotDimension" member="childLevel" label="ChildLevel" />, por ejemplo:

### Configuración de valores

Una configuración de valor requiere un **member** que coincida con un campo de los **data** proporcionados, o puede definir una función **aggregator** personalizada para escenarios personalizados más complejos. De forma predeterminada, hay 4 agregaciones predefinidas que se pueden utilizar según el tipo de datos del campo de datos:

- `PivotNumericAggregate` - para campos numéricos.
  Contiene las siguientes funciones de agregación: `SUM`, `AVG`, `MIN`, `MAX`, `COUNT`.
- `PivotDateAggregate` - para campos de fecha.
  Contiene las siguientes funciones de agregación: `LATEST`, `EARLIEST`, `COUNT`.
- `PivotTimeAggregate` - para campos de hora.
  Contiene las siguientes funciones de agregación: `LATEST`, `EARLIEST`, `COUNT`.
- `PivotAggregate` - para cualquier otro tipo de datos. Esta es la agregación base.
  Contiene las siguientes funciones de agregación: `COUNT`.

La función de agregación actual se puede cambiar en tiempo de ejecución mediante el menú desplegable del chip de valor. De forma predeterminada, muestra una lista de agregaciones disponibles basada en el tipo de datos del campo. También se puede establecer una lista personalizada de agregaciones mediante la propiedad <ApiLink type="PivotValue" member="aggregateList" label="AggregateList" />, por ejemplo:

### Propiedad Enable

<ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> es la interfaz que describe el estado actual del componente <ApiLink type="PivotGrid" />. Con ella, el desarrollador puede declarar campos de los datos como **rows**, **columns**, **filters** o **values**. La configuración permite habilitar o deshabilitar cada uno de estos elementos por separado. Solo los elementos habilitados se incluyen en el estado actual de la cuadrícula dinámica. El componente <ApiLink type="PivotDataSelector" /> utiliza la misma configuración y muestra una lista de todos los elementos: habilitados y deshabilitados. Para cada uno de ellos hay una casilla de verificación en el estado correspondiente. Los usuarios finales pueden ajustar fácilmente el estado dinámico alternando los diferentes elementos mediante estas casillas de verificación.
La propiedad `Enable` controla si un <ApiLink type="PivotDimension" /> o <ApiLink type="PivotValue" /> determinado está activo y participa en la vista dinámica representada por la cuadrícula dinámica.

### Generar configuración automáticamente

La propiedad <ApiLink type="PivotGrid" member="autoGenerateConfig" label="AutoGenerateConfig" /> genera automáticamente dimensiones y valores basados en los campos del origen de datos:

- Campos numéricos:
  - Creados como <ApiLink type="PivotValue" /> usando el agregador `PivotNumericAggregate.sum`.
  - Se agregan a la colección de valores y se habilitan de forma predeterminada.

- Campos no numéricos:
  - Creados como <ApiLink type="PivotDimension" />.
  - Deshabilitados de forma predeterminada.
  - Se agregan a la colección de columnas.

- Campos de fecha(solo el primer campo `date` está habilitado; los demás campos `date` aplican la regla de campos no numéricos):
  - Creados como <ApiLink type="PivotDateDimension" />
  - Habilitados de forma predeterminada
  - agregados a la colección de filas.

Esta característica permite a los desarrolladores crear rápidamente una vista dinámica sin especificar manualmente dimensiones y valores. Con un selector dinámico junto a la cuadrícula dinámica, los usuarios pueden habilitar y reordenar dimensiones y valores según sea necesario.

### Claves de cálculo de valores dinámicos

La cuadrícula dinámica proporciona una personalización de los campos de claves de objeto que utiliza para realizar sus cálculos dinámicos.
Se puede ver una vista más detallada de cómo se utilizan a continuación en los datos de ejemplo, donde se pueden ver valores ya agregados:

```json
[
    {
        ProductCategory: 'All', AllProducts: 'All Products', All: 1000, 'All-Bulgaria': 774, 'All-USA': 829, 'All-Uruguay': 524,
        AllProducts_records: [
            { ProductCategory: 'Clothing', 'All-Bulgaria': 774, 'All-USA': 296, 'All-Uruguay': 456 },
            { ProductCategory: 'Bikes', 'All-Uruguay': 68 },
            { ProductCategory: 'Accessories', 'All-USA': 293 },
            { ProductCategory: 'Components', 'All-USA': 240 }
        ]
    }
];
```

Todos estos se almacenan en la propiedad **pivotKeys**, que forma parte de <ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> y se puede utilizar para cambiar las claves dinámicas predeterminadas.

- **children** - Campo que almacena elementos secundarios para la construcción de jerarquías. Representa un mapa de valores agrupados y todos los pivotGridRecords que se basan en ese valor. Se puede utilizar en escenarios muy específicos, donde existe la necesidad de hacer algo al crear las jerarquías. No es necesario cambiarlo para el uso común.
- **records** - Campo que almacena una referencia a los registros de datos originales. Se puede ver en el ejemplo anterior: **AllProducts\_records**. Evite establecer campos en los datos con el mismo nombre que esta propiedad. Si sus registros de datos tienen la propiedad **records**, puede especificar un valor diferente y único para ella usando **pivotKeys**.
- **aggregations** - Campo que almacena valores de agregación. Se aplica al crear las jerarquías y tampoco debe cambiarse para escenarios comunes.
- **level** - Campo que almacena el nivel de dimensión basado en su jerarquía. Evite establecer campos en los datos con el mismo nombre que esta propiedad. Si sus registros de datos tienen la propiedad **level**, puede especificar un valor diferente y único para ella usando **pivotKeys**.
- **columnDimensionSeparator** - Separador utilizado al generar los valores de campo de columna únicos. Es el guion(**-**) del valor de ejemplo: **All-Bulgaria**.
- **rowDimensionSeparator** - Separador utilizado al generar los valores de campo de fila únicos. Se utiliza al crear los campos **records** y **level**.

Los valores predeterminados son:

## Problemas conocidos y limitaciones

| Limitación                                                                              | Descripción                                                                                                                                                                          |
| --------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| No se admite establecer columnas de forma declarativa.                                         | La cuadrícula dinámica genera sus columnas basándose en la configuración <ApiLink type="PivotGrid" member="columns" label="Columns" />, por lo que no se admite establecerlas de forma declarativa, como en la cuadrícula base. Dichas columnas se ignoran. |
| Establecer valores duplicados de la propiedad <ApiLink type="PivotDimension" member="memberName" label="MemberName" /> o <ApiLink type="PivotValue" member="member" label="Member" /> para dimensiones/valores. | Estas propiedades deben ser únicas para cada dimensión/valor. La duplicación puede provocar la pérdida de datos del resultado final.                                                            |
| La selección de filas solo se admite en modo **Single**.                                     | Actualmente no se admite la selección múltiple.                                                                                                                                       |

<PlatformBlock for="Angular">

\| La combinación de miembros de dimensión distingue entre mayúsculas y minúsculas| La cuadrícula dinámica crea grupos y combina los mismos valores (distinguiendo entre mayúsculas y minúsculas). Pero las dimensiones proporcionan <ApiLink type="PivotDimension" member="memberFunction" label="MemberFunction" /> y esto se puede cambiar allí; el resultado de <ApiLink type="PivotDimension" member="memberFunction" label="MemberFunction" /> se compara y se usa como valor de visualización.|

</PlatformBlock>
