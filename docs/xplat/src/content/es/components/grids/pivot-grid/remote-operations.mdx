# {Platform} Operaciones remotas de la cuadrícula dinámica

En escenarios en los que los datos dinámicos ya están agrupados y agregados desde un servicio remoto y no es necesario realizar más procesamiento en el cliente, la cuadrícula dinámica puede configurarse para usar una estrategia vacía personalizada que omitirá el procesamiento de datos en el cliente y le permitirá mostrar directamente los datos tal como están:

<PlatformBlock for="Angular">

```typescript
public pivotConfigHierarchy: IPivotConfiguration = {
    columnStrategy: NoopPivotDimensionsStrategy.instance(),
    rowStrategy: NoopPivotDimensionsStrategy.instance(),
}
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```typescript
public pivotConfigHierarchy: IgcPivotConfiguration = {
    columnStrategy: IgcNoopPivotDimensionsStrategy.instance(),
    rowStrategy: IgcNoopPivotDimensionsStrategy.instance(),
}
```

</PlatformBlock>

El siguiente ejemplo muestra cómo controlar escenarios en los que los datos ya están agregados y cómo debe ser su estructura:

<Sample src="/{PivotGridSample}/remote" height={530} alt="{Platform} Pivot Grid Custom Predefined Aggregations Example" />

Los usuarios tienen la posibilidad de lograr determinados escenarios proporcionando a la cuadrícula dinámica datos ya agregados.
Existen algunos requisitos sobre cómo deben ser los datos y algunos aspectos específicos relacionados con las jerarquías en la vista dinámica. Por ejemplo, para declarar una jerarquía en la dimensión **filas**:

```typescript
rows: [
    {
        memberName: 'AllProducts',
        memberFunction: () => 'All Products',
        enabled: true,
        childLevel: {
            memberName: 'ProductCategory',
            enabled: true
        }
    }
]
```

Y un ejemplo de los datos agregados sería:

```typescript
public aggregatedData = [
    {
        ProductCategory: 'All', AllProducts: 'All Products', All: 1000, 'All-Bulgaria': 774, 'All-USA': 829, 'All-Uruguay': 524, AllProducts_records: [
            { ProductCategory: 'Clothing', 'All-Bulgaria': 774, 'All-USA': 296, 'All-Uruguay': 456 },
            { ProductCategory: 'Bikes', 'All-Uruguay': 68 },
            { ProductCategory: 'Accessories', 'All-USA': 293 },
            { ProductCategory: 'Components', 'All-USA': 240 }
        ]
    }
];
```

La cuadrícula dinámica proporciona los campos de claves de objeto que usa para realizar sus cálculos dinámicos.

- <ApiLink type="PivotKeys" member="children" label="Children" /> - Campo que almacena elementos secundarios para la creación de jerarquías. Representa un mapa de valores agrupados y todos los pivotGridRecords que se basan en ese valor. Puede utilizarse en escenarios muy específicos en los que sea necesario hacer algo al crear las jerarquías. No es necesario cambiarlo para el uso común.
- <ApiLink type="PivotKeys" member="records" label="Records" /> - Campo que almacena la referencia a los registros de datos originales. Puede verse en el ejemplo anterior: **AllProducts\_records**. Evite establecer campos en los datos con el mismo nombre que esta propiedad. Si sus registros de datos tienen la propiedad **records**, puede especificar un valor diferente y único para ella mediante <ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" />.
- <ApiLink type="PivotKeys" member="aggregations" label="Aggregations" /> - Campo que almacena valores de agregación. Se aplica al crear las jerarquías y tampoco debe cambiarse en escenarios comunes.
- <ApiLink type="PivotKeys" member="level" label="Level" /> - Campo que almacena el nivel de dimensión en función de su jerarquía. Evite establecer campos en los datos con el mismo nombre que esta propiedad. Si sus registros de datos tienen la propiedad **level**, puede especificar un valor diferente y único para ella mediante <ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" />.
- <ApiLink type="PivotKeys" member="columnDimensionSeparator" label="ColumnDimensionSeparator" /> - Separador usado al generar los valores únicos de campo de columna. Es el guion(**-**) del ejemplo anterior: **All-Bulgaria**.
- <ApiLink type="PivotKeys" member="rowDimensionSeparator" label="RowDimensionSeparator" /> - Separador usado al generar los valores únicos de campo de fila. Es el guion bajo(**\_**) del ejemplo anterior: **AllProducts\_records**. Se usa al crear los campos <ApiLink type="PivotKeys" member="records" label="Records" /> y <ApiLink type="PivotKeys" member="level" label="Level" />.

Todos estos se almacenan en la propiedad <ApiLink type="PivotConfiguration" member="pivotKeys" label="PivotKeys" />, que forma parte de <ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> y puede usarse para cambiar las claves dinámicas predeterminadas.
Estos valores predeterminados son:

```typescript
export const DEFAULT_PIVOT_KEYS = {
    aggregations: 'aggregations', records: 'records', children: 'children', level: 'level',
    rowDimensionSeparator: '_', columnDimensionSeparator: '-'
};
```

Establecer <ApiLink type="NoopPivotDimensionsStrategy" /> para <ApiLink type="PivotConfiguration" member="columnStrategy" label="ColumnStrategy" /> y <ApiLink type="PivotConfiguration" member="rowStrategy" label="RowStrategy" /> omite la agrupación y agregación de datos realizadas por las canalizaciones de datos, pero la cuadrícula dinámica aún necesita declaraciones para las filas, columnas, valores y filtros a fin de representar la vista dinámica según lo esperado:

<PlatformBlock for="Angular">

```typescript
public pivotConfig: IPivotConfiguration = {
    rows: [
        {
            memberName: 'AllProducts',
            memberFunction: () => 'All Products',
            enabled: true,
            childLevel: {
                memberName: 'ProductCategory',
                enabled: true
            }
        }
    ],
    columns: [
        {
            memberName: 'All',
            enabled: true,
            childLevel: {
                memberName: 'Country',
                enabled: true
            }
        }
    ],
    values: [
        {
            member: 'UnitsSold',
            aggregate: {
                aggregator: IgxPivotNumericAggregate.sum,
                key: 'sum',
                label: 'Sum'
            },
            enabled: true
        },
    ]
}
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```typescript
public pivotConfig: IgcPivotConfiguration = {
    rows: [
        {
            memberName: 'AllProducts',
            memberFunction: () => 'All Products',
            enabled: true,
            childLevel: {
                memberName: 'ProductCategory',
                enabled: true
            }
        }
    ],
    columns: [
        {
            memberName: 'All',
            enabled: true,
            childLevel: {
                memberName: 'Country',
                enabled: true
            }
        }
    ],
    values: [
        {
            member: 'UnitsSold',
            aggregate: {
                aggregator: IgcPivotNumericAggregate.sum,
                key: 'sum',
                label: 'Sum'
            },
            enabled: true
        },
    ]
}
```

</PlatformBlock>

Es importante que los datos coincidan con la configuración. Para obtener los mejores resultados, no deben incluirse campos adicionales en los datos agregados y no debe dejarse sin declarar como filas o columnas ningún campo de los datos proporcionados. El componente <ApiLink type="PivotGrid" /> construye sus datos en función de <ApiLink type="PivotGrid" member="pivotConfiguration" label="PivotConfiguration" /> y se espera que la configuración y los datos agregados coincidan según corresponda.

De forma similar, para otras operaciones de datos remotos como ordenación y filtrado, el procesamiento de datos puede omitirse estableciendo las estrategias vacías relacionadas: <ApiLink type="GridBaseDirective" member="filterStrategy" label="FilterStrategy" />, <ApiLink type="GridBaseDirective" member="sortStrategy" label="SortStrategy" />:

<PlatformBlock for="Angular">

```html
<{PivotGridSelector} [filterStrategy]="noopFilterStrategy" [sortStrategy]="noopSortStrategy">
</{PivotGridSelector}>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<{PivotGridSelector} filter-strategy="noopFilterStrategy" sort-strategy="noopSortStrategy">
</{PivotGridSelector}>
```

</PlatformBlock>

```typescript
public noopFilterStrategy = NoopFilteringStrategy.instance();
public noopSortStrategy = NoopSortingStrategy.instance();
```
