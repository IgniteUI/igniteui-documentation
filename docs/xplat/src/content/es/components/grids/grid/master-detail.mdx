# {Platform} Cuadrícula maestro-detalle

El componente <ApiLink type="Grid" /> admite especificar una plantilla de detalles que muestra detalles adicionales para una fila concreta al expandir/contraer su contenido. Cuando se especifica, cada registro actúa como maestro, que al expandirse muestra una plantilla de detalles personalizable con datos contextuales para el registro actual.

Este modo resulta útil cuando necesita mostrar datos de estilo maestro-detalle en una estructura jerárquica.

## Configuración

Para configurar <ApiLink type="Grid" /> para que se muestre en modo maestro-detalle, debe especificar una plantilla para la cuadrícula:

<PlatformBlock for="Angular">

```html
<{ComponentSelector}>
         <ng-template GridDetail let-dataItem>
              <!-- Custom detail template content here -->
        </ng-template>
</{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```ts
constructor() {
  var grid = this.grid = document.getElementById('grid') as IgcGridComponent;
  this._bind = () => {
    grid.data = this.customersData;
    grid.detailTemplate = this.masterDetailTemplate;
  }
  this._bind();
}
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
    <IgbGrid DetailTemplateScript="DetailTemplate"  AutoGenerate=true  Data=northwindEmployees></IgbGrid>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrGrid detailTemplate={masterDetailTemplate} autoGenerate={false} data={nwindData}></IgrGrid>
```

</PlatformBlock>

El contexto de la plantilla son los datos del registro maestro, de modo que los valores del registro maestro pueden mostrarse en la plantilla de detalles. Por ejemplo:

<PlatformBlock for="Angular">

```html
<{ComponentSelector}>
         <ng-template GridDetail let-dataItem>
            <div *ngIf="dataItem.Category">
                <header>{{dataItem.Category.CategoryName}}</header>
                <span>{{dataItem.Category.Description}}</span>
            </div>
        </ng-template>
</{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```ts
    public masterDetailTemplate = (ctx: IgcGridMasterDetailContext) => {
        var data = ctx.implicit;
        return html` <div class="contact-container">
        <span><strong>Name:</strong> ${data.ContactName}</span> <br/>
        <span><strong>Title:</strong> ${data.ContactTitle}</span> <br/>
        <span><strong>Company:</strong> ${data.CompanyName}</span> <br/>
    </div>`;
    }
```

</PlatformBlock>

## API

## Navegación con teclado

- Cuando el foco está en una fila de detalles:

  - <kbd>🡑</kbd> - navega una fila hacia arriba y enfoca una celda de la fila anterior.
  - <kbd>🡓</kbd> - navega una fila hacia abajo y enfoca una celda de la fila siguiente.
  - <kbd>PESTAÑA</kbd> - Permite que el foco se desplace al siguiente elemento enfocable dentro de la plantilla si hay elementos enfocables; de lo contrario, se desplaza a la siguiente fila de la cuadrícula.
  - <kbd>MAYÚS</kbd> + <kbd>PESTAÑA</kbd> - mueve el foco a la fila anterior.

- Cuando el foco está en una fila de datos con expansor:
  - <kbd>ALT</kbd> + <kbd>🡒</kbd> o <kbd>ALT</kbd> + <kbd>🡓</kbd> - expande la fila.
  - <kbd>ALT</kbd> + <kbd>🡐</kbd> o <kbd>ALT</kbd> + <kbd>🡑</kbd> - contrae la fila.

## Problemas conocidos y limitaciones

| Limitaciones conocidas                                                                                                                                                  | Descripción                                                                                          |
| ------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------------------------- |
| La navegación con Tab dentro de la plantilla de detalles personalizada puede no actualizar la posición de desplazamiento de la cuadrícula maestra en caso de que el siguiente elemento enfocado esté fuera del área visible. | La navegación con Tab dentro de la plantilla de detalles personalizada queda a cargo del navegador.                          |
| La plantilla de detalles no se exportará a Excel.                                                                                                                    | Dado que la plantilla de detalles puede contener cualquier tipo de contenido, no podemos exportarla a Excel de forma predeterminada. |
| La función de búsqueda no resaltará elementos de la plantilla de detalles.                                                                                          |                                                                                                      |
