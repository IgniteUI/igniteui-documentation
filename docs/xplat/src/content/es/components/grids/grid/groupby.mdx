# {Platform} Agrupar por en la cuadrícula

El comportamiento Agrupar por de {ProductName} en {Platform} {GridName} crea filas de datos agrupadas en función de los valores de columna. Agrupar por en <ApiLink type="Grid" /> permite visualizar los grupos en una estructura jerárquica. Las filas de datos agrupadas se pueden expandir o contraer, y el orden de agrupación se puede cambiar mediante la UI o la API. Cuando la selección de filas está habilitada, se representa un selector de fila de Agrupar por en el área más a la izquierda de la fila de grupo. En caso de que la propiedad <ApiLink type="Grid" member="rowSelection" label="RowSelection" /> esté establecida en single, las casillas de verificación se deshabilitan y solo sirven como indicación del grupo donde se coloca la selección. Si la propiedad <ApiLink type="Grid" member="rowSelection" label="RowSelection" /> está establecida en multiple, al hacer clic sobre el selector de fila de Agrupar por se seleccionan todos los registros que pertenecen a este grupo.

## Ejemplo de Agrupar por en la cuadrícula {Platform}

Este ejemplo presenta las capacidades de agrupación de una gran cantidad de datos. Arrastrar los encabezados de columna a la parte superior (área de agrupación) permite a los usuarios ver los datos de la columna seleccionada en una estructura jerárquica. Pueden agrupar por varios campos arrastrando más encabezados de columna a la parte superior. Estas opciones de agrupación son útiles cuando se tienen tablas con numerosas filas y columnas donde los usuarios quieren presentar los datos de una manera mucho más rápida y visualmente aceptable.

## Estado de agrupación inicial

Es posible definir la agrupación inicial de la cuadrícula asignando una matriz de expresiones a la propiedad <ApiLink type="Grid" member="groupingExpressions" label="GroupingExpressions" /> de la cuadrícula.

```typescript
    ];
}
```

### API de expandir/contraer

Además de las expresiones de agrupación, también puede controlar los estados de expansión de las filas de grupo. Se almacenan en una propiedad separada del componente <ApiLink type="Grid" />, <ApiLink type="Grid" member="groupingExpansionState" label="GroupingExpansionState" />, que es una colección de <ApiLink type="GroupByExpandState" />. Cada estado de expansión se define de forma única por el nombre del campo para el que se crea y el valor que representa para cada nivel de agrupación, es decir, el identificador es una matriz jerárquica de <ApiLink type="GroupByKey" />.

Al igual que con <ApiLink type="Grid" member="groupingExpressions" label="GroupingExpressions" />, establecer una lista de <ApiLink type="GroupByExpandState" /> directamente en <ApiLink type="Grid" member="groupingExpansionState" label="GroupingExpansionState" /> cambiará la expansión en consecuencia. Además, <ApiLink type="Grid" /> expone un método <ApiLink type="Grid" member="toggleGroup" label="ToggleGroup" /> que alterna un grupo mediante la instancia del registro de grupo o a través de la propiedad <ApiLink type="RowDirective" member="expanded" label="Expanded" /> de la fila.

<PlatformBlock for="WebComponents">

```typescript
const groupRow = this.grid.getRowByIndex(0).groupRow;
grid.toggleGroup(groupRow);
```

```typescript
const groupRow = this.grid.getRowByIndex(0);
groupRow.expanded = false;
```

</PlatformBlock>

<PlatformBlock for="React">

```typescript

gridRef.current.toggleGroup(groupRow);
```


</PlatformBlock>

Puede controlar si los grupos deben crearse expandidos o no mediante la propiedad <ApiLink type="Grid" member="groupsExpanded" label="GroupsExpanded" />.

### API para seleccionar/deseleccionar todas las filas de un grupo

La selección/deselección de todas las filas de un grupo está disponible mediante los métodos de API <ApiLink type="Grid" member="selectRowsInGroup" label="SelectRowsInGroup" /> y <ApiLink type="Grid" member="deselectRowsInGroup" label="DeselectRowsInGroup" />.

El fragmento de código siguiente se puede usar para seleccionar todas las filas dentro de un grupo mediante el método de instancia de registro de grupo <ApiLink type="Grid" member="selectRowsInGroup" label="SelectRowsInGroup" />. Además, el segundo parámetro de este método es una propiedad booleana mediante la cual puede elegir si la selección de filas anterior se borrará o no. La selección anterior se conserva de forma predeterminada.

## Plantillas

### Plantillas de fila de grupo

La fila de grupo, excepto la UI de expandir/contraer, admite plantillas por completo. De forma predeterminada, representa un icono de agrupación y muestra el nombre del campo y el valor que representa. El contexto con el que se representa la plantilla es de tipo <ApiLink type="GroupByRecord" />.

Como ejemplo, la siguiente plantilla haría que el resumen de las filas de grupo fuera más detallado:

</PlatformBlock>


</PlatformBlock>

## Agrupar por en la cuadrícula {Platform} con paginación

Las filas de grupo participan en el proceso de paginación junto con las filas de datos. Cuentan para el tamaño de página de cada página. Las filas contraídas no se incluyen en el proceso de paginación. Cualquier operación de expansión o contracción obliga a la paginación a volver a calcular el recuento de páginas y ajustar el índice de página si es necesario.
Los grupos que abarcan varias páginas se dividen entre ellas. La fila de grupo solo es visible en la página en la que comienza y no se repite en páginas posteriores. La información de resumen de las filas de grupo se calcula en función de todo el grupo y no se ve afectada por la paginación.

## Navegación por teclado

La UI de agrupación admite las siguientes interacciones de teclado:

- Para filas de grupo (el foco debe estar en la fila o en la celda de expandir/contraer)
  - <kbd>ALT</kbd> + <kbd>DERECHA</kbd> - Expande el grupo
  - <kbd>ALT</kbd> + <kbd>IZQUIERDA</kbd> - Contrae el grupo
  - <kbd>ESPACIO</kbd> - selecciona todas las filas del grupo, si la propiedad <kbd>rowSelection</kbd> está establecida en multiple

- Para componentes <ApiLink pkg="core" type="Chip" /> de grupo en el área de agrupar por (el foco debe estar en el chip)
  - <kbd>MAYÚS</kbd> + <kbd>IZQUIERDA</kbd> - mueve el chip enfocado a la izquierda, cambiando el orden de agrupación, si es posible
  - <kbd>MAYÚS</kbd> + <kbd>DERECHA</kbd> - mueve el chip enfocado a la derecha, cambiando el orden de agrupación, si es posible
  - <kbd>ESPACIO</kbd> - cambia la dirección de ordenación
  - <kbd>ELIMINAR</kbd> - desagrupa el campo
  - Los elementos independientes del chip también pueden recibir foco y se puede interactuar con ellos usando la tecla <kbd>ENTRAR</kbd>.

## Agrupar por personalizado en la cuadrícula {Platform}

La cuadrícula permite definir agrupación personalizada por columna o por expresión de agrupación, lo que proporciona agrupación basada en una condición personalizada. Esto resulta útil cuando necesita agrupar por objetos complejos o para otros escenarios específicos de la aplicación.

<PlatformBlock for="Angular">

<DocsAside type="info">

Para implementar una agrupación personalizada, primero los datos deben ordenarse adecuadamente. Debido a esto, es posible que también necesite aplicar una estrategia de ordenación personalizada que extienda la base `DefaultSortingStrategy`. Después de ordenar los datos, los grupos personalizados se pueden determinar especificando un <ApiLink type="GroupingExpression" member="groupingComparer" label="GroupingComparer" /> para la columna o para la expresión de agrupación específica.

</DocsAside>

Además, extrae los valores de la fecha necesarios para la comparación.

Se define una función <PlatformBlock for="React"><ApiLink type="GroupingExpression" member="groupingComparer" label="GroupingComparer" /></PlatformBlock><ApiLink type="GroupingExpression" member="groupingComparer" label="GroupingComparer" /> para las expresiones de agrupación, que determina los elementos que pertenecen al mismo grupo según el modo de agrupación seleccionado. Los valores de los datos ordenados para los que esta función devuelve 0 se marcan como parte del mismo grupo.

```typescript
grid.groupingExpressions = [
    { fieldName: 'OrderDate', dir: SortingDirection.Desc,
    groupingComparer: (a, b) => {
            const dateA = this.getParsedDate(a);
            const dateB = this.getParsedDate(b);
            if (this.groupByMode === 'Month') {
                return dateA.month === dateB.month ? 0 : -1;
            } else if (this.groupByMode === "Year") {
                return dateA.year === dateB.year ? 0 : -1;
            } else if (this.groupByMode === 'Week') {
                return this.getWeekOfDate(a) === this.getWeekOfDate(b) ? 0 : -1;
            }
            return dateA.day === dateB.day && dateA.month === dateB.month ? 0 : -1;
        }
    }
];
```

### Importación del tema global

Para comenzar la personalización de la característica Agrupar por, debe importar el archivo `index`, donde se encuentran todas las funciones de estilo y mixins.

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

### Definición de un tema personalizado

A continuación, cree un tema nuevo que extienda `grid-theme` y acepte los parámetros necesarios para personalizar Agrupar por según se desee. También debe extender `chip-theme`, porque se usa en la característica Agrupar por.

```scss

$custom-theme: grid-theme(
    /* Group By properties that affect styling */
    $group-row-background: #494949,
    $group-row-selected-background: #383838,
    $group-label-column-name-text: #f8f8f8,
    $group-label-icon: #FFCD0F,
    $group-label-text: #f8f8f8,
    $group-count-background: #FFCD0F,

    $expand-icon-color: #FFCD0F,

    $cell-active-border-color: #FFCD0F,
    $row-selected-background: #fff6d3,
    $row-selected-text-color: #000,
    $drop-indicator-color: #FFCD0F
    /* add other features properties here... */
);

    $background: #494949,
    $text-color: #f8f8f8,
    $hover-text-color: #e7e7e7

```

### Definición de una paleta de colores personalizada

En el enfoque que describimos anteriormente, los valores de color estaban codificados de forma rígida. Como alternativa, puede lograr mayor flexibilidad usando las funciones `palette` y `color`.
`palette` genera una paleta de colores basada en los colores primarios y secundarios proporcionados.

```scss
$black-color: #292826;
$yellow-color: #FFCD0F;

$custom-palette: palette(
  $primary: $black-color,
  $secondary: $yellow-color
);
```

Después de generar una paleta personalizada, la función `color` se puede usar para obtener diferentes variedades de los colores primarios y secundarios.

```scss
$custom-theme: grid-theme(
    $group-row-background: color($custom-palette, "primary", 300),
    $group-row-selected-background: color($custom-palette, "primary", 400),
    $group-label-column-name-text:contrast-color($custom-palette, "primary", 500),
    $group-label-icon: color($custom-palette, "secondary", 600),
    $group-label-text:contrast-color($custom-palette, "primary", 500),
    $group-count-background: color($custom-palette, "secondary", 600),
    $group-count-text-color: color($custom-palette, "primary", 400),
    $expand-icon-color: color($custom-palette, "secondary", 600),
    $expand-icon-hover-color: color($custom-palette, "secondary", 300),
    $cell-active-border-color: color($custom-palette, "secondary", 600)
);

$custom-chips-theme: chip-theme(
    $background: color($custom-palette, "primary", 300),
    $text-color:contrast-color($custom-palette, "primary", 500),
    $hover-text-color:contrast-color($custom-palette, "primary", 600)
);
```

### Definición de esquemas personalizados

Puede ir aún más lejos y crear una estructura flexible que tenga todos los beneficios de un [**esquema**](../themes/sass/schemas.md). El **esquema** es la receta de un tema.
Extienda uno de los dos esquemas predefinidos que se proporcionan para cada componente. En nuestro caso, usaríamos `$_light_grid`.

```scss
$custom-grid-schema: extend($_light-grid,(
    group-row-background: (-color:('secondary', 100)),
    group-row-selected-background: (-color:('primary', 400)),
    group-label-column-name-text: (-color:('primary', 600)),
    group-label-icon: (-color:('primary', 600)),
    group-label-text: (-color:('secondary', 700)),
    group-count-background: (-color:('primary', 600)),
    group-count-text-color: (-color:('secondary', 400)),
    expand-icon-color: (-color:('primary', 600)),
    expand-icon-hover-color: (-color:('primary', 400))
));
```

Para que se aplique el esquema personalizado, se debe extender `light` o los globales `dark`. Todo el proceso consiste en proporcionar a un componente un esquema personalizado y agregarlo posteriormente al tema del componente correspondiente.

```scss
$my-custom-schema: extend($light-schema, (
    -grid: $custom-grid-schema
));

$custom-theme: grid-theme(
    $palette: $custom-palette,
    $schema: $my-custom-schema
);
```

### Aplicación del tema personalizado

La forma más sencilla de aplicar su tema es con una instrucción `sass` `@include` en el archivo de estilos global:

```scss
@include grid($custom-theme);
@include chip($custom-chips-theme);
```

### Tema de componente con ámbito

Para que el tema personalizado afecte solo a un componente específico, puede mover todos los estilos que acaba de definir desde el archivo de estilos global al archivo de estilos del componente personalizado (incluida la importación del archivo `index`).

De esta manera, debido a [ViewEncapsulation](https://{Platform}.io/api/core/Component#encapsulation) de {Platform}, sus estilos se aplicarán solo a su componente personalizado.

 <DocsAside type="info">

Si el componente usa una ViewEncapsulation [emulada](../themes/styles.md#view-encapsulation), es necesario penetrar esta encapsulación usando `::ng-deep` para aplicar estilos a la cuadrícula.

 </DocsAside>

 <DocsAside type="info">

Envuelva la instrucción dentro de un selector `:host` para evitar que sus estilos afecten a elementos fuera de nuestro componente:

 </DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-theme);
        @include chip($custom-chips-theme);
    }
}
```
