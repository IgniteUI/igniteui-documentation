# {Platform} {ComponentTitle} Actualizaciones de datos en vivo

La característica Actualizaciones de datos en vivo de {ProductName} en {Platform} {ComponentTitle} se utiliza para habilitar actualizaciones en tiempo real o casi en tiempo real de los datos mostrados dentro de la cuadrícula. Esto resulta extremadamente útil en aplicaciones donde los datos cambian constantemente, como rastreadores del mercado de valores, marcadores deportivos en vivo o paneles de IoT (Internet de las cosas). <ApiLink type="{ComponentName}" /> puede gestionar miles de actualizaciones por segundo, sin dejar de responder a las interacciones del usuario.

## {Platform} Ejemplo de actualización de datos en vivo

El ejemplo siguiente demuestra el rendimiento de {ComponentTitle} cuando todos los registros se actualizan varias veces por segundo. Utilice los controles de UI para elegir el número de registros cargados y la frecuencia de las actualizaciones.
Envíe los mismos datos al [gráfico de columnas](../../charts/types/column-chart.md) para experimentar las potentes capacidades de gráficos de Ignite UI for Angular. El botón `Chart` mostrará los datos de precios de categorías por región para las filas seleccionadas, y el botón de columna `Chart` mostrará lo mismo para la fila actual.

## Enlace de datos y actualizaciones

Un servicio proporciona datos al componente cuando se carga la página y cuando se utiliza el controlador deslizante para obtener un determinado número de registros. Aunque en un escenario real los datos actualizados se consumirían desde el servicio, aquí los datos se actualizan en el código. Esto se hace para mantener la demostración simple y centrarse en su objetivo principal: demostrar el rendimiento de la cuadrícula.

## Plantillas

La actualización de la vista funciona de la misma manera para columnas con una plantilla predeterminada y para columnas con una plantilla personalizada. Sin embargo, se recomienda mantener las plantillas personalizadas relativamente simples. A medida que aumenta el número de elementos en la plantilla, también aumenta el impacto negativo en el rendimiento.

## Fuente de datos en vivo con componentes Dock Manager e igxGrid

El propósito de esta demostración es presentar un panel de pantalla financiera con un flujo de datos en tiempo real mediante un backend de concentrador [SignalR](https://dotnet.microsoft.com/apps/aspnet/signalr).
Como puede ver, el componente igxGrid gestiona con facilidad las actualizaciones de alta frecuencia desde el servidor. El código de la aplicación ASP.NET Core que utiliza SignalR se puede encontrar en este [repositorio público de GitHub](https://github.com/IgniteUI/finjs-web-api).

<Sample src="/{ComponentSample}/finjs-dock-manager" height={700} alt="{Platform} {ComponentTitle} Live data Update Example with a service" />

### Iniciar la conexión del concentrador

signal-r.service gestiona la conectividad y las actualizaciones de los parámetros gestionables expuestos **frecuencia**, **volumen** y **conmutador de estado de actualización en vivo** (Iniciar/Detener).

```ts
this.hubConnection = new signalR.HubConnectionBuilder()
        .configureLogging(signalR.LogLevel.Trace)
        .withUrl('https://www.infragistics.com/angular-apis/webapi/streamHub')
        .build();
    this.hubConnection
        .start()
        .then(() => {
            this.hasRemoteConnection = true;
            this.registerSignalEvents();
            this.broadcastParams(interval, volume, live, updateAll);
        })
        .catch(() => {});
```

En función de la frecuencia especificada, las cuadrículas recibirán del servidor un total de 30 nuevas actualizaciones. Se aplican clases cellStyle específicas a las tres columnas que gestionan los cambios (Precio, Cambio y Cambio en porcentaje).

### Frecuencia de actualización y volumen de datos

Al utilizar el panel de acciones de la izquierda, puede gestionar la frecuencia de la fuente de datos y el volumen de los datos solicitados. Todas las cuadrículas utilizan el mismo origen de datos. Puede utilizar los demás elementos de acción para **detener la fuente de datos**, cambiar el **tema de la aplicación** o agregar **dinámicamente un contenedor DockSlot** con un igxGrid.

Utilizamos el método `updateParameters` para solicitar un nuevo conjunto de datos con cierta frecuencia. Este método forma parte de la [implementación del concentrador de transmisión](https://github.com/IgniteUI/finjs-web-api/blob/master/WebAPI/Models/StreamHub.cs#L18) de SignalR.

```ts
this.hubConnection.invoke('updateparameters', frequency, volume, live, updateAll)
    .then(() => console.log('requestLiveData', volume))
    .catch(err => {
        console.error(err);
    });
```
