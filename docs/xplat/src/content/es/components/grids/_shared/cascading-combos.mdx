# {Platform} {ComponentTitle} con combos en cascada

La funcionalidad de edición de {ComponentTitle} ofrece la oportunidad de usar componentes Combobox en cascada. Al seleccionar el valor en cualquier <ApiLink type="Combo" /> anterior, los usuarios recibirán solo los datos que sean relevantes para su selección dentro del siguiente componente Combobox {Platform}.

## Configuración

Para habilitar la edición de columnas, asegúrese de que la propiedad <ApiLink type="Column" member="editable" /> esté establecida en `true`.

Una vez habilitada la edición de columnas, puede empezar agregando su <ApiLink type="Combo" />. Tenga en cuenta que aquí, para tener disponible una única selección, deberá establecer la propiedad <ApiLink type="Combo" member="singleSelect" />.

- <ApiLink type="Combo" member="displayKey" /> - Obligatorio para matrices de objetos - Especifica qué propiedad se utilizará para el texto de los elementos. Si no se especifica ningún valor para <ApiLink type="Combo" member="displayKey" />, el combo usará el <ApiLink type="Combo" member="valueKey" /> especificado (si lo hay).

Para controlar el cambio de selección, necesitamos el evento de cambio. Los argumentos del evento emitido contienen información sobre la selección anterior al cambio, la selección actual y los elementos que se agregaron o eliminaron. Por lo tanto, filtrará los valores en función de la selección del combo anterior.

<PlatformBlock for="Blazor">

```javascript
//In Javascript
igRegisterScript("CountryChange", (ctx) => {
    const value = e.detail.newValue;
    cell.update(value);
    const nextCombo = document.getElementById("region_" + cell.id.rowID);
    const nextProgress = document.getElementById("progress_region_" + cell.id.rowID);
    if (value === "") {
        nextCombo.deselect(nextCombo.value);
        nextCombo.disabled = true;
        nextCombo.data = [];
    } else {
        nextProgress.style.display = "block";
        setTimeout(() => {
            nextProgress.style.display = "none";
            nextCombo.disabled = false;
            nextCombo.data = this.regions.filter(x => x.Country === value);
        }, 2000);
    }
});
```

```tsx
    const onCountryChange = (rowId: string, event: CustomEvent) => {
        const regionCombo = getComboRef(`region_${rowId}`).current;
        const cityCombo = getComboRef(`city_${rowId}`).current;
        const regions = regions;
        const newValue = event.detail.newValue[0];

        if (newValue === undefined) {
            regionCombo.deselect(regionCombo.value);
            regionCombo.disabled = true;
            regionCombo.data = [];

            cityCombo.deselect(regionCombo.value);
            cityCombo.disabled = true;
            cityCombo.data = [];
        } else {
            regionCombo.disabled = false;
            regionCombo.data = regions.filter(x => x.Country === newValue);

            cityCombo.deselect(cityCombo.value);
            cityCombo.disabled = true;
            cityCombo.data = [];
        }
    }
```

Para controlar el cambio de selección, necesitamos `selectionChanging`. Los argumentos del evento emitido, `IComboSelectionChangingEventArgs`, contienen información sobre la selección anterior al cambio, la selección actual y los elementos que se agregaron o eliminaron. Por lo tanto, filtrará los valores en función de la selección del combo anterior.

```html
<igx-combo [data]="countriesData" (selectionChanging)="countryChanging($event)"></igx-combo>
```

```typescript
public countryChanging(event: IComboSelectionChangeEventArgs) {
    if (event.added.length) {
        event.newSelection = event.added;
    }
}
```

## Problemas conocidos y limitaciones

| Limitación                                              | Descripción                                                                                                                    |
| ------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| La lista desplegable del combo puede ocultarse detrás de otros elementos de la UI. | Debido al orden de apilamiento de los elementos en la cuadrícula, la lista desplegable del combo puede ocultarse detrás de otros elementos como encabezados, pies de página, etc. |
