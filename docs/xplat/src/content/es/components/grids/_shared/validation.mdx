# {Platform} {ComponentTitle} Edición y validación

El {Platform} {ComponentName} proporciona edición que cuenta con un mecanismo integrado de validación de la entrada del usuario al editar celdas/filas. Extiende la funcionalidad de validación de formularios para permitir una integración más sencilla con una funcionalidad conocida. Cuando cambia el estado del editor, se aplican indicadores visuales a la celda editada en `{ComponentName}`.

## Configuración

### Configurar mediante configuración basada en plantillas

Extendemos algunos validadores de formularios de {Platform} para que trabajen directamente con <ApiLink type="Column" />. Los mismos validadores están disponibles como atributos para establecerse declarativamente en <ApiLink type="Column" />. Los siguientes validadores son compatibles de forma predeterminada:

- Required
- Min
- Max
- Email
- MinLength
- MaxLength
- Pattern

## API del servicio de validación

La cuadrícula expone un servicio de validación mediante la propiedad <ApiLink type="{ComponentName}" member="validation" />.

Ese servicio tiene las siguientes APIs públicas:

- `Valid` - devuelve si el estado de validación de la cuadrícula es válido.
- `GetInvalid` - devuelve registros con estados no válidos.
- `Clear` - borra el estado de un registro por id o borra todo el estado si no se proporciona ningún id.
- <ApiLink type="{ComponentName}" member="markAsTouched" /> - marca el registro/campo relacionado como tocado.

Los estados no válidos persistirán hasta que los errores de validación en ellos se corrijan según la regla de validación o se borren.

## Desencadenadores de validación

La validación se desencadenará en los siguientes escenarios:

- Durante la edición mediante el editor de celdas según el <ApiLink type="GridValidationTrigger" /> de la cuadrícula. Ya sea en `Change` mientras se escribe en el editor, o en `Blur` cuando el editor pierde el foco o se cierra.
- Al actualizar celdas/filas mediante la API: <ApiLink type="{ComponentName}" member="updateRow" />, <ApiLink type="{ComponentName}" member="updateCell" />, etc.
- Al usar la edición por lotes y la API `Undo`/`Redo` del servicio de transacciones.

<DocsAside type="info">

La validación no se desencadenará para registros que no se hayan editado mediante la entrada del usuario o mediante la API de edición. Los indicadores visuales en la celda solo se mostrarán si la entrada relacionada se considera tocada, ya sea mediante la interacción del usuario o mediante la API <ApiLink type="{ComponentName}" member="markAsTouched" /> del servicio de validación.

</DocsAside>

## Opciones de personalización de la validación de {Platform} {ComponentTitle}

### Establecer un validador personalizado

Puede definir su propia directiva de validación para usarla en un <ApiLink type="Column" /> en la plantilla.

```ts
@Directive({
    selector: '[phoneFormat]',
    providers: [{ provide: NG_VALIDATORS, useExisting: PhoneFormatDirective, multi: true }]
})
export class PhoneFormatDirective extends Validators {
    @Input('phoneFormat')
    public phoneFormatString = '';

    public validate(control: AbstractControl): ValidationErrors | null {
        return this.phoneFormatString ? phoneFormatValidator(new RegExp(this.phoneFormatString, 'i'))(control)
            : null;
    }
}
```

Una vez que se defina y agregue en el módulo de su aplicación, puede establecerla declarativamente en una columna determinada de la cuadrícula:

```html
<igx-column phoneFormat="\+\d{1}\-(?!0)(\d{3})\-(\d{3})\-(\d{4})\b"></igx-column>
```

### Cambiar la plantilla de error predeterminada

Puede definir su propia plantilla de error personalizada que se mostrará en la información sobre herramientas de error cuando la celda entre en un estado no válido. Esto es útil en escenarios en los que desea agregar su propio mensaje de error personalizado o cambiar de otro modo el aspecto o el contenido del mensaje.

```html
<igx-column>
  <ng-template igxCellValidationError let-cell='cell' let-defaultErr="defaultErrorTemplate">
      <ng-container *ngTemplateOutlet="defaultErr">
      </ng-container>
      <div *ngIf="cell.validation.errors?.['phoneFormat']">
        Please enter correct phone format
      </div>
  </ng-template>
</igx-column>
```

### Impedir salir del modo de edición en un estado no válido

En ese escenario, puede usar los eventos <ApiLink type="{ComponentName}" member="cellEdit" /> o <ApiLink type="{ComponentName}" member="rowEdit" /> y cancelar el evento en caso de que el nuevo valor no sea válido.

Los argumentos de ambos eventos tienen una propiedad `Valid` y se pueden cancelar en consecuencia.

### Validación entre campos

En algunos escenarios, la validación de un campo puede depender del valor de otro campo en el registro.

En ese caso, se puede usar un validador personalizado para comparar los valores del registro mediante su <PlatformBlock for="Angular"><ApiLink type="IGridFormGroupCreatedEventArgs" member="formGroup" label="FormGroup" /></PlatformBlock><PlatformBlock for="WebComponents,Blazor"><ApiLink type="FormGroup" /></PlatformBlock> compartido.

<ComponentBlock for="Grid">

El siguiente ejemplo demuestra una validación entre campos de diferentes campos del mismo registro. Comprueba la validez de las fechas en comparación con la fecha actual y entre la fecha de activación y la fecha de creación del registro, así como la proporción de acuerdos ganados/perdidos para cada empleado. Todos los errores se recopilan en una columna anclada independiente que muestra que el registro no es válido y muestra los errores relacionados.

Las siguientes líneas de código muestran la función de validador entre campos, que contiene las comparaciones y establece los errores relacionados con ellas.

<PlatformBlock for="Blazor">

```razor
private rowValidator(): ValidatorFn {
    return (formGroup: FormGroup): ValidationErrors | null => {
        let returnObject = {};
        const createdOnRecord = formGroup.get('created_on');
        const lastActiveRecord = formGroup.get('last_activity');
        const winControl = formGroup.get('deals_won');
        const loseControl = formGroup.get('deals_lost');
        const actualSalesControl = formGroup.get('actual_sales');

        // Validate dates
        const curDate = new Date();
        if (new Date(createdOnRecord.value) > curDate) {
            // The created on date should not be greater than current date.
            returnObject['createdInvalid'] =  true;
        }
        if (new Date(lastActiveRecord.value) > curDate) {
            // The last active date should not be greater than current date.
            returnObject['lastActiveInvalid'] = true;
        }
        if (new Date(createdOnRecord.value) > new Date(lastActiveRecord.value)) {
            // The created on date should not be greater than last active date.
            returnObject['createdLastActiveInvalid'] = true;
        }

        // Validate deals
        const dealsRatio = this.calculateDealsRatio(winControl.value, loseControl.value);
        if (actualSalesControl.value === 0 && dealsRatio > 0) {
            // If the actual sales value is 0 but there are deals made.
            returnObject['salesZero'] = true;
        }
        if (actualSalesControl.value > 0 && dealsRatio === 0) {
            // If the deals ratio based on deals won is 0 but the actual sales is bigger than 0.
            returnObject['salesNotZero'] = true;
        }

        return returnObject;
    };
}

public calculateDealsRatio(dealsWon, dealsLost) {
    if (dealsLost === 0) return dealsWon + 1;
    return Math.round(dealsWon / dealsLost * 100) / 100;
}

```

Los validadores entre campos se pueden agregar al formGroup en el evento `FormGroupCreated`. En ellos se pueden comparar varios campos para determinar su validez.

```ts
  public formCreateCustomerHandler(event: IGridFormGroupCreatedEventArgs) {
        const formGroup = event.formGroup;
        formGroup.addValidators(this.addressValidator());
    }

    public formCreateOrderHandler(event: IGridFormGroupCreatedEventArgs) {
        const formGroup = event.formGroup;
        formGroup.addValidators(this.dateValidator());
    }

    public addressValidator(): ValidatorFn {
        return (control: AbstractControl): ValidationErrors | null => {
            const formGroup = control;
            let returnObject = {};
            const city = formGroup.get('City');
            const country = formGroup.get('Country');
            const validCities = this.countryData.get(country.value);
            if (!validCities || !validCities[city.value]) {
                returnObject['invalidAddress'] = true;
            }
            return returnObject;
        }
    }

    public dateValidator(): ValidatorFn {
        return (control: AbstractControl): ValidationErrors | null => {
            const formGroup = control;
            let returnObject = {};
            const orderDate = formGroup.get('OrderDate').value;
            const shippedDate = formGroup.get('ShippedDate').value;
            if (new Date(shippedDate) < new Date(orderDate)) {
                returnObject['invalidRange'] = true;
            }
            return returnObject;
        }
    }
```

```ts
    public isRowValid(cell: IgxGridCell) {
        const hasErrors = !!cell.row.validation.errors || cell.row.cells.some(x => !!x.validation.errors);
        return !hasErrors;
    }

    public stateMessage(cell: IgxGridCell) {
        const messages = [];
        const row = cell.row;
        if  (row.validation.errors?.invalidAddress) {
            messages.push('The address information is invalid. City does not match the Country.');
        }
        if  (row.validation.errors?.invalidRange) {
            messages.push('The ShippedDate cannot be before the OrderDate.');
        }
        const cellValidationErrors = row.cells.filter(x => !!x.validation.errors);
        if (cellValidationErrors && cellValidationErrors.length > 0) {
            const fields = cellValidationErrors.map(x => x.column.field).join(',');
            messages.push('The following fields are required: ' + fields);
        }

        if (messages.length === 0) {
            // no errors
            return ['Valid'];
        }
        return messages;
    }
```
