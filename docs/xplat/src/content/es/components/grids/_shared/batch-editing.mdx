# {Platform} {ComponentTitle} Edición por lotes y transacciones

<ComponentBlock for="Grid">

La característica de edición por lotes de <ApiLink type="{ComponentName}" /> se basa en <ApiLink type="TransactionService" />. Consulte el tema [**Jerarquía de clases del servicio de transacciones**](../transaction-classes.md) para ver una descripción general de <ApiLink type="TransactionService" /> y detalles sobre cómo se implementa.

</ComponentBlock>

<ComponentBlock for="TreeGrid">

La característica de edición por lotes de <ApiLink type="{ComponentName}" /> se basa en `HierarchicalTransactionService`. Consulte el tema [**Jerarquía de clases del servicio de transacciones**](../transaction-classes.md) para ver una descripción general de `HierarchicalTransactionService` y detalles sobre cómo se implementa.

</ComponentBlock>

<ComponentBlock for="HierarchicalGrid">

Para usar `HierarchicalTransactionService` con <ApiLink type="{ComponentName}" />, pero hacer que acumule registros de transacciones separados para cada isla, se debe proporcionar en su lugar una fábrica de servicios. Hay una exportada y lista para su uso como `HierarchicalTransactionServiceFactory`.

</ComponentBlock>

A continuación se muestra un ejemplo detallado de cómo se habilita la edición por lotes para el componente <ApiLink type="{ComponentName}" />.

## {Platform} {ComponentTitle} Ejemplo de edición por lotes y transacciones

El siguiente ejemplo demuestra un escenario en el que <ApiLink type="{ComponentName}" /> tiene <ApiLink type="{ComponentName}" member="batchEditing" /> habilitado y la edición de filas habilitada. Esto último garantizará que se agregue una transacción después de confirmar la edición de toda la fila.

## Uso

Debe habilitar <ApiLink type="{ComponentName}" member="batchEditing" /> desde su {ComponentTitle}:

<PlatformBlock for="WebComponents">

```html
<{ComponentSelector} [data]="data" [batchEditing]="true">
</{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<{ComponentSelector} Data=data BatchEditing="true">
</{ComponentSelector}>
```

</PlatformBlock>

Esto garantizará que se proporcione una instancia adecuada del servicio `Transaction` para <ApiLink type="{ComponentName}" />. El <ApiLink type="TransactionService" /> adecuado se proporciona mediante un `TransactionFactory`.

<PlatformBlock for="Angular">

Puede obtener más información sobre esta implementación interna en el [tema de transacciones](../transaction-classes.md#transaction-factory).

</PlatformBlock>

Después de habilitar la edición por lotes, defina un <ApiLink type="{ComponentName}" /> con una fuente de datos enlazada y <ApiLink type="{ComponentName}" member="rowEditable" /> establecido en true, y enlace:

```ts
constructor() {
    var grid = this.grid = document.getElementById('grid') as IgcGridComponent;
    var undo = this.undo = document.getElementById('undo') as HTMLButtonElement;
    var redo = this.redo = document.getElementById('redo') as HTMLButtonElement;
    var commit = this.commit = document.getElementById('commit') as HTMLButtonElement;
}

this._bind = () => {
    grid.data = this.data;
    undo.addEventListener('click', this.OnUndoClick);
    redo.addEventListener('click', this.OnRedoClick);
    commit.addEventListener('click', this.OnCommitClick);
}
this._bind();

private OnCommitClick() {
    if (this.grid.transactions.getAggregatedChanges(false).length > 0)
        this.grid.transactions.commit(this.data);
}

private OnUndoClick() {
    if (this.grid.transactions.canUndo)
        this.grid.transactions.undo();
}

private OnRedoClick() {
    if (this.grid.transactions.canRedo)
        this.grid.transactions.redo();
}
```

export class GridBatchEditingSampleComponent {
    @ViewChild('grid', { read: IgxGridComponent }) public gridRowEditTransaction: IgxGridComponent;

    public undo() {
        /* exit edit mode and commit changes */
        this.grid.endEdit(true);
        this.grid.transactions.undo();
    }

    public redo() {
        /* exit edit mode and commit changes */
        this.grid.endEdit(true);
        this.grid.transactions.redo()

    public commit() {
        this.grid.transactions.commit(this.data);
        this.toggle.close();

export class HierarchicalGridBatchEditingSampleComponent {
    public undo(grid: IgxHierarchicalGridComponent) {
        /* exit edit mode and commit changes */
        grid.endEdit(true);
        grid.transactions.undo();
    }

    public redo(grid: IgxHierarchicalGridComponent) {
        /* exit edit mode and commit changes */
        grid.endEdit(true);
        grid.transactions.redo();

    public commit() {
        this.hierarchicalGrid.transactions.commit(this.data);
        this.childGrid.hgridAPI.getChildGrids().forEach((grid) => {
            grid.transactions.commit(grid.data);
        });

<DocsAside type="info">

La API de transacciones no gestionará el final de la edición y deberá hacerlo por su cuenta. De lo contrario, `{ComponentName}` permanecería en modo de edición. Una forma de hacerlo es llamando a `EndEdit` en el método correspondiente.

</DocsAside>

<ComponentBlock for="TreeGrid">

Eliminar un nodo primario en <ApiLink type="{ComponentName}" /> tiene algunas particularidades. Si está usando datos jerárquicos, los elementos secundarios se eliminarán al eliminar su primario. Si está usando datos planos, puede establecer el comportamiento deseado mediante la propiedad `CascadeOnDelete` de <ApiLink type="{ComponentName}" />. Esta propiedad indica si los registros secundarios deben eliminarse cuando se elimina su primario (de forma predeterminada, está establecida en **true**).

</ComponentBlock>

<DocsAside type="info">

Deshabilitar la propiedad <ApiLink type="{ComponentName}" member="rowEditable" /> modificará `{ComponentName}` para crear transacciones al cambiar una celda y no expondrá la superposición de edición de filas en la UI.

</DocsAside>
