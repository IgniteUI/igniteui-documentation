# {Platform} {ComponentTitle} Anclaje de columnas

La característica de anclaje de columnas {ProductName} en {Platform} {ComponentTitle} permite a los desarrolladores bloquear columnas específicas en el orden deseado, garantizando su visibilidad en todo momento incluso cuando los usuarios se desplazan horizontalmente por `{ComponentName}`. Hay una UI integrada para el anclaje de columnas, accesible mediante la barra de herramientas {Platform} {ComponentTitle}. Además, los desarrolladores tienen la flexibilidad de crear una interfaz de usuario personalizada que cambie el estado de anclaje de las columnas.

## API de anclaje de columnas

El anclaje de columnas se controla mediante la propiedad <ApiLink type="Column" member="pinned" /> de <ApiLink type="Column" />. Las columnas ancladas se representan en el lado izquierdo de <ApiLink type="{ComponentName}" /> de forma predeterminada y permanecen fijas durante el desplazamiento horizontal de las columnas no ancladas en el cuerpo de <ApiLink type="{ComponentName}" />.

Ambos métodos devuelven un valor booleano que indica si su operación respectiva se realizó correctamente o no. Normalmente, el motivo por el que fallan es que la columna ya está en el estado deseado.

## Posición de anclaje

Puede cambiar la posición de anclaje de columnas mediante la opción de configuración <ApiLink type="Column" member="pinningPosition" />. Le permite establecer la posición de las columnas en Start o End.
Cuando se establece en End, las columnas se representan al final de la cuadrícula, después de las columnas no ancladas. Las columnas no ancladas se pueden desplazar horizontalmente, mientras que las columnas ancladas permanecen fijas a la derecha.

## UI personalizada de anclaje de columnas

Puede definir su UI personalizada y cambiar el estado de anclaje de las columnas mediante la API relacionada.

Supongamos que, en lugar de una barra de herramientas, desea definir iconos de anclaje en los encabezados de columna en los que el usuario final pueda hacer clic para cambiar el estado de anclaje de una columna concreta.

Esto se puede hacer creando una plantilla de encabezado para las columnas con un icono personalizado.

igRegisterScript("WebGridPinHeaderTemplate", (ctx) => {
    var html = window.igTemplating.html;
    window.toggleColumnPin = function toggleColumnPin(field) {
        var grid = document.getElementsByTagName["igc-grid"](0);
        var col = grid.getColumnByName(field);
        col.pinned = !col.pinned;
        grid.markForCheck();
    }
    return html`<div>

## Estilo

<ApiLink type="{ComponentName}" /> permite aplicar estilos mediante la [biblioteca de temas {ProductName}](../themes/styles.md). <ApiLink type="function-grid-theme" label="theme" kind="sass" module="index.html" /> de la cuadrícula expone una amplia variedad de propiedades, que permiten la personalización de todas las características de la cuadrícula.

En los pasos siguientes, recorreremos los pasos para personalizar el estilo de anclaje de la cuadrícula.

### Importar tema global

Para comenzar la personalización de la característica de anclaje, debe importar el archivo `index`, donde se encuentran todas las funciones de estilo y mixins.

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

### Definir tema personalizado

A continuación, cree un tema nuevo que extienda <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> y acepte los parámetros necesarios para personalizar la característica de anclaje según se desee.

```scss
$custom-theme: grid-theme(
    /* Pinning properties that affect styling */
    $pinned-border-width: 5px,
    $pinned-border-style: double,
    $pinned-border-color: #FFCD0F,
    $cell-active-border-color: #FFCD0F
    /* add other features properties here... */
);
```

### Definir una paleta de colores personalizada

En el enfoque descrito anteriormente, los valores de color se codificaron de forma fija. Como alternativa, puede lograr una mayor flexibilidad usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.
`igx-palette` genera una paleta de colores basada en los colores primario y secundario proporcionados.

```scss
$primary-color: #292826;
$secondary-color: #ffcd0f;

$custom-palette: palette(
 $primary: $primary-color,
 $secondary: $secondary-color
);
```

Después de generar una paleta personalizada, la función `igx-color` se puede usar para obtener diferentes variedades de los colores primario y secundario.

```scss
$custom-theme: grid-theme(
    $pinned-border-width: 5px,
    $pinned-border-style: double,
    $pinned-border-color: color($custom-palette, "secondary", 500),
    $cell-active-border-color: color($custom-palette, "secondary", 500)
);
```

`$custom-theme` contiene las mismas propiedades que la de la sección anterior, pero esta vez los colores no están codificados de forma fija. En su lugar, se usó la `igx-palette` personalizada y los colores se obtuvieron mediante sus colores primario y secundario, con una variante de color determinada.

### Definir esquemas personalizados

Puede ir aún más lejos y crear una estructura flexible que tenga todos los beneficios de un [**esquema**](../themes/sass/schemas.md). El **esquema** es la receta de un tema.
Extienda uno de los dos esquemas predefinidos que se proporcionan para cada componente. En nuestro caso, usaríamos `$_light_grid`.

```scss
$custom-grid-schema: extend($_light-grid,(
    pinned-border-width: 5px,
    pinned-border-style: double,
    pinned-border-color: color: ("secondary", 500),
    cell-active-border-color: color: ("secondary", 500)
));
```

Para que se aplique el esquema personalizado, se debe extender `light` o `dark` global. Todo el proceso consiste realmente en proporcionar a un componente un esquema personalizado y añadirlo después al tema del componente correspondiente.

```scss
$my-custom-schema: extend($light-schema, (
    igx-grid: $custom-grid-schema
));
$custom-theme: grid-theme(
    $palette: $custom-palette,
    $schema: $my-custom-schema
);
```

### Aplicar el tema personalizado

La forma más sencilla de aplicar su tema es con una instrucción `sass` `@include` en el archivo de estilos globales:

```scss
@include grid($custom-theme);
```

### Tema de componente con ámbito

Para que el tema personalizado afecte solo a un componente específico, puede mover todos los estilos que acaba de definir desde el archivo de estilos globales al archivo de estilos del componente personalizado (incluida la importación del archivo `index`).

De esta forma, debido a [ViewEncapsulation](https://angular.io/api/core/Component#encapsulation) de Angular, sus estilos se aplicarán solo a su componente personalizado.

 <DocsAside type="info">

Si el componente usa una ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario atravesar esta encapsulación usando `::ng-deep` para aplicar estilo a la cuadrícula.

 </DocsAside>

 <DocsAside type="info">

Envuelva la instrucción dentro de un selector `:host` para evitar que sus estilos afecten a elementos fuera de nuestro componente:

 </DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-theme);
    }
}
```
