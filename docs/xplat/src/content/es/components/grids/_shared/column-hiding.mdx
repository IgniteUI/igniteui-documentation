# {Platform} {ComponentTitle} Ocultación de columnas

<PlatformBlock for="Angular">

{ProductName} <ApiLink type="{ComponentName}" /> proporciona un <ApiLink type="ColumnActionsComponent" /> con un `ColumnHidingDirective` que permite a los usuarios realizar la ocultación de columnas directamente a través de la interfaz de usuario o mediante el componente {Platform}.

</PlatformBlock>

El {ProductName} tiene una UI integrada de ocultación de columnas, que se puede usar a través de la barra de herramientas {Platform} {ComponentTitle} para cambiar el estado visible de las columnas. Los desarrolladores tienen la flexibilidad de definir la UI de ocultación de columnas en cualquier lugar de la página según sea necesario. La característica de ocultación de columnas de {Platform} {ComponentTitle} es especialmente útil cuando se desea reducir el tamaño de la cuadrícula y eliminar la necesidad de desplazarse con tabulación por campos redundantes.

## UI de ocultación de columnas de la barra de herramientas

La UI integrada de ocultación de columnas se coloca dentro de un <ApiLink pkg="core" type="Dropdown" /> en la barra de herramientas de <ApiLink type="{ComponentName}" />. Podemos mostrar/ocultar la UI de ocultación de columnas mediante este menú desplegable exacto.

Para este propósito, todo lo que tenemos que hacer es establecer tanto <ApiLink type="GridToolbarActions" /> como <ApiLink type="GridToolbarHiding" /> dentro de <ApiLink type="{ComponentName}" />.

<ComponentBlock for="Grid">

<PlatformBlock for="Angular">

```html
<igx-grid [data]="localdata">
    <igx-grid-toolbar>
        <igx-grid-toolbar-actions>
            <igx-grid-toolbar-hiding></igx-grid-toolbar-hiding>
        </igx-grid-toolbar-actions>
    </igx-grid-toolbar>
</igx-grid>
```

El <ApiLink type="{ComponentName}" /> nos proporciona algunas propiedades útiles cuando se trata de usar la UI de ocultación de columnas de la barra de herramientas.

Al usar la propiedad <ApiLink type="GridToolbarHiding" member="title" label="Title" />, estableceremos el título que se muestra dentro del botón desplegable de la barra de herramientas.

Al usar las propiedades <ApiLink type="GridToolbarHiding" member="title" label="Title" /> y <ApiLink type="GridToolbarHiding" member="prompt" label="Prompt" />, estableceremos el título y el aviso de filtro que se muestran dentro del menú desplegable de la barra de herramientas.

## Estilización

Para comenzar con la estilización del componente de acciones de columna, necesitamos importar el archivo de índice, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

Al usar el enfoque más sencillo, creamos un nuevo tema que extiende `column-actions-theme` y acepta los parámetros `$title-color` y `$background-color`.

```scss
$custom-column-actions-theme: column-actions-theme(
    $background-color: steelblue,
    $title-color: gold
);
```

Como se observa, `column-actions-theme` solo controla los colores del contenedor de acciones de columna, pero no afecta los botones, las casillas de verificación ni el grupo de entrada dentro de él. Supongamos que también queremos estilizar los botones, así que crearemos un nuevo tema de botón:

```scss
$custom-button: button-theme($flat-text-color: gold, $disabled-color: black);
```

En este ejemplo solo cambiamos el color del texto de los botones planos y el color del botón deshabilitado, pero `button-theme` proporciona muchos más parámetros para controlar el estilo del botón.

El último paso es **incluir** los mixins de componentes, cada uno con su tema respectivo:

```scss
@include column-actions($custom-column-actions-theme);
.igx-column-actions {
    @include button($custom-button);
}
```

> **Nota**
> Delimitamos el mixin **igx-button** dentro de `.igx-column-actions`, de modo que solo se estilicen los botones de ocultación de columnas. De lo contrario, otros botones de la cuadrícula también se verían afectados.

> **Nota**
> Si el componente está usando un `Emulated`ViewEncapsulation, es necesario `penetrate` esta encapsulación usando `::ng-deep`:

```scss
:host {
    ::ng-deep {
        @include column-actions($custom-column-actions-theme);
        .igx-column-actions {
            @include button($custom-button);
        }
    }
}
```

### Definir una paleta de colores

En lugar de codificar de forma rígida los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en términos de colores usando las funciones `igx-palette` y `igx-color`.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$yellow-color: gold;
$blue-color: steelblue;

$custom-palette: palette($primary: $blue-color, $secondary: $yellow-color);
```

Y luego con `igx-color` podemos recuperar fácilmente el color de la paleta.

```scss
$custom-column-actions-theme: column-actions-theme(
    $palette: $custom-palette,
    $title-color: color($custom-palette, "secondary", 400),
    $background-color: color($custom-palette, "primary", 200)
);

$custom-button: button-theme(
    $palette: $custom-palette,
    $flat-text-color: color($custom-palette, "secondary", 400),
    $disabled-color: black
);
```

> **Nota**
> `igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema \`Paletas]\(themes/sass/palettes.md) para obtener orientación detallada sobre cómo usarlas.

### Usar esquemas

Al avanzar más con el motor de temas, puede construir una estructura sólida y flexible que se beneficie de los [**esquemas**](themes/sass/schemas.md). Un **esquema** es una receta de un tema.

```scss
// Extending the dark column actions schema
$custom-column-actions-schema: extend($_dark-column-actions,
    (
        title-color:(
           color: ("secondary", 400)
        ),
        background-color:(
           color: ("primary", 200)
        )
    )
);
// Extending the dark button schema
$custom-button-schema: extend($_dark-button,
    (
        flat-text-color:(
           color: ("secondary", 500)
        ),
        disabled-color:(
           color: ("primary", 700)
        )
    )
);
```

Para aplicar nuestros esquemas personalizados, tenemos que **extender** uno de los globales `light` o `dark`, que básicamente señala los componentes con un esquema personalizado, y después agregarlo a los temas de componentes respectivos:

```scss
// Extending the global dark-schema
$custom-dark-schema: extend($dark-schema,(
    igx-column-actions: $custom-column-actions-schema,
    igx-button: $custom-button-schema
));

// Defining column-actions-theme with the global dark schema
$custom-column-actions-theme: column-actions-theme(
  $palette: $custom-palette,
  $schema: $custom-dark-schema
);

// Defining button-theme with the global dark schema
$custom-button: button-theme(
  $palette: $custom-palette,
  $schema: $custom-dark-schema
);
```

No olvide incluir los temas de la misma manera que se demostró anteriormente.
