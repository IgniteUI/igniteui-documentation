## Virtualización remota

## Desplazamiento infinito

Un diseño popular para escenarios que requieren obtener datos por fragmentos desde un punto de conexión es el denominado desplazamiento infinito. Para las cuadrículas de datos, se caracteriza por el aumento continuo de los datos cargados, desencadenado cuando el usuario final se desplaza hasta la parte inferior. Los siguientes párrafos explican cómo puede usar la API disponible para lograr fácilmente el desplazamiento infinito en <ApiLink type="{ComponentName}" />.

Para implementar el desplazamiento infinito, debe obtener los datos en fragmentos. Los datos que ya se han obtenido deben almacenarse localmente y debe determinar la longitud de un fragmento y cuántos fragmentos hay. También debe llevar un seguimiento del índice de la última fila de datos visible en la cuadrícula. De esta manera, mediante las propiedades <ApiLink type="ForOfState" member="chunkSize" /> y <ApiLink type="ForOfState" member="chunkSize" />, puede determinar si el usuario se desplaza hacia arriba y debe mostrarle datos ya obtenidos, o si se desplaza hacia abajo y debe obtener más datos del punto de conexión.

Lo primero que debe hacer es obtener el primer fragmento de los datos. Establecer la propiedad <ApiLink type="{ComponentName}" member="totalItemCount" /> es importante, ya que permite que la cuadrícula dimensione correctamente su barra de desplazamiento.

<PlatformBlock for="Angular">

```typescript
public ngAfterViewInit() {
    this._remoteService.loadDataForPage(this.page, this.pageSize, (request) => {
        if (request.data) {
            this.grid.totalItemCount = this.page * this.pageSize;
            this.grid.data = this._remoteService.getCachedData({startIndex: 0, chunkSize: 10});
            this.totalItems = request.data['@odata.count'];
            this.totalPageCount = Math.ceil(this.totalItems / this.pageSize);
            this.grid.isLoading = false;
        }
    });
}

public handlePreLoad() {
    const isLastChunk = this.grid.totalItemCount ===
                        this.grid.virtualizationState.startIndex + this.grid.virtualizationState.chunkSize;
    // when last chunk reached load another page of data
    if (isLastChunk) {
        if (this.totalPageCount === this.page) {
            this.grid.data = this._remoteService.getCachedData(this.grid.virtualizationState);
            return;

                this.LocalData = data.ToList();
                this.grid.IsLoading = false;
                this.grid.TotalItemCount = Math.Min(this.Page * this.PageSize, this.TotalItems);

## Ordenación/filtrado remotos

Para proporcionar ordenación y filtrado remotos, debe suscribirse a las salidas <ApiLink type="{ComponentName}" member="dataPreLoad" />, <ApiLink type="{ComponentName}" member="sortingExpressionsChange" /> y <ApiLink type="{ComponentName}" member="filteringExpressionsTreeChange" />, para realizar la solicitud adecuada en función de los argumentos recibidos, así como establecer la propiedad pública <ApiLink type="{ComponentName}" /> <ApiLink type="{ComponentName}" member="totalItemCount" /> con la información correspondiente procedente del servicio.

También aprovecharemos la función `debounceTime` de **rxjs**, que emite un valor desde el Observable de origen solo después de que haya transcurrido un intervalo de tiempo determinado sin otra emisión de origen. De esta manera, la operación remota se desencadenará solo cuando haya transcurrido la cantidad de tiempo especificada sin que el usuario la interrumpa.

```typescript
const DEBOUNCE_TIME = 300;
public ngAfterViewInit() {
    this.grid.dataPreLoad.pipe(
        debounceTime(DEBOUNCE_TIME),
        takeUntil(this.destroy$)
    ).subscribe(() => {
        this.processData();
    });

    this.grid.filteringExpressionsTreeChange.pipe(
        debounceTime(DEBOUNCE_TIME),
        takeUntil(this.destroy$)
    ).subscribe(() => {
        this.processData(true);
    });

    this.grid.sortingExpressionsChange.pipe(
        debounceTime(DEBOUNCE_TIME),
        takeUntil(this.destroy$)
    ).subscribe(() => {
        this.processData();
    });
}
```

## Estrategia de valores únicos de columna

Los elementos de la lista dentro del diálogo de filtrado estilo Excel representan los valores únicos de la columna correspondiente. <ApiLink type="{ComponentName}" /> genera estos valores basándose en su origen de datos de forma predeterminada. En el caso del filtrado remoto, los datos de la cuadrícula no contienen todos los datos del servidor. Para proporcionar manualmente los valores únicos y cargarlos bajo demanda, podemos aprovechar la entrada <ApiLink type="{ComponentName}" /> de <ApiLink type="{ComponentName}" member="uniqueColumnValuesStrategy" label="UniqueColumnValuesStrategy" />. Esta entrada es en realidad un método que proporciona tres argumentos:

- <ApiLink type="Column" />  - La instancia de columna correspondiente.
- <ApiLink type="FilteringExpressionsTree" /> - El árbol de expresiones de filtrado, que se reduce según la columna correspondiente.
- `Done` - Callback que debe llamarse con los valores de columna recién generados cuando se recuperan del servidor.

El desarrollador puede generar manualmente los valores únicos de columna necesarios basándose en la información que proporcionan los argumentos <ApiLink type="Column" /> y <ApiLink type="FilteringExpressionsTree" /> y, a continuación, invocar el callback `Done`.

<DocsAside type="info">

Cuando se proporciona la entrada `UniqueColumnValuesStrategy`, no se utilizará el proceso predeterminado de generación de valores únicos en el filtrado estilo Excel.

</DocsAside>

## Paginación remota

<ComponentBlock for="Grid">

La funcionalidad de paginación puede funcionar con datos remotos. Para demostrarlo, primero declaremos nuestro servicio, que será responsable de obtener los datos. Necesitaremos el recuento de todos los elementos de datos para calcular el recuento de páginas. Esta lógica se añadirá a nuestro servicio.

<PlatformBlock for="Angular">

```typescript
@Injectable()
export class RemotePagingService {
    public remoteData: BehaviorSubject<any[]>;
    public dataLenght: BehaviorSubject<number> = new BehaviorSubject(0);
    public url = 'https://www.igniteui.com/api/products';

    constructor(private http: HttpClient) {
        this.remoteData = new BehaviorSubject([]) as any;
    }

    public getData(index?: number, perPage?: number): any {
        let qS = '';

        if (perPage) {
            qS = `?$skip=${index}&$top=${perPage}&$count=true`;
        }

        this.http
            .get(`${this.url + qS}`).pipe(
                map((data: any) => data)
            ).subscribe((data) => this.remoteData.next(data));
    }

    public getDataLength(): any {
        return this.http.get(this.url).pipe(
            map((data: any) => data.length)
        );
    }
}

    public static getDataWithPaging(pageIndex?: number, pageSize?: number) {
        return fetch(RemotePagingService.buildUrl(RemotePagingService.CUSTOMERS_URL, pageIndex, pageSize))
        .then((result) => result.json())

    private static buildUrl(baseUrl: string, pageIndex?: number, pageSize?: number) {
        let qS = "";
        if (baseUrl) {
                qS += `${baseUrl}`;

        // Add pageIndex and size to the query string if they are defined
        if (pageIndex !== undefined) {
            qS += `?pageIndex=${pageIndex}`;
            if (pageSize !== undefined) {
                qS += `&size=${pageSize}`;
            }
        } else if (pageSize !== undefined) {
            qS += `?perPage=${pageSize}`;
        }

        return `${qS}`;
    }
}

Necesitaremos el recuento de todos los elementos de datos para calcular el recuento de páginas. Esta lógica se añadirá a nuestro servicio.

```typescript
@Injectable()
export class RemotePagingService {
    public remoteData: BehaviorSubject<any[]>;
    public dataLenght: BehaviorSubject<number> = new BehaviorSubject(0);
    public url = 'https://www.igniteui.com/api/products';

    constructor(private http: HttpClient) {
        this.remoteData = new BehaviorSubject([]) as any;
    }

    public getData(index?: number, perPage?: number): any {
        let qS = '';

        if (perPage) {
            qS = `?$skip=${index}&$top=${perPage}&$count=true`;
        }

        this.http
            .get(`${this.url + qS}`).pipe(
                map((data: any) => data)
            ).subscribe((data) => this.remoteData.next(data));
    }

    public getDataLength(): any {
        return this.http.get(this.url).pipe(
            map((data: any) => data.length)
        );
    }
}

### Paginación remota con plantilla predeterminada

Si desea usar la plantilla de paginación predeterminada, debe establecer la propiedad <ApiLink type="GridBaseDirective" member="totalRecords" label="TotalRecords" /> del Paginator; solo entonces la cuadrícula podrá calcular el número total de páginas basándose en el total de registros remotos. Al realizar una paginación remota, el Paginator pasará a la cuadrícula solo los datos de la página actual, por lo que la cuadrícula no intentará paginar el origen de datos proporcionado. Por eso debemos establecer la propiedad <ApiLink type="GridBaseDirective" member="pagingMode" label="PagingMode" /> de la cuadrícula en <ApiLink type="GridPagingMode" member="Remote" label="GridPagingMode.Remote" />. También es necesario suscribirse a los eventos `PagingDone` o `PerPageChange` para obtener los datos desde su servicio remoto; depende del caso de uso qué evento se utilizará.

A continuación encontrará los métodos que hemos definido para implementar nuestras propias acciones de página `next` y `previous`.

```typescript
@ViewChild('grid1', { static: true }) public grid1: IgxGridComponent;

public ngAfterViewInit() {
    this.grid1.isLoading = true;
    this.remoteService.getData(0, this.perPage);
}

public nextPage() {
    this.firstPage = false;
    this.page++;
    const skip = this.page * this.perPage;
    const top = this.perPage;
    this.remoteService.getData(skip, top);
    if (this.page + 1 >= this.totalPages) {
        this.lastPage = true;
    }
    this.setNumberOfPagingItems(this.page, this.totalPages);
}

public previousPage() {
    this.lastPage = false;
    this.page--;
    const skip = this.page * this.perPage;
    const top = this.perPage;
    this.remoteService.getData(skip, top);
    if (this.page <= 0) {
        this.firstPage = true;
    }
    this.setNumberOfPagingItems(this.page, this.totalPages);
}

public paginate(page: number, recalculate = false) {
    this.page = page;
    const skip = this.page * this.perPage;
    const top = this.perPage;
    if (recalculate) {
        this.totalPages = Math.ceil(this.totalCount / this.perPage);
    }
    this.setNumberOfPagingItems(this.page, this.totalPages);
    this.remoteService.getData(skip, top);
    this.buttonDeselection(this.page, this.totalPages);
}

### Paginación remota con edición por lotes

Con los ejemplos hasta ahora aclaramos cómo configurar <ApiLink type="{ComponentName}" /> con datos remotos. Ahora, centrémonos en habilitar la edición por lotes para la cuadrícula siguiendo el [tema/guía de edición por lotes](batch-editing.md).

Antes de continuar con el ejemplo, conviene aclarar el caso de uso actual. Cuando la paginación se realiza en el servidor, la cuadrícula contiene los datos solo de la página actual y, si añadimos nuevas filas, las filas recién añadidas (con edición por lotes) se concatenarán con los datos actuales que contiene la cuadrícula. Por lo tanto, si el servidor no devuelve datos para una página determinada, el origen de datos de la cuadrícula estará compuesto solo por las filas recién añadidas, que la cuadrícula paginará basándose en la configuración de paginación definida (page, perPage).

```typescript
public ngOnInit() {
    this._dataLengthSubscriber = this.remoteService.getDataLength().subscribe((data) => {
        this.totalCount = data;
        this._recordOnServer = data;
        this._totalPagesOnServer = Math.floor(this.totalCount / this.perPage);
        this.grid1.isLoading = false;
    });
}

Para manejar correctamente este caso de uso, necesitamos implementar cierta lógica personalizada.

Primero, debemos conocer el número total de registros que hay en el servidor. Con eso, calculamos el número total de páginas de datos en el servidor y, basándonos en su valor, implementaremos la lógica de paginación personalizada.

```typescript
public paginate(page: number) {
    this.grid1.endEdit(true);
    if (page > this._totalPagesOnServer) {
        if (this.page !== this._totalPagesOnServer) {
            const skipEl = this._totalPagesOnServer * this.perPage;
            this.remoteService.getData(skipEl, this.perPage);
        }
        this.page = page - this._totalPagesOnServer;
        this.page = page;
        return;
    } else {
        this.page = 0;
    }
    this.page = page;
    const skip = this.page * this.perPage;
    this.remoteService.getData(skip, this.perPage);
}

## Problemas conocidos y limitaciones

- Cuando la cuadrícula no tiene establecido <ApiLink type="{ComponentName}" member="primaryKey" /> y los escenarios de datos remotos están habilitados (cuando la paginación, la ordenación, el filtrado o el desplazamiento desencadenan solicitudes a un servidor remoto para recuperar los datos que se mostrarán en la cuadrícula), una fila perderá el siguiente estado después de que se complete una solicitud de datos:

- Selección de fila

- Expandir/contraer fila

- Edición de fila

- Anclado de fila
