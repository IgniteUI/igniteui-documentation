# {Platform} {ComponentTitle} Información general de paginación

La característica de paginación de {ProductName} en {Platform} {ComponentTitle} se usa para dividir un conjunto grande de datos en una secuencia de páginas que tienen contenido similar. La paginación de la cuadrícula de React mejora la experiencia del usuario y la interacción con los datos. La paginación de <ApiLink type="{ComponentName}" /> se puede configurar mediante un componente independiente proyectado en el árbol de la cuadrícula definiendo una etiqueta <ApiLink type="Paginator" />, de forma similar a la adición de una columna. Como en cualquier tabla {Platform}, la paginación en {Platform} {ComponentTitle} admite una plantilla para páginas personalizadas.

## Paginación con Agrupar por

Las filas de grupo participan en el proceso de paginación junto con las filas de datos. Cuentan para el tamaño de página de cada página. Las filas contraídas no se incluyen en el proceso de paginación.

La integración entre la paginación y Agrupar por se describe en el tema [Agrupar por](groupby.md#{PlatformLower}-grid-group-by-with-paging).

</ComponentBlock>

## Estilo de paginación en {Platform}

Para comenzar a aplicar estilos al paginador, necesitamos importar el archivo `index`, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Siguiendo el enfoque más sencillo, creamos un nuevo tema que extiende `PaginatorTheme` y acepta los parámetros `$text-color`, `$background-color` y `$border-color`.

```scss
$dark-paginator: paginator-theme(
    $text-color: #F4D45C,
    $background-color: #575757,
    $border-color: #292826
);
```

Como se ve, `PaginatorTheme` solo controla los colores del contenedor de paginación, pero no afecta a los botones de la UI del paginador. Para aplicar estilos a esos botones, creemos un nuevo tema de botón:

```scss
$dark-button: button-theme(
    $icon-color: #FFCD0F,
    $icon-hover-color: #292826,
    $icon-hover-background: #FFCD0F,
    $icon-focus-color: #292826,
    $icon-focus-background: #FFCD0F,
    $disabled-color: #16130C
);
```

En este ejemplo solo cambiamos el color y el fondo del icono, y el color del botón deshabilitado, pero `ButtonTheme` proporciona muchos más parámetros para controlar el estilo del botón.

El último paso es **incluir** los mixins de componentes, cada uno con su tema respectivo:

```scss
@include grid-paginator($dark-grid-paginator);
.igx-grid-paginator__pager {
    @include button($dark-button);
}
```

<DocsAside type="info">

Delimitamos el alcance del mixin <ApiLink pkg="core" type="Button" /> dentro de `.igx-paginator__pager`, para que solo se apliquen estilos a los botones del paginador. De lo contrario, otros botones de la cuadrícula también se verían afectados.

</DocsAside>

<DocsAside type="info">

Si el componente está usando una ViewEncapsulation [Emulada](../themes/styles.md#view-encapsulation), es necesario `penetrate` esta encapsulación usando `::ng-deep`:

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include paginator($dark-paginator);
        .igx-paginator__pager {
            @include button($dark-button);
        }
    }
}
```

### Definición de una paleta de colores

En lugar de codificar de forma fija los valores de color como acabamos de hacer, podemos lograr mayor flexibilidad en cuanto a colores usando las funciones `Palette` y <ApiLink pkg="core" type="Color" />.

`Palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$yellow-color: #F9D342;
$black-color: #292826;

$dark-palette: palette($primary: $black-color, $secondary: $yellow-color);
```

Y luego, con <ApiLink pkg="core" type="Color" />, podemos recuperar fácilmente un color de la paleta.

```scss
$dark-paginator: paginator-theme(
    $palette: $dark-palette,
    $text-color: color($dark-palette, "secondary", 400),
    $background-color: color($dark-palette, "primary", 200),
    $border-color: color($dark-palette, "primary", 500)
);

$dark-button: button-theme(
    $palette: $dark-palette,
    $icon-color: color($dark-palette, "secondary", 700),
    $icon-hover-color: color($dark-palette, "primary", 500),
    $icon-hover-background: color($dark-palette, "secondary", 500),
    $icon-focus-color: color($dark-palette, "primary", 500),
    $icon-focus-background: color($dark-palette, "secondary", 500),
    $disabled-color: color($dark-palette, "primary", 700)
);
```

<DocsAside type="info">

<ApiLink pkg="core" type="Color" /> y `Palette` son funciones eficaces para generar y recuperar colores. Consulte el tema [Paletas](../themes/sass/palettes.md) para obtener orientación detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Avanzando más con el motor de temas, puede crear una estructura robusta y flexible que se beneficie de los [esquemas](../themes/sass/schemas.md). Un esquema es una receta de un tema.

Extienda uno de los dos esquemas predefinidos, que se proporcionan para cada componente; en este caso, los esquemas `DarkPagination` y `DarkButton`:

```scss
// Extending the dark paginator schema
$dark-paginator-schema: extend($_dark-pagination,
        (
            text-color:(
               color: ("secondary", 400)
            ),
            background-color:(
               color: ("primary", 200)
            ),
            border-color:(
               color: ("primary", 500)
            )
        )
);
// Extending the dark button schema
$dark-button-schema: extend($_dark-button,
        (
            icon-color:(
               color: ("secondary", 700)
            ),
            icon-hover-color:(
               color: ("primary", 500)
            ),
            icon-hover-background:(
               color: ("secondary", 500)
            ),
            icon-focus-color:(
               color: ("primary", 500)
            ),
            icon-focus-background:(
               color: ("secondary", 500)
            ),
            disabled-color:(
               color: ("primary", 700)
            )
        )
);
```

Para aplicar nuestros esquemas personalizados, tenemos que **extender** uno de los globales `Light` o `Dark`, que básicamente señala los componentes con un esquema personalizado, y después agregarlo a los temas de los componentes correspondientes:

```scss
// Extending the global dark-schema
$custom-dark-schema: extend($dark-schema,(
    igx-paginator: $dark-paginator-schema,
    igx-button: $dark-button-schema
));

// Definingpaginator-theme with the global dark schema
$dark-paginator: paginator-theme(
  $palette: $dark-palette,
  $schema: $custom-dark-schema
);

// Defining button-theme with the global dark schema
$dark-button: button-theme(
  $palette: $dark-palette,
  $schema: $custom-dark-schema
);
```

No olvide incluir los temas de la misma manera que se demostró anteriormente.
