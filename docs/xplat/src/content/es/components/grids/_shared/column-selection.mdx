# {Platform} {ComponentTitle} Información general sobre la selección de columnas

La característica de selección de columnas {Platform} {ComponentTitle} en {ProductName} ofrece una forma simplificada y similar a Excel de seleccionar y resaltar una columna completa con un solo clic. Puede habilitarse mediante la entrada <ApiLink type="{ComponentName}" member="columnSelection" />. Gracias a la API enriquecida, la característica permite manipular fácilmente el estado de selección, extraer datos de las fracciones seleccionadas, realizar operaciones de análisis de datos y crear visualizaciones.

## Uso básico

La característica de selección de columnas puede habilitarse mediante la entrada <ApiLink type="{ComponentName}" member="columnSelection" />, que acepta valores <ApiLink type="GridSelectionMode" />.

## Interacciones

El modo de selección predeterminado es `None`. Si se establece en `Single` o `Multiple`, todas las columnas presentadas serán <ApiLink type="Column" member="selectable" />. Dicho esto, para seleccionar una columna, solo es necesario hacer clic en una, lo que la marcará como <ApiLink type="Column" member="selected" />. Si la columna no es seleccionable, no se aplicará ningún estilo de selección en el encabezado al pasar el cursor.

<DocsAside type="info">

La característica [Encabezados de varias columnas](multi-column-headers.md) no se refleja en la entrada <ApiLink type="{ComponentName}" member="selectable" />. El <ApiLink type="ColumnGroupComponent" /> es <ApiLink type="{ComponentName}" member="selectable" /> si al menos uno de sus elementos secundarios tiene habilitado el comportamiento de selección. Además, el componente se marca como <ApiLink type="{ComponentName}" member="selected" /> si todos sus descendientes <ApiLink type="{ComponentName}" member="selectable" /> son <ApiLink type="{ComponentName}" member="selected" />.

</DocsAside>

## Combinaciones de teclado

<DocsAside type="info">

Las combinaciones de teclado están disponibles solo cuando la entrada <ApiLink type="{ComponentName}" member="columnSelection" /> de la cuadrícula está establecida en `multiple`.

</DocsAside>

Hay dos escenarios para la navegación con teclado de la característica **Selección de columnas**:

- Selección de varias columnas: mantener presionadas <kbd>CTRL</kbd> + <kbd>clic</kbd> en cada celda de encabezado **seleccionable**.
- Selección de rango de columnas: mantener presionadas <kbd>MAYÚS</kbd> + <kbd>clic</kbd> selecciona todas las columnas **seleccionables** intermedias.

## Manipulaciones de API

La **API** proporciona algunas capacidades adicionales cuando se trata de las columnas **no visibles**, de modo que cada columna **oculta** puede marcarse como <ApiLink type="Column" member="selected" /> estableciendo el **setter** correspondiente.

<DocsAside type="info">

La instrucción anterior también se aplica a <ApiLink type="ColumnGroupComponent" />, excepto que cuando se cambia la propiedad <ApiLink type="{ComponentName}" member="selected" />, cambia el estado de sus descendientes.

</DocsAside>

Puede encontrar más información sobre las manipulaciones de la API en la sección [Referencias de API](#api-references).

## Estilos

Además de los temas predefinidos, la cuadrícula puede personalizarse aún más estableciendo algunas de las [propiedades CSS](../theming-grid.md) disponibles.
En caso de que quiera cambiar algunos de los colores, primero debe establecer un `class` para la cuadrícula:

<PlatformBlock for="WebComponents">

```html
<{ComponentSelector} class="grid"></{ComponentSelector}>
```

</PlatformBlock>

<PlatformBlock for="React">

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<{ComponentSelector} class="grid"></{ComponentSelector}>
```

</PlatformBlock>

A continuación, establezca las propiedades CSS relacionadas en esta clase:

```css
.grid {
    --ig-grid-row-selected-background: #0062A3;
    --ig-grid-row-selected-text-color: #ecaa53;
    --ig-grid-row-selected-hover-background: #0062A3;
    --ig-grid-header-selected-text-color: #ecaa53;
    --ig-grid-header-selected-background: #0062A3;
    --ig-grid-row-selected-hover-text-color: #ecaa53;
    --ig-grid-row-selected-hover-background: #0062A3;
}
```

## Estilos

Antes de profundizar en las opciones de estilos, deben importarse el módulo principal y todos los mixins de componentes.

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

<ComponentBlock for="Grid">

> Tenga en cuenta que la [selección de filas](row-selection.md) y la [selección de columnas](column-selection.md) no pueden manipularse de forma independiente. Dependen del mismo `variables`.

</ComponentBlock>

Dicho esto, continuemos y cambiemos los estilos de **selección** y **hover**. <br/>

Siguiendo el enfoque más simple, definamos nuestro **tema** personalizado.

<ComponentBlock for="TreeGrid">

```scss
$custom-grid-theme: grid-theme(
    $row-selected-background: #011627,
    $row-selected-text-color: #ECAA53,
    $row-selected-hover-background: #011627,
    $header-selected-text-color: #ECAA53,
    $header-selected-background: #011627,
    $expand-icon-color: #ECAA53,
    $expand-icon-hover-color: #B64B80
);
```

</ComponentBlock>

<ComponentBlock for="Grid, HierarchicalGrid">

```scss
$custom-grid-theme: grid-theme(
    $row-selected-background: #011627,
    $row-selected-text-color: #ECAA53,
    $row-selected-hover-background: #011627,
    $header-selected-text-color: #ECAA53,
    $header-selected-background: #011627
);
```

</ComponentBlock>

<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> acepta varios parámetros, pero estos son los cinco responsables de cambiar la apariencia de todas las columnas seleccionadas:

- **$row-selected-background** - establece el fondo de la fracción seleccionada.
- **$row-selected-text-color** - establece el color del texto de la fracción seleccionada
- **$row-selected-hover-background** - establece el color de la celda o del conjunto de celdas sobre el que se pasa el cursor.
- **$header-selected-text-color** - establece el color del texto del encabezado de columna seleccionado
- **$header-selected-background** - establece el color de fondo del encabezado de columna seleccionado.

### Uso de variables CSS

El último paso es incluir el tema `{ComponentSelector}` personalizado.

```scss
@include css-vars($custom-grid-theme)
```

### Sobrescritura del tema base

Para aplicar estilos a componentes para Internet Explorer 11, tenemos que usar un enfoque diferente, ya que no admite variables CSS.

<DocsAside type="info">

Si el componente usa ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario aplicar `penetrate` a esta encapsulación usando `::ng-deep`. Para evitar que el tema personalizado se filtre a otros componentes, asegúrese de haber incluido el selector `:host` antes de `::ng-deep`.

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-column-selection-theme);
    }
}
```
