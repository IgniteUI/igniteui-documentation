# {ComponentTitle} Reordenación y movimiento de columnas

La característica de movimiento de columnas {Platform} {ComponentTitle} en {ProductName} permite reordenar columnas de forma rápida y sencilla. Esto puede hacerse mediante la API de movimiento de columnas o arrastrando y soltando los encabezados en otra posición mediante gestos de ratón o táctiles. En {Platform} {ComponentTitle}, puede habilitar el movimiento de columnas para columnas fijadas y no fijadas, así como para [encabezados de varias columnas](multi-column-headers.md).

<DocsAside type="info">

La reordenación entre columnas y grupos de columnas solo se permite cuando están en el mismo nivel de la jerarquía y ambos están en el mismo grupo. El movimiento se permite entre columnas/grupos de columnas, si son columnas de nivel superior.

</DocsAside>

<DocsAside type="info">

Si un encabezado de columna tiene una plantilla y el movimiento de columnas está habilitado, o la columna correspondiente se puede agrupar, entonces los elementos con plantilla deben tener el atributo **draggable** establecido en **false**.

</DocsAside>

<PlatformBlock for="Angular">

Esto permite adjuntar controladores para cualquier evento emitido por el elemento; de lo contrario, el evento es consumido por la directiva `igxDrag`.

</PlatformBlock>

<DocsAside type="info">

Si el área fijada supera su ancho máximo permitido (80 % del ancho total de <ApiLink type="{ComponentName}" />), una indicación visual notifica al usuario final que la operación de soltar está prohibida y que no es posible fijar. Esto significa que no se le permitirá soltar una columna en el área fijada.

</DocsAside>

## API

Además de la funcionalidad de arrastrar y soltar, la característica de movimiento de columnas también proporciona métodos de API para permitir mover una columna o reordenar columnas mediante programación:

<ApiLink type="GridBaseDirective" member="moveColumn" /> - Mueve una columna antes o después de otra columna (un destino). El primer parámetro es la columna que se va a mover, y el segundo parámetro es la columna de destino. También acepta un tercer parámetro opcional `Position` (que representa un valor <ApiLink type="DropPosition" />), que determina si colocar la columna antes o después de la columna de destino.

<PlatformBlock for="React, Angular, WebComponents">

```typescript
// Move the ID column after the Name column
const idColumn = grid.getColumnByName("ID");
const nameColumn = grid.getColumnByName("Name");

grid.moveColumn(idColumn, nameColumn, DropPosition.AfterDropTarget);

```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
    public async void HandleClick()
    {
        IgbColumn Col1 = await this.grid.GetColumnByVisibleIndexAsync(0);
        IgbColumn Col2 = await this.grid.GetColumnByVisibleIndexAsync(1);
        this.Grid.MoveColumn(Col1,Col2, DropPosition.AfterDropTarget);
    }
```

</PlatformBlock>

<ApiLink type="Column" member="move" label="Move" /> - Mueve una columna a un índice visible especificado. Si el parámetro de índice pasado no es válido (es negativo o supera el número de columnas), o si no se permite mover la columna a este índice (si está dentro de otro grupo), no se realiza ninguna operación.

<PlatformBlock for="WebComponents">

```typescript
// Move the ID column at 3rd position.
const idColumn = grid.getColumnByName("ID");
idColumn.move(3);
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
    public async void HandleClick()
    {
        IgbColumn Col1 = await this.grid.GetColumnByVisibleIndexAsync(0);
        this.Col1.Move(3);
    }
```

</PlatformBlock>

Tenga en cuenta que, al usar la característica de movimiento de columnas, el evento <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingEnd" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingEnd" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingEnd" /></PlatformBlock> se emitirá si la operación se realizó correctamente. También tenga en cuenta que, en comparación con la funcionalidad de arrastrar y soltar, el uso de la característica de movimiento de columnas no requiere establecer la propiedad <ApiLink type="{ComponentName}" member="moving" /> en true.

## Eventos

Hay varios eventos relacionados con el movimiento de columnas para proporcionar un medio de intervenir en las operaciones de arrastrar y soltar de las columnas. Estos son <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingStart" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingStart" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingStart" /></PlatformBlock>, `ColumnMoving` y <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingEnd" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingEnd" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingEnd" /></PlatformBlock>.

Puede suscribirse al evento <PlatformBlock for="Angular,WebComponents"><ApiLink type="GridComponentEventMap" member="columnMovingEnd" /></PlatformBlock><PlatformBlock for="React"><ApiLink type="Grid" member="onColumnMovingEnd" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink type="Grid" member="columnMovingEnd" /></PlatformBlock> de <ApiLink type="{ComponentName}" /> para implementar lógica personalizada cuando una columna se suelta en una nueva posición.     var dataGrid = this.dataGrid = document.getElementById('dataGrid') as {ComponentName}Component;

## Aplicación de estilos

Para empezar a aplicar estilos a los encabezados de movimiento de columnas <ApiLink type="{ComponentName}" />, debemos importar el archivo `index`, donde residen todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

Siguiendo el enfoque más sencillo, creamos un nuevo tema que extiende <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> y acepta los parámetros `$ghost-header-background`, `$ghost-header-text-color` y `$ghost-header-icon-color`.

```scss
// Define dark theme for the column moving
$dark-grid-column-moving-theme: grid-theme(
    $ghost-header-text-color: #F4D45C,
    $ghost-header-background: #575757,
    $ghost-header-icon-color: #f4bb5c
);
```

El último paso es **incluir** los mixins de componentes con su tema correspondiente:

```scss
@include grid($dark-grid-column-moving-theme);
```

<DocsAside type="info">

Según la estrategia de [**encapsulación de vistas**](/components/themes/sass/component-themes.html#view-encapsulation) del componente, puede ser necesario `penetrate` esta encapsulación usando `::ng-deep`

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($dark-grid-column-moving-theme);
    }
}
```

### Definición de una paleta de colores

En lugar de codificar de forma rígida los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en términos de colores usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.

**igx-palette** genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$yellow-color: #F4D45C;
$black-color: #575757;

$dark-palette: palette($primary: $yellow-color, $secondary: $black-color);
```

Y luego, con <ApiLink type="function-igx-color" label="**igx-color**" kind="sass" module="index.html" />, podemos recuperar fácilmente un color de la paleta.

```scss
$dark-grid-column-moving-theme: grid-theme(
    $palette: $dark-palette,
    $ghost-header-text-color: color($dark-palette, "primary", 400),
    $ghost-header-background: color($dark-palette, "secondary", 200),
    $ghost-header-icon-color: color($dark-palette, "primary", 500)
);
```

<DocsAside type="info">

El color y la paleta son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](/components/themes/palettes.html) para obtener orientación detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Avanzando con el motor de temas, puede crear una estructura robusta y flexible que se beneficie de los [esquemas](/components/themes/sass/schemas.html). Un **esquema** es una receta de un tema.

Extienda uno de los dos esquemas predefinidos que se proporcionan para cada componente, en este caso: <ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" />.

```scss
// Extending the dark grid schema
$dark-grid-column-moving-schema: extend($_light-grid,
        (
            ghost-header-text-color:(
               color: ("primary", 400)
            ),
            ghost-header-background:(
               color: ("secondary", 200)
            ),
            ghost-header-icon-color:(
               color: ("primary", 500)
            )
        )
);
```

Para aplicar nuestro esquema personalizado, debemos **extender** uno de los globales (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> o <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />), lo que básicamente señala los componentes con un esquema personalizado, y después agregarlo al tema del componente correspondiente:

```scss
// Extending the global dark-schema
$custom-light-schema: extend($light-schema,(
    igx-grid: $dark-grid-column-moving-schema,
));

// Defining dark-grid-theme with the global dark schema
$dark-grid-column-moving-theme: grid-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);
```

No olvide incluir el tema de la misma manera que se demostró anteriormente.
