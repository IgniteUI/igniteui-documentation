## Uso

<PlatformBlock for="Angular">

El método <ApiLink type="GridState" member="getState" /> devuelve el estado de la cuadrícula en una cadena JSON serializada, por lo que los desarrolladores pueden simplemente tomarla y guardarla en cualquier almacenamiento de datos (base de datos, nube, localStorage del navegador, etc.). El método acepta un primer parámetro opcional `Serialize`, que determina si <ApiLink type="GridState" member="getState" /> devolverá un objeto <ApiLink type="IGridState" /> o una cadena JSON serializada.

El desarrollador puede elegir obtener solo el estado de una característica o características determinadas, pasando el nombre de la característica o una matriz con nombres de características como segundo argumento.

</PlatformBlock>

<PlatformBlock for="React, WebComponents">

El método <ApiLink type="GridState" member="getState" /> devuelve el estado de la cuadrícula en un objeto <ApiLink type="GridStateInfo" />, que contiene toda la información de estado. Es posible que se requieran pasos adicionales para guardarlo.

</PlatformBlock>

Las API de punto único fáciles de usar permiten lograr una funcionalidad completa de persistencia de estado en solo unas pocas líneas de código. **Copie y pegue el código de abajo**: guardará el estado de la cuadrícula en el objeto `LocalStorage` del navegador cada vez que el usuario salga de la página actual. Siempre que el usuario regrese a la página principal, se restaurará el estado de la cuadrícula. Ya no será necesario configurar esas complejas expresiones de filtrado avanzado y ordenación cada vez para obtener los datos que desea: hágalo una vez y deje que el código de abajo haga el resto por sus usuarios:

<PlatformBlock for="Angular">

```typescript
@ViewChild(IgxGridStateDirective, { static: true })
public state!: IgxGridStateDirective;

public ngOnInit() {
    this.router.events.pipe(take(1)).subscribe((event: NavigationStart) => {
        this.saveGridState();
    });
}

public ngAfterViewInit() {
    this.restoreGridState();
}

public saveGridState() {
    const state = this.state.getState() as string;
    window.sessionStorage.setItem('grid1-state', state);
}

public restoreGridState() {
    const state = window.sessionStorage.getItem('grid1-state');
    this.state.setState(state);
}

```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```typescript
constructor() {
    window.addEventListener("load", () => { this.restoreGridState(); });
    window.addEventListener("beforeunload", () => { this.saveGridState(); });
}

// Using methods that work with IgcGridStateInfo object.
public saveGridState() {
    const state = this.gridState.getState();
    window.localStorage.setItem('grid-state', JSON.stringify(state));
}

public restoreGridState() {
    const state = window.localStorage.getItem('grid-state');
    if (state) {
        this.gridState.applyState(JSON.parse(state));
    }
}

// Or using string alternative methods.
public saveGridStateString() {
    const state = this.gridState.getStateAsString();
    window.localStorage.setItem('grid-state', state);
}

public restoreGridStateString() {
    const state = window.localStorage.getItem('grid-state');
    if (state) {
        this.gridState.applyStateFromString(state);
    }
}
```

</PlatformBlock>

## Restauración de columnas

<ApiLink type="GridState" /> no conservará plantillas de columnas, formateadores de columnas, etc. de forma predeterminada (consulte [limitaciones](state-persistence.md#limitations)). Restaurar cualquiera de estos elementos se puede lograr con código a nivel de aplicación. Mostremos cómo hacer esto para columnas con plantilla:

1 - Defina una variable de referencia de plantilla (en el ejemplo siguiente es `#activeTemplate`) y asigne un controlador de eventos para el evento `ColumnInit`:

<PlatformBlock for="Angular">

```html
<igx-grid id="grid" #grid igxGridState (columnInit)="onColumnInit($event)">
    <igx-column [field]="'IsActive'" header="IsActive">

            <igx-checkbox [checked]="val"></igx-checkbox>
        </ng-template>
    </igx-column>
</igx-grid>
```

</PlatformBlock>

```ts
constructor() {
    var grid = this.grid = document.getElementById('grid') as IgcGridComponent;
    var isActiveCol = this.isActive = document.getElementById('isActive') as IgcColumnComponent;
    this.onColumnInit = this.onColumnInit.bind(this);

    this._bind = () => {
        grid.data = this.data;
        grid.addEventListener("columnInit", this.onColumnInit);
        isActiveCol.bodyTemplate = this.activeTemplate;
    }
    this._bind();
}

public activeTemplate = (ctx: IgcCellTemplateContext) => {
    return html`<igc-checkbox checked="${ctx.cell.value}"></igc-checkbox>`;
```

## Limitaciones

<ComponentBlock for="HierarchicalGrid">

- Al restaurar todas las características de la cuadrícula a la vez (usando la API `applyState` sin parámetros), las propiedades de columna de la cuadrícula raíz podrían restablecerse a sus valores predeterminados. Si esto sucede, restaure las columnas o la característica de selección de columnas por separado después de eso:

```typescript
state.applyState(gridState);
state.applyState(gridState.columns);
state.applyState(gridState.columnSelection);
```
