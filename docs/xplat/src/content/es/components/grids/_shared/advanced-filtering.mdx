# {Platform} {ComponentTitle} Filtrado avanzado

El Filtrado avanzado de {ProductName} en {Platform} {ComponentTitle} le permite manipular datos proporcionándole un cuadro de diálogo donde puede crear diferentes grupos con condiciones de filtrado en todas las columnas de <ApiLink type="{ComponentName}" />.

## Ejemplo de Filtrado avanzado de {Platform} {ComponentTitle}

<Sample src="/{ComponentSample}/advanced-filtering-options" height={530} alt="{Platform} {ComponentTitle} Advanced Filtering Example" />

## Interacción

Para abrir el cuadro de diálogo de filtrado avanzado, se debe hacer clic en el botón **Filtrado avanzado** de la barra de herramientas de la cuadrícula. Si no se aplica ningún filtro avanzado, debe comenzar creando un grupo de condiciones de filtrado vinculadas con **Y** u **O**. Después de eso, puede agregar condiciones de filtrado o subgrupos.

Para agregar una condición de filtrado, debe seleccionar cualquiera de las columnas de <ApiLink type="Column" member="filterable" />, un operando basado en el <ApiLink type="Column" member="dataType" /> de la columna y un valor si el operando no es unario. Una vez confirmada la condición, aparece un chip con la información de la condición. Al pasar el cursor sobre el chip o hacer clic en él, tiene las opciones de modificarlo o agregar otra condición o grupo justo después.

Si selecciona más de un chip de condición de filtrado, aparece un menú contextual con opciones para crear un grupo o eliminar los filtros. Si elige crear un grupo con las condiciones seleccionadas, el grupo recién creado aparecerá donde estaba colocada la condición seleccionada situada más arriba.

Para seleccionar un grupo, también puede hacer clic en su línea vertical, cuyo color se basa en la condición de enlace **Y** u **O**. Si se selecciona un solo grupo, obtiene un menú contextual con opciones para cambiar su lógica de filtrado, desagruparlo o eliminarlo.

Para filtrar los datos una vez que haya terminado de crear las condiciones y los grupos de filtrado, debe hacer clic en el botón **Aplicar**. Si ha modificado el filtro avanzado, pero no desea conservar los cambios, debe hacer clic en el botón **Cancelar**. También puede borrar el filtro avanzado haciendo clic en el botón **Borrar filtro**.

El filtrado avanzado genera un <ApiLink type="{ComponentName}" member="filteringExpressionsTree" /> que se almacena en la propiedad de entrada <ApiLink type="{ComponentName}" member="advancedFilteringExpressionsTree" />. Puede usar la propiedad <ApiLink type="{ComponentName}" member="advancedFilteringExpressionsTree" /> para establecer un estado inicial del filtrado avanzado.

<PlatformBlock for="Angular">

```typescript
ngAfterViewInit(): void {
    const tree = new FilteringExpressionsTree(FilteringLogic.And);
    tree.filteringOperands.push({
        fieldName: 'ID',
        condition: IgxStringFilteringOperand.instance().condition('contains'),
        searchVal: 'a',
        ignoreCase: true
    });
    const subTree = new FilteringExpressionsTree(FilteringLogic.Or);
    subTree.filteringOperands.push({
        fieldName: 'ContactTitle',
        condition: IgxStringFilteringOperand.instance().condition('doesNotContain'),
        searchVal: 'b',
        ignoreCase: true
    });
    subTree.filteringOperands.push({
        fieldName: 'CompanyName',
        condition: IgxStringFilteringOperand.instance().condition('startsWith'),
        searchVal: 'c',
        ignoreCase: true
    });
    tree.filteringOperands.push(subTree);

    this.@@igObjectRef.advancedFilteringExpressionsTree = tree;
}

```

</PlatformBlock>

## Filtrado avanzado externo

Como se ve en la demostración anterior, el cuadro de diálogo de filtrado avanzado se hospeda en una superposición sobre <ApiLink type="{ComponentName}" />. Cuando la configuración del cuadro de diálogo esté lista, las acciones de aplicar o cerrar ocultarán ese cuadro de diálogo. Hay una manera de hacer que ese cuadro de diálogo permanezca visible, y es usarlo como un componente independiente. En la demostración siguiente, el cuadro de diálogo de filtrado avanzado se declara por separado de <ApiLink type="{ComponentName}" />.

## Aplicación de estilos

Para empezar a aplicar estilos al cuadro de diálogo de Filtrado avanzado, necesitamos importar el archivo **index**, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

El cuadro de diálogo de filtrado avanzado toma su color de fondo del tema de la cuadrícula, usando el parámetro **filtering-row-background**. Para cambiar el fondo, necesitamos crear un tema personalizado:

```scss
$custom-grid: grid-theme(
    $filtering-row-background: #FFCD0F
);
```

Dado que tenemos otros componentes dentro del cuadro de diálogo de filtrado avanzado, como botones, chips, menús desplegables y entradas, necesitamos crear un tema independiente para cada uno:

```scss
$custom-button: button-theme(
    $disabled-color: gray,

);

$custom-button-group: button-group-theme(
    $item-background:  #292826,

);

$custom-input-group: input-group-theme(
    $box-background: #4a4a4a,

);

$custom-chip: chip-theme(
    $background: #FFCD0F,

);

$custom-drop-down: drop-down-theme(
    $background-color: #292826,

);
```

En este ejemplo solo cambiamos algunos de los parámetros para los componentes enumerados, pero los temas <ApiLink type="function-button-theme" label="button-theme" kind="sass" module="index.html" />, <ApiLink type="function-button-group-theme" label="button-group-theme" kind="sass" module="index.html" />, <ApiLink type="function-chip-theme" label="chip-theme" kind="sass" module="index.html" />, <ApiLink type="function-drop-down-theme" label="drop-down-theme" kind="sass" module="index.html" />, <ApiLink type="function-input-group-theme" label="input-group-theme" kind="sass" module="index.html" /> proporcionan muchos más parámetros para controlar sus respectivos estilos.

El último paso es **incluir** los mixins de componentes, cada uno con su tema respectivo. También agregaremos algunos estilos para otros elementos dentro del cuadro de diálogo de filtrado avanzado.

```scss
@include grid($custom-grid);
igx-advanced-filtering-dialog {
    @include button($custom-button);
    @include button-group($custom-button-group);
    @include input-group($custom-input-group);
    @include chip($custom-chip);
    @include drop-down($custom-drop-down);
    .igx-filter-empty__title {
        color: #FFCD0F
    }
    .igx-advanced-filter__header {
        color: #FFCD0F
    }
    .igx-filter-tree__expression-actions igx-icon {
        color: #FFCD0F
    }
    .igx-filter-tree__expression-actions igx-icon:hover {
        color: #ffe482
    }
    .igx-filter-tree__expression-actions igx-icon:focus {
        color: #ffe482
    }
    .igx-filter-contextual-menu {
        border: 1px solid #FFCD0F
    }
    .igx-filter-contextual-menu__close-btn {
        position: absolute !important;
        background: #292826 !important;
        border-color: #FFCD0F !important;
    }
    .igx-input-group__input::placeholder {
        color: gray;
    }
}
```

<DocsAside type="info">

Delimitamos la mayoría de los mixins de los componentes dentro de `igx-advanced-filtering-dialog`, de modo que estos temas personalizados afecten solo a los componentes anidados en el cuadro de diálogo de filtrado avanzado. De lo contrario, otros botones, chips, entradas y menús desplegables de la aplicación también se verían afectados.

</DocsAside>

### Definición de una paleta de colores

En lugar de codificar de forma rígida los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en cuanto a colores usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$yellow-color: #FFCD0F;
$black-color: #292826;
$dark-palette: palette($primary: $yellow-color, $secondary: $black-color);
```

Y luego, con <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />, podemos recuperar fácilmente el color de la paleta.

```scss
$custom-grid: grid-theme(
    $filtering-row-background: color($dark-palette, "secondary", 400)
);

$custom-button: button-theme(
    $disabled-color: color($dark-palette, "secondary", 100),

);

$custom-button-group: button-group-theme(
    $item-background: color($dark-palette, "secondary", 400),

);

$custom-input-group: input-group-theme(
    $box-background: color($dark-palette, "secondary", 200),

);

$custom-chip: chip-theme(
    $background: color($dark-palette, "primary", 400),

);

$custom-drop-down: drop-down-theme(
    $background-color: color($dark-palette, "secondary", 400),

);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](../themes/sass/palettes.md) para obtener orientación detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Al avanzar más con el motor de temas, puede crear una estructura sólida y flexible que se beneficie de los [**esquemas**](../themes/sass/schemas.md). Un **esquema** es una receta de un tema.

Extienda uno de los dos esquemas predefinidos, que se proporcionan para cada componente; en este caso, los esquemas <ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" />, <ApiLink type="variable-_light-button" label="light-button" kind="sass" module="index.html" />, <ApiLink type="variable-_light-button-group" label="light-button-group" kind="sass" module="index.html" />, <ApiLink type="variable-_light-chip" label="light-chip" kind="sass" module="index.html" />, <ApiLink type="variable-_light-input-group" label="light-input-group" kind="sass" module="index.html" /> y <ApiLink type="variable-_light-drop-down" label="light-drop-down" kind="sass" module="index.html" />:

```scss
$grid-dark-palette: palette($primary: #11bd7b, $secondary: #e32057, $info: $black-color);

$custom-grid-schema: extend($_light-grid,
    (
        filtering-row-background:(
           color: ("info")
        )
    )
);

$custom-button-schema: extend($_light-button,
    (
        disabled-color:(
           color: ("secondary", 100)
        ),

    )
);

$custom-button-group-schema: extend($_light-button-group,
    (
        item-background:(
           color: ("secondary", 400)
        ),

    )
);

$custom-input-group-schema: extend($_light-input-group,
    (
        box-background:(
           color: ("secondary", 200)
        ),

    )
);

$custom-chip-schema: extend($_light-chip,
    (
        background:(
           color: ("primary", 400)
        ),

    )
);

$custom-drop-down-schema: extend($_light-drop-down,
    (
        background-color:(
           color: ("secondary", 400)
        ),

    )
);
```

Para aplicar nuestros esquemas personalizados, tenemos que **extender** uno de los globales (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> o <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />), que básicamente señala los componentes con un esquema personalizado, y después agregarlo a los temas de los componentes correspondientes:

```scss
$custom-light-schema: extend($light-schema,(
    igx-grid: $custom-grid-schema,
    igx-button: $custom-button-schema,
    igx-button-group: $custom-button-group-schema,
    igx-input-group: $custom-input-group-schema,
    igx-chip: $custom-chip-schema,
    igx-drop-down: $custom-drop-down-schema
));

$custom-grid: grid-theme(
    $palette: $grid-dark-palette,
    $schema: $custom-light-schema
);

$custom-button: button-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-button-group: button-group-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-input-group: input-group-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-chip: chip-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-drop-down: drop-down-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);
```

No olvide incluir los temas del mismo modo que se mostró anteriormente.
