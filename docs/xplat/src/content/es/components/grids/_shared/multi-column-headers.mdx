# {Platform} {ComponentTitle} Información general de encabezados de varias columnas

La característica de encabezados de varias columnas de {ProductName} en {Platform} {ComponentTitle} le permite agrupar columnas colocándolas bajo un encabezado múltiple común. Cada grupo de encabezados de varias columnas en <ApiLink type="{ComponentName}" /> podría ser una representación de combinaciones entre otros grupos o columnas. Esta característica es especialmente útil al trabajar con grandes conjuntos de datos donde desplazarse horizontalmente podría resultar engorroso.

## {Platform} {ComponentTitle} Ejemplo de encabezados de varias columnas

Cada <ApiLink type="ColumnGroup" /> admite [movimiento](column-moving.md), [anclaje](column-pinning.md) y [ocultación](column-hiding.md).

<DocsAside type="info">

Cuando hay un conjunto de columnas y grupos de columnas, el anclaje funciona solo para los padres de columnas de nivel superior. Más específicamente, no se permite el anclaje por grupos de columnas anidados o columnas. <br />
El movimiento entre columnas y grupos de columnas solo está permitido cuando están en el mismo nivel de la jerarquía y ambos están en el mismo `group`. <br />
Cuando `columns/column-groups` no están envueltas por el `group` actual, lo que significa que son `columns` **de nivel superior**, se permite el movimiento entre columnas visibles completas.

</DocsAside>

## Plantilla de encabezado de varias columnas

<PlatformBlock for="Angular">

Cada uno de los grupos de columnas de la cuadrícula se puede crear con plantilla por separado. El grupo de columnas espera una etiqueta `ng-template` decorada con la directiva `igxHeader`.
El `ng-template` se proporciona con el objeto del grupo de columnas como contexto.

```html
<igx-column-group header="General Information">
    <ng-template igxHeader let-columnGroup>
        {{ columnGroup.header | uppercase }}
    </ng-template>
</igx-column-group>
```

Si desea reutilizar una única plantilla para varios grupos de columnas, podría establecer la propiedad <ApiLink type="ColumnGroup" member="headerTemplate" label="HeaderTemplate" /> del grupo de columnas de esta manera:

```html
<ng-template #columnGroupHeaderTemplate let-columnGroup>
    {{ columnGroup.header | uppercase }}
</ng-template>

<igx-column-group header="General Information" [headerTemplate]="columnGroupHeaderTemplate">
</igx-column-group>
<igx-column-group header="Address Information" [headerTemplate]="columnGroupHeaderTemplate">
</igx-column-group>
```

</PlatformBlock>

## Estilos

Para comenzar a aplicar estilos al comportamiento de ordenación, necesitamos importar el archivo `index`, donde residen todas las funciones de tema y los mixins de componentes:

```scss
@import '~igniteui-angular/lib/core/styles/themes/index';
```

Siguiendo el enfoque más sencillo, creamos un nuevo tema que extiende <ApiLink type="function-igx-grid-theme" label="igx-grid-theme" kind="sass" module="index.html" /> y acepta los parámetros `$header-background`, `$header-text-color`, `$header-border-width`, `$header-border-style` y `$header-border-color`.

```scss
$custom-theme: igx-grid-theme(
    $header-background: #e0f3ff,
    $header-text-color: #e41c77,
    $header-border-width: 1px,
    $header-border-style: solid,
    $header-border-color: rgba(0, 0, 0, 0.08)
);
```

El último paso es **incluir** los mixins de componentes:

```scss
@include igx-grid($custom-theme);
```

<DocsAside type="info">

Si el componente está utilizando una ViewEncapsulation [Emulada](../themes/styles.md#view-encapsulation), es necesario `penetrate` esta encapsulación usando `::ng-deep`:

</DocsAside>

```scss
:host {
   ::ng-deep {
       @include igx-grid($custom-theme);
   }
}
```

### Definición de una paleta de colores

En lugar de codificar de forma fija los valores de color como acabamos de hacer, podemos lograr mayor flexibilidad en términos de colores usando las funciones `igx-palette` y `igx-color`.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$light-blue-color: #e0f3ff;
$deep-pink-color: #e41c77;

$custom-palette: igx-palette($primary: $light-blue-color, $deep-pink-color);
```

Y luego, con `igx-color`, podemos recuperar fácilmente un color de la paleta.

```scss
$custom-theme: igx-grid-theme(
    $header-background: igx-color($custom-palette, "primary", 500),
    $header-text-color: igx-color($custom-palette, "secondary", 500),
    $header-border-width: 1px,
    $header-border-style: solid,
    $header-border-color: igx-color($custom-palette, "grays", 200)
);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](../themes/palette.md) para obtener una guía detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Avanzando más con el motor de temas, puede crear una estructura robusta y flexible que se beneficie de los [esquemas](../themes/schemas.md). Un esquema es una receta de un tema.

Extienda uno de los dos esquemas predefinidos, que se proporcionan para cada componente, en este caso: `_light-grid`:

```scss
// Extending the light grid schema
$custom-grid-schema: extend($_light-grid,
    (
        header-background: (igx-color:('primary', 500)),
        header-text-color: (igx-color:('secondary', 500)),
        header-border-width: 1px,
        header-border-style: solid,
        header-border-color: (igx-color:('grays', 200))
    )
);
```

Para aplicar nuestro esquema personalizado, tenemos que **extender** uno de los globales `light` o `dark`, que básicamente señala los componentes con un esquema personalizado, y después agregarlo a los temas de los componentes respectivos:

```scss
Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid: $custom-grid-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: igx-grid-theme(
  $palette: $custom-palette,
  $schema: $my-custom-schema
);
```

No olvide incluir los temas de la misma manera que se demostró anteriormente.
