## Uso

**Grupos de columnas contraíbles** forma parte de la característica de encabezados multicolumna, que proporciona una forma de contraer/expandir un grupo de columnas a un conjunto de datos más pequeño. Cuando se contrae un grupo de columnas, se mostrará al usuario final un subconjunto de las columnas y las demás columnas hijas del grupo se ocultarán. Cada columna contraída/expandida puede estar enlazada al origen de datos de la cuadrícula, o puede no estar enlazada y, por lo tanto, ser calculada.

Para definir un grupo de columnas como contraíble, debe establecer la propiedad <ApiLink type="ColumnGroup" member="collapsible" /> en <ApiLink type="ColumnGroup" /> en **true**.

Debe definir la propiedad <ApiLink type="ColumnState" member="visibleWhenCollapsed" /> en al menos dos columnas hijas. Al menos una columna debe estar visible cuando el grupo está contraído (<ApiLink type="ColumnState" member="visibleWhenCollapsed" /> establecido en **true**) y al menos una columna debe estar oculta cuando el grupo está expandido (<ApiLink type="ColumnState" member="visibleWhenCollapsed" /> establecido en `false`); de lo contrario, la **funcionalidad contraíble se deshabilitará**. Si <ApiLink type="ColumnState" member="visibleWhenCollapsed" /> no se especifica para algunas de las columnas hijas, esta columna siempre estará visible independientemente de si el estado principal está expandido o contraído.

Veamos el marcado siguiente:

En resumen, cada columna hija tiene tres estados:

- Puede estar siempre visible, sin importar el estado expandido de su elemento principal.
- Puede estar visible cuando su elemento principal está contraído.
- Puede estar oculta cuando su elemento principal está contraído.

El estado inicial del grupo de columnas especificado como contraíble es <ApiLink type="ColumnGroup" member="expanded" /> establecido en **true**, pero puede cambiar fácilmente este comportamiento estableciéndolo en **false**.

<PlatformBlock for="Blazor">

```razor
 <IgbColumnGroup @ref="infoColumn" Header="Customer Information" Collapsible="true">
    <IgbColumn Field="CustomerName" Header="Full name" VisibleWhenCollapsed="true"></IgbColumn>
    <IgbColumn Field="CustomerID" Header="Customer ID" VisibleWhenCollapsed="false"></IgbColumn>
    <IgbColumn Field="FirstName" Header="First Name" VisibleWhenCollapsed="false"></IgbColumn>
    <IgbColumn Field="LastName" Header="Last Name" VisibleWhenCollapsed="false"></IgbColumn>
    <IgbColumnGroup @ref="addressColumn" Header="Customer Address">
        <IgbColumn Field="Country" Header="Country" Sortable="true"></IgbColumn>
        <IgbColumn Field="City" Header="City" Sortable="true"></IgbColumn>
    </IgbColumnGroup>
 </IgbColumnGroup>

@code {
    private IgbColumnGroup infoColumn;
    private IgbColumnGroup addressColumn;

    public RenderFragment<IgbColumnTemplateContext> ColumnIndicatorTemplate = (context) =>
    {
        string icon = context.Column.Expanded ? "remove" : "add";
        return @<IgbIcon IconName="@icon" Collection="material"></IgbIcon>;
    };

    protected override void OnAfterRender(bool firstRender)
    {
        this.infoColumn.CollapsibleIndicatorTemplate = this.ColumnIndicatorTemplate;
        this.addressColumn.CollapsibleIndicatorTemplate = this.ColumnIndicatorTemplate;
    }
}

```

</PlatformBlock>

### Uso de una propiedad

Puede definir una plantilla personalizada de expandir/contraer y proporcionarla a cada uno de los grupos de columnas contraíbles mediante la propiedad <ApiLink type="ColumnGroup" member="collapsibleIndicatorTemplate" />. Compruebe el marcado siguiente:

```html
<ng-template #indTemplate let-column="column">
    <igx-icon [attr.draggable]="false" >{{column.expanded ? 'remove' : 'add'}} </igx-icon>
</ng-template>
<igx-column-group header="Customer Information" [collapsible]="true" [collapsibleIndicatorTemplate]="indTemplate">
    <igx-column field="CustomerName" header="Fullname" [dataType]="'string'" [visibleWhenCollapsed]="true"></igx-column>
    <igx-column field="CustomerID" header="Customer ID" [dataType]="'string'" [visibleWhenCollapsed]="false"></igx-column>
    <igx-column-group header="Customer Address" [collapsible]="true" [collapsibleIndicatorTemplate]="indTemplate">
        <igx-column field="Country" header="Country" [dataType]="'string'" [sortable]="true" [visibleWhenCollapsed]="true"></igx-column>
        <igx-column field="City" header="City" [dataType]="'string'" [sortable]="true" [visibleWhenCollapsed]="false"></igx-column>
    </igx-column-group>
</igx-column-group>
```

### Uso de la directiva igxCollapsibleIndicator

Otra forma de lograr este comportamiento es usar la directiva igxCollapsibleIndicator como se muestra en el ejemplo siguiente:

```html
<igx-column-group header="Customer Information" [collapsible]="true">

    <igx-column field="CustomerID" header="Customer ID" [dataType]="'string'" [visibleWhenCollapsed]="false"></igx-column>
    <igx-column-group header="Customer Address" [collapsible]="true">
        <igx-column field="Country" header="Country" [dataType]="'string'" [sortable]="true" [visibleWhenCollapsed]="true"></igx-column>
        <igx-column field="City" header="City" [dataType]="'string'" [sortable]="true" [visibleWhenCollapsed]="false"></igx-column>
    </igx-column-group>
</igx-column-group>
```

> **Nota**
> Tenga en cuenta que, inicialmente, la opción de contraer grupo tiene prioridad sobre columna oculta: si declaró que su columna debe estar oculta mediante la propiedad
> hidden y tiene un grupo definido donde debe mostrarse la misma columna, la columna se mostrará.
