# {Platform} {ComponentTitle} Barra de herramientas

La barra de herramientas {ProductName} es un contenedor para operaciones de UI en {Platform} {ComponentTitle}. La barra de herramientas {Platform} se encuentra en la parte superior del componente {Platform}, es decir, <ApiLink type="{ComponentName}" />, y coincide con su tamaño horizontal. El contenedor de la barra de herramientas puede alojar cualquier contenido personalizado o conjunto de controles de UI predefinidos. El conjunto predeterminado para {Platform} {ComponentTitle} incluye:

- Ocultación de columnas
- Anclaje de columnas
- Exportación a Excel
- Filtrado avanzado

La barra de herramientas y los componentes de UI predefinidos admiten eventos {Platform} y exponen API para los desarrolladores.

## Barra de herramientas con cuadrículas secundarias

<PlatformBlock for="Angular">

Debido a ciertas limitaciones en cómo se implementan las cuadrículas secundarias de un IgxHierarchicalGrid y en cómo funciona el ámbito de DI, hay una salvedad al
usar la barra de herramientas en el ámbito de las cuadrículas secundarias. Al definir un componente de barra de herramientas dentro de las etiquetas `igx-row-island`, asegúrese siempre de usar IgxGridToolbar y pasar la instancia de cuadrícula proporcionada como una propiedad de entrada a la propia barra de herramientas.
Esto garantizará que siempre tenga la instancia de cuadrícula correcta en el ámbito de su plantilla:

```html
<igx-hierarchical-grid>
    <!--...-->
    <igx-row-island>
        <!--
            You can name the binding from igxGridToolbar however you want. Just make sure to use
            it inside the template if you need to access the grid instance.
        -->
        <igx-grid-toolbar [grid]="gridRef" *igxGridToolbar="let gridRef">
            <igx-grid-toolbar-title>Child toolbar {{ gridRef.parentIsland.level }}</igx-grid-toolbar-title>
        </igx-grid-toolbar>
    </igx-row-island>
    <!--...-->
</igx-hierarchical-grid>
```

</PlatformBlock>

<PlatformBlock for="WebComponents, React">

Debido a ciertas limitaciones en cómo se implementan las cuadrículas secundarias de un `{ComponentSelector}` y en cómo funciona el ámbito de DI, para definir un componente de barra de herramientas dentro de `{RowIslandSelector}`, use la propiedad de entrada `ToolbarTemplate`. Esto permite que las cuadrículas secundarias creen sus propias instancias independientes de barra de herramientas:

## Características

La barra de herramientas es excelente para separar la lógica y las interacciones que afectan a la cuadrícula en su conjunto.

Como se muestra arriba, se puede configurar para proporcionar componentes predeterminados para controlar la ocultación de columnas, el anclaje de columnas, el filtrado avanzado y la exportación de datos desde la cuadrícula.

Estas características se pueden habilitar de forma independiente entre sí siguiendo un patrón similar al componente de tarjeta de la suite {ProductName}.

A continuación se enumeran las características principales de la barra de herramientas con código de ejemplo para cada una de ellas.

Al igual que con el resto de las acciones de la barra de herramientas, la exportación se proporciona mediante un <ApiLink type="GridToolbarExporter" /> de forma inmediata.

<PlatformBlock for="Angular">

El componente de exportación usa el servicio correspondiente para el formato de datos de destino <ApiLink type="ExcelExporterService" /> y `CSVExporterService`. Si necesita repasar la DI en {Platform}, consulte la [guía oficial](https://{Platform}.io/guide/dependency-injection). Aquí hay un fragmento de ejemplo que muestra cómo habilitar todos los servicios de exportación para su aplicación.

```typescript
// app.module.ts

import { IgxExcelExporterService, IgxCsvExporterService } from 'igniteui-{Platform}';

@NgModule({
    //...
    providers: [IgxExcelExporterService, IgxCsvExporterService ]
})
export class AppModule { ... }
```

<DocsAside type="info">

En v12.2.1 y versiones posteriores, los servicios de exportador se proporcionan en root, lo que significa que ya no necesita declararlos en los proveedores de AppModule.

## Estilo

Para comenzar a aplicar estilos a la barra de herramientas, necesitamos importar el archivo de índice, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-{Platform}/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-{Platform}/lib/core/styles/themes/index';
```

Primero, creemos una nueva paleta.

```scss
$my-dark-palette: palette(
    $primary: #2466ff,
    $secondary: #FFCD0F,
    $surface: #2a2b2f,
    $grays: #fff,
);

$my-dark-color: color($my-dark-palette, 'surface');
```

Ahora, cree un nuevo tema que extienda <ApiLink type="function-grid-toolbar-theme" label="grid-toolbar-theme" kind="sass" module="index.html" /> y modifique los parámetros `$background-color` y `$title-text-color`.

```scss
$dark-grid-toolbar-theme: grid-toolbar-theme(
    $palette: $my-dark-palette,
    $background-color: $my-dark-color,
    $title-text-color: color($my-dark-palette, 'secondary'),
    $dropdown-background: $my-dark-color,
);
```

Para aplicar tema a los menús de acciones de columna de la barra de herramientas, tenemos que cambiar el tema del componente `column-actions-theme`.

```scss
$dark-column-actions-theme: column-actions-theme(
    $palette: $my-dark-palette,
    $title-color: color($my-dark-palette, 'secondary'),
    $background-color: color($my-dark-palette, 'surface')
);
```

Dado que las acciones de columna usan otros componentes: igx-button, igx-checkbox e igx-input-group, necesitamos cambiar sus temas para que coincidan con nuestro nuevo tema de barra de herramientas.

```scss
$dark-button-theme: button-theme(
    $palette: $my-dark-palette,
    $outlined-background: color($my-dark-palette, 'secondary'),
    $outlined-hover-background: color($my-dark-palette, 'grays', 100),
    $outlined-hover-text-color: color($my-dark-palette, 'secondary')
);

$dark-checkbox-theme: checkbox-theme(
    $palette: $my-dark-palette,
    $tick-color: $my-dark-color,
);

$dark-input-group-theme: input-group-theme(
    $palette: $my-dark-palette
);
```

El último paso es **incluir** los temas recién creados.

```scss
:host {
    @include grid-toolbar($dark-grid-toolbar-theme);
    @include column-actions($dark-column-actions-theme);
    @include checkbox($dark-checkbox-theme);
    @include input-group($dark-input-group-theme);
    @include button($dark-button-theme);
}
```

<DocsAside type="info">

Si `$legacy-support` está establecido en `false(default)`, incluya las variables CSS del componente de esta forma:

</DocsAside>

```scss
:host {
    @include css-vars($dark-grid-toolbar-theme);
    @include css-vars($dark-column-actions-theme);
    @include css-vars($dark-checkbox-theme);
    @include css-vars($dark-input-group-theme);
    @include css-vars($dark-button-theme);
}
```

<DocsAside type="info">

Si el componente usa una ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario `penetrate` esta encapsulación usando `::ng-deep`:

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid-toolbar($dark-grid-toolbar-theme);
        @include column-actions($dark-column-actions-theme);
        @include checkbox($dark-checkbox-theme);
        @include input-group($dark-input-group-theme);
        @include button($dark-button-theme);
    }
}
```
