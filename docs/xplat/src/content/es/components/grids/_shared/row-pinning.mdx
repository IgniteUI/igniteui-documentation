# {Platform} {ComponentTitle} Anclaje de filas

La característica de anclaje de filas de {ProductName} en {Platform} {ComponentTitle} permite anclar una o varias filas en la parte superior o inferior de la cuadrícula. El anclaje de filas permite a los usuarios finales anclar filas en un orden determinado, duplicándolas en un área especial que siempre está visible incluso cuando desplazan el <ApiLink type="{ComponentName}" /> verticalmente. {Platform} {ComponentTitle} tiene una UI de anclaje de filas integrada, que se habilita inicializando un componente <ApiLink type="ActionStrip" /> en el contexto de {ComponentTitle}. Además, puede definir una UI personalizada y cambiar el estado de anclaje de las filas mediante la API de anclaje de filas.

## API de anclaje de filas

El anclaje de filas se controla mediante la entrada <ApiLink type="RowDirective" member="pinned" /> de `Row`. Las filas ancladas se representan en la parte superior de <ApiLink type="{ComponentName}" /> de forma predeterminada y permanecen fijas durante el desplazamiento vertical de las filas no ancladas en el cuerpo de <ApiLink type="{ComponentName}" />.

<PlatformBlock for="Angular, WebComponents">

```typescript
this.grid.getRowByIndex(0).pinned = true;
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
gridRef.current.getRowByIndex(0).pinned = true;
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp
this.Grid.PinRowAsync("ALFKI", 0);
```

</PlatformBlock>

También puede usar los métodos <ApiLink type="{ComponentName}" /> o <ApiLink type="{ComponentName}" member="pinRow" label="PinRow" /> de <ApiLink type="{ComponentName}" member="unpinRow" label="UnpinRow" /> para anclar o desanclar registros por su ID:

<PlatformBlock for="Angular, WebComponents">

```typescript

this.grid.unpinRow('ALFKI');
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx

```

</PlatformBlock>

<PlatformBlock for="Blazor">

```csharp

this.Grid.UnpinRowAsync("ALFKI");
```

</PlatformBlock>

Tenga en cuenta que el ID de fila es el valor de la clave principal, definido por <ApiLink type="{ComponentName}" member="primaryKey" /> de la cuadrícula, o la propia instancia del registro. Ambos métodos devuelven un valor booleano que indica si su operación correspondiente se ha realizado correctamente o no. Normalmente, el motivo por el que fallan es que la fila ya se encuentra en el estado deseado.

Una fila se ancla debajo de la última fila anclada. El orden de las filas ancladas se puede cambiar suscribiéndose al evento <ApiLink type="{ComponentName}" member="rowPinning" /> y cambiando la propiedad <ApiLink type="PinRowEventArgs" /> de los argumentos del evento al índice de posición deseado.

## Posición de anclaje

Puede cambiar la posición de anclaje de filas mediante la opción de configuración <ApiLink type="{ComponentName}" member="pinning" />. Le permite establecer la posición del área de anclaje en Superior o Inferior.
Cuando se establece en Inferior, las filas ancladas se representan en la parte inferior de la cuadrícula, después de las filas no ancladas. Las filas no ancladas se pueden desplazar verticalmente, mientras que las filas ancladas permanecen fijas en la parte inferior.

### Mediante arrastre de filas

Supongamos que desea poder arrastrar y soltar directamente una fila entre las filas ancladas y no ancladas para cambiar su estado de anclaje.
Esto se puede lograr habilitando la característica de arrastre de filas y anclando/desanclando las filas mediante la API al soltar.

Primero, la cuadrícula debe marcarse como área de colocación usando la directiva `igxDrop` y la funcionalidad de arrastre de filas debe habilitarse mediante la opción `rowDraggable`.

```html
<igx-grid [data]="data" [autoGenerate]="true" [rowDraggable]="true"
    [primaryKey]="'ID'" igxDrop (dropped)="onDropAllowed($event)">
</igx-grid>
```

Luego, el evento `dropped` se puede usar para controlar la lógica de reordenación y anclaje/desanclaje.

```typescript
public onDropAllowed(args) {
    const event = args.originalEvent;
    let currRowPinnedIndex;
    const currRowIndex = this.getCurrentRowIndex(this.grid.rowList.toArray(),
        { x: event.clientX, y: event.clientY });
    if (currRowIndex === -1) { return; }

    const currRowID = this.getCurrentRowID(this.grid.rowList.toArray(),
        { x: event.clientX, y: event.clientY });

    const currentRow = this.grid.rowList.toArray().find((r) => r.rowID === currRowID);
    if (currentRow.pinned) {
        currRowPinnedIndex = this.grid.pinnedRows.indexOf(this.grid.pinnedRows.find((r) => r.rowID === currRowID));
    }
    // remove the row that was dragged and place it onto its new location
    this.grid.deleteRow(args.dragData.key);
    this.data.splice(currRowIndex, 0, args.dragData.data);
    if (currentRow.pinned && !args.dragData.pinned) {
        this.grid.pinRow(args.dragData.key, currRowPinnedIndex);
    } else if (!currentRow.pinned && args.dragData.pinned) {
        this.grid.unpinRow(args.dragData.key);
    } else if (currentRow.pinned && args.dragData.pinned) {
        this.grid.unpinRow(args.dragData.key);
        this.grid.pinRow(args.dragData.key, currRowPinnedIndex);
    }
}
```

Esto permitiría reordenar las filas y moverlas entre las colecciones de filas ancladas y no ancladas.

## Limitaciones del anclaje de filas

- Solo se pueden anclar los registros que existen en el origen de datos.
- El estado de anclaje de filas no se exporta a Excel. La cuadrícula se exporta como si no se hubiera aplicado ningún anclaje de filas.
- Las copias de las filas ancladas en el área desplazable de la cuadrícula son una parte integral de cómo otras características de la cuadrícula logran su funcionalidad en presencia de filas ancladas y, por lo tanto, no se puede deshabilitar su creación ni se pueden eliminar.
- Dado que la selección de filas funciona completamente con los ID de fila, al seleccionar filas ancladas también se seleccionan sus copias (y viceversa). Además, la selección de rango (por ejemplo, usando <kbd>MAYÚS</kbd> + clic) dentro del área anclada funciona de la misma manera que seleccionar un rango de filas dentro del área desplazable. La selección resultante incluye todas las filas intermedias, incluso si no están ancladas actualmente. Obtener las filas seleccionadas mediante la API solo devuelve una única instancia de cada registro seleccionado.

<PlatformBlock for="Angular">

- Debido a cómo se almacenan internamente las filas ancladas para que puedan aparecer tanto en las áreas ancladas como no ancladas de la cuadrícula, el anclaje de filas no se admite cuando los registros de la cuadrícula se recuperan desde un endpoint remoto bajo demanda (virtualización remota).
- Cuando la cuadrícula no tiene establecido ningún <ApiLink type="{ComponentName}" member="primaryKey" /> y los escenarios de datos remotos están habilitados (cuando la paginación, la ordenación, el filtrado o el desplazamiento desencadenan solicitudes a un servidor remoto para recuperar los datos que se mostrarán en la cuadrícula), una fila perderá el siguiente estado después de que se complete una solicitud de datos:
  - Selección de filas
  - Expansión/contracción de filas
  - Edición de filas
  - Anclaje de filas

</PlatformBlock>

A continuación, cree un nuevo tema que extienda <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> y acepte los parámetros necesarios para personalizar la característica de anclaje de filas según se desee.

```scss
$custom-grid-theme: grid-theme(
    /* Pinning properties that affect styling */
    $pinned-border-width: 5px,
    $pinned-border-style: double,
    $pinned-border-color: #FFCD0F,
    $cell-active-border-color: #FFCD0F
    /* add other features properties here... */
);
```

### Uso de variables CSS

El último paso es pasar el tema personalizado de la cuadrícula:

```scss
@include css-vars($custom-grid-theme);
```

### Uso de mixins

Para aplicar estilos a componentes para Internet Explorer 11, debe usar un enfoque diferente, ya que no admite variables CSS.

Si el componente usa una ViewEncapsulation [Emulada](../themes/styles.md#view-encapsulation), es necesario `penetrate` esta encapsulación usando `::ng-deep`. Sin embargo, para evitar que el tema personalizado se filtre a otros componentes, asegúrese de incluir el selector `:host` antes de `::ng-deep`:

```scss
:host {
    ::ng-deep {
        // Pass the custom grid theme to the `igx-grid` mixin
        @include grid($custom-grid-theme);
    }
}
```
