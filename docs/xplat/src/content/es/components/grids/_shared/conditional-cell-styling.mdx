# {Platform} {ComponentTitle} Estilo condicional

La característica de estilo condicional de {ProductName} en {Platform} {ComponentTitle} permite aplicar estilos personalizados a nivel de fila o de celda.  La funcionalidad de estilo condicional <ApiLink type="{ComponentName}" /> se utiliza para enfatizar visualmente o resaltar datos que cumplen ciertos criterios, lo que facilita a los usuarios identificar información importante o tendencias dentro de la cuadrícula.

## {ComponentTitle} Estilo condicional de filas

El componente <ApiLink type="{ComponentName}" /> en {ProductName} proporciona dos formas de aplicar **estilo condicional a las filas** según reglas personalizadas.

- Configurando la entrada <ApiLink type="{ComponentName}" member="rowClasses" /> en el componente <ApiLink type="{ComponentName}" />;
- Configurando la entrada <ApiLink type="{ComponentName}" member="rowStyles" /> en el componente <ApiLink type="{ComponentName}" />;

Más adelante en este tema abordaremos ambas con más detalle.

### Uso de clases de fila

Puede aplicar estilos condicionalmente a las filas de <ApiLink type="{ComponentName}" /> configurando la entrada <ApiLink type="{ComponentName}" member="rowClasses" /> y definiendo reglas personalizadas.

### Uso de estilos de fila

El control <ApiLink type="{ComponentName}" /> expone la propiedad <ApiLink type="{ComponentName}" member="rowStyles" />, que permite aplicar estilos condicionales a las filas de datos. De forma similar a <ApiLink type="{ComponentName}" member="rowClasses" />, acepta un literal de objeto donde las claves son propiedades de estilo y los valores son expresiones para evaluación. Además, puede aplicar estilos normales (sin ninguna condición).

> La firma de devolución de llamada para <ApiLink type="{ComponentName}" member="rowStyles" /> y <ApiLink type="{ComponentName}" member="rowClasses" /> es:

### Uso de estilos de celda

Las columnas exponen la propiedad <ApiLink type="Column" member="cellStyles" label="CellStyles" />, que permite aplicar estilos condicionales a las celdas de la columna. De forma similar a <ApiLink type="Column" member="cellClasses" />, acepta un literal de objeto donde las claves son propiedades de estilo y los valores son expresiones para evaluación. Además, puede aplicar estilos normales con facilidad (sin ninguna condición).

Definamos nuestros estilos:

}

## Problemas conocidos y limitaciones

- Si hay celdas enlazadas a la misma condición (de diferentes columnas) y se actualiza una celda, las demás celdas no se actualizarán según el nuevo valor, si se cumple la condición.

<PlatformBlock for="Angular, WebComponents">

Se debe realizar una comprobación para aplicar los cambios al resto de las celdas. El ejemplo siguiente muestra cómo hacerlo.

```ts
public backgroundClasses = {
    myBackground: (rowData: any, columnKey: string) => {
        return rowData.Col2 < 10;
    }
};

public editDone(evt) {
    this.Col1.cellClasses = {...this.backgroundClasses};
}
```

  <igx-column field="Col2" dataType="number" [editable]="true" [cellClasses]="backgroundClasses"></igx-column>
