# {Platform} {ComponentTitle} Edición de celdas

La edición de celdas de {ProductName} en {Platform} {ComponentTitle} proporciona una gran capacidad de manipulación de datos del contenido de celdas individuales dentro del componente {Platform} {ComponentTitle} y viene con una API potente para operaciones CRUD de React. Es una característica fundamental en aplicaciones como hojas de cálculo, tablas de datos y cuadrículas de datos, que permite a los usuarios agregar, editar o actualizar datos dentro de celdas específicas.
De forma predeterminada, la cuadrícula en {ProductName} se utiliza en la edición de celdas. Y debido a la **plantilla predeterminada de edición de celdas**, habrá diferentes editores según el tipo de datos de la columna Parte superior del formulario.

Además, puede definir sus propias plantillas personalizadas para acciones de actualización de datos y para sobrescribir el comportamiento predeterminado para confirmar y descartar cualquier cambio.

## Edición de celdas

### Edición mediante UI

Puede entrar en modo de edición para una celda específica cuando una celda editable está enfocada de una de las siguientes formas:

- con doble clic;
- con un solo clic: un solo clic entrará en modo de edición solo si la celda seleccionada anteriormente estaba en modo de edición y la celda seleccionada actualmente es editable. Si la celda seleccionada anteriormente no estaba en modo de edición, un solo clic seleccionará la celda sin entrar en modo de edición;
- al presionar la tecla <kbd>ENTRAR</kbd>;
- al presionar la tecla <kbd>F2</kbd>;

Puede salir del modo de edición **sin confirmar** los cambios de una de las siguientes formas:

- al presionar la tecla <kbd>Escape</kbd>;
- cuando realiza operaciones de **ordenación**, **filtrado**, **búsqueda** y **ocultación**;

Puede salir del modo de edición y **confirmar** los cambios de una de las siguientes formas:

- al presionar la tecla <kbd>ENTRAR</kbd>;
- al presionar la tecla <kbd>F2</kbd>;
- al presionar la tecla <kbd>PESTAÑA</kbd>;
- con un solo clic en otra celda: cuando haga clic en otra celda en <ApiLink type="{ComponentName}" />, se enviarán sus cambios.
- operaciones como paginación, cambio de tamaño, fijación o movimiento saldrán del modo de edición y se enviarán los cambios.

<DocsAside type="info">

La celda permanece en modo de edición cuando se desplaza vertical u horizontalmente o hace clic fuera de <ApiLink type="{ComponentName}" />. Esto es válido tanto para la edición de celdas como para la edición de filas.

</DocsAside>

### {ComponentTitle} Edición al estilo de Excel

El uso de la edición al estilo de Excel permite al usuario navegar por las celdas igual que lo haría usando Excel, y editarlas con mucha rapidez.

La implementación de esta funcionalidad personalizada se puede realizar utilizando los eventos de <ApiLink type="{ComponentName}" />. Primero nos conectamos a los eventos keydown de la cuadrícula y, desde allí, podemos implementar dos funcionalidades:

<PlatformBlock for="React">

```tsx
const gridRef = useRef<IgrGrid>();
useEffect(() => {
    gridRef.current.addEventListener("keydown", handleKeyDown);
    return () => {
        gridRef.current.removeEventListener("keydown", handleKeyDown);
    };
}, []);
<IgrGrid ref={gridRef} autoGenerate={false} data={nwindData} primaryKey="ProductID">
</IgrGrid>
```

<DocsAside type="info">

Estamos utilizando el evento keydown nativo del navegador en lugar del evento sintético onKeyDown de React. Cuando una celda entra en modo de edición y se presiona la tecla ENTER para moverse a la siguiente fila, la característica de edición de la cuadrícula actualiza el valor de la celda y cierra el modo de edición. Como resultado, el elemento de entrada utilizado para la edición se elimina del DOM. Debido a las optimizaciones del sistema de eventos de React, el evento sintético onKeyDown no se propaga hasta la cuadrícula porque el elemento ya no existe en el árbol de React en ese momento. Por lo tanto, es necesario usar el escuchador de eventos nativo para garantizar el comportamiento esperado.

</DocsAside>

</PlatformBlock>

public keydownHandler(event) {
  const key = event.keyCode;
  const grid = this.grid;
  const activeElem = grid.navigation.activeNode;

  if ((key >= 48 && key <= 57) ||
      (key >= 65 && key <= 90) ||
      (key >= 97 && key <= 122)) {
        // Number or Alphabet upper case or Alphabet lower case
        const columnName = grid.getColumnByVisibleIndex(activeElem.column).field;
        const cell = grid.getCellByColumn(activeElem.row, columnName);
        if (cell && !grid.crudService.cellInEditMode) {
            grid.crudService.enterEditMode(cell);
            cell.editValue = event.key;
        }
    }
}

function handleKeyDown(event: KeyBoardEvent) {
    const code = event.code;
    const grid = event.currentTarget as IgrGrid;
    const activeElem = grid.selectedCells[0];

    if ((event.code >= "Digit0" && event.code <= "Digit9") || (event.code >= "KeyA" && event.code <= "KeyZ")
        || (event.code >= "Numpad0" && event.code <= "Numpad9" && event.code !== "Enter" && event.code !== "NumpadEnter")) {
        if (activeElem && !activeElem.editMode) {
            activeElem.editMode = true;
            activeElem.editValue = event.key;
        } else if (activeElem && activeElem.editMode) {
            event.preventDefault();
            activeElem.editValue = activeElem.editValue + event.key;
        }
    }
}

### Ejemplo de estilo

<Sample src="/{ComponentSample}/cell-editing-styling" height={650} alt="{Platform} {ComponentTitle} Cell Editing Styling Example" />

<PlatformBlock for="Angular">

<ApiLink type="{ComponentName}" /> permite aplicar estilo a sus celdas mediante la [biblioteca de temas de {ProductName}](../themes/styles.md). <ApiLink type="function-grid-theme" label="theme" kind="sass" module="index.html" /> de la cuadrícula expone una amplia variedad de propiedades, que permiten a los usuarios aplicar estilo a muchos aspectos diferentes de la cuadrícula.

En los pasos siguientes, repasaremos cómo puede aplicar estilo a la celda de la cuadrícula en modo de edición y cómo puede delimitar el alcance de esos estilos.

Para usar la [biblioteca de temas de Ignite UI](../themes/styles.md), primero debemos importar el archivo de tema `index` en nuestros estilos globales:

### Importar la biblioteca de estilos

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to {ProductName} version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Ahora podemos usar todas las funciones expuestas por el motor de temas de {ProductName}.

### Definir una paleta

Después de haber importado correctamente el archivo de índice, creamos una paleta personalizada que podemos usar. Definamos dos colores que nos gusten y usémoslos para crear una paleta con [igx-palette](../themes/palettes.md):

```scss
$white: #fff;
$blue: #4567bb;

$color-palette: palette($primary: $white, $secondary: $blue);
```

### Definir temas

Ahora podemos definir el tema usando nuestra paleta. Las celdas reciben estilo mediante <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" />, por lo que podemos usarlo para generar un tema para nuestro <ApiLink type="{ComponentName}" />:

```scss
$custom-grid-theme: grid-theme(
    $cell-editing-background: $blue,
    $cell-edited-value-color: $white,
    $cell-active-border-color: $white,
    $edit-mode-color: color($color-palette, "secondary", 200)
);
```

### Aplicar el tema

La forma más sencilla de aplicar nuestro tema es con una instrucción `sass` `@include` en el archivo de estilos globales:

```scss
@include grid($custom-grid-theme);
```

De esta manera, el tema se aplicará a **todas** las cuadrículas de nuestra aplicación. Si deseamos aplicar este estilo personalizado solo a un componente específico, debemos delimitar el alcance del tema.

### Tema de componente con alcance delimitado

Para que el tema personalizado afecte solo a nuestro componente específico, podemos mover todos los estilos que acabamos de definir desde el archivo de estilos globales al archivo de estilos de nuestro componente personalizado (incluida la [importación](#importing-style-library) del archivo `index`).

De esta manera, debido a [ViewEncapsulation](https://angular.io/api/core/Component#encapsulation) de {Platform}, nuestros estilos se aplicarán solo a nuestro componente personalizado.

 <DocsAside type="info">

Si el componente utiliza un ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario penetrar esta encapsulación usando `::ng-deep` para aplicar estilo a la cuadrícula.

 </DocsAside>

 <DocsAside type="info">

Envolvemos la instrucción dentro de un selector `:host` para evitar que nuestros estilos afecten a elementos fuera de nuestro componente:

 </DocsAside>

```scss
:host {
    ::ng-deep {
            @include grid($custom-grid-theme);
        }
    }
}
```
