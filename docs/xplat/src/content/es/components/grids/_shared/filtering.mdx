# {Platform} {ComponentTitle} Filtrado

El Filtrado de {ProductName} en {Platform} {ComponentTitle} es una característica que permite mostrar u ocultar datos de forma selectiva según criterios o condiciones específicos. Hay un contenedor de datos enlazado mediante el cual el componente <ApiLink type="{ComponentName}" /> proporciona una API de filtrado completa y todas las capacidades de filtrado. Los tipos de filtrado disponibles aquí son tres:

- Filtrado rápido
- [Filtrado estilo Excel](excel-style-filtering.md)
- [Filtrado avanzado](advanced-filtering.md)

## Configuración

Para especificar si el filtrado está habilitado y qué modo de filtrado debe utilizarse, <ApiLink type="{ComponentName}" /> expone las siguientes propiedades: <ApiLink type="{ComponentName}" member="allowFiltering" />, <ApiLink type="{ComponentName}" member="allowAdvancedFiltering" />, <ApiLink type="{ComponentName}" member="filterMode" /> y <ApiLink type="Column" member="filterable" />.

La propiedad <ApiLink type="{ComponentName}" member="allowFiltering" /> le permite especificar las siguientes opciones:

- **false**: el filtrado para la cuadrícula correspondiente estará deshabilitado. Este es el valor predeterminado.
- **true**: el filtrado para la cuadrícula correspondiente estará habilitado.

La propiedad <ApiLink type="{ComponentName}" member="allowAdvancedFiltering" /> le permite especificar las siguientes opciones:

- **false**: el filtrado avanzado para la cuadrícula correspondiente estará deshabilitado. Este es el valor predeterminado.
- **true**: el filtrado avanzado para la cuadrícula correspondiente estará habilitado.

La propiedad <ApiLink type="{ComponentName}" member="filterMode" /> le permite especificar las siguientes opciones:

- **QuickFilter**: una UI de filtrado sencilla. Este es el valor predeterminado.
- **ExcelStyleFilter**: una UI de filtrado similar a Excel.

La propiedad <ApiLink type="Column" member="filterable" /> le permite especificar las siguientes opciones:

- **true**: el filtrado para la columna correspondiente estará habilitado. Este es el valor predeterminado.
- **false**: el filtrado para la columna correspondiente estará deshabilitado.

## Interacción

Para abrir la fila de filtro de una columna concreta, debe hacerse clic en el chip "Filtro" situado debajo de su encabezado. Para agregar condiciones, debe elegir el operando de filtro mediante el menú desplegable situado a la izquierda de la entrada e introducir un valor. Para las columnas **number** y **date**, "Igual a" está seleccionado de forma predeterminada; para **string**, "Contiene"; y para **boolean**, "Todos". Al presionar "Enter" se confirma la condición y ahora puede agregar otra. Hay un menú desplegable entre los chips de "condición", que determina el operador lógico entre ellos; "AND" está seleccionado de forma predeterminada. Para eliminar una condición, puede hacer clic en el botón "X" del chip, y para editarla debe seleccionar el chip; la entrada se rellenará con los datos del chip. Mientras la fila de filtro está abierta, puede hacer clic en el encabezado de cualquier columna filtrable para seleccionarla y poder agregar condiciones de filtro para ella.

Mientras se hayan aplicado algunas condiciones de filtrado a una columna y la fila de filtro esté cerrada, puede eliminar las condiciones haciendo clic en el botón de cierre del chip, o puede abrir la fila de filtro seleccionando cualquiera de los chips. Cuando no hay espacio suficiente para mostrar todas las condiciones, se muestra un icono de filtro con una insignia que indica cuántas condiciones más hay. También se puede hacer clic en él para abrir la fila de filtro.

## Uso

Hay una estrategia de filtrado predeterminada proporcionada de fábrica, así como todas las condiciones de filtrado estándar, que el desarrollador puede reemplazar por su propia implementación. Además, hemos proporcionado una forma de integrar fácilmente sus propias condiciones de filtrado personalizadas. Actualmente, <ApiLink type="{ComponentName}" /> no solo proporciona una UI de filtrado sencilla, sino también opciones de filtrado más complejas. Según el <ApiLink type="Column" member="dataType" /> establecido de la columna, el conjunto correcto de **operaciones de filtrado** se carga dentro del menú desplegable de la UI de filtro. Además, puede establecer las propiedades <ApiLink type="FilteringExpression" member="ignoreCase" label="IgnoreCase" /> y <ApiLink type="BooleanFilteringOperand" member="condition" label="Condition" /> inicial.

La característica de filtrado se habilita para el componente <ApiLink type="{ComponentName}" /> estableciendo la entrada <ApiLink type="{ComponentName}" member="allowFiltering" /> en **true**. El <ApiLink type="{ComponentName}" member="filterMode" /> predeterminado es `QuickFilter` y **no puede** cambiarse en tiempo de ejecución. Para deshabilitar esta característica para una columna determinada, establezca la entrada <ApiLink type="Column" member="filterable" /> en **false**.

Puede filtrar cualquier columna o una combinación de columnas mediante la API <ApiLink type="{ComponentName}" />. <ApiLink type="{ComponentName}" /> expone varios métodos para esta tarea: <ApiLink type="Filter" />, `FilterGlobal` y `ClearFilter`.

- <ApiLink type="Filter" />: filtra una sola columna o una combinación de columnas.

Se exponen cinco clases de operandos de filtrado:

- <ApiLink type="FilteringOperand" />: este es un operando de filtrado base, que puede heredarse al definir condiciones de filtrado personalizadas.
- <ApiLink type="BooleanFilteringOperand" /> define todas las condiciones de filtrado predeterminadas para el tipo **boolean**.
- <ApiLink type="NumberFilteringOperand" /> define todas las condiciones de filtrado predeterminadas para el tipo **numeric**.
- <ApiLink type="StringFilteringOperand" /> define todas las condiciones de filtrado predeterminadas para el tipo **string**.
- <ApiLink type="DateFilteringOperand" /> define todas las condiciones de filtrado predeterminadas para el tipo **date**.

```typescript
// Single column filtering

// Filter the `ProductName` column for values which `contains` the `myproduct` substring, ignoring case
this.grid.filter('ProductName', 'myproduct', IgcStringFilteringOperand.instance().condition('contains'), true);
```

Los únicos parámetros obligatorios son la clave de campo de la columna y el término de filtrado. Tanto la condición como la distinción entre mayúsculas y minúsculas se inferirán de las propiedades de la columna si no se proporcionan. En el caso de filtrado múltiple, el método acepta una matriz de expresiones de filtrado.

## Operandos de filtrado personalizados

Puede personalizar el menú de filtrado agregando, eliminando o modificando los operandos de filtrado. De forma predeterminada, el menú de filtrado contiene ciertos operandos según el tipo de datos de la columna (<ApiLink type="BooleanFilteringOperand" />, <ApiLink type="DateFilteringOperand" />, <ApiLink type="NumberFilteringOperand" /> y <ApiLink type="StringFilteringOperand" />). Puede extender estas clases o su clase base <ApiLink type="FilteringOperand" /> para cambiar el comportamiento de los elementos del menú de filtrado.

En el ejemplo siguiente, inspeccione los menús de filtros de las columnas “Product Name” y “Discontinued”. Para el filtro de la columna “Discontinued”, hemos limitado el número de operandos a Todos, Verdadero y Falso. Para el filtro de la columna “Product Name”, hemos modificado la lógica de los operandos Contiene y No contiene para realizar una búsqueda con distinción entre mayúsculas y minúsculas, y también hemos agregado los operandos Vacío y No vacío.

Para hacerlo, extienda <ApiLink type="StringFilteringOperand" /> y <ApiLink type="BooleanFilteringOperand" />, modifique las operaciones y su lógica, y establezca la entrada `filters` de la columna en los nuevos operandos.

<PlatformBlock for="Angular">

```typescript
// grid-custom-filtering.component.ts

export class GridCustomFilteringComponent {
    public caseSensitiveFilteringOperand = CaseSensitiveFilteringOperand.instance();
    public booleanFilteringOperand = BooleanFilteringOperand.instance();
}

export class CaseSensitiveFilteringOperand extends IgxStringFilteringOperand {
    private constructor() {
        super();
        const customOperations = [
            {
                iconName: 'contains',
                isUnary: false,
                logic: (target: string, searchVal: string, ignoreCase?: boolean) => {
                    ignoreCase = false;
                    const search = IgxStringFilteringOperand.applyIgnoreCase(searchVal, ignoreCase);
                    target = IgxStringFilteringOperand.applyIgnoreCase(target, ignoreCase);
                    return target.indexOf(search) !== -1;
                },
                name: 'Contains (case sensitive)'
            },
            {
                iconName: 'does_not_contain',
                isUnary: false,
                logic: (target: string, searchVal: string, ignoreCase?: boolean) => {
                    ignoreCase = false;
                    const search = IgxStringFilteringOperand.applyIgnoreCase(searchVal, ignoreCase);
                    target = IgxStringFilteringOperand.applyIgnoreCase(target, ignoreCase);
                    return target.indexOf(search) === -1;
                },
                name: 'Does Not Contain (case sensitive)'
            }
        ];

        const emptyOperators = [
            // 'Empty'
            this.operations[6],
            // 'Not Empty'
            this.operations[7]
        ];

        this.operations = customOperations.concat(emptyOperators);
    }
}

export class BooleanFilteringOperand extends IgxBooleanFilteringOperand {
    private constructor() {
        super();
        this.operations = [
            // 'All'
            this.operations[0],
            // 'TRUE'
            this.operations[1],
            // 'FALSE'
            this.operations[2]
        ];
    }
}
```

```typescript
                iconName: 'does_not_contain',
                isUnary: false,
                logic: (target: string, searchVal: string, ignoreCase?: boolean) => {
                    ignoreCase = false;
                    const search = IgcStringFilteringOperand.applyIgnoreCase(searchVal, ignoreCase);
                    target = IgcStringFilteringOperand.applyIgnoreCase(target, ignoreCase);
                    return target.indexOf(search) === -1;
                },
                name: 'Does Not Contain (case sensitive)'
            }
        ];

        const emptyOperators = [
            // 'Empty'
            this.operations[6],
            // 'Not Empty'
            this.operations[7]
        ];

        this.operations = customOperations.concat(emptyOperators);
    }
}

export class BooleanFilteringOperand extends IgcBooleanFilteringOperand {
    private constructor() {
        super();
        this.operations = [
            // 'All'
            this.operations[0],
            // 'TRUE'
            this.operations[1],
            // 'FALSE'
            this.operations[2]
        ];
    }
}
```

## Estilos

Para comenzar a aplicar estilos a la fila de filtrado, necesitamos importar el archivo `index`, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Siguiendo el enfoque más sencillo, creamos un nuevo tema que extiende `grid-theme` y acepta los parámetros `$filtering-row-text-color`, `$filtering-row-background`, `$filtering-header-text-color` y `$filtering-header-background`.

```scss
$custom-grid: grid-theme(
    $filtering-row-text-color: #292826,
    $filtering-row-background: #FFCD0F,
    $filtering-header-text-color: #292826,
    $filtering-header-background: #FFCD0F
);
```

Como se ve, `grid-theme` solo controla los colores de la fila de filtrado y del encabezado de columna respectivo que se está filtrando. Obviamente, tenemos muchos más componentes dentro de la fila de filtrado, como un grupo de entrada, chips, botones y otros. Para aplicarles estilos, necesitamos crear un tema separado para cada uno, así que creemos un nuevo tema de grupo de entrada y un nuevo tema de botón:

```scss
$dark-input-group: input-group-theme(
    $box-background: #FFCD0F,
    $idle-text-color: #292826,
    $focused-text-color: #292826,
    $filled-text-color: #292826
);

$dark-button: button-theme(
    $flat-background: #FFCD0F,
    $flat-text-color: #292826,
    $flat-hover-background: #292826,
    $flat-hover-text-color: #FFCD0F
);
```

En este ejemplo solo cambiamos algunos de los parámetros para el grupo de entrada y el botón, pero `input-group-theme` y `button-theme` proporcionan muchos más parámetros para controlar sus estilos respectivos.

El último paso es **incluir** los mixins de componentes, cada uno con su tema respectivo. También estableceremos la propiedad de color para el marcador de posición de la entrada.

```scss
@include grid($custom-grid);
.igx-grid__filtering-row {
    @include button($dark-button);
    @include input-group($dark-input-group);

    .igx-input-group__input::placeholder {
        color: #FFCD0F;
    }
}
```

<DocsAside type="info">

Limitamos el alcance de los mixins **igx-button** y **igx-input-group** dentro de `.igx-grid__filtering-row`, de modo que solo se apliquen estilos a los botones de la fila de filtrado y a su grupo de entrada. De lo contrario, otros botones y grupos de entrada de la cuadrícula también se verían afectados.

</DocsAside>

 <DocsAside type="info">

Si el componente utiliza una ViewEncapsulation `Emulated`, es necesario `penetrate` esta encapsulación utilizando `::ng-deep`:

 </DocsAside>

```scss
:host {
     ::ng-deep {
        @include grid($custom-grid);
        .igx-grid__filtering-row {
            @include button($dark-button);
            @include input-group($dark-input-group);

            .igx-input-group__input::placeholder {
                color: #FFCD0F;
            }
        }
    }
}
```

### Definición de una paleta de colores

En lugar de codificar de forma rígida los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en términos de colores utilizando las funciones `igx-palette` y `igx-color`.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$yellow-color: #FFCD0F;
$black-color: #292826;

$dark-palette: palette($primary: $black-color, $secondary: $yellow-color);
```

Y luego, con `igx-color`, podemos recuperar fácilmente un color de la paleta.

```scss
$custom-grid: grid-theme(
    $filtering-row-text-color: color($dark-palette, "primary", 400),
    $filtering-row-background: color($dark-palette, "secondary", 400),
    $filtering-header-text-color: color($dark-palette, "primary", 400),
    $filtering-header-background: color($dark-palette, "secondary", 400)
);

$dark-input-group: input-group-theme(
    $box-background: color($dark-palette, "secondary", 400),
    $idle-text-color: color($dark-palette, "primary", 400),
    $focused-text-color: color($dark-palette, "primary", 400),
    $filled-text-color: color($dark-palette, "primary", 400)
);

$dark-button: button-theme(
    $flat-background: color($dark-palette, "secondary", 400),
    $flat-text-color: color($dark-palette, "primary", 400),
    $flat-hover-background: color($dark-palette, "primary", 400),
    $flat-hover-text-color: color($dark-palette, "secondary", 400)
);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](../themes/sass/palettes.md) para obtener orientación detallada sobre cómo utilizarlas.

</DocsAside>

### Uso de esquemas

Avanzando más con el motor de temas, puede crear una estructura sólida y flexible que se beneficie de [Esquemas](../themes/sass/schemas.md). Un **esquema** es una receta de un tema.

Extienda uno de los dos esquemas predefinidos, que se proporcionan para cada componente; en este caso, los esquemas `light-grid`, `light-input-group` y `light-button`:

```scss
// Extending the light grid schema
$custom-grid-schema: extend($_light-grid,
    (
        filtering-row-text-color:(
           color: ("primary", 400)
        ),
        filtering-row-background:(
           color: ("secondary", 400)
        ),
        filtering-header-text-color:(
           color: ("primary", 400)
        ),
        filtering-header-background:(
           color: ("secondary", 400)
        )
    )
);

// Extending the light input group schema
$custom-input-group-schema: extend($_light-input-group,
    (
        box-background:(
           color: ("secondary", 400)
        ),
        idle-text-color:(
           color: ("primary", 400)
        ),
        focused-text-color:(
           color: ("primary", 400)
        ),
        filled-text-color:(
           color: ("primary", 400)
        )
    )
);

// Extending the light button schema
$custom-button-schema: extend($_light-button,
    (
        flat-background:(
           color: ("secondary", 400)
        ),
        flat-text-color:(
           color: ("primary", 400)
        ),
        flat-hover-background:(
           color: ("primary", 400)
        ),
        flat-hover-text-color:(
           color: ("secondary", 400)
        )
    )
);
```

Para aplicar nuestros esquemas personalizados, tenemos que **extender** uno de los globales (`light` o `dark`), que básicamente señala los componentes con un esquema personalizado, y después agregarlo a los temas de componentes respectivos:

```scss
// Extending the global light-schema
$custom-light-schema: extend($light-schema,(
    igx-grid: $custom-grid-schema,
    igx-input-group: $custom-input-group-schema,
    igx-button: $custom-button-schema
));

// Defining grid-theme with the global light schema
$custom-grid: grid-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);

// Defining button-theme with the global light schema
$custom-button: button-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);

// Defining input-group-theme with the global light schema
$custom-input-group: input-group-theme(
  $palette: $dark-palette,
  $schema: $custom-light-schema
);
```

No olvide incluir los temas de la misma manera que se demostró anteriormente.

## Limitaciones conocidas

<DocsAside type="info">

Algunos navegadores, como Firefox, no logran analizar separadores decimales específicos de una región al considerarlos separadores de agrupación, lo que provoca que no sean válidos. Al introducir dichos valores para el valor de filtro de una columna numérica, solo la parte válida del número se aplicará a la expresión de filtrado. Para obtener más información, consulte el [problema](https://bugzilla.mozilla.org/show_bug.cgi?id=1199665) de Firefox.

</DocsAside>

### Cambios incompatibles en 6.1.0

- Se elimina la propiedad <ApiLink type="{ComponentName}" /> `filteringExpressions`. Use <ApiLink type="FilteringExpressionsTree" /> en su lugar.
- Se elimina el método `filter_multiple`. Use el método <ApiLink type="Filter" /> y la propiedad <ApiLink type="FilteringExpressionsTree" /> en su lugar.
- El método <ApiLink type="Filter" /> tiene una nueva firma. Ahora acepta los siguientes parámetros:
  - <ApiLink type="Column" member="name" label="Name" />: el nombre de la columna que se filtrará.
  - `Value`: el valor que se utilizará para filtrar.
  - `ConditionOrExpressionTree` (opcional): este parámetro acepta un objeto de tipo <ApiLink type="FilteringOperation" /> o <ApiLink type="FilteringExpressionsTree" />. Si solo se necesita filtrado simple, se podría pasar una operación de filtrado como argumento. En caso de filtrado avanzado, se podría pasar como argumento un árbol de expresiones que contenga lógica de filtrado compleja.
  - <ApiLink type="FilteringExpression" member="ignoreCase" label="IgnoreCase" /> (opcional): si el filtrado distingue entre mayúsculas y minúsculas o no.
- El evento `FilteringDone` ahora tiene solo un parámetro de tipo <ApiLink type="FilteringExpressionsTree" /> que contiene el estado de filtrado de la columna filtrada.
- operandos de filtrado: la propiedad de condición <ApiLink type="FilteringExpression" /> ya no es una referencia directa a un método de condición de filtrado; en su lugar, es una referencia a un <ApiLink type="FilteringOperation" />.
- <ApiLink type="ColumnComponent" /> ahora expone una propiedad <ApiLink type="Column" member="filters" label="Filters" />, que toma una referencia de clase <ApiLink type="FilteringOperand" />.
- Se pueden proporcionar filtros personalizados a las columnas de {ComponentTitle} rellenando la propiedad <ApiLink type="BooleanFilteringOperand" member="operations" label="Operations" /> de <ApiLink type="FilteringOperand" /> con operaciones de tipo <ApiLink type="FilteringOperation" />.
