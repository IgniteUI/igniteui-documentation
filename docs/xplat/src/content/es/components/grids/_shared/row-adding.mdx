# Agregado de filas de {Platform} {ComponentTitle}

<ComponentBlock for="Grid, HierarchicalGrid">

La función de agregado de filas de {ProductName} en {Platform} {ComponentTitle} permite a los usuarios introducir y enviar nuevos registros de datos sin navegar a un formulario o página independientes. Con <ApiLink type="{ComponentName}" />, los usuarios pueden manipular datos mediante el agregado de filas en línea y una API eficaz para operaciones CRUD.
Agregue un componente <ApiLink type="ActionStrip" /> con acciones de edición habilitadas en la plantilla de la cuadrícula. Después, pase el cursor sobre una fila y use el botón proporcionado. Finalmente, presione <kbd>ALT</kbd> + <kbd>+</kbd> para generar la UI de agregado de filas.

</ComponentBlock>

<ComponentBlock for="TreeGrid">

> **Nota**:
> Establecer la clave primaria es obligatorio para las operaciones de agregado de filas.

> **Nota**:
> Todas las columnas, excepto la de clave primaria, son editables de forma predeterminada en la UI de agregado de filas. Si desea deshabilitar la edición para una columna específica, debe establecer la entrada de la columna <ApiLink type="Column" member="editable" /> en `false`.

<ComponentBlock for="Grid, HierarchicalGrid">

> **Nota**:
> La entrada <ApiLink type="GridEditingActions" /> que controla la visibilidad del botón de agregar fila puede usar el contexto de la franja de acciones (que es de tipo <ApiLink type="RowType" /> para ajustar con precisión para qué registros se muestra el botón.

</ComponentBlock>

> Las entradas <ApiLink type="GridEditingActions" /> que controlan la visibilidad de los botones de agregar fila y agregar elemento secundario pueden usar el contexto de la franja de acciones (que es de tipo <ApiLink type="RowType" /> para ajustar con precisión para qué registros se muestran los botones.

</ComponentBlock>

El `BaseTransactionService` interno se proporciona automáticamente para <ApiLink type="{ComponentName}" />. Conserva los cambios de celda pendientes hasta que se envía o se cancela el estado de la fila.

## Iniciar el agregado de filas mediante programación

<ApiLink type="{ComponentName}" /> permite generar mediante programación la UI de agregar fila usando dos métodos públicos diferentes. Uno que acepta un ID de fila para especificar la fila debajo de la cual debe generarse la UI y otro que funciona por índice. Puede usar estos métodos para generar la UI en cualquier lugar dentro de la vista de datos actual. No se admite cambiar la página ni especificar una fila que, por ejemplo, esté filtrada.

<ComponentBlock for="Grid, HierarchicalGrid">

El uso de <ApiLink type="{ComponentName}" member="beginAddRowById" /> requiere que especifique la fila que se usará como contexto para la operación mediante su `RowID` (PK). Luego, el método funciona como si el usuario final hubiera hecho clic en el botón de la franja de acciones de agregar fila para la fila especificada, generando la UI debajo de ella. También puede hacer que la UI se genere como la primera fila de la cuadrícula pasando `null` para el primer parámetro.

<PlatformBlock for="WebComponents">

```typescript
this.grid.beginAddRowById('ALFKI');  // Spawns the add row UI under the row with PK 'ALFKI'
this.grid.beginAddRowById(null);     // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="React">

```typescript
gridRef.current.beginAddRowById('ALFKI');  // Spawns the add row UI under the row with PK 'ALFKI'
gridRef.current.beginAddRowById(null);     // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
@code {
    await this.grid.BeginAddRowByIdAsync('ALFKI', false);  // Spawns the add row UI under the row with PK 'ALFKI'
    await this.grid.BeginAddRowByIdAsync(null, false);     // Spawns the add row UI as the first record
}
```

</PlatformBlock>

El método <ApiLink type="{ComponentName}" member="beginAddRowByIndex" /> funciona de forma similar, pero requiere que especifique el índice en el que debe generarse la UI. Los valores permitidos van de 0 al tamaño de la vista de datos - 1.

<PlatformBlock for="WebComponents">

```typescript
this.grid.beginAddRowByIndex(10);   // Spawns the add row UI at index 10
this.grid.beginAddRowByIndex(0);    // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="React">

```typescript
gridRef.current.beginAddRowByIndex(10);   // Spawns the add row UI at index 10
gridRef.current.beginAddRowByIndex(0);    // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
@code {
    await this.grid.BeginAddRowByIndexAsync(10);   // Spawns the add row UI at index 10
    await this.grid.BeginAddRowByIndexAsync(0);    // Spawns the add row UI as the first record
}
```

</PlatformBlock>

</ComponentBlock>

<ComponentBlock for="TreeGrid">

El uso de <ApiLink type="{ComponentName}" member="beginAddRowById" /> requiere que especifique la fila que se usará como contexto para la operación mediante su `RowID` (PK). Luego, el método funciona como si el usuario final hubiera hecho clic en el botón de la franja de acciones de agregar fila para la fila especificada, generando la UI debajo de ella. El segundo parámetro controla si la fila se agrega como elemento secundario de la fila de contexto o como elemento del mismo nivel. También puede hacer que la UI se genere como la primera fila de la cuadrícula pasando `null` para el primer parámetro.

<PlatformBlock for="WebComponents">

```typescript
this.treeGrid.beginAddRowById('ALFKI', true);   // Spawns the add row UI to add a child for the row with PK 'ALFKI'
this.treeGrid.beginAddRowById(null);            // Spawns the add row UI as the first record
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
@code {
    await this.treeGrid.BeginAddRowByIdAsync('ALFKI', true);  // Spawns the add row UI to add a child for the row with PK 'ALFKI'
    await this.treeGrid.BeginAddRowByIdAsync(null, false);     // Spawns the add row UI as the first record
}
```

</PlatformBlock>

El método <ApiLink type="{ComponentName}" member="beginAddRowByIndex" /> funciona de forma similar, pero la fila que se usará como contexto se especifica por índice.

## Posicionamiento

- La posición predeterminada de la UI de agregar fila es debajo de la fila para la que el usuario final hizo clic en el botón de agregar fila.

- <ApiLink type="{ComponentName}" /> se desplaza para mostrar completamente la UI de agregar fila automáticamente.

- La superposición de la UI de agregar fila mantiene su posición durante el desplazamiento.

## Comportamiento

La UI de agregar fila tiene el mismo comportamiento que la de edición de filas, ya que están diseñadas para proporcionar una experiencia de edición coherente a los usuarios finales. Consulte el tema [Edición de filas de {ComponentTitle}](row-editing.md) para obtener más información.

Después de agregar una nueva fila mediante la UI de agregado de filas, su posición y/o visibilidad se determina por el estado de ordenación, filtrado y agrupación de <ApiLink type="{ComponentName}" />. En un <ApiLink type="{ComponentName}" /> que no tenga ninguno de estos estados aplicado, aparece como el último registro. Se muestra brevemente una snackbar que contiene un botón que el usuario final puede usar para desplazar <ApiLink type="{ComponentName}" /> hasta su posición si no está visible.

## Navegación con el teclado

- <kbd>ALT</kbd> + <kbd>+</kbd> - Entra en modo de edición para agregar una fila

<ComponentBlock for="TreeGrid">

- <kbd>ALT</kbd> + <kbd>MAYÚS</kbd> + <kbd>+</kbd> - Entra en modo de edición para agregar un elemento secundario

</ComponentBlock>

- <kbd>ESC</kbd> sale del modo de agregado de filas sin enviar ningún cambio

- <kbd>PESTAÑA</kbd> mueve el foco de una celda editable de la fila a la siguiente y de la celda editable más a la derecha a los botones CANCEL y DONE. La navegación desde el botón DONE va a la celda editable más a la izquierda dentro de la fila que se está editando actualmente.

## Integración de funciones

- Cualquier operación de agregado de filas se detendrá si se modifica la vista de datos de <ApiLink type="{ComponentName}" />. Se envían todos los cambios realizados por el usuario final. Las operaciones que cambian la vista de datos incluyen, entre otras, la ordenación, la agrupación, el filtrado, la paginación, etc.

- Los resúmenes se actualizan después de que finaliza la operación de agregar fila. Lo mismo es válido para las demás funciones dependientes de la vista de datos, como la ordenación, el filtrado, etc.

- Al generar la UI para <ApiLink type="{ComponentName}" />, cualquier diseño secundario actualmente expandido para una fila para la que el usuario final haga clic en el botón de agregar fila se contrae.

## Escenarios remotos

En la mayoría de los escenarios de datos remotos, la asignación de la clave primaria se produce en la solicitud de creación del servidor. En este caso, los registros agregados en el cliente no tendrán el valor final de la clave primaria hasta que se guarden en la base de datos del servidor. En ese caso, la forma recomendada de manejar esta actualización en <ApiLink type="{ComponentName}" /> es la siguiente:

- Si <ApiLink type="{ComponentName}" /> no usa transacciones.

  Una vez que la solicitud de creación se complete correctamente y devuelva los datos del registro agregado, puede reemplazar el id de ese registro en la instancia local del registro de datos.

- Si <ApiLink type="{ComponentName}" /> usa transacciones.

  Una vez que la solicitud de creación o la solicitud de actualización por lotes se complete correctamente y devuelva las instancias de registros agregadas (con sus ids generados por la base de datos), las transacciones ADD relacionadas deben eliminarse del registro de transacciones usando el método API `Clear`. Esto es necesario porque la transacción local tendrá un campo id generado, que puede diferir del creado en la base de datos, por lo que deben eliminarse. Luego puede agregar los registros pasados en la respuesta a la instancia de datos local.

Esto garantizará que los ids generados de forma remota siempre se reflejen en los datos locales, y que las operaciones posteriores de actualización/eliminación apunten a los ids de registro correctos.

</PlatformBlock>
