# Resúmenes de {Platform} {ComponentTitle}

La característica de resúmenes de {ProductName} en {Platform} {ComponentTitle} funciona a nivel de columna como pie de grupo. Los resúmenes de {Platform} {ComponentName} son una característica eficaz que permite al usuario ver información de la columna en un contenedor independiente con un conjunto predefinido de elementos de resumen predeterminados, según el tipo de datos dentro de la columna o mediante la implementación de una plantilla personalizada en <ApiLink type="{ComponentName}" />.

## Ejemplo de información general de resúmenes de {Platform} {ComponentTitle}

<Sample src="/{ComponentSample}/data-summary-options" height={650} alt="{Platform} {ComponentTitle} data summary options" />

<DocsAside type="info">

El resumen de la columna es una **función de todos los valores de la columna**, a menos que se aplique filtrado; en ese caso, el resumen de la columna será una **función de los valores del resultado filtrado**

</DocsAside>

Los resúmenes de <ApiLink type="{ComponentName}" /> también se pueden habilitar a nivel de columna en {ProductName}, lo que significa que puede activarlos solo para las columnas que necesite. Los resúmenes de <ApiLink type="{ComponentName}" /> le ofrecen un conjunto predefinido de resúmenes predeterminados, según el tipo de datos de la columna, para que pueda ahorrar algo de tiempo:

Para `string` y `boolean` <ApiLink type="Column" member="dataType" />, está disponible la siguiente función:

- Recuento

Para los tipos de datos `number`, `currency` y `percent`, están disponibles las siguientes funciones:

- Recuento
- Mín.
- Máx.
- Promedio
- Suma

Para el tipo de datos `date`, están disponibles las siguientes funciones:

- Recuento
- Más antiguo
- Más reciente

Todos los tipos de datos de columna disponibles se pueden encontrar en el [tema Tipos de columna](column-types.md#default-template).

Los resúmenes de <ApiLink type="{ComponentName}" /> se habilitan por columna estableciendo la propiedad <ApiLink type="ColumnState" member="hasSummary" /> en **true**. También es importante tener en cuenta que los resúmenes de cada columna se resuelven según el tipo de datos de la columna. En <ApiLink type="{ComponentName}" />, el tipo de datos de columna predeterminado es `string`, por lo que si desea resúmenes específicos de `number` o `date`, debe especificar la propiedad <ApiLink type="Column" member="dataType" /> como `number` o `date`. Tenga en cuenta que los valores de resumen se mostrarán localizados, según <ApiLink type="GridBaseDirective" member="locale" label="Locale" /> de la cuadrícula y <ApiLink type="Column" member="pipeArgs" /> de la columna.

## Resúmenes personalizados de {ComponentTitle}

Si estas funciones no satisfacen sus requisitos, puede proporcionar un resumen personalizado para las columnas específicas.

<PlatformBlock for="WebComponents">

Para lograrlo, debe invalidar una de las clases base <ApiLink type="SummaryOperand" />, <ApiLink type="NumberSummaryOperand" /> o <ApiLink type="DateSummaryOperand" /> según el tipo de datos de la columna y sus necesidades. De esta forma, puede redefinir la función existente o agregar nuevas funciones. La clase <ApiLink type="SummaryOperand" /> proporciona la implementación predeterminada solo para el método <ApiLink type="SummaryOperand" member="count" label="Count" />. <ApiLink type="NumberSummaryOperand" /> extiende <ApiLink type="SummaryOperand" /> y proporciona implementación para <ApiLink type="NumberSummaryOperand" member="min" label="Min" />, <ApiLink type="NumberSummaryOperand" member="max" label="Max" />, <ApiLink type="NumberSummaryOperand" member="sum" label="Sum" /> y <ApiLink type="NumberSummaryOperand" member="average" label="Average" />. <ApiLink type="DateSummaryOperand" /> extiende <ApiLink type="SummaryOperand" /> y además le proporciona <ApiLink type="DateSummaryOperand" member="earliest" label="Earliest" /> y <ApiLink type="DateSummaryOperand" member="latest" label="Latest" />.

</PlatformBlock>

<PlatformBlock for="Angular">

```typescript
import { IgxSummaryResult, IgxSummaryOperand, IgxNumberSummaryOperand, IgxDateSummaryOperand } from 'igniteui-angular';

class MySummary extends IgxNumberSummaryOperand {
    constructor() {
        super();
    }

    operate(data?: any[]): IgxSummaryResult[] {
        const result = super.operate(data);
        result.push({
            key: 'test',
            label: 'Test',
            summaryResult: data.filter(rec => rec > 10 && rec < 30).length
        });
        return result;
    }
}

```

</PlatformBlock>

Ahora puede acceder a todos los datos de {ComponentTitle} dentro del resumen de columna personalizado. ## Formato de resúmenes

De forma predeterminada, los resultados de resumen producidos por los operandos de resumen integrados se localizan y formatean según <ApiLink type="GridBaseDirective" member="locale" label="Locale" /> de la cuadrícula y <ApiLink type="Column" member="pipeArgs" /> de la columna. Al usar operandos personalizados, <ApiLink type="GridBaseDirective" member="locale" label="Locale" /> y <ApiLink type="Column" member="pipeArgs" /> no se aplican. Si desea cambiar la apariencia predeterminada de los resultados de resumen, puede aplicarles formato mediante la propiedad <ApiLink type="Column" member="summaryFormatter" label="SummaryFormatter" />.

```typescript
public dateSummaryFormat(summary: IgxSummaryResult, summaryOperand: IgxSummaryOperand): string {
    const result = summary.summaryResult;
    if (summaryOperand instanceof IgxDateSummaryOperand && summary.key !== 'count'
        && result !== null && result !== undefined) {
        const pipe = new DatePipe('en-US');
        return pipe.transform(result,'MMM YYYY');
    }
    return result;
}
```

## Resúmenes con Agrupar por

Cuando ha agrupado por columnas, <ApiLink type="{ComponentName}" /> le permite cambiar la posición del resumen y el modo de cálculo mediante las propiedades <ApiLink type="{ComponentName}" member="summaryCalculationMode" label="SummaryCalculationMode" /> y <ApiLink type="{ComponentName}" member="summaryPosition" label="SummaryPosition" />. Junto con estas dos propiedades, <ApiLink type="{ComponentName}" /> expone una propiedad <ApiLink type="{ComponentName}" member="showSummaryOnCollapse" label="ShowSummaryOnCollapse" /> que le permite determinar si la fila de resumen permanece visible cuando la fila de grupo a la que hace referencia está contraída.

Los valores disponibles de la propiedad <ApiLink type="{ComponentName}" member="summaryCalculationMode" label="SummaryCalculationMode" /> son:

- <ApiLink type="GridSummaryCalculationMode" label="RootLevelOnly" />: los resúmenes se calculan solo para el nivel raíz.
- <ApiLink type="GridSummaryCalculationMode" label="ChildLevelsOnly" />: los resúmenes se calculan solo para los niveles secundarios.
- <ApiLink type="GridSummaryCalculationMode" label="RootAndChildLevels" />: los resúmenes se calculan tanto para el nivel raíz como para los niveles secundarios. Este es el valor predeterminado.

Los valores disponibles de la propiedad <ApiLink type="{ComponentName}" member="summaryPosition" label="SummaryPosition" /> son:

- <ApiLink type="GridSummaryPosition" label="Top" />: la fila de resumen aparece antes de los elementos secundarios de la fila de agrupación.
- <ApiLink type="GridSummaryPosition" label="Bottom" />: la fila de resumen aparece después de los elementos secundarios de la fila de agrupación. Este es el valor predeterminado.

La propiedad <ApiLink type="{ComponentName}" member="showSummaryOnCollapse" label="ShowSummaryOnCollapse" /> es booleana. Su valor predeterminado está establecido en **false**, lo que significa que la fila de resumen se ocultaría cuando la fila de grupo está contraída. Si la propiedad se establece en **true**, la fila de resumen permanece visible cuando la fila de grupo está contraída.

<DocsAside type="info">

La propiedad <ApiLink type="{ComponentName}" member="summaryPosition" label="SummaryPosition" /> se aplica solo a los resúmenes de nivel secundario. Los resúmenes de nivel raíz siempre aparecen fijos en la parte inferior de <ApiLink type="{ComponentName}" />.

</DocsAside>

## Estilos

Para comenzar a aplicar estilos al comportamiento de ordenación, debemos importar el archivo `index`, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Siguiendo el enfoque más sencillo, creamos un nuevo tema que extiende <ApiLink type="function-grid-summary-theme" label="grid-summary-theme" kind="sass" module="index.html" /> y acepta los parámetros `$background-color`, `$focus-background-color`, `$label-color`, `$result-color`, `$pinned-border-width`, `$pinned-border-style` y `$pinned-border-color`.

```scss
$custom-theme: grid-summary-theme(
    $background-color: #e0f3ff,
    $focus-background-color: rgba( #94d1f7, .3 ),
    $label-color: #e41c77,
    $result-color: black,
    $pinned-border-width: 2px,
    $pinned-border-style: dotted,
    $pinned-border-color: #e41c77
);
```

El último paso es **incluir** los mixins de componentes:

```scss
@include grid-summary($custom-theme);
```

<DocsAside type="info">

Si el componente usa una ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario `penetrate` esta encapsulación mediante `::ng-deep`:

</DocsAside>

```scss
:host {
   ::ng-deep {
       @include grid-summary($custom-theme);
   }
}
```

### Definición de una paleta de colores

En lugar de codificar de forma rígida los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en términos de colores usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$blue-color: #7793b1;
$green-color: #00ff2d;

$my-custom-palette: palette($primary: $blue-color, $secondary: $green-color);
```

Y luego, con <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />, podemos recuperar fácilmente el color de la paleta.

```scss
$custom-theme: grid-summary-theme(
    $background-color: color($my-custom-palette, "primary", 700),
    $focus-background-color: color($my-custom-palette, "primary", 800),
    $label-color: color($my-custom-palette, "secondary", 500),
    $result-color: color($my-custom-palette, "grays", 900),
    $pinned-border-width: 2px,
    $pinned-border-style: dotted,
    $pinned-border-color: color($my-custom-palette, "secondary", 500)
);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones eficaces para generar y recuperar colores. Consulte el tema [Paletas](../themes/palettes.md) para obtener orientación detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Al avanzar con el motor de temas, puede crear una estructura robusta y flexible que se beneficie de los [**esquemas**](../themes/sass/schemas.md). Un **esquema** es una receta de un tema.

Extienda uno de los dos esquemas predefinidos, que se proporcionan para cada componente, en este caso: <ApiLink type="variable-_light-grid-summary" label="_light-grid-summary" kind="sass" module="index.html" />:

```scss
// Extending the light grid summary schema
$my-summary-schema: extend($_light-grid-summary,
    (
        background-color: (igx-color: ('primary', 700)),
        focus-background-color: (igx-color: ('primary', 800)),
        label-color: (igx-color: ('secondary', 500)),
        result-color: (igx-color: ('grays', 900)),
        pinned-border-width: 2px,
        pinned-border-style: dotted,
        pinned-border-color: (igx-color: ('secondary', 500))
    )
);
```

Para aplicar nuestro esquema personalizado, debemos **extender** uno de los globales (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> o <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />), que básicamente indica los componentes con un esquema personalizado, y después agregarlo a los temas de los componentes correspondientes:

```scss
// Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid-summary: $my-summary-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: grid-summary-theme(
    $palette: $my-custom-palette,
    $schema: $my-custom-schema
);
```

No olvide incluir los temas de la misma forma que se demostró anteriormente.
