# Ordenación de {Platform} {ComponentTitle}

La característica de ordenación de datos de {ProductName} en {Platform} {ComponentTitle} se habilita a nivel de columna, lo que significa que <ApiLink type="{ComponentName}" /> puede tener una combinación de columnas ordenables y no ordenables. Realizar acciones de ordenación de {Platform} le permite cambiar el orden de visualización de los registros según criterios especificados.

<PlatformBlock for="Angular">

<DocsAside type="info">

Hasta ahora, la agrupación y la ordenación funcionaban de forma conjunta. En la versión 13.2, se introduce un nuevo comportamiento que desacopla la agrupación de la ordenación. Por ejemplo, borrar la agrupación no borrará las expresiones de ordenación en la cuadrícula ni viceversa. Aun así, si una columna está tanto ordenada como agrupada, las expresiones agrupadas tienen prioridad.

</DocsAside>

</PlatformBlock>

## Ordenación mediante la API

Puede ordenar cualquier columna o una combinación de columnas mediante la API de <ApiLink type="{ComponentName}" /> usando el método <ApiLink type="{ComponentName}" member="sort" label="Sort" />:


</PlatformBlock>

</PlatformBlock>

<PlatformBlock for="Angular">

```typescript

// Perform sorting on both the ProductName and Price columns.

    { fieldName: 'ProductName', dir: SortingDirection.Asc, ignoreCase: true },
```

## Estilos

Para comenzar a aplicar estilos al comportamiento de ordenación, necesitamos importar el archivo `index`, donde se encuentran todas las funciones de tema y mixins de componentes:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Siguiendo el enfoque más sencillo, creamos un nuevo tema que extiende <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> y acepta los parámetros `$sorted-header-icon-color` y `sortable-header-icon-hover-color`.

```scss
$custom-theme: grid-theme(
    $sorted-header-icon-color: #ffb06a,
    $sortable-header-icon-hover-color: black
);
```

El último paso es **incluir** los mixins de componentes:

```scss
 @include grid($custom-theme);
```

<DocsAside type="info">

Si el componente usa una ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario aplicar `penetrate` a esta encapsulación usando `::ng-deep`:

</DocsAside>

```scss
:host {
   ::ng-deep {
       @include grid($custom-theme);
   }
}
```

### Definición de una paleta de colores

En lugar de codificar directamente los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en cuanto a colores usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$black-color: black;
$orange-color: #ffb06a;

$custom-palette: palette($primary: $black-color, $secondary: $orange-color);
```

Y luego con <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" /> podemos recuperar fácilmente un color de la paleta.

```scss
$custom-theme: grid-theme(
    $sorted-header-icon-color: color($custom-palette, "secondary", 500),
    $sortable-header-icon-hover-color: color($custom-palette, "primary", 500)
);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](../themes/sass/palettes.md) para obtener orientación detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Yendo más allá con el motor de temas, puede construir una estructura robusta y flexible que se beneficie de los [**esquemas**](../themes/sass/schemas.md). Un **esquema** es una receta de un tema.

Extienda uno de los dos esquemas predefinidos que se proporcionan para cada componente, en este caso: <ApiLink type="variable-_light-grid" label="_light-grid" kind="sass" module="index.html" />:

```scss
// Extending the light grid schema
$custom-grid-schema: extend($_light-grid,
    (
        sorted-header-icon-color: (igx-color:('secondary', 500)),
        sortable-header-icon-hover-color: (igx-color:('primary', 500))
    )
);
```

Para aplicar nuestro esquema personalizado, debemos **extender** uno de los globales (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> o <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />), lo que básicamente indica los componentes con un esquema personalizado, y después agregarlo a los temas de los componentes correspondientes:

```scss
// Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid: $custom-grid-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: grid-theme(
  $palette: $custom-palette,
  $schema: $my-custom-schema
);
```

No olvide incluir los temas de la misma manera que se demostró anteriormente.
