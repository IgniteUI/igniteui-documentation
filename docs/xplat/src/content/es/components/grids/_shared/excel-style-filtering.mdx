# Filtrado de Excel en {Platform} {ComponentTitle}

El {Platform} {ComponentTitle} expone una característica de filtrado de estilo Excel que proporciona una UI de filtrado similar a Excel. Simplifica el proceso de trabajar con grandes conjuntos de datos. La idea principal es ayudarles a filtrar los datos más relevantes, a la vez que se eliminan las entradas irrelevantes.

## Ejemplo de filtrado de estilo Excel de {Platform} {ComponentTitle}

## Uso

Para activar el filtrado de estilo Excel del componente <ApiLink type="{ComponentName}" />, se deben establecer dos entradas. <ApiLink type="{ComponentName}" member="allowFiltering" /> debe establecerse en **true** y <ApiLink type="{ComponentName}" member="filterMode" /> debe establecerse en el valor `ExcelStyleFilter`.

## Interacciones

Para abrir el menú de filtro de una columna concreta, se debe hacer clic en el icono de filtro {Platform} del encabezado. Además, puede usar la combinación <kbd>CTRL</kbd> + <kbd>MAYÚS</kbd> + <kbd>L</kbd> en un encabezado seleccionado. Si la columna se puede ordenar, anclar, mover, seleccionar u ocultar junto con la funcionalidad de filtrado, habrá botones disponibles para las características que estén activadas.

Si no se aplica ningún filtro, todos los elementos de la lista estarán seleccionados. Se pueden filtrar desde la entrada situada encima de la lista. Para filtrar los datos, puede seleccionar/anular la selección de los elementos de la lista y hacer clic en el botón Aplicar, o presionar <kbd>ENTRAR</kbd>. El filtrado aplicado mediante los elementos de la lista crea expresiones de filtro con el operador `equals` y el operador lógico entre las expresiones es `OR`.

Si escribe algo en el cuadro de búsqueda y aplica el filtro, solo se seleccionarán los elementos que coincidan con los criterios de búsqueda. Sin embargo, si desea agregar elementos a los filtrados actualmente, debe seleccionar la opción **Agregar selección actual al filtro**.

Si desea borrar el filtro, puede marcar la opción **Seleccionar todo** y luego hacer clic en el botón Aplicar.

Para aplicar un filtro con diferentes expresiones, puede hacer clic en el **Filtro de texto**, lo que abrirá un submenú con todos los operadores de filtro disponibles para la columna concreta. Al seleccionar uno de ellos, se abrirá el cuadro de diálogo de filtro personalizado, donde puede agregar tantas expresiones como desee con diferentes operadores de filtro y lógicos. También hay un botón de borrado, que puede borrar el filtro.

## Plantillas

<PlatformBlock for="Angular">

Si desea personalizar aún más el menú de filtro de estilo Excel sin deshabilitar las características de columna, podría usar plantillas personalizadas. El menú de filtro de estilo Excel proporciona dos directivas para plantillas:

- `IgxExcelStyleColumnOperationsTemplateDirective` - vuelve a crear la plantilla del área con todas las operaciones de columna, como ordenación, anclaje, etc.
- `IgxExcelStyleFilterOperationsTemplateDirective` - vuelve a crear la plantilla del área con todas las operaciones específicas de filtro.

Puede volver a crear la plantilla solo de una de esas áreas o de ambas. Puede colocar cualquier contenido personalizado dentro de esas directivas o puede usar cualquiera de nuestros componentes de filtrado de estilo Excel integrados.

El código siguiente muestra cómo definir un menú de filtro de estilo Excel personalizado usando los componentes `excel-style-header`, `excel-style-sorting` y `excel-style-search`.

</PlatformBlock>

<PlatformBlock for="WebComponents, Blazor, React">

Si desea personalizar aún más el menú de filtro de estilo Excel, puede usar la propiedad <ApiLink type="{ComponentName}" member="excelStyleHeaderIconTemplate" /> para definir una plantilla personalizada para el icono del encabezado del menú.

El código siguiente muestra cómo personalizar el menú de filtro de estilo Excel usando <ApiLink type="{ComponentName}" member="excelStyleHeaderIconTemplate" />:

```javascript
        return html`<img height="15px" width="15px" src="http://static.infragistics.com/xplatform/images/grid/propeller-logo.svg" title="Continued" alt="Continued" />`
```

## Salida externa

El [z-index](https://developer.mozilla.org/en-US/docs/Web/CSS/z-index) de {ComponentTitle} crea un contexto de apilamiento separado para cada cuadrícula en el DOM. Esto garantiza que todos los elementos descendientes de la cuadrícula se representen según lo previsto, sin superponerse entre sí.
Sin embargo, los elementos que salen de la cuadrícula (por ejemplo, el filtro de estilo Excel) entrarán en conflicto con elementos externos con el mismo `z-index` (por ejemplo, tener dos cuadrículas una debajo de otra), lo que dará como resultado una representación incorrecta. La solución para este problema es establecer la propiedad `Outlet` en una directiva de salida externa que permita que los elementos superpuestos aparezcan siempre en la parte superior.

## Estilos

Para comenzar a aplicar estilos al cuadro de diálogo de filtrado de estilo Excel, necesitamos importar el archivo `index`, donde se encuentran todas las funciones de tema y los mixins de componentes:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

El cuadro de diálogo de filtrado de estilo Excel toma su color de fondo del tema de la cuadrícula, usando el parámetro `filtering-row-background`. Para cambiar el fondo, necesitamos crear un tema personalizado:

```scss
$custom-grid: grid-theme(
    $filtering-row-background: #FFCD0F
);
```

Obviamente, tenemos muchos más componentes dentro del cuadro de diálogo de filtrado similar a Excel, como botones, casillas de verificación, una lista e incluso un desplegable. Para aplicarles estilos, necesitamos crear un tema separado para cada uno:

```scss
$dark-button: button-theme(
    $background: #FFCD0F,
    $foreground: #292826,
    $hover-background: #292826,
    $hover-foreground: #FFCD0F
);

$dark-input-group: input-group-theme(
    $box-background: #FFCD0F,
    $idle-text-color: #292826,
    $focused-text-color: #292826,
    $filled-text-color: #292826
);

$custom-list: list-theme(
    $background: #FFCD0F
);

$custom-checkbox: checkbox-theme(
    $empty-color: #292826,
    $fill-color: #292826,
    $tick-color: #FFCD0F,
    $label-color: #292826
);

$custom-drop-down: drop-down-theme(
    $background-color: #FFCD0F,
    $item-text-color: #292826,
    $hover-item-background: #292826,
    $hover-item-text-color: #FFCD0F
);
```

En este ejemplo solo cambiamos algunos de los parámetros de los componentes enumerados, pero los temas <ApiLink type="function-button-theme" label="button-theme" kind="sass" module="index.html" />, <ApiLink type="function-checkbox-theme" label="checkbox-theme" kind="sass" module="index.html" />, <ApiLink type="function-drop-down-theme" label="drop-down-theme" kind="sass" module="index.html" />, <ApiLink type="function-input-group-theme" label="input-group-theme" kind="sass" module="index.html" />, <ApiLink type="function-list-theme" label="list-theme" kind="sass" module="index.html" /> proporcionan muchos más parámetros para controlar sus estilos respectivos.

El último paso es **incluir** los mixins de componentes, cada uno con su tema respectivo. También estableceremos la propiedad de color para el marcador de posición de la entrada.

```scss
@include drop-down($custom-drop-down);
@include grid($custom-grid);
.igx-excel-filter, .igx-excel-filter__secondary {
    @include button($dark-button);
    @include input-group($dark-input-group);
    @include list($custom-list);
    @include checkbox($custom-checkbox);
    .igx-input-group__input::placeholder {
        color: #FFCD0F;
    }
}
```

<DocsAside type="info">

Limitamos el alcance de la mayoría de los mixins de los componentes dentro de `.igx-excel-filter` y `.igx-excel-filter__secondary`, de modo que estos temas personalizados afecten solo a los componentes anidados en el cuadro de diálogo de filtrado de estilo Excel y todos sus subcuadros de diálogo. De lo contrario, otros botones, casillas de verificación, grupos de entrada y listas también se verían afectados.

</DocsAside>

<DocsAside type="info">

Si el componente usa una ViewEncapsulation [Emulated](../themes/styles.md#view-encapsulation), es necesario `penetrate` esta encapsulación usando `::ng-deep`:

</DocsAside>

```scss
:host {
    ::ng-deep {
        @include drop-down($custom-drop-down);
        @include grid($custom-grid);
        .igx-excel-filter, .igx-excel-filter__secondary {
            @include button($dark-button);
            @include input-group($dark-input-group);
            @include list($custom-list);
            @include checkbox($custom-checkbox);
            .igx-input-group__input::placeholder {
                color: #FFCD0F;
            }
        }
    }
}
```

### Definir una paleta de colores

En lugar de codificar de forma rígida los valores de color como acabamos de hacer, podemos lograr una mayor flexibilidad en cuanto a colores usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.

`igx-palette` genera una paleta de colores basada en los colores primario y secundario que se pasan:

```scss
$yellow-color: #FFCD0F;
$black-color: #292826;

$dark-palette: palette($primary: $black-color, $secondary: $yellow-color);
```

Y luego, con <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />, podemos recuperar fácilmente un color de la paleta.

```scss
$custom-grid: grid-theme(
    $filtering-row-background: #FFCD0F
);

$dark-button: button-theme(
    $background: color($dark-palette, "secondary", 400),
    $foreground: color($dark-palette, "primary", 400),
    $hover-background: color($dark-palette, "primary", 400),
    $hover-foreground: color($dark-palette, "secondary", 400)
);

$dark-input-group: input-group-theme(
    $box-background: color($dark-palette, "secondary", 400),
    $idle-text-color: color($dark-palette, "primary", 400),
    $focused-text-color: color($dark-palette, "primary", 400),
    $filled-text-color: color($dark-palette, "primary", 400)
);

$custom-list: list-theme(
    $background: color($dark-palette, "secondary", 400)
);

$custom-checkbox: checkbox-theme(
    $empty-color: color($dark-palette, "primary", 400),
    $fill-color: color($dark-palette, "primary", 400),
    $tick-color: color($dark-palette, "secondary", 400),
    $label-color: color($dark-palette, "primary", 400)
);

$custom-drop-down:drop-down-theme(
    $background-color: color($dark-palette, "secondary", 400),
    $item-text-color: color($dark-palette, "primary", 400),
    $hover-item-background: color($dark-palette, "primary", 400),
    $hover-item-text-color: color($dark-palette, "secondary", 400)
);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](../themes/sass/palettes.md) para obtener una guía detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Avanzando más con el motor de temas, puede crear una estructura sólida y flexible que se beneficie de los [**esquemas**](../themes/sass/schemas.md). Un **esquema** es una receta de un tema.

Extienda uno de los dos esquemas predefinidos, que se proporcionan para cada componente, en este caso: los esquemas <ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" />, <ApiLink type="variable-_light-input-group" label="light-input-group" kind="sass" module="index.html" />, <ApiLink type="variable-_light-button" label="light-button" kind="sass" module="index.html" />, <ApiLink type="variable-_light-list" label="light-list" kind="sass" module="index.html" />, <ApiLink type="variable-_light-checkbox" label="light-checkbox" kind="sass" module="index.html" /> y <ApiLink type="variable-_light-drop-down" label="light-drop-down" kind="sass" module="index.html" />:

```scss
$custom-grid-schema: extend($_light-grid,
    (
        filtering-row-background:(
           color: ("secondary", 400)
        )
    )
);

$custom-button-schema: extend($_light-button,
    (
        flat-background:(
           color: ("secondary", 400)
        ),
        flat-text-color:(
           color: ("primary", 400)
        ),
        flat-hover-background:(
           color: ("primary", 400)
        ),
        flat-hover-text-color:(
           color: ("secondary", 400)
        ),

        raised-background:(
           color: ("secondary", 400)
        ),
        raised-text-color:(
           color: ("primary", 400)
        ),
        raised-hover-background:(
           color: ("primary", 400)
        ),
        raised-hover-text-color:(
           color: ("secondary", 400)
        )
    )
);

$custom-input-group-schema: extend($_light-input-group,
    (
        box-background:(
           color: ("secondary", 400)
        ),
        idle-text-color:(
           color: ("primary", 400)
        ),

           color: ("primary", 400)

$custom-list-schema: extend($_light-list,
```

Para aplicar nuestros esquemas personalizados, tenemos que **extender** uno de los globales (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> o <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />), lo que básicamente señala los componentes con un esquema personalizado, y después agregarlo a los temas de componentes correspondientes:

```scss
$custom-light-schema: extend($light-schema,(
   grid: $custom-grid-schema,
   button: $custom-button-schema,
   input-group: $custom-input-group-schema,
   list: $custom-list-schema,
   checkbox: $custom-checkbox-schema,
   drop-down: $custom-drop-down-schema
));

$custom-grid: grid-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-button: button-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-input-group: input-group-theme(
    $palette: $dark-palette,
    $schema: $custom-light-schema
);

$custom-list: list-theme(
    $palette: $dark-palette,

);

    $palette: $dark-palette,
    $schema: $custom-light-schema
```
