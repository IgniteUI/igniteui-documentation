# {Platform}  {ComponentTitle} Información general sobre el cambio de tamaño de columnas

La característica de cambio de tamaño de columnas de {ProductName} en {Platform} {ComponentTitle} permite a los usuarios ajustar fácilmente el ancho de las columnas de <ApiLink type="{ComponentName}" />. De forma predeterminada, verán un indicador temporal de cambio de tamaño mientras esté en curso la operación de cambio de tamaño mediante arrastre. Hay varias opciones de cambio de tamaño disponibles: cambiar el tamaño de columnas en píxeles/porcentajes, restringir el cambio de tamaño de columnas, ajustar automáticamente el tamaño de las columnas con doble clic y ajustar automáticamente el tamaño de las columnas durante la inicialización.

Todos estos escenarios son compatibles con la característica **Cambio de tamaño de columnas**. De forma predeterminada, si una columna no tiene establecido un ancho, se ajusta al espacio disponible con el ancho establecido en píxeles.

<DocsAside type="info">

Hay una ligera diferencia en la forma en que funciona el cambio de tamaño para las columnas establecidas en píxeles y porcentajes.

</DocsAside>

**Píxeles**

El cambio de tamaño de columnas con ancho en píxeles funciona sumando o restando directamente la cantidad horizontal del movimiento del ratón al tamaño de la columna.

**Porcentajes**

Al cambiar el tamaño de columnas con ancho en porcentajes, la cantidad horizontal del movimiento del ratón en píxeles se traduce aproximadamente a su cantidad porcentual relativa al ancho de la cuadrícula. Las columnas siguen siendo responsivas y cualquier cambio de tamaño futuro de la cuadrícula también se reflejará en las columnas.

## Restringir el cambio de tamaño de columnas

## Ajuste automático del tamaño de columnas con doble clic

Cada columna puede **ajustarse automáticamente** haciendo doble clic en el lado derecho del encabezado: el tamaño de la columna se ajustará al valor de celda visible actualmente más largo, incluido el propio encabezado. Este comportamiento está habilitado de forma predeterminada; no se necesita configuración adicional. Sin embargo, la columna no se ajustará automáticamente en caso de que <ApiLink type="ColumnState" member="maxWidth" /> esté establecido en esa columna y el nuevo ancho supere ese valor <ApiLink type="ColumnState" member="maxWidth" />. En este caso, el tamaño de la columna se ajustará según el valor preestablecido de <ApiLink type="ColumnState" member="maxWidth" />.

También puede ajustar automáticamente el tamaño de una columna de forma dinámica mediante el método expuesto <ApiLink type="Column" member="autosize" /> en <ApiLink type="Column" />.

<ComponentBlock for="Grid, TreeGrid">


</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbColumn Width="auto"></IgbColumn>
```

Cuando la columna se inicializa por primera vez en la vista, resuelve su ancho al tamaño de la celda visible más larga o del encabezado. Tenga en cuenta que las celdas que están fuera de las filas visibles no se incluyen.

Este enfoque está más optimizado para el rendimiento que el ajuste automático posterior a la inicialización y se recomienda especialmente en los casos en los que necesita ajustar automáticamente el tamaño de un gran número de columnas.

El enfoque más sencillo para lograr esto es crear un nuevo tema que extienda <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> y acepte muchos parámetros, así como el parámetro `$resize-line-color`.

```scss
$custom-grid-theme: grid-theme(
```


### Definir una paleta de colores

En lugar de codificar de forma rígida los valores de color, podemos lograr una mayor flexibilidad en términos de colores mediante el uso de las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.

`igx-palette` genera una paleta de colores basada en el color primario y secundario especificado:

```scss
$primary-color: #0288D1;
$secondary-color: #BDBDBD;

$custom-theme-palette: palette($primary: $primary-color, $secondary: $secondary-color);
```

Y luego, con <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />, podemos recuperar fácilmente el color de la paleta.

```scss
$custom-grid-theme: grid-theme(
    $palette: $custom-theme-palette,
    $resize-line-color: color($custom-theme-palette, 'secondary', 500)
);
```

<DocsAside type="info">

`igx-color` y `igx-palette` son funciones potentes para generar y recuperar colores. Consulte el tema [Paletas](../themes/sass/palettes.md) para obtener orientación detallada sobre cómo usarlas.

</DocsAside>

### Uso de esquemas

Yendo más allá con el motor de temas, puede crear una estructura sólida y flexible que se beneficie de los [**esquemas**](../themes/sass/schemas.md). Un **esquema** es una receta de un tema.

Extienda el esquema predefinido proporcionado para cada componente; en este caso, el esquema <ApiLink type="variable-_light-grid" label="light-grid" kind="sass" module="index.html" />:

```scss
// Extending the light grid schema
$light-grid-schema: extend($_light-grid,
    (
        resize-line-color: (
           color: ('secondary', 500)
            ),
        header-background: (
           color: ("primary", 100)
            ),
        header-text-color: (
           color: ("primary", 600)
            )
    )
);
```

Para aplicar nuestro esquema personalizado, debemos **extender** uno de los globales (<ApiLink type="variable-light-schema" label="light" kind="sass" module="index.html" /> o <ApiLink type="variable-dark-schema" label="dark" kind="sass" module="index.html" />), lo que básicamente indica los componentes con un esquema personalizado, y después agregarlo al tema del componente correspondiente:

```scss
// Extending the global light-schema
$custom-light-grid-schema: extend($light-schema,(
    igx-grid: $light-grid-schema
));

// Specifying the palette and schema of the custom grid theme
$custom-grid-theme: grid-theme(
    $palette: $custom-theme-palette,
    $schema: $custom-light-grid-schema
);
```

No olvide incluir el tema de la misma manera que se demostró anteriormente.
