# Carga bajo demanda de la cuadrícula jerárquica

La <ApiLink type="HierarchicalGrid" /> de Ignite UI for {Platform} permite una representación rápida al solicitar la cantidad mínima de datos que se debe recuperar del servidor para que el usuario pueda ver el resultado en la vista e interactuar con los datos visibles lo antes posible. Inicialmente solo se recuperan y se representan los datos de la cuadrícula raíz; solo después de que el usuario expande una fila que contiene una cuadrícula secundaria, recibirá los datos de esa cuadrícula secundaria concreta. Este mecanismo, también conocido como carga bajo demanda, se puede configurar fácilmente para funcionar con cualquier dato remoto.

En este tema se muestra cómo configurar la carga bajo demanda creando un proveedor de servicio remoto que se comunica con un servicio remoto ya disponible. Aquí está la demostración funcional y más adelante la revisaremos paso a paso y describiremos el proceso de creación.

## Proveedor de servicio remoto

Primero prepararemos nuestro proveedor de servicio para estar listos para obtener los datos que necesitaríamos para la cuadrícula jerárquica.

### Obtención de datos básicos

<PlatformBlock for="Angular">

Nos comunicaremos con nuestro servicio backend mediante el protocolo HTTP utilizando la interfaz XMLHttpRequest que proporcionan los navegadores. Para lograrlo más fácilmente, usaremos el módulo [`HttpClient`](https://angular.io/api/common/http/HttpClient) de Angular, que ofrece una API HTTP de cliente simplificada. De esta manera, para obtener nuestros datos necesitaremos este método sencillo en nuestro servicio:

```typescript
public getData(dataState): Observable<any[]> {
    return this.http.get(this.buildUrl(dataState)).pipe(
        map(response => response),
    );
}
```

Como puede ver, `this.http` será una referencia a nuestro módulo `HttpCLient`, y `buildUrl()` será el método que generará nuestra URL en función de los datos que hemos recibido. Asignamos nuestra respuesta y devolvemos un Observable, ya que esto se ejecuta de forma asíncrona. De esta manera, más adelante podemos suscribirnos a él, procesarlo más en nuestra aplicación y pasarlo a nuestra cuadrícula.

</PlatformBlock>

### Construcción de nuestra URL de solicitud

A continuación definiremos cómo debemos construir nuestra URL para la solicitud GET. Aquí es donde podremos obtener los datos para nuestra cuadrícula principal, pero también para cualquier cuadrícula secundaria dentro de ella. Usaremos los datos `Customers` de este [tema](https://data-northwind.indigo.design/swagger/index.html) para nuestro nivel raíz y usaremos `Orders` y `Details` para los niveles inferiores. <PlatformBlock for="Angular">

Definiremos todo esto en una interfaz llamada `IDataState`. Un ejemplo:

```typescript
export interface IDataState {
    key: string;
    parentID: any;
    parentKey: string;
    rootLevel: boolean;
}

//...
public buildUrl(dataState: IDataState): string {
    let qS = "";
    if (dataState) {
        if (dataState.rootLevel) {
            qS += `${dataState.key}`;
        } else {
            qS += `${dataState.parentKey}/${dataState.parentID}/${dataState.key}`;
        }
    }
    return `${this.url}${qS}`;
}
//...
```

Establecer los datos para cualquier elemento secundario que se haya expandido es un poco diferente. Cuando una fila se expande por primera vez, se representa un nuevo <ApiLink type="HierarchicalGrid" /> secundario para ella y necesitamos obtener la referencia de la cuadrícula recién creada para establecer sus datos. Por eso cada componente <ApiLink type="RowIsland" /> proporciona el evento `GridCreated` que se desencadena cuando se crea una nueva cuadrícula secundaria para esa isla de filas específica. Podemos usarlo para obtener la referencia que necesitamos para la nueva cuadrícula, solicitar sus datos al servicio y aplicarlos.

Podemos usar un solo método para todas las islas de filas, ya que construimos nuestro servicio de modo que solo necesite información sobre si es el nivel raíz, la clave de la isla de filas, la clave principal de la fila primaria y su identificador único. Toda esta información se puede acceder directamente desde los argumentos del evento o desde la isla de filas responsable de desencadenar el evento.

Como no tiene ningún elemento primario, solo podemos pasar que `rootLevel` es **true**, y la clave correspondiente, al `getData` de nuestro servicio. Este último paso tiene como objetivo mejorar la experiencia del usuario informándole de que los datos aún se están cargando, para que no tenga que mirar una cuadrícula vacía mientras tanto. Por eso <ApiLink type="HierarchicalGrid" /> admite un indicador de carga que puede mostrarse mientras la cuadrícula está vacía. Si se reciben nuevos datos, el indicador de carga se ocultará y los datos se representarán.

### Configuración de la indicación de carga

<ApiLink type="HierarchicalGrid" /> puede mostrar un indicador de carga estableciendo la propiedad <ApiLink type="GridBaseDirective" member="isLoading" label="IsLoading" /> en **true** mientras no haya datos. Necesitamos establecerla inicialmente para la cuadrícula raíz y también al crear nuevas cuadrículas secundarias, hasta que se carguen sus datos. Siempre podríamos establecerla en **true** en nuestra plantilla, pero queremos ocultarla y mostrar que la cuadrícula no tiene datos si el servicio devuelve un array vacío estableciéndola en **false**.

En este caso, la versión final de nuestra configuración se vería así:

<PlatformBlock for="Angular">

```typescript
import { AfterViewInit, Component, ViewChild } from "@angular/core";
import {
    IGridCreatedEventArgs,
    IgxHierarchicalGridComponent,
    IgxRowIslandComponent
} from "igniteui-angular";
import { RemoteLoDService } from "../services/remote-lod.service";

@Component({
    providers: [RemoteLoDService],
    selector: "app-hierarchical-grid-lod",
    styleUrls: ["./hierarchical-grid-lod.component.scss"],
    templateUrl: "./hierarchical-grid-lod.component.html"
})
export class HierarchicalGridLoDSampleComponent implements AfterViewInit {
    @ViewChild("hGrid")
    public hGrid: IgxHierarchicalGridComponent;

    constructor(private remoteService: RemoteLoDService) { }

    public ngAfterViewInit() {
        this.hGrid.isLoading = true;
        this.remoteService.getData({ parentID: null, rootLevel: true, key: "Customers" }).subscribe((data) => {
            this.hGrid.isLoading = false;
            this.hGrid.data = data;
            this.hGrid.cdr.detectChanges();
        });
    }

    public gridCreated(event: IGridCreatedEventArgs, _parentKey: string) {
        const dataState = {
            key: event.owner.key,
            parentID: event.parentID,
            parentKey: _parentKey,
            rootLevel: false
        };
        event.grid.isLoading = true;
        this.remoteService.getData(dataState).subscribe(
            (data) => {
                event.grid.isLoading = false;
                event.grid.data = data;
                event.grid.cdr.detectChanges();
            }
        );
    }
}
```
