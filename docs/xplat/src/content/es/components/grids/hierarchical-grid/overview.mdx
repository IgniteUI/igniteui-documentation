# Descripción general y configuración de la cuadrícula de datos jerárquica

La cuadrícula de datos jerárquica de {ProductName} se utiliza para mostrar y manipular datos tabulares jerárquicos. Enlace rápidamente sus datos con muy poco código o use una variedad de eventos para personalizar distintos comportamientos. Este componente proporciona un amplio conjunto de características, como selección de datos, filtrado al estilo de Excel, ordenación, paginación, plantillas, movimiento de columnas, fijación de columnas, exportación a Excel y CSV, y mucho más. La cuadrícula jerárquica se basa en el componente de cuadrícula plana y amplía su funcionalidad al permitir que los usuarios expandan o contraigan las filas de la cuadrícula primaria, mostrando las cuadrículas secundarias correspondientes cuando se necesita información más detallada.

## Uso de la cuadrícula de datos jerárquica de {Platform}

### Enlace de datos

El **{HierarchicalGridSelector}** deriva de {GridSelector} y comparte la mayor parte de su funcionalidad. La diferencia principal es que permite definir varios niveles de jerarquía. Se configuran mediante una etiqueta independiente dentro de la definición de **{HierarchicalGridSelector}**, llamada **{RowIslandSelector}**. El componente **{RowIslandSelector}** define la configuración de cada cuadrícula secundaria para el nivel concreto. También se admiten varias islas de filas por nivel.
La cuadrícula jerárquica admite dos formas de enlazarse a los datos:

### Uso de datos jerárquicos

Si la aplicación carga todos los datos jerárquicos como una matriz de objetos que hacen referencia a matrices secundarias de objetos, entonces la cuadrícula jerárquica puede configurarse para leerlos y enlazarse a ellos automáticamente. A continuación se muestra un ejemplo de un origen de datos jerárquico con la estructura adecuada:

### Uso de la carga bajo demanda

La mayoría de las aplicaciones están diseñadas para cargar inicialmente la menor cantidad de datos posible, lo que da como resultado tiempos de carga más rápidos. En esos casos, {HierarchicalGridSelector} puede configurarse para permitir que servicios creados por el usuario le proporcionen datos bajo demanda.

<PlatformBlock for="Angular">

```html

        <igx-row-island [key]="'Details'" [primaryKey]="'productId'" [autoGenerate]="true" (gridCreated)="gridCreated($event, 'Orders')">
```

```typescript
//  hierarchicalGridSample.component.ts

@Component({...})
export class HierarchicalGridLoDSampleComponent implements AfterViewInit {
    @ViewChild("hGrid")
    public hGrid: IgxHierarchicalGridComponent;

    constructor(private remoteService: RemoteLoDService) { }

    public ngAfterViewInit() {
        this.hGrid.isLoading = true;
        this.remoteService.getData({ parentID: null, rootLevel: true, key: "Customers" }).subscribe((data) => {
            this.hGrid.isLoading = false;
            this.hGrid.data = data;
            this.hGrid.cdr.detectChanges();
        });
    }

    public gridCreated(event: IGridCreatedEventArgs, _parentKey: string) {
        const dataState = {
            key: event.owner.key,
            parentID: event.parentID,
            parentKey: _parentKey,
            rootLevel: false
        };
        event.grid.isLoading = true;
        this.remoteService.getData(dataState).subscribe(
            (data) => {
                event.grid.isLoading = false;
                event.grid.data = data;
                event.grid.cdr.detectChanges();
            }
        );
    }
}
```

```typescript
// remote-load-on-demand.service.ts

@Injectable()
export class RemoteLoDService {
    public url = `https://data-northwind.indigo.design/`;

    constructor(private http: HttpClient) { }

    public getData(dataState: any): Observable<any[]> {
        return this.http.get(this.buildUrl(dataState)).pipe(
            map((response) => response)
        );
    }

    public buildUrl(dataState) {
        let qS = "";
        if (dataState) {
            if (dataState.rootLevel) {
                qS += `${dataState.key}`;
            } else {
                qS += `${dataState.parentKey}/${dataState.parentID}/${dataState.key}`;
            }
        }
        return `${this.url}${qS}`;
    }
}
```

```ts
export class HierarchicalGridLoadOnDemand {
    constructor() {
        const hierarchicalGrid = document.getElementById("hGrid") as IgcHierarchicalGridComponent;
        const ordersRowIsland = document.getElementById("ordersRowIsland");
        const orderDetailsRowIsland = document.getElementById("orderDetailsRowIsland");

        ordersRowIsland.addEventListener("gridCreated", (event: any) => {
            this.gridCreated(event, "Customers");

        orderDetailsRowIsland.addEventListener("gridCreated", (event: any) => {
            this.gridCreated(event, "Orders");

        hierarchicalGrid.isLoading = true;

            hierarchicalGrid.isLoading = false;
            hierarchicalGrid.data = data;
            hierarchicalGrid.markForCheck();
```

## Ocultar/mostrar indicadores de expansión de fila

Si dispone de una forma de proporcionar información sobre si una fila tiene elementos secundarios antes de expandirla, podría usar la propiedad de entrada <ApiLink type="HierarchicalGridBaseDirective" member="hasChildrenKey" label="HasChildrenKey" />. De este modo, podría proporcionar una propiedad booleana desde los objetos de datos que indique si se debe mostrar un indicador de expansión.

Tenga en cuenta que no es necesario establecer la propiedad <ApiLink type="HierarchicalGridBaseDirective" member="hasChildrenKey" label="HasChildrenKey" />. En caso de no proporcionarla, se mostrarán indicadores de expansión para cada fila.

Además, si desea mostrar u ocultar el indicador del encabezado para expandir o contraer todo, puede usar la propiedad <ApiLink type="HierarchicalGridBaseDirective" member="showExpandAll" label="ShowExpandAll" />.
Esta UI está deshabilitada de forma predeterminada por motivos de rendimiento y no se recomienda habilitarla en cuadrículas con muchos datos o en cuadrículas con carga bajo demanda.

Las siguientes características de la cuadrícula funcionan por nivel de cuadrícula, lo que significa que cada instancia de cuadrícula las administra de forma independiente del resto de las cuadrículas:

- Ordenación
- Filtrado
- Paginación
- Encabezados de varias columnas
- Ocultación
- Fijación
- Movimiento
- Resúmenes
- Búsqueda

Las características de selección y navegación funcionan globalmente para todo el {HierarchicalGridTitle}

- Selección
  La selección no permite que haya celdas seleccionadas en dos cuadrículas secundarias diferentes al mismo tiempo.
- Navegación
  Al navegar con <kbd>↑</kbd> + <kbd>↓</kbd>, si el elemento siguiente/anterior es una cuadrícula secundaria, la navegación continuará en la cuadrícula secundaria relacionada, marcando la celda relacionada como seleccionada y enfocada. Si la celda secundaria está fuera del puerto de visualización visible actual, se desplaza hasta quedar a la vista para que la celda seleccionada siempre sea visible.

## Estilos

igxHierarchicalGrid permite aplicar estilos mediante la [biblioteca de temas de Ignite UI for Angular](../themes/sass/component-themes.md). El <ApiLink type="function-grid-theme" label="theme" kind="sass" module="index.html" /> de la cuadrícula expone una amplia variedad de propiedades que permiten personalizar todas las características de la cuadrícula.

En los pasos siguientes, recorreremos los pasos para personalizar los estilos de igxHierarchicalGrid.

### Importar el tema global

Para comenzar la personalización de la cuadrícula jerárquica, debe importar el archivo `index`, donde se encuentran todas las funciones y mixins de estilo.

```scss
@import '~igniteui-angular/lib/core/styles/themes/index'
```

### Definir un tema personalizado

A continuación, cree un nuevo tema que amplíe <ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> y acepte los parámetros necesarios para personalizar la cuadrícula jerárquica según se desee.

 <DocsAside type="info">

No existe una función específica de cuadrícula jerárquica `sass`.

 </DocsAside>

```scss
$custom-theme: grid-theme(
  $cell-active-border-color: #ffcd0f,
  $cell-selected-background: #6f6f6f,
  $row-hover-background: #f8e495,
  $row-selected-background: #8d8d8d,
  $header-background: #494949,
  $header-text-color: #fff,
  $expand-icon-color: #ffcd0f,
  $expand-icon-hover-color: #e0b710,
  $resize-line-color: #ffcd0f,
  $row-highlight: #ffcd0f
);
```

### Definir una paleta de colores personalizada

En el enfoque descrito anteriormente, los valores de color se codificaron de forma fija. Como alternativa, puede lograr una mayor flexibilidad usando las funciones <ApiLink type="function-igx-palette" label="igx-palette" kind="sass" module="index.html" /> y <ApiLink type="function-igx-color" label="igx-color" kind="sass" module="index.html" />.
`igx-palette` genera una paleta de colores basada en los colores primario y secundario proporcionados.

```scss
$black-color: #494949;
$yellow-color: #FFCD0F;

$custom-palette: palette(
 $primary: $black-color,
 $secondary: $yellow-color
);
```

Después de generar una paleta personalizada, se puede usar la función `igx-color` para obtener diferentes variedades de los colores primario y secundario.

```scss
$custom-theme: grid-theme(
    $cell-active-border-color: (igx-color($custom-palette, "secondary", 500)),
    $cell-selected-background: (igx-color($custom-palette, "primary", 300)),
    $row-hover-background: (igx-color($custom-palette, "secondary", 100)),
    $row-selected-background: (igx-color($custom-palette, "primary", 100)),
    $header-background: (igx-color($custom-palette, "primary", 500)),
    $header-text-color: (igx-contrast-color($custom-palette, "primary", 500)),
    $expand-icon-color: (igx-color($custom-palette, "secondary", 500)),
    $expand-icon-hover-color: (igx-color($custom-palette, "secondary", 600)),
    $resize-line-color: (igx-color($custom-palette, "secondary", 500)),
    $row-highlight: (igx-color($custom-palette, "secondary", 500))
);
```

### Definir esquemas personalizados

Puede ir aún más lejos y crear una estructura flexible que tenga todos los beneficios de un [**esquema**](../themes/sass/schemas.md). El **esquema** es la receta de un tema.
Amplíe uno de los dos esquemas predefinidos que se proporcionan para cada componente. En nuestro caso, usaremos `$_light_grid`.

```scss
$custom-grid-schema: extend($_light-grid,(
    cell-active-border-color: (igx-color:('secondary', 500)),
    cell-selected-background: (igx-color:('primary', 300)),
    row-hover-background: (igx-color:('secondary', 100)),
    row-selected-background: (igx-color:('primary', 100)),
    header-background: (igx-color:('primary', 500)),
    header-text-color: (igx-contrast-color:('primary', 500)),
    expand-icon-color: (igx-color:('secondary', 500)),
    expand-icon-hover-color: (igx-color:('secondary', 600)),
    resize-line-color: (igx-color:('secondary', 500)),
    row-highlight: (igx-color:('secondary', 500))
));
```

Para que se aplique el esquema personalizado, se debe ampliar alguno de los globales `light` o `dark`. Todo el proceso consiste, en realidad, en proporcionar un componente con un esquema personalizado y agregarlo después al tema del componente correspondiente.

```scss
$my-custom-schema: extend($light-schema, (
    igx-grid: $custom-grid-schema
));

$custom-theme: grid-theme(
    $palette: $custom-palette,
    $schema: $my-custom-schema
);
```

### Aplicar el tema personalizado

La forma más sencilla de aplicar su tema es con una instrucción `sass` `@include` en el archivo de estilos global:

```scss
@include grid($custom-theme);
```

### Tema de componente con ámbito

Para que el tema personalizado afecte solo a un componente específico, puede mover todos los estilos que acaba de definir desde el archivo de estilos global al archivo de estilos del componente personalizado (incluida la importación del archivo `index`).

De este modo, debido a [ViewEncapsulation](https://angular.io/api/core/Component#encapsulation) de Angular, sus estilos se aplicarán solo a su componente personalizado.

 <DocsAside type="info">

Si el componente usa una ViewEncapsulation [`Emulated`](../themes/sass/component-themes.md#view-encapsulation), es necesario atravesar esta encapsulación usando `::ng-deep` para aplicar estilos a la cuadrícula.

 </DocsAside>

 <DocsAside type="info">

Envuelva la instrucción dentro de un selector `:host` para evitar que sus estilos afecten a elementos fuera de nuestro componente:

 </DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-theme);
    }
}
```
