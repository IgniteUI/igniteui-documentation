# {Platform} Icon Button Overview

The {ProductName} Icon Button component allows developers to use registered icons as buttons in their application. It carries all features of the [icon](../layouts/icon.mdx) component but adds features from the [button](./button.mdx) component as well.

## Usage


<PlatformBlock for="WebComponents">


First, you need to install the {ProductName} by running the following command:

```cmd
npm install {PackageWebComponents}
```

Before using the <ApiLink pkg="core" type="IconButton" />, you need to register it as follows:

```ts
import { defineComponents, IgcIconButtonComponent } from "igniteui-webcomponents";
import 'igniteui-webcomponents/themes/light/bootstrap.css';

defineComponents(IgcIconButtonComponent);
```

For a complete introduction to the {ProductName}, read the [**Getting Started**](../general-getting-started.mdx) topic.


</PlatformBlock>



<PlatformBlock for="React">


First, you need to the install the corresponding {ProductName} npm package by running the following command:

```cmd
npm install igniteui-react
```

You will then need to import the <ApiLink pkg="core" type="IconButton" />, its necessary CSS, and register its module, like so:

```tsx
import { IgrIconButton } from 'igniteui-react';
import 'igniteui-webcomponents/themes/light/bootstrap.css';
```

</PlatformBlock>



<PlatformBlock for="Blazor">


Before using the <ApiLink pkg="core" type="IconButton" />, you need to register it as follows:


```csharp
// in Program.cs file

builder.Services.AddIgniteUIBlazor(typeof(IgbIconButtonModule));
```

You will also need to link an additional CSS file to apply the styling to the <ApiLink pkg="core" type="IconButton" /> component. The following needs to be placed in the **wwwroot/index.html** file in a **Blazor Web Assembly** project or the **Pages/_Host.cshtml** file in a **Blazor Server** project:

```razor
<link href="_content/IgniteUI.Blazor/themes/light/bootstrap.css" rel="stylesheet" />
```


</PlatformBlock>

The simplest way to start using the <ApiLink pkg="core" type="IconButton" /> is as follows:

```

```

</PlatformBlock>

## Examples

### Variant

Similar to the regular button components, the icon button supports several variants - `flat` (default), `contained`, and `outlined`; To change the icon button type set the <ApiLink pkg="core" type="IconButton" member="variant" label="variant" /> attribute of the icon button.

<Sample src="/inputs/icon-button/variant" height={100} alt="{Platform} Icon Button Example" />



<PlatformBlock for="React">

```tsx
<IgrIconButton name="search" variant="contained"></IgrIconButton>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<igc-icon-button name="search" variant="contained"></igc-icon-button>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbIconButton IconName="search" Variant="@IconButtonVariant.Contained" Collection="material" />
```

</PlatformBlock>

### Size

The size of the button can be changed by utilizing the `--ig-size` CSS variable to any of the three supported sizes: `--ig-size-small`, `--ig-size-medium`, `--ig-size-large`(default).

<Sample src="/inputs/icon-button/size" height={100} alt="{Platform} Icon Button Example" />



<PlatformBlock for="React">

```tsx
<IgrIconButton className="size-medium" name="thumb-up"></IgrIconButton>
```

</PlatformBlock>
<PlatformBlock for="React">
```css
.size-medium {
    --ig-size: var(--ig-size-medium);
}
```
</PlatformBlock>

<PlatformBlock for="WebComponents, Blazor">
```css
igc-icon-button {
    --ig-size: var(--ig-size-medium);
}
```
</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbIconButton IconName="thumb-up" Collection="material" Size="@SizableComponentSize.Medium" />
```

</PlatformBlock>

### Type

The icon button component will change its internal structure from `<button>` to an `<a>` type element when the <ApiLink pkg="core" type="IconButton" member="href" label="href" /> attribute is set. In that case the icon button can be thought of as a regular link. Setting the <ApiLink pkg="core" type="IconButton" member="href" label="href" /> attribute will allow you to also set the <ApiLink pkg="core" type="IconButton" member="rel" label="rel" />, <ApiLink pkg="core" type="IconButton" member="target" label="target" />, and <ApiLink pkg="core" type="IconButton" member="download" label="download" /> attributes of the icon button.

<PlatformBlock for="WebComponents">

```html
<igc-icon-button
  name="thumb-up"
  collection="material"
  href="https://duckduckgo.com"
  target="_blank">
</igc-icon-button>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrIconButton name="thumb-up" collection="material" href="https://duckduckgo.com" target="_blank">
</IgrIconButton>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbIconButton IconName="thumb-up" Collection="material" Href="https://duckduckgo.com" Target="_blank" />
```

</PlatformBlock>

### Mirrored

Some icons need to look a little different when used in Right-to-Left(RTL) mode. For that reason we provide a `mirrored` attribute that, when set, flips the icon button horizontally.

<PlatformBlock for="React">

```tsx
<IgrIconButton name="thumb-up" mirrored={true}></IgrIconButton>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<igc-icon-button name="thumb-up" mirrored></igc-icon-button>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbIconButton IconName="thumb-up" Collection="material" Mirrored="true" />
```

</PlatformBlock>

## Styling

The <ApiLink pkg="core" type="IconButton" /> component exposes two CSS parts - `base` and `icon` that allow you to style the wrapping element (`<button>` or `<a>`) and the wrapped `<igc-icon>` element.

```css
igc-icon-button[variant="contained"]:not([disabled])::part(base) {
  padding: 12px;
  background-color: var(--ig-success-500);
}

igc-icon-button::part(icon) {
  --size: 22px;
  color: var(--ig-success-500-contrast);
}
```
