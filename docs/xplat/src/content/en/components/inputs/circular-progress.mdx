# {Platform} Circular Progress Component

The {ProductName} Circular Progress Indicator provides a compact visual representation of determinate or indeterminate progress.

## Overview

### When to Use

Use circular progress for compact progress feedback.

### When Not to Use

Use <ApiLink pkg="core" type="LinearProgress" /> when progress should span the width of its container.

## Anatomy

The Circular Progress consists of a circular track, a progress indicator, and an optional label.

```text
Circular Progress
├── track
├── progress indicator
└── label/value
```

## Best Practices

- Use an indeterminate indicator only when the completion time is unknown.
- For determinate progress, provide meaningful `max` and `value` values.

## Properties

| Name | Type | Description |
| --- | --- | --- |
| `max` | `number` | Sets the maximum value. |
| `value` | `number` | Sets the current value. |
| `variant` | `StyleVariant` | Sets the visual variant. |
| `indeterminate` | `boolean` | Shows progress without a determinate value. |
| `hideLabel` | `boolean` | Shows or hides the default label. |
| `labelFormat` | `string` | Formats the default label. |

## Styling


## Accessibility

### Keyboard Interaction

Progress indicators are not interactive and should not receive keyboard focus.

### Screen Readers / ARIA

Provide surrounding text or a label that explains what is progressing.

### Accessibility Compliance

Verify status text and contrast in the application; this topic does not make a product-level WCAG conformance claim.
