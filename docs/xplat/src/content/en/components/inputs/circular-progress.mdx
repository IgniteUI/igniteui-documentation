# {Platform} Circular Progress Overview

The {ProductName} Circular Progress Indicator component provides a visual indicator of an application’s process as it changes. The circular indicator updates its appearance as its state changes.

## Usage

### Progress Types

You can set the type of your indicator, using the <ApiLink pkg="core" type="CircularProgress" member="variant" label="Variant" /> attribute. There are five types of circular progress indicators - **primary** (default), **error**, **success**, **info**, and **warning**.

<PlatformBlock for="React">

```tsx
<IgrCircularProgress value="100" variant="success"></IgrCircularProgress>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<igc-circular-progress value="100" variant="success"></igc-circular-progress>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbCircularProgress Value=100 Variant=@ProgressBaseVariant.Success  />
 ```

</PlatformBlock>

### Indeterminate Progress

If you want to track a process that is not determined precisely, you can set the <ApiLink pkg="core" type="CircularProgress" member="indeterminate" label="Indeterminate" /> property. Also, you can hide the default label of the {ProductName} <ApiLink pkg="core" type="CircularProgress" /> by setting the <ApiLink pkg="core" type="CircularProgress" member="hideLabel" label="HideLabel" /> property and customize the progress indicator default label via the exposed <ApiLink pkg="core" type="CircularProgress" member="labelFormat" label="LabelFormat" /> property.

<PlatformBlock for="React">

```tsx
<IgrCircularProgress value="100" indeterminate="true"></IgrCircularProgress>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<igc-circular-progress value="100" indeterminate="true"></igc-circular-progress>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbCircularProgress Value=100 Indeterminate=true/>
```

</PlatformBlock>

### Gradient Progress

Customizing the progress bar in order to use a color gradient instead of a solid color could be done via the exposed `gradient` slot and <ApiLink pkg="core" type="CircularGradient" /> which defines the gradient stops.

<Sample src="/inputs/circular-progress-indicator/dynamic" height={200} alt="{Platform} Circular Progress Dynamic Example" />


<DocsAside type="info">
For each <ApiLink pkg="core" type="CircularGradient" /> defined as gradient slot of {ProductName} <ApiLink pkg="core" type="CircularProgress" /> a [SVG stop](https://developer.mozilla.org/en-US/docs/Web/SVG/Element/stop) element would be created. The values passed as `color`, `offset` and `opacity` would be set as stop-color, offset and stop-opacity of the SVG element without further validations.
</DocsAside>

<PlatformBlock for="React">

```tsx
<IgrCircularProgress >
    <IgrCircularGradient slot="gradient" offset="0%" color="#ff9a40">
    </IgrCircularGradient>
    <IgrCircularGradient slot="gradient" offset="50%" color="#1eccd4">
    </IgrCircularGradient>
    <IgrCircularGradient slot="gradient" offset="100%" color="#ff0079">
    </IgrCircularGradient>
</IgrCircularProgress>
```

</PlatformBlock>

```html
    <igc-circular-gradient slot="gradient" offset="100%" color="#ff0079"></igc-circular-gradient>
</igc-circular-progress>
```

</PlatformBlock>

## Styling

The <ApiLink pkg="core" type="CircularProgress" /> component exposes CSS parts for almost all of its inner elements:

|Name|Description|
|--|--|
| `svg`               | The progress SVG element.                 |
| `gradient_start`     | The progress linear-gradient start color. |
| `gradient_end`       | The progress linear-gradient end color.   |
| `track`              | The progress ring's track area.           |
| `fill`               | The progress indicator area.              |
| `label`              | The progress label.                       |
| `value`              | The progress label value.                 |
| `indeterminate`      | The progress indeterminate state.         |
| `primary`            | The progress indicator primary state.     |
| `danger`             | The progress indicator error state.       |
| `warning`            | The progress indicator warning state.     |
| `info`               | The progress indicator info state.        |
| `success`            | The progress indicator success state.     |

Using this CSS parts we have almost full control over the Circular Progress styling.

```css

igc-circular-progress {
  margin: 20px;
  --diameter: 50px;
}

igc-circular-progress::part(gradient_end),
igc-circular-progress::part(gradient_start) {
  stop-color: var(--ig-success-200);
}

igc-circular-progress::part(track) {
  stroke: var(--ig-gray-400);
}

```
