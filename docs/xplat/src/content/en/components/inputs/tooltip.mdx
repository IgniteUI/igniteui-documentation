# {Platform} Tooltip

The {ProductName} Tooltip component provides a way to display a tooltip for a specific element. A tooltip is a popup that displays information related to an element, usually when the element receives keyboard focus or when the mouse hovers over it.

### Getting Started


<PlatformBlock for="WebComponents">



To start using the `Tooltip`, first, you need to install the Ignite UI for Web Components by running the following command:

npm install {PackageWebComponents}

## Usage

### Tooltip target

To attach a tooltip to the desired element, use the <ApiLink type="Tooltip" member="anchor" label="Anchor" /> property of the <ApiLink type="Tooltip" /> and set it to the ID of the target element.

<PlatformBlock for="WebComponents">

```html
<igc-button id="target-button">Hover me</igc-button>
<igc-tooltip anchor="target-button">
  Congrats you've hovered the button!
</igc-tooltip>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrButton id="target-button">Hover me</IgrButton>
<IgrTooltip anchor="target-button">
  Congrats you have hovered the button!
</IgrTooltip>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbButton id="target-button">Hover me</IgbButton>
<IgbTooltip Anchor="target-button">
  Congrats you have hovered the button!
</IgbTooltip>
```

</PlatformBlock>


<PlatformBlock for="WebComponents, React">


You can also specify the target by passing the element instance directly:

</PlatformBlock>


<PlatformBlock for="WebComponents">

```html
<igc-tooltip id="tooltip">
  Congrats you've hovered the button!
</igc-tooltip>
<igc-button id="hover-button">Hover me</igc-button>
```

</PlatformBlock>

### Show/Hide delay settings

If you want to control the delay before showing and hiding the <ApiLink type="Tooltip" />, you can use the <ApiLink type="Tooltip" member="showDelay" label="ShowDelay" /> and <ApiLink type="Tooltip" member="hideDelay" label="HideDelay" /> properties. Both properties accept a number value representing time in milliseconds.

<PlatformBlock for="WebComponents">

```html
<igc-tooltip show-delay="600" hide-delay="800">
  Her name is Madelyn James.
</igc-tooltip>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrTooltip showDelay="600" hideDelay="800">
  Her name is Madelyn James.
</IgrTooltip>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbTooltip ShowDelay="600" HideDelay="800">
  Her name is Madelyn James.
</IgbTooltip>
```

</PlatformBlock>

> [!NOTE]
> It's important to note that the Tooltip API methods — <ApiLink type="Tooltip" member="show" label="Show" />, <ApiLink type="Tooltip" member="hide" label="Hide" />, and <ApiLink type="Tooltip" member="toggle" label="Toggle" /> — DO NOT take the <ApiLink type="Tooltip" member="showDelay" label="ShowDelay" /> and <ApiLink type="Tooltip" member="hideDelay" label="HideDelay" /> properties into account. They act immediately when invoked.

### Placement

The <ApiLink type="Tooltip" /> can also be positioned relative to its target element with ease. All you need to do is use the <ApiLink type="Tooltip" member="placement" label="Placement" /> property along with one of the <ApiLink type="PopoverPlacement" kind="enum" prefixed={false} /> options.

If the <ApiLink type="Tooltip" member="placement" label="Placement" /> property is not set, the default value is `Bottom`, which places the <ApiLink type="Tooltip" /> below the target element.

Additionally, you can make the <ApiLink type="Tooltip" /> "sticky" using the <ApiLink type="Tooltip" member="sticky" label="Sticky" /> property, which adds a close button and keeps the <ApiLink type="Tooltip" /> visible until the user closes it manually - either by clicking the close button or pressing the `Esc` key. This behavior overrides the default hover behavior, preventing the <ApiLink type="Tooltip" /> from disappearing when the user stops hovering over the target element.

The <ApiLink type="Tooltip" /> also includes an optional arrow indicator that can be configured via the <ApiLink type="Tooltip" member="withArrow" label="WithArrow" /> property. The arrow visually connects the tooltip to its anchor element and its position automatically adjusts based on the tooltip's <ApiLink type="Tooltip" member="placement" label="Placement" />.

<PlatformBlock for="WebComponents">

```html
<igc-button id="target-button">Hover me</igc-button>
<igc-tooltip anchor="target-button" placement="top-start" sticky with-arrow>
  Congrats you've hovered the button!
</igc-tooltip>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrButton id="target-button">Hover me</IgrButton>
<IgrTooltip anchor="target-button" placement="top-start" sticky withArrow={true}>
  Congrats you have hovered the button!
</IgrTooltip>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbButton id="target-button">Hover me</IgbButton>
<IgbTooltip Anchor="target-button" Placement="PopoverPlacement.TopStart" Sticky="true" DisableArrow="true">
  Congrats you have hovered the button!
</IgbTooltip>
```

</PlatformBlock>

### Triggers

By default, the <ApiLink type="Tooltip" /> is triggered only while hovering over the target element. However, you can change this behavior using the <ApiLink type="Tooltip" member="showTriggers" label="ShowTriggers" /> and <ApiLink type="Tooltip" member="hideTriggers" label="HideTriggers" /> properties, which allow you to control when the <ApiLink type="Tooltip" /> appears and disappears. These properties accept event names as values—such as `click`, `focus`, or `keypress`—letting you trigger the `Tooltip` in different scenarios.

### Additional Properties

Apart from the properties we've already covered, the <ApiLink type="Tooltip" /> component offers a variety of additional properties that allow you to further configure its behavior, position, and appearance.

|Name|Type|Description|
|--|--|--|
| `Open` | boolean | Determines whether the tooltip is visible. |
| `WithArrow` | boolean | Determines whether to render an arrow indicator for the tooltip. |
| `Offset` | number | Sets the pixel distance between the tooltip and its `Anchor`. |

### Methods

In addition to its configurable properties, the <ApiLink type="Tooltip" /> also exposes three methods that you can use:

|Name|Description|
|--|--|
| `Show` | Displays the tooltip if it’s not already shown. If a target is provided, it sets the target as a transient `Anchor`. |
| `Hide` | Hides the tooltip if it’s not already hidden. |
| `Toggle` |  Toggles the tooltip between the shown and hidden states. |

## Accessibility & ARIA Support

The <ApiLink type="Tooltip" /> is built with accessibility in mind and includes the following ARIA attributes:

- `role` - When the tooltip is in its default behavior, `role="tooltip"` is applied. If the <ApiLink type="Tooltip" member="sticky" label="Sticky" /> property is enabled, the role changes to `status`.
- `inert` - Dynamically toggled based on visibility. When the tooltip is hidden, it becomes inert.
- `aria-atomic` - Set to true, ensuring that the entire tooltip content is announced when it changes.
- `aria-live` - Set to polite, indicating to screen readers that updates should be announced only when the user is idle.

## Styling

The <ApiLink type="Tooltip" /> component exposes two CSS parts that you can use for styling:

|Name|Description|
|--|--|
| `base` | The base wrapper of the tooltip component. |
| `top, right, bottom, left ...` | The area containing the tooltip arrow. The part name matches the value of the tooltip placement property. |

```css
igc-tooltip::part(base) {
  background-color: var(--ig-primary-500);
  color: var(--ig-primary-500-contrast);
}

igc-tooltip::part(bottom) {
  border-bottom-color: var(--ig-primary-500);
}
```
