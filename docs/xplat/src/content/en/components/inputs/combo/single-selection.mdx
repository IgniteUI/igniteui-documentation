# {Platform} Single Selection ComboBox

The {Platform} `ComboBox` supports single-selection mode and quick filtering of the list of items via the main input prompt. Users can quickly type in the item they are looking for and be presented with a list of options. Upon pressing the enter key, the first highlighted match will be selected.

## {Platform} Single Selection Example

To enable single-selection and quick filtering, set the <ApiLink type="Combo" member="singleSelect" label="SingleSelect" /> property on the <ApiLink type="Combo" />. The user experience and keyboard navigation will mostly stay the same, but instead of having to type in your search query into a special filtering box above the list of options, the main input box will be used.

## Selection API

The selection API for a ComboBox with the <ApiLink type="Combo" member="singleSelect" label="SingleSelect" /> property applied mostly remains the same, however, there are some important differences compared to ComboBoxes that don't have this property set.

The main difference is that only one item can be selected at any time. For example, if you have specified a <ApiLink type="Combo" member="valueKey" label="ValueKey" /> for your combo component, passing more than one item to the <ApiLink type="Combo" member="select" label="Select" />/<ApiLink type="Combo" member="deselect" label="Deselect" /> methods will have no effect. This also means that any previously selected items will automatically get deselected upon making a new selection.

Here's how to select/deselect an item programmatically in a single selection combo.

### Selecting items


<PlatformBlock for="WebComponents">


```ts
// select the item matching the 'BG01' value of the value key field.
combo.select('BG01');
```

</PlatformBlock>


<PlatformBlock for="Blazor">

```razor
<IgbCombo SingleSelect @ref="Combo"></IgbCombo>

@code {
    private IgbCombo Combo;

    this.Combo.Select(new object[] { "UK01" });
}
```

</PlatformBlock>


<PlatformBlock for="React">


```tsx
// select the item matching the 'BG01' value of the value key field.
comboRef.current.select('BG01');
```

</PlatformBlock>


To deselect an item without making a new selection, call the <ApiLink type="Combo" member="deselect" label="Deselect" /> method.

#### Deselecting items


<PlatformBlock for="WebComponents">


```ts
// deselect the item matching the 'BG01' value of the value key field.
combo.deselect('BG01');
```

</PlatformBlock>


<PlatformBlock for="Blazor">

```razor

```

</PlatformBlock>


<PlatformBlock for="React">


```tsx

```

</PlatformBlock>


## Disabled features

Naturally, some configuration options will have no effect in a single selection ComboBox.

### Placeholder

Assigning a value to the <ApiLink type="Combo" member="placeholderSearch" label="PlaceholderSearch" /> property will yield no result since the filtering input that usually is placed above the list of options will not be present in a single selection ComboBox.

### Auto-focusing the list of options

Setting the <ApiLink type="Combo" member="autofocusList" label="AutofocusList" /> option on a single selection ComboBox will also have no effect.

## Keyboard Navigation

The keyboard navigation should behave the same as with a non-single selection ComboBox, except for the fact that now the main input plays the role of a filtering prompt and so all keyboard actions that apply to the filtering/search input are moved to the main input prompt.

## Other Features

All other features will behave the same as in a non-single selection ComboBox component.
