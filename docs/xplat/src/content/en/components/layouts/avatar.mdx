# Avatar Component

The {ProductName} Avatar represents a user, entity, or object with an image, initials, or custom content.

Use the avatar to provide a compact visual identity in lists, cards, profile menus, and activity feeds.

## Anatomy

The avatar is a single host element that applies image semantics and renders one of the supported content patterns.

<Image src={avatarAnatomy} alt="{Platform} Avatar anatomy" class="responsive-img" style="margin: 50px 0;" />

<PlatformBlock for="WebComponents, React, Blazor">

```text
igc-avatar[role="image"]         // host - exposes the avatar
└─ div[part="base"]              // avatar wrapper
   ├─ span[part="initials"]      // rendered when `initials` is set
   ├─ slot                       // rendered when `initials` is not set
   └─ img[part="image"]          // rendered while `src` is set and loads
```

</PlatformBlock>

<PlatformBlock for="Angular">

```text
igx-avatar[role="img"]           // host - exposes the avatar
└─ one of, by priority:
   ├─ div.igx-avatar__image      // `src` set - image painted as a background
   ├─ igx-icon                   // `icon` set
   ├─ span                       // `initials` set - first two characters
   └─ ng-content                 // custom projected content
```

</PlatformBlock>

## Usage

Render an avatar with an image source, initials, or custom content in the default slot.

### Variants

<PlatformBlock for="WebComponents, React, Blazor">

Set only the content source you intend to show. The avatar renders <ApiLink pkg="core" type="Avatar" member="initials" label="Initials" /> when they are set, otherwise it renders default slot content, and it also renders an image element while <ApiLink pkg="core" type="Avatar" member="src" label="Src" /> is set and loads successfully.

</PlatformBlock>

<PlatformBlock for="Angular">

Set only the content source you intend to show. The Angular avatar renders `src` first, then `icon`, then `initials`, and falls back to projected custom content when none of those inputs are set.

</PlatformBlock>

```html

  alt="A photo of Ana Zane">
```

### Do/Don't

**When to use:** Use the avatar when a UI needs a small representation of a person, organization, object, or account, such as a profile image, initials, or an icon. Keep a consistent avatar strategy within the same UI region so repeated identities are easy to scan.

<PlatformBlock for="WebComponents, React, Blazor">

**When not to use:** Use the [Badge](../inputs/badge.mdx) component when you need to show a count, status, or notification indicator instead of representing an entity. Badges can also decorate avatars when both identity and status need to appear together.

</PlatformBlock>

<PlatformBlock for="Angular">

**When not to use:** Use the [Badge](../badge.mdx) component when you need to show a count, status, or notification indicator instead of representing an entity. Badges can also decorate avatars when both identity and status need to appear together.

</PlatformBlock>

## Properties

The avatar exposes a small set of inputs for its content and shape.

<PlatformBlock for="WebComponents, React, Blazor">

| Name | Type | Default | Description |
| -- | -- | -- | -- |
| <ApiLink pkg="core" type="Avatar" member="alt" label="Alt" /> | `string` | n/a | Sets alternative text for the image avatar. |
| <ApiLink pkg="core" type="Avatar" member="initials" label="Initials" /> | `string` | n/a | Sets text initials rendered when no image is displayed. |
| <ApiLink pkg="core" type="Avatar" member="shape" label="Shape" /> | `"square" \| "rounded" \| "circle"` | `"square"` | Sets the avatar shape. |
| <ApiLink pkg="core" type="Avatar" member="src" label="Src" /> | `string` | n/a | Sets the image source URL. |

</PlatformBlock>

| <ApiLink type="Avatar" member="shape" label="shape" /> | `"square" \| "rounded" \| "circle"` | `"square"` | Sets the avatar shape. |
| <ApiLink type="Avatar" member="size" label="size" /> | `"small" \| "medium" \| "large"` | `"small"` | Sets the avatar size. |

## Accessibility

The avatar is a non-interactive identity visual with accessible image semantics.

### Keyboard Interaction

The avatar does not receive focus and has no keyboard interaction.

| Key | Action |
| -- | -- |
| n/a | The avatar is not keyboard interactive. |

### Screen Readers / ARIA

The avatar initializes with image semantics and a default accessible label of `avatar`.

<PlatformBlock for="WebComponents, React, Blazor">

- Set `alt` when a `src` image represents a specific person, entity, or object.
- Add an explicit `aria-label` when projected custom content needs a different accessible name.
- Treat decorative avatars as redundant when adjacent text already identifies the same entity.

</PlatformBlock>

<PlatformBlock for="Angular">

- Use surrounding text to identify the represented person, entity, or object when an image avatar conveys identity.
- Treat decorative avatars as redundant when adjacent text already identifies the same entity.

</PlatformBlock>

### Accessibility Compliance

Infragistics documents {ProductName} accessibility support for Section 508 and WCAG 2.1 guideline areas in the [Accessibility Compliance](../interactivity/accessibility-compliance.mdx) topic.

<PlatformBlock for="WebComponents, React, Blazor">

| Criterion | How the component complies |
| -- | -- |
| [1.1.1 Non-text Content](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content) | The avatar has an accessible label, and image avatars can receive specific alternative text through `alt`. |
| [4.1.2 Name, Role, Value](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value) | The component initializes with image semantics and exposes content-related ARIA information. |

</PlatformBlock>

<PlatformBlock for="Angular">

| Criterion | How the component complies |
| -- | -- |
| [1.1.1 Non-text Content](https://www.w3.org/WAI/WCAG22/Understanding/non-text-content) | The avatar has an accessible label and can be paired with surrounding text that identifies the represented entity. |
| [4.1.2 Name, Role, Value](https://www.w3.org/WAI/WCAG22/Understanding/name-role-value) | The component initializes with `role="img"` and a default accessible label. |

</PlatformBlock>

Your responsibilities:

- Keep sufficient contrast between the avatar background and text or icon color when overriding styles.
- Avoid duplicating the same identity announcement when adjacent text already names the person or entity.

<PlatformBlock for="WebComponents, React, Blazor">

- Provide a descriptive label or `alt` text when the avatar conveys identity.

</PlatformBlock>

<PlatformBlock for="Angular">

- Make sure nearby text identifies the represented entity when the avatar image conveys identity.

</PlatformBlock>
