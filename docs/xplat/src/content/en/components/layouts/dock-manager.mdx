# Dock Manager Component

The Infragistics {Platform} Dock Manager arranges application views in panes that users can dock, float, resize, pin, hide, and persist.

## Overview

Dock Manager gives you a desktop-style workspace where multiple tools, documents, and supporting views can share the same page and still remain user-configurable at runtime.

### When to Use

Use Dock Manager when the page needs a complex workspace with multiple panes, floating windows, tabbed documents, runtime docking, and layout persistence.

### When Not to Use

Do not use Dock Manager for simple fixed pane resizing or lightweight tab switching. For simpler split layouts, use the [Splitter](./splitter.md) component, and for switching between related views inside one region, use the [Tabs](./tabs.md) component.

## Anatomy

Dock Manager renders a pane hierarchy where content panes are grouped inside split panes, tab groups, document hosts, and optional floating windows.

### Pane Types

The dock manager layout is composed from the following pane types:

- <ApiLink type="ContentPane" />
- <ApiLink type="SplitPane" />
- <ApiLink type="TabGroupPane" />
- <ApiLink type="DocumentHost" />

### Docking Indicators

Dragging a floating pane can show different docking targets depending on the hovered location.

#### Root Docking

Root docking docks a pane against the outer edges of the dock manager and makes it a direct child of the layout root pane.

<Image src={dockmanagerRootDocking} alt="Dock Manager root docking indicators" class="responsive-img" />

#### Pane Docking

Pane docking docks a floating pane around an existing content pane or tab group, or groups it into tabs.

<Image src={dockmanagerPaneDocking} alt="Dock Manager pane docking indicators" class="responsive-img" />

#### Document Host Docking

Document host docking adds targets that dock relative to the active document region.

<Image src={dockmanagerDocumentHostDocking} alt="Dock Manager document host docking indicators" class="responsive-img" />

#### Splitter Docking

Splitter docking inserts a pane directly into an existing split boundary for more precise placement.

<Image src={dockmanagerSplitterDocking} alt="Dock Manager splitter docking indicators" class="responsive-img" />

#### Edge Docking

Edge docking appears when root docking is disabled and lets users insert panes at the start or end of the root split direction.

<Image src={dockmanagerEdgeDocking} alt="Dock Manager edge docking indicators" class="responsive-img" />

## Usage

Render a dock manager, assign a layout object, then configure pane types, docking behavior, layout updates, persistence, events, and localization.

### Render the Component

Add the dock manager host first, then assign its layout and matching slotted content.

<PlatformBlock for="WebComponents">

```html
<igc-dockmanager id="dockManager">
</igc-dockmanager>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrDockManager id="dockManager">
</IgrDockManager>
```

</PlatformBlock>

<DocsAside type="info">
Dock Manager relies on Shadow DOM and slots, so it does not support older non-Chromium browsers such as Internet Explorer 11 and Edge 18 and earlier.
</DocsAside>

### Define the Initial Layout

Set the <ApiLink type="DockManager" member="layout" label="Layout" /> property with a root pane and child panes to describe the workspace structure.

<PlatformBlock for="WebComponents">

```ts
import { IgcDockManagerPaneType, IgcSplitPaneOrientation, IgcDockManagerComponent } from 'igniteui-dockmanager';

this.dockManager = document.getElementById('dockManager') as IgcDockManagerComponent;
this.dockManager.layout = {
    rootPane: {
        type: IgcDockManagerPaneType.splitPane,
        orientation: IgcSplitPaneOrientation.horizontal,
        panes: [
            {
                type: IgcDockManagerPaneType.contentPane,
                contentId: 'content1',
                header: 'Pane 1'
            }
        ]
    }
};
```

</PlatformBlock>

<PlatformBlock for="React">


### Configure Content Panes

Use content panes for the actual tools, views, and document surfaces inside the docked layout.

The <ApiLink type="ContentPane" member="header" label="Header" /> property provides the visible pane title. Use <ApiLink type="ContentPane" member="headerId" label="HeaderId" />, <ApiLink type="ContentPane" member="tabHeaderId" label="TabHeaderId" />, and <ApiLink type="ContentPane" member="unpinnedHeaderId" label="UnpinnedHeaderId" /> when different header slot content is needed for the pane header, tab header, or unpinned header.

<PlatformBlock for="WebComponents">

```ts
const contentPane = {
    type: IgcDockManagerPaneType.contentPane,
    contentId: 'content1',
    header: 'Pane 1',
    tabHeaderId: 'tabHeader1'
};
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
const contentPane = {
    type: IgrDockManagerPaneType.contentPane,
    contentId: 'content1',
    header: 'Pane 1',
```

Use <ApiLink type="ContentPane" member="isPinned" label="IsPinned" /> to unpin a pane, <ApiLink type="ContentPane" member="unpinnedLocation" label="UnpinnedLocation" /> to choose its edge, and the `allowClose`, `allowPinning`, `allowDocking`, `allowFloating`, `documentOnly`, `disabled`, and `hidden` properties to control runtime behavior for each pane.

### Add Floating Panes

Use floating panes when a view should open in a detached window above the main docked layout.

Floating pane definitions are stored in <ApiLink type="DockManagerLayout" member="floatingPanes" label="FloatingPanes" />.

<PlatformBlock for="WebComponents">

```ts
const layout = {
    rootPane: {
        // ...
    },
    floatingPanes: [
        {
            type: IgcDockManagerPaneType.splitPane,
            orientation: IgcSplitPaneOrientation.horizontal,
            floatingLocation: { x: 80, y: 80 },
            floatingWidth: 200,
            floatingHeight: 150,
            floatingResizable: true,
            panes: [
                {
                    type: IgcDockManagerPaneType.contentPane,
                    contentId: 'content1',
                    header: 'Floating Pane 1'
                }
            ]
        }
    ]
};
```

### Configure Docking Behavior

Dock Manager lets end users pin, unpin, resize, close, float, move, dock, and maximize panes at runtime, and those changes are reflected in the <ApiLink type="DockManager" member="layout" label="Layout" /> object.

Set `allowRootDock` to `false` to disable root docking and enable edge docking. Set `allowSplitterDock` to `false` when the layout should not allow precise splitter insertion while dragging a pane.

### Update the Layout at Runtime

Mutate the layout object when you need to add, remove, or rearrange panes programmatically, then reassign the layout object so the UI reflects the changes.

<PlatformBlock for="WebComponents">

```ts
const splitPane = this.dockManager.layout.rootPane.panes[0];
const contentPane = splitPane.panes[0];
this.dockManager.removePane(contentPane);
this.dockManager.layout = { ...this.dockManager.layout };
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx

const contentPane = splitPane.panes[0];
this.dockManager.removePane(contentPane);
this.dockManager.layout = { ...this.dockManager.layout };
```

</PlatformBlock>

### Save and Restore Layouts

Persist the current layout by serializing the <ApiLink type="DockManager" member="layout" label="Layout" /> property, then restore it by assigning the parsed object back to the dock manager.

<PlatformBlock for="WebComponents">

```ts
private savedLayout: string;

private saveLayout() {
    this.savedLayout = JSON.stringify(this.dockManager.layout);
}

private loadLayout() {
    this.dockManager.layout = JSON.parse(this.savedLayout);
}
```

</PlatformBlock>

<PlatformBlock for="React">


### Localize Strings

Localize the strings used for context menus, tooltips, and ARIA attributes with either registered language bundles or per-instance resource strings.

Dock Manager reads the page `lang` attribute by default and falls back to English when the language is missing or unsupported.

<PlatformBlock for="WebComponents">

```ts
import { registerI18n } from 'igniteui-i18n-core';
import { DockManagerResourceStringsES } from 'igniteui-i18n-resources';

registerI18n(DockManagerResourceStringsES, 'es');
```

```ts
import { addResourceStrings } from 'igniteui-dockmanager';

const dockManagerStringsFr: IgcDockManagerResourceStrings = {
  close: 'Fermer',
  // ...
};

addResourceStrings('fr', dockManagerStringsFr);
```

</PlatformBlock>

Use the <ApiLink type="DockManager" member="resourceStrings" label="ResourceStrings" /> property when you want to override strings directly for a specific dock manager instance.

## Best Practices

Use Dock Manager for intentional workspaces where end-user layout control is a feature, not incidental complexity.

- Keep pane headers concise so floating windows, tabs, and unpinned headers remain easy to scan.
- Use a document host for document-like surfaces and reserve content panes outside it for supporting tools and navigation.
- Persist user-customized layouts when the same workspace is expected across sessions.
- Disable docking modes your app does not need so the drag-and-drop experience stays understandable.
- Give slotted content a predictable width and height so resizing panes does not produce clipped or collapsed content.

## Accessibility

Dock Manager supports extensive keyboard-driven pane navigation and docking, but accessibility documentation for pane semantics, announcements, and conformance should stay explicit because this component manages a complex workspace model.

### Keyboard Interaction

Use the following keyboard shortcuts to navigate panes, move documents, and close the active pane.

| Key | Action |
| -- | -- |
| <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Up Arrow</kbd> | Docks to the global top. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Down Arrow</kbd> | Docks to the global bottom. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Right Arrow</kbd> | Docks to the global right. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>Left Arrow</kbd> | Docks to the global left. |
| <kbd>Shift</kbd> + <kbd>Up Arrow</kbd> | With multiple tabs in a tab group, splits the view and docks the focused tab above. |
| <kbd>Shift</kbd> + <kbd>Down Arrow</kbd> | With multiple tabs in a tab group, splits the view and docks the focused tab below. |
| <kbd>Shift</kbd> + <kbd>Right Arrow</kbd> | With multiple tabs in a tab group, splits the view and docks the focused tab right. |
| <kbd>Shift</kbd> + <kbd>Left Arrow</kbd> | With multiple tabs in a tab group, splits the view and docks the focused tab left. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>F6</kbd> or <kbd>Cmd/Ctrl</kbd> + <kbd>Right Arrow</kbd> | Focuses the next tab in the document host. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>F6</kbd> or <kbd>Cmd/Ctrl</kbd> + <kbd>Left Arrow</kbd> | Focuses the previous tab in the document host. |
| <kbd>Alt</kbd> + <kbd>F6</kbd> | Focuses the next content pane. |
| <kbd>Alt</kbd> + <kbd>Shift</kbd> + <kbd>F6</kbd> | Focuses the previous content pane. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>F7</kbd> or <kbd>Cmd/Ctrl</kbd> + <kbd>F8</kbd> | Starts pane navigation from the first document forward. |
| <kbd>Alt</kbd> + <kbd>F7</kbd> or <kbd>Alt</kbd> + <kbd>F8</kbd> | Starts pane navigation from the first pane forward. |
| <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>F7</kbd> or <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>F8</kbd> | Starts pane navigation from the last document backward. |
| <kbd>Alt</kbd> + <kbd>Shift</kbd> + <kbd>F7</kbd> or <kbd>Alt</kbd> + <kbd>Shift</kbd> + <kbd>F8</kbd> | Starts pane navigation from the last pane backward. |
| <kbd>Alt</kbd> + <kbd>F3</kbd> | Closes the active pane. |
