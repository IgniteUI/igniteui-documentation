# Carousel Component

The {ProductName} Carousel is a slideshow component for presenting slides, cards, and other page-like content in a sequential viewport.

## Overview

The carousel displays one slide at a time and provides built-in navigation, indicators, animations, and automatic transitions for sequential content.

### When to Use

Use the carousel when users move through a series of slides, images, or short content panels in order and the previous or next action is part of the experience.

### When Not to Use

Do not use the carousel when users need to compare several items at once or jump directly among many sections without stepping through intermediate slides.

## Best Practices

Use the carousel to support a focused sequence, not to hide unrelated content behind motion.

- Keep slide content concise so users can understand one view before moving to the next.
- Avoid autoplay for dense or task-critical content unless users can easily pause and reorient.
- Use custom indicators only when they communicate slide identity better than the default indicators.
- Ensure each slide image or card has meaningful alternative text and visible context.
- Prefer direct navigation patterns over a carousel when users need random access to many items.

## Accessibility

The carousel provides roles, labels, live-region behavior, and keyboard navigation for both slide selection and previous or next navigation.

### Keyboard Interaction

Use the keyboard shortcuts below to operate the navigation buttons and indicators.

| Key | Action |
| -- | -- |
| <kbd>Space</kbd> or <kbd>Enter</kbd> on a navigation button | Moves to the previous or next slide, depending on the focused button. |
| <kbd>Left Arrow</kbd> on an indicator | Moves to the previous slide, or the next slide in right-to-left mode. |
| <kbd>Right Arrow</kbd> on an indicator | Moves to the next slide, or the previous slide in right-to-left mode. |
| <kbd>Home</kbd> on an indicator | Moves to the first slide, or the last slide in right-to-left mode. |
| <kbd>End</kbd> on an indicator | Moves to the last slide, or the first slide in right-to-left mode. |

### Screen Readers / ARIA

The carousel uses explicit roles and labels for the region, indicator list, indicators, and slides.

- The carousel container uses the `region` role and `aria-roledescription="carousel"`.
- The indicators container uses the `tablist` role, and each indicator uses the `tab` role.
- Each slide uses the `tabpanel` role and `aria-roledescription="slide"`.
- The navigation buttons use `aria-label` values of `Previous slide` and `Next slide`.
- The slides container uses `aria-live`, which is `polite` by default and switches to `off` while automatic playback is active.
- Slides use generated labels in the form `{index + 1} of {total}`, and indicators use generated labels in the form `Slide {index + 1}`.

Your responsibilities:

- Provide meaningful text alternatives for slide images and other non-text content.
- Ensure custom indicator and navigation content remains understandable without relying on color alone.
- Verify focus order and contrast when you place interactive controls inside slides.
- Evaluate autoplay timing against your application's accessibility requirements.
