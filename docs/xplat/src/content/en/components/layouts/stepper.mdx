# Stepper Component

The {ProductName} Stepper is a workflow component for guiding users through a sequence of ordered steps.

## Overview

The stepper breaks a process into distinct stages so users can track progress, understand their position, and complete each step in sequence.

### When to Use

Use the stepper when users move through a structured multi-step flow such as checkout, onboarding, or setup.

### When Not to Use

Do not use the stepper when users need freeform navigation across unrelated sections or when the sequence is too short to justify a dedicated progress pattern.

When linear mode is enabled, the current non-optional step must be valid before the user can continue to the next one.

<DocsAside type="info">
Optional step validity is not taken into account in order to move forward.
</DocsAside>

### Navigate Programmatically

Use the stepper methods when your application controls movement between steps.

The stepper provides:

- `navigateTo`
- `next`
- `prev`
- `reset`

<DocsAside type="info">
The `reset` method returns the stepper to its initial state and activates the first step, but it does not clear the step content.
</DocsAside>

### Change the Step Type

Set <ApiLink pkg="core" type="Stepper" member="stepType" label="StepType" /> to render full headers, indicators only, or titles only.

When `stepType` is `full`, use the `titlePosition` property to place the title before, after, above, or below the indicator.

<Sample src="/layouts/stepper/steptypes" height={300} alt="{Platform} Step type example" />

### Adapt Content Height

Set `--body-grid-rows` to `0fr` when you want the content region height to follow the active step instead of matching the tallest step.

```css
igc-stepper {
  --body-grid-rows: 0fr;
}
```

<Sample src="/layouts/stepper/step-height" height={490} alt="{Platform} Stepper content height example" />

### Change Stepper Animations

Set <ApiLink pkg="core" type="Stepper" member="horizontalAnimation" label="HorizontalAnimation" /> and <ApiLink pkg="core" type="Stepper" member="verticalAnimation" label="VerticalAnimation" /> to control the transition style for each orientation.

Set `animationDuration` to change the transition duration, or set the animation type to `none` to disable animations.

## Best Practices

Use the stepper to support a real sequence, not as a decorative progress marker for unrelated sections.

- Keep step titles short and specific so users can quickly understand the flow.
- Use subtitles only when they add context that the title cannot carry alone.
- Reserve linear mode for flows that truly require completion in order.
- Keep optional, invalid, and complete states aligned with real business rules so progress feedback stays trustworthy.

## Methods

Use the stepper methods when you need to move between steps from code.

| Name | Description |
| -- | -- |
| `navigateTo` | Activates the step at the given index. |
| `next` | Activates the next non-disabled step. |
| `prev` | Activates the previous non-disabled step. |
| `reset` | Returns the stepper to its initial active step. |

## Accessibility

The stepper provides keyboard step navigation and stateful progress feedback for sequential workflows.

### Keyboard Interaction

Use the keyboard shortcuts below to move focus and activate steps.

| Key | Action |
| -- | -- |
| <kbd>Tab</kbd> | Moves focus to the next tabbable element. |
| <kbd>Shift</kbd> + <kbd>Tab</kbd> | Moves focus to the previous tabbable element. |
| <kbd>Down Arrow</kbd> | Moves focus to the header of the next accessible step when the stepper is vertically oriented. |
| <kbd>Up Arrow</kbd> | Moves focus to the header of the previous accessible step when the stepper is vertically oriented. |
| <kbd>Left Arrow</kbd> | Moves focus to the header of the previous accessible step in both orientations. |
| <kbd>Right Arrow</kbd> | Moves focus to the header of the next accessible step in both orientations. |
| <kbd>Home</kbd> | Moves focus to the first enabled step header. |
| <kbd>End</kbd> | Moves focus to the last enabled step header. |
| <kbd>Enter</kbd> or <kbd>Space</kbd> | Activates the currently focused step. |
