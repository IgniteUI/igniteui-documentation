# {Platform} Card Overview

The {ProductName} Card displays text, images, icons, and buttons in a visually rich presentation that can serve as an entry point to more detailed information. Cards can be used to create a multimedia dashboard.

## Usage

Cards allow you to easily display content composed of different types of objects or similar objects whose size and supported actions can vary.

You will notice a few things above. First, to _tag_ an element as a header title, we need to place it between the <ApiLink pkg="core" type="CardHeader" /> tags and set its slot name to `title`. Conversely, to define a subtitle, we need to set the slot name to `subtitle`. When slotting content into the `title` and `subtitle` slots, we recommend using `<span>` rather than heading elements (`<h1>`–`<h6>`). Heading elements carry built-in styling (such as font size, line height, and margins) that can interfere with the component's intended typography and layout. A `<span>` provides a neutral container that inherits the component’s styles cleanly.

Any image or video we want to show in the card, we wrap inside the <ApiLink pkg="core" type="CardMedia" /> tags. The <ApiLink pkg="core" type="CardMedia" /> allows us to size the content placed inside so that it maintains its aspect ratio while filling the element’s entire content box. If the object's aspect ratio does not match the aspect ratio of its box, then the object will be clipped to fit.

You can place anything inside the <ApiLink pkg="core" type="CardContent" /> tags. Usually text goes there.

Finally, the <ApiLink pkg="core" type="CardActions" /> is where you'd place any actionable items, like buttons.

### Media, Thumbs, and Avatars

If you want to display an image or icon in the card header next to the title and subtitle, assign the element's slot attribute to `thumbnail`. For the best results, we recommend using an [`igc-avatar`](../layouts/avatar.mdx) component for the `thumbnail` slot, as it integrates seamlessly with the card header's layout and styling.

Taking the card above as an example, we can edit the contents of the <ApiLink pkg="core" type="CardHeader" /> and add an avatar with `slot="thumbnail"`:

<PlatformBlock for="WebComponents">

```html
<igc-card-header>
    <igc-avatar slot="thumbnail" src="path/to/image" initials="TS"></igc-avatar>
    <span slot="title">Title</span>
    <span slot="subtitle">Subtitle</span>
</igc-card-header>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrCardHeader>
    <IgrAvatar slot="thumbnail" src="path/to/image" initials="TS" />
    <span slot="title">Title</span>
    <span slot="subtitle">Subtitle</span>
</IgrCardHeader>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbCardHeader>
    <IgbAvatar slot="thumbnail" Src="path/to/image" Initials="TS" />
    <span slot="title">Title</span>
    <span slot="subtitle">Subtitle</span>
</IgbCardHeader>
```

</PlatformBlock>

The above example will show the avatar alongside the title and subtitle in the card header.

### Outlined cards
The card has an `outlined` attribute which, if set, removes any shadows from the card, replacing them with a thin border to separate the card from the background.

### Horizontal Layout

By default all sections of the card (header, content, media, actions) are laid out vertically. This is nice when we have a lot of vertical space. Say we wanted to lay out the sections in the card horizontally. We can achieve such a layout with some simple CSS.

Here's an example of an outlined horizontal card:

<PlatformBlock for="WebComponents">

```html
<igc-card outlined>
    <div class="card-horizontal">
        <div>
            <igc-card-header>
                <img src="ROZES-Under-the-Grave.jpg" slot="thumbnail">
                <span slot="title">Rozes</span>
                <span slot="subtitle">Under the Grave (2016)</span>
            </igc-card-header>
            <igc-card-content>
                <p>As I have always said: I write what’s real and what’s true,
                    even if it means throwing myself under the bus.</p>
            </igc-card-content>
        </div>
        <div class="divider"></div>
        <igc-card-actions>
            <igc-icon-button name="previous"></igc-icon-button>
            <igc-icon-button name="play"></igc-icon-button>
            <igc-icon-button name="next"></igc-icon-button>
        </igc-card-actions>
    </div>
</igc-card>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor

        <div>
            <IgbCardHeader>
```

```tsx

            <IgrCardHeader>
```

</PlatformBlock>

We are using an additional `div` element to bundle the <ApiLink pkg="core" type="CardHeader" /> and <ApiLink pkg="core" type="CardContent" /> together, keeping them aligned vertically, and applying the `.card-horizontal` class to the wrapping `div` element to align the two sections of the card horizontally.

The styles that `.card-horizontal` class applies are:

```css
.card-horizontal {
    display: flex;
    flex-direction: row;
    flex: 1 1 0%;
}

.card-horizontal img {
    width: 64px;
    height: 64px;
}

.card-horizontal igc-card-actions {
    justify-content: center;
}
```

### Card Actions

The card actions area allows additional configuration to what we have already mentioned.
The area is intended to hold actionable items, like buttons. There are two slots available that can be used to position content: `start` and `end`. We can assign these slots to any element placed inside the <ApiLink pkg="inputs" type="CardActions" />, and it will be positioned accordingly, either at the `start` or the `end` of the area. These slots can be used with any type of content, but we highly recommend sticking to actionable elements, or wrappers of actionable elements, as shown in the example below:

<PlatformBlock for="WebComponents">

```html
<igc-card-actions>
    <igc-button slot="end">
        <igc-ripple></igc-ripple>
        Read more
    </igc-button>
    <div slot="start">
        <igc-icon-button name="twitter">
            <igc-ripple></igc-ripple>
        </igc-icon-button>
        <igc-icon-button name="facebook">
            <igc-ripple></igc-ripple>
        </igc-icon-button>
    </div>
</igc-card-actions>
```

</PlatformBlock>

<PlatformBlock for="Blazor">

```razor
<IgbCardActions>
    <IgbButton slot="end">
        <IgbRipple />
        Read more
    </IgbButton>
    <div slot="start">
        <IgbIconButton name="twitter">
            <IgbRipple />
        </IgbIconButton>
        <IgbIconButton name="facebook" >
            <IgbRipple />
        </IgbIconButton>
    </div>
</IgbCardActions>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrCardActions>
    <IgrButton slot="end">
        <span>Read more</span>
        <IgrRipple />
    </IgrButton>

            <IgrRipple />
        </IgrIconButton>

        </IgrIconButton>
    </div>
</IgrCardActions>
```

</PlatformBlock>

You can also add more content in-between by simply omitting the slot property and let the elements go to the default slot.

## Styling

Since the card is a container that wraps different elements, styling is done by styling its building blocks - the <ApiLink pkg="core" type="CardHeader" />, <ApiLink pkg="core" type="CardContent" />, <ApiLink pkg="core" type="CardMedia" /> and <ApiLink pkg="core" type="CardActions" /> sub-components.

```css
igc-card {
  background-color: var(--ig-secondary-900);
}

igc-card-content,
igc-card-header::part(title) {
  color: var(--ig-primary-500-contrast);
}

igc-card-header > *[slot="subtitle"] {
  color: var(--ig-warn-500);
  opacity: 0.9;
}

igc-icon-button::part(base) {
  background-color: var(--ig-primary-300);
}
```
