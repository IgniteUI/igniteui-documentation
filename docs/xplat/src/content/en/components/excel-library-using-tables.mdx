# {Platform} Using Tables

The Infragistics {Platform} Excel Engine's <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" /> functionality allows you to format your data in rows and columns The data in a worksheet table can be managed independently from the data in the other rows and columns in a <ApiLink pkg="excel" prefixed={false} type="Worksheet" />.

## Adding a Table to a Worksheet
Worksheet tables in the Infragistics {Platform} Excel Engine are represented by the <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" /> object and are added in the worksheet's <ApiLink pkg="excel" type="Worksheet" member="tables" label="Tables" prefixed={false} suffix={false} /> collection. In order to add a table, you need to invoke the `Add` method on this collection. In this method, you can specify the region in which you would like to add a table, whether or not the table should contain headers, and optionally, specify the table's style as a <ApiLink pkg="excel" type="WorksheetTableStyle" prefixed={false} suffix={false} /> object.

The following code demonstrates how you can add a table with headers to a <ApiLink pkg="excel" prefixed={false} type="Worksheet" /> spanning a region of A1 to G10, where A1 to G1 will be the column headers:

```ts
var workbook = new Workbook(WorkbookFormat.Excel2007);
var worksheet = this.workbook.worksheets().add("Sheet1");

worksheet.tables().add("A1:G10", true);
```

Once you have added a table, you can modify it by adding or deleting rows and columns by calling the <ApiLink pkg="excel" type="WorksheetTable" member="insertColumns" label="InsertColumns" prefixed={false} suffix={false} />, <ApiLink pkg="excel" type="WorksheetTable" member="insertDataRows" label="InsertDataRows" prefixed={false} suffix={false} />, <ApiLink pkg="excel" type="WorksheetTable" member="deleteColumns" label="DeleteColumns" prefixed={false} suffix={false} />, or <ApiLink pkg="excel" type="WorksheetTable" member="deleteDataRows" label="DeleteDataRows" prefixed={false} suffix={false} /> methods on the <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" />. You can also set a new table range by using the <ApiLink pkg="excel" type="WorksheetTable" member="resize" label="Resize" prefixed={false} suffix={false} /> method of the table.

The following code snippet shows the usage of these methods:

```ts
var workbook = new Workbook(WorkbookFormat.Excel2007);
var worksheet = workbook.worksheets().add("Sheet1");
var table = worksheet.tables().add("A1:G10", true);

//Will add 5 columns at index 1.
table.insertColumns(1, 5);

//Will add 5 rows at index 0.
table.insertDataRows(0, 5);

//Will delete 5 columns starting at index 1.
table.deleteColumns(1, 5);

//Will delete 5 rows starting at index 0.
table.deleteDataRows(0, 5);

//Will resize the table to be in the region of A1:G15.
table.resize("A1:G15");
```

## Filtering Tables
Filtering is done by applying a filter on a column in the <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" />. When the filter is applied on a column, all filters in the table will be reevaluated to determine which rows meet the criteria of all filters applied.

If the data in the table is subsequently changed or you change the `Hidden` property of the rows, the filter conditions will not automatically reevaluate. The filter conditions in a table are only reapplied when table column filters are added, removed, modified, or when the <ApiLink pkg="excel" type="WorksheetTable" member="reapplyFilters" label="ReapplyFilters" prefixed={false} suffix={false} /> method is called on the table.

The following are the filter types available to the columns of your <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" />:

- <ApiLink pkg="excel" type="AverageFilter" prefixed={false} suffix={false} /> - Cells can be filtered based on whether they are above or below the average value of all cells in the column.
- <ApiLink pkg="excel" type="CustomFilter" prefixed={false} suffix={false} /> - Cells can be filtered based on one or more custom conditions.
- <ApiLink pkg="excel" type="DatePeriodFilter" prefixed={false} suffix={false} /> - Only cells with dates in a specific month or quarter of any year will be displayed.
- <ApiLink pkg="excel" type="FillFilter" prefixed={false} suffix={false} /> - Only cells with a specific fill will be displayed.
- <ApiLink pkg="excel" type="FixedValuesFilter" prefixed={false} suffix={false} /> - Cells which only match specific display values or which fall within a specific group of dates/times will be displayed.
- <ApiLink pkg="excel" type="FontColorFilter" prefixed={false} suffix={false} /> - Only cells with a specific font color will be displayed.
- <ApiLink pkg="excel" type="RelativeDateRangeFilter" prefixed={false} suffix={false} /> - Cells with date values can be filtered based on whether they occur within a relative time range of the date when the filter was applied, such as the next day or previous quarter.
- <ApiLink pkg="excel" type="TopOrBottomFilter" prefixed={false} suffix={false} /> - This filter allows for filtering the top or bottom N values. It also allows filtering the top or bottom N% values.
- <ApiLink pkg="excel" type="YearToDateFilter" prefixed={false} suffix={false} /> - Cells with date values can be filtered if they occur between the start of the year and the date on which the filter was applied.

The following code snippet demonstrates how to apply an "above average" filter to a <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" />'s first column:

```ts
var workbook = new Workbook(WorkbookFormat.Excel2007);
var worksheet = workbook.worksheets().add("Sheet1");
var table = worksheet.tables().add("A1:G10", true);

table.columns(0).applyAverageFilter(AverageFilterType.AboveAverage);
```

## Sorting Tables
Sorting is done by setting a sorting condition on a table column. When a sorting condition is set on a column, all sorting conditions in the table will be reevaluated to determine the order of the cells in the table. When cells need to be moved to meet their sort criteria, the entire row of cells in the table is moved as a unit.

If the data in the table is subsequently changed, the sort conditions do not automatically reevaluate. The sort conditions in a table are only reapplied when sort conditions are added, removed, modified, or when the <ApiLink pkg="excel" type="WorksheetTable" member="reapplySortConditions" label="ReapplySortConditions" prefixed={false} suffix={false} /> method is called on the table. When sorting conditions are reevaluated, only the visible cells are sorted. All cells in hidden rows are kept in place.

In addition to accessing sort conditions from the table columns, they are also exposed off the <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" />'s SortSettings property's <ApiLink pkg="excel" type="SortSettings" member="sortConditions" label="SortConditions" prefixed={false} suffix={false} exclude="Angular,React,WebComponents" /> collection. This is an ordered collection of columns/sort condition pairs. The order of this collection is the precedence of the sorting.

The following sort condition types are available to set on columns:

- <ApiLink pkg="excel" type="OrderedSortCondition" prefixed={false} suffix={false} /> - Sort cells in an ascending or descending order based on their value.
- <ApiLink pkg="excel" type="CustomListSortCondition" prefixed={false} suffix={false} /> - Sort cells in a defined order based on their text or display value. For example, this might be useful for sorting days as they appear on a calendar, rather than alphabetically.
- <ApiLink pkg="excel" type="FillSortCondition" prefixed={false} suffix={false} /> - Sort cells based on whether their fill is a specific pattern or gradient.
- <ApiLink pkg="excel" type="FontColorSortCondition" prefixed={false} suffix={false} /> - Sort cells based on whether their font is a specific color.

There is also a <ApiLink pkg="excel" type="SortSettings" member="caseSensitive" label="CaseSensitive" prefixed={false} suffix={false} exclude="Angular,React,WebComponents" /> property on the SortSettings of the <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" /> to determine whether strings should be sorted case sensitively or not.

The following code snippet demonstrates how to apply an <ApiLink pkg="excel" type="OrderedSortCondition" prefixed={false} suffix={false} /> to a <ApiLink pkg="excel" prefixed={false} type="WorksheetTable" />:

```ts
var workbook = new Workbook(WorkbookFormat.Excel2007);
var worksheet = this.workbook.worksheets().add("Sheet1");
var table = worksheet.tables().add("A1:G10", true);

table.columns(0).sortCondition = new OrderedSortCondition(SortDirection.Ascending);

//Alternative:
table.sortSettings.sortConditions().addItem(table.columns(0), new OrderedSortCondition(SortDirection.Ascending));
```
