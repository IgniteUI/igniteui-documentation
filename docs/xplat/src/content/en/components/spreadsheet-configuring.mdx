# {Platform} Configuring Spreadsheet

The {Platform} Spreadsheet component allows the user to configure many different aspects of the control. This includes, but is not limited to, editing of the cells, the visibility of gridlines and headers, protection, zoom level, and various other properties related to the Excel worksheet.

## Configuring Cell Editing

When a user edits a cell value and confirms the new input, the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control has the ability to navigate to cells adjacent to the currently active cell on press of the <kbd>ENTER</kbd> key, depending on the configuration of the spreadsheet.

In order to enable this <kbd>ENTER</kbd> key navigation, you can set the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="isEnterKeyNavigationEnabled" label="IsEnterKeyNavigationEnabled" suffix={false} exclude="Angular,Blazor,WebComponents" /> property to **true**. If set to false, the active cell will stay the same when pressing the <kbd>ENTER</kbd> key.

You can also configure the direction of the adjacent cell navigated to on press of the <kbd>ENTER</kbd> key by setting the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="enterKeyNavigationDirection" label="EnterKeyNavigationDirection" suffix={false} exclude="Angular,Blazor,WebComponents" /> property to `Down`, `Up`, `Left` or `Right`.

The following code snippets demonstrate the above:

<PlatformBlock for="Angular">

```html
<igx-spreadsheet isEnterKeyNavigationEnabled=true
    enterKeyNavigationDirection="Left">
</igx-spreadsheet>
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<igc-spreadsheet is-enter-key-navigation-enabled="true"
    enter-key-navigation-direction="Left">
</igc-spreadsheet>
```

</PlatformBlock>

<PlatformBlock for="React">
```ts
import { SpreadsheetEnterKeyNavigationDirection } from 'igniteui-react-spreadsheet';
```


</PlatformBlock>

<PlatformBlock for="WebComponents">
```ts
import { SpreadsheetEnterKeyNavigationDirection } from 'igniteui-webcomponents-spreadsheet';
```


</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrSpreadsheet isEnterKeyNavigationEnabled={true} enterKeyNavigationDirection={SpreadsheetEnterKeyNavigationDirection.Left} />
```

</PlatformBlock>

```ts
this.spreadsheet.isEnterKeyNavigationEnabled = true;
this.spreadsheet.enterKeyNavigationDirection = SpreadsheetEnterKeyNavigationDirection.Left;
```

## Configuring Navigation

The <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control allows you to configure navigation between a worksheet's cells by configuring whether or not the control is in "end mode." End mode is the functionality where, on press of an arrow key, the active cell will be moved from the current cell to the end of the row or column where data exists in the adjacent cells, depending on the direction of the arrow key pressed. This functionality is good for navigating to the end of large blocks of data very quickly.

For example, if you are in end mode, and you click in a large 100x100 block of data, and press the <kbd>→</kbd> arrow key, this will navigate to the right end of the row that you are in to the furthest right column with data. After this operation, the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> will pop out of end mode.

End mode goes into effect at runtime when the user presses the <kbd>END</kbd> key, but it can be configured programmatically by setting the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="isInEndMode" label="IsInEndMode" suffix={false} exclude="Angular,Blazor,WebComponents" /> property of the spreadsheet control.

The following code snippets demonstrate the above, in that the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> will begin in end mode:

<PlatformBlock for="Angular">

```html
<igx-spreadsheet isInEndMode=true></igx-spreadsheet>
```

</PlatformBlock>

<PlatformBlock for="React">

```tsx
<IgrSpreadsheet isInEndMode={true} />
```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html

```

</PlatformBlock>

```ts
this.spreadsheet.isInEndMode = true;
```

## Configuring Protection

The <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> will respect the protection of a workbook on a worksheet-by-worksheet basis. Configuration for a worksheet's protection can be configured by calling the `Protect()` method on the worksheet to protect it, and the `Unprotect()` method to unprotect it.

You can activate or deactivate protection on the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control's currently active worksheet by using the code below:

```ts
this.spreadsheet.activeWorksheet.protect();
this.spreadsheet.activeWorksheet.unprotect();
```

## Configuring Selection

The <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control allows you to configure the type of selection allowed in the control then modifier keys (<kbd>SHIFT</kbd> or <kbd>CTRL</kbd>) are pressed by the user. This is done by setting the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="selectionMode" label="SelectionMode" suffix={false} exclude="Angular,Blazor,WebComponents" /> property of the spreadsheet to one of the following values:

- `AddToSelection`: New cell ranges are added to the <ApiLink pkg="spreadsheet" type="SpreadsheetSelection" suffix={false} prefixed={false} exclude="Blazor" /> object's <ApiLink pkg="spreadsheet" type="SpreadsheetSelection" member="cellRanges" label="CellRanges" suffix={false} prefixed={false} exclude="Blazor" /> collection without needing to hold down the <kbd>CTRL</kbd> key when dragging via the mouse and a range is added with the first arrow key navigation after entering the mode. One can enter the mode by pressing <kbd>SHIFT</kbd> + <kbd>F8</kbd>.
- `ExtendSelection`: The selection range in the <ApiLink pkg="spreadsheet" type="SpreadsheetSelection" suffix={false} prefixed={false} exclude="Blazor" /> object's <ApiLink pkg="spreadsheet" type="SpreadsheetSelection" member="cellRanges" label="CellRanges" suffix={false} prefixed={false} exclude="Blazor" /> collection representing the active cell is updated as one uses the mouse to select a cell or navigating via the keyboard.
- `Normal`: The selection is replaced when dragging the mouse to select a cell or range of cells. Similarly when navigating via the keyboard a new selection is created. One may add a new range by holding the <kbd>CTRL</kbd> key and using the mouse and one may alter the selection range containing the active cell by holding the <kbd>SHIFT</kbd> key down while clicking with the mouse or navigating with the keyboard such as with the arrow keys.

The
<ApiLink type="SpreadsheetSelection" pkg="spreadsheet" prefixed={false} suffix={false} exclude="Blazor" />
object mentioned in the descriptions above can be obtained by using the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="activeSelection" label="ActiveSelection" suffix={false} exclude="Angular,Blazor,WebComponents" /> property of the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control.

The following code snippets demonstrate configuration of the selection mode:

The selection of the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control can also be set or obtained programmatically. For single selection, you can set the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="activeCell" label="ActiveCell" suffix={false} exclude="Angular,Blazor,WebComponents" /> property Multiple selection is done through the
<ApiLink type="SpreadsheetSelection" pkg="spreadsheet" prefixed={false} suffix={false} exclude="Blazor" />
object that is returned by the <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control's <ApiLink pkg="spreadsheet" type="Spreadsheet" member="activeSelection" label="ActiveSelection" suffix={false} exclude="Angular,Blazor,WebComponents" /> property.

The
<ApiLink type="SpreadsheetSelection" pkg="spreadsheet" prefixed={false} suffix={false} exclude="Blazor" />
object has an `AddCellRange()` method that allows you to programmatically add a range of cells to the selection of the spreadsheet in the form of a new  <ApiLink type="SpreadsheetCellRange" pkg="spreadsheet" prefixed={false} suffix={false} exclude="Blazor" /> object.

The following code snippet demonstrates adding a cell range to the spreadsheet's selection:

```ts
this.spreadsheet.activeSelection.addCellRange(new SpreadsheetCellRange(2, 2, 5, 5));
```

## Configuring Tab Bar Area

The <ApiLink pkg="spreadsheet" type="Spreadsheet" suffix={false} exclude="Angular,Blazor,WebComponents" /> control respects the configuration of the visibility and width of the tab bar area from the <ApiLink pkg="excel" type="Workbook" member="windowOptions" label="WindowOptions" prefixed={false} suffix={false} /> of the currently active <ApiLink pkg="spreadsheet" type="Spreadsheet" member="workbook" label="Workbook" suffix={false} exclude="Angular,Blazor,WebComponents" /> via the `TabBarWidth` and `TabBarVisibility` properties, respectively.

The tab bar area is the area that visualizes the worksheet names as tabs in the control.

You can configure the tab bar's visibility and width using the following code snippet:

```ts
this.spreadsheet.workbook.windowOptions.tabBarVisible = false;

this.spreadsheet.workbook.windowOptions.tabBarWidth = 200;
```

## Configuring Zoom Level

The {Platform} Spreadsheet component supports zooming in and out by configuring its <ApiLink pkg="spreadsheet" type="Spreadsheet" member="zoomLevel" label="ZoomLevel" suffix={false} exclude="Angular,Blazor,WebComponents" /> property. The zoom level can be a maximum of 400% and a minimum of 10%.

Setting this property to a number represents the percentage as a whole number, so setting the <ApiLink pkg="spreadsheet" type="Spreadsheet" member="zoomLevel" label="ZoomLevel" suffix={false} exclude="Angular,Blazor,WebComponents" /> to 100 is equivalent to setting it to 100%.

The following code snippets show how to configure the spreadsheet's zoom level:

<PlatformBlock for="Angular">


</PlatformBlock>

<PlatformBlock for="React">

```tsx

```

</PlatformBlock>

<PlatformBlock for="WebComponents">

```html
<igc-spreadsheet zoom-level="200"></igc-spreadsheet>
```

</PlatformBlock>

```ts
this.spreadsheet.zoomLevel = 200;
```
