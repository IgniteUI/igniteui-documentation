# {Platform} {ComponentTitle} Summaries

The {ProductName} Summaries feature in {Platform} {ComponentTitle} functions on a per-column level as group footer. {Platform} {ComponentName} summaries is powerful feature which enables the user to see column information in a separate container with a predefined set of default summary items, depending on the type of data within the column or by implementing a custom  template in the <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} />.

## {Platform} {ComponentTitle} Summaries Overview Example

<Sample src="/{ComponentSample}/data-summary-options" height={650} alt="{Platform} {ComponentTitle} data summary options" />


<DocsAside type="info">
The summary of the column is a **function of all column values**, unless filtering is applied, then the summary of the column will be **function of the filtered result values**
</DocsAside>

<ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> summaries can also be enabled on a per-column level in {ProductName}, which means that you can activate it only for columns that you need. <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> summaries gives you a predefined set of default summaries, depending on the type of data in the column, so that you can save some time:


For `string` and `boolean` <ApiLink pkg="grids" type="Column" member="dataType" />, the following function is available:
- Count

For `number`, `currency` and `percent` data types, the following functions are available:

- Count
- Min
- Max
- Average
- Sum

For `date` data type, the following functions are available:
- Count
- Earliest
- Latest

All available column data types could be found in the official [Column types topic](column-types.md#default-template).

<ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> summaries are enabled per-column by setting <ApiLink pkg="grids" type="Column" member="hasSummary" /> property to **true**. It is also important to keep in mind that the summaries for each column are resolved according to the column data type. In the <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> the default column data type is `string`, so if you want `number` or `date` specific summaries you should specify the <ApiLink pkg="grids" type="Column" member="dataType" /> property as `number` or `date`. Note that the summary values will be displayed localized, according to the grid <PlatformBlock for="Angular,React,WebComponents"><ApiLink pkg="grids" kind="interface" type="GridBaseDirective" member="locale" label="Locale" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink pkg="grids" kind="class" type="GridBaseDirective" member="locale" label="Locale" /></PlatformBlock> and column <ApiLink pkg="grids" type="Column" member="pipeArgs" />.

## Custom {ComponentTitle} Summaries

If these functions do not fulfill your requirements you can provide a custom summary for the specific columns.


<PlatformBlock for="WebComponents">
In order to achieve this you have to override one of the base classes <ApiLink pkg="grids" type="SummaryOperand" suffix={false} exclude="Blazor" />, <ApiLink pkg="grids" type="NumberSummaryOperand" suffix={false} exclude="Blazor" /> or <ApiLink pkg="grids" type="DateSummaryOperand" suffix={false} exclude="Blazor" /> according to the column data type and your needs. This way you can redefine the existing function or you can add new functions. <ApiLink pkg="grids" type="SummaryOperand" suffix={false} exclude="Blazor" /> class provides the default implementation only for the <ApiLink pkg="grids" type="SummaryOperand" exclude="Blazor" member="count" label="Count" suffix={false} /> method. <ApiLink pkg="grids" type="NumberSummaryOperand" suffix={false} exclude="Blazor" /> extends <ApiLink pkg="grids" type="SummaryOperand" suffix={false} exclude="Blazor" /> and provides implementation for the <ApiLink pkg="grids" type="NumberSummaryOperand" exclude="Blazor" member="min" label="Min" suffix={false} />, <ApiLink pkg="grids" type="NumberSummaryOperand" exclude="Blazor" member="max" label="Max" suffix={false} />, <ApiLink pkg="grids" type="NumberSummaryOperand" exclude="Blazor" member="sum" label="Sum" suffix={false} /> and <ApiLink pkg="grids" type="NumberSummaryOperand" exclude="Blazor" member="average" label="Average" suffix={false} />. <ApiLink pkg="grids" type="DateSummaryOperand" suffix={false} exclude="Blazor" /> extends <ApiLink pkg="grids" type="SummaryOperand" suffix={false} exclude="Blazor" /> and additionally gives you <ApiLink pkg="grids" type="DateSummaryOperand" exclude="Blazor" member="earliest" label="Earliest" suffix={false} /> and <ApiLink pkg="grids" type="DateSummaryOperand" exclude="Blazor" member="latest" label="Latest" suffix={false} />.

</PlatformBlock>

<PlatformBlock for="Angular">
```typescript
import { IgxSummaryResult, IgxSummaryOperand, IgxNumberSummaryOperand, IgxDateSummaryOperand } from 'igniteui-angular';

class MySummary extends IgxNumberSummaryOperand {
    constructor() {
        super();
    }

    operate(data?: any[]): IgxSummaryResult[] {
        const result = super.operate(data);
        result.push({
            key: 'test',
            label: 'Test',
            summaryResult: data.filter(rec => rec > 10 && rec < 30).length
        });
        return result;
    }
}

```
</PlatformBlock>

## Formatting summaries
By default, summary results, produced by the built-in summary operands, are localized and formatted according to the grid <ApiLink pkg="grids" kind="interface" type="GridBaseDirective" member="locale" label="Locale" /> and column <ApiLink pkg="grids" type="Column" member="pipeArgs" />. When using custom operands, the <ApiLink pkg="grids" kind="interface" type="GridBaseDirective" member="locale" label="Locale" /> and <ApiLink pkg="grids" type="Column" member="pipeArgs" /> are not applied. If you want to change the default appearance of the summary results, you may format them using the <ApiLink pkg="grids" type="Column" member="summaryFormatter" label="SummaryFormatter" /> property.

```typescript
public dateSummaryFormat(summary: IgxSummaryResult, summaryOperand: IgxSummaryOperand): string {
    const result = summary.summaryResult;
    if (summaryOperand instanceof IgxDateSummaryOperand && summary.key !== 'count'
        && result !== null && result !== undefined) {
        const pipe = new DatePipe('en-US');
        return pipe.transform(result,'MMM YYYY');
    }
    return result;
}
```

## Summaries with Group By

When you have grouped by columns, the <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> allows you to change the summary position and calculation mode using the <ApiLink pkg="grids" type="{ComponentName}" member="summaryCalculationMode" label="SummaryCalculationMode" prefixed={false} /> and <ApiLink pkg="grids" type="{ComponentName}" member="summaryPosition" label="SummaryPosition" prefixed={false} /> properties. Along with these two properties the <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} /> exposes and <ApiLink pkg="grids" type="{ComponentName}" member="showSummaryOnCollapse" label="ShowSummaryOnCollapse" prefixed={false} /> property which allows you to determine whether the summary row stays visible when the group row that refers to is collapsed.

The available values of the <ApiLink pkg="grids" type="{ComponentName}" member="summaryCalculationMode" label="SummaryCalculationMode" prefixed={false} /> property are:

- <PlatformBlock for="Angular,React,WebComponents"><ApiLink pkg="grids" type="GridSummaryCalculationMode" kind="type" prefixed={false} label="RootLevelOnly" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink pkg="grids" type="GridSummaryCalculationMode" kind="enum" prefixed={false} label="RootLevelOnly" /></PlatformBlock> - Summaries are calculated only for the root level.
- <PlatformBlock for="Angular,React,WebComponents"><ApiLink pkg="grids" type="GridSummaryCalculationMode" kind="type" prefixed={false} label="ChildLevelsOnly" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink pkg="grids" type="GridSummaryCalculationMode" kind="enum" prefixed={false} label="ChildLevelsOnly" /></PlatformBlock> - Summaries are calculated only for the child levels.
- <PlatformBlock for="Angular,React,WebComponents"><ApiLink pkg="grids" type="GridSummaryCalculationMode" kind="type" prefixed={false} label="RootAndChildLevels" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink pkg="grids" type="GridSummaryCalculationMode" kind="enum" prefixed={false} label="RootAndChildLevels" /></PlatformBlock> - Summaries are calculated for both root and child levels. This is the default value.

The available values of the <ApiLink pkg="grids" type="{ComponentName}" member="summaryPosition" label="SummaryPosition" prefixed={false} /> property are:

- <PlatformBlock for="Angular,React,WebComponents"><ApiLink pkg="grids" type="GridSummaryPosition" kind="type" prefixed={false} label="Top" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink pkg="grids" type="GridSummaryPosition" kind="enum" prefixed={false} label="Top" /></PlatformBlock> - The summary row appears before the group by row children.
- <PlatformBlock for="Angular,React,WebComponents"><ApiLink pkg="grids" type="GridSummaryPosition" kind="type" prefixed={false} label="Bottom" /></PlatformBlock><PlatformBlock for="Blazor"><ApiLink pkg="grids" type="GridSummaryPosition" kind="enum" prefixed={false} label="Bottom" /></PlatformBlock> - The summary row appears after the group by row children. This is the default value.

The <ApiLink pkg="grids" type="{ComponentName}" member="showSummaryOnCollapse" label="ShowSummaryOnCollapse" prefixed={false} /> property is boolean. Its default value is set to **false**, which means that the summary row would be hidden when the group row is collapsed. If the property is set to **true** the summary row stays visible when group row is collapsed.


<DocsAside type="info">
The <ApiLink pkg="grids" type="{ComponentName}" member="summaryPosition" label="SummaryPosition" prefixed={false} /> property applies only for the child level summaries. The root level summaries appear always fixed at the bottom of the <ApiLink pkg="grids" type="{ComponentName}" prefixed={false} />.
</DocsAside>

## Keyboard Navigation

The summary rows can be navigated with the following keyboard interactions:

- <kbd>UP</kbd> - navigates one cell up.
- <kbd>DOWN</kbd> - navigates one cell down.
- <kbd>LEFT</kbd> - navigates one cell left.
- <kbd>RIGHT</kbd> - navigates one cell right.
- <kbd>CTRL</kbd> + <kbd>LEFT</kbd> or <kbd>HOME</kbd> - navigates to the leftmost cell.
- <kbd>CTRL</kbd> + <kbd>RIGHT</kbd> or <kbd>END</kbd> - navigates to the rightmost cell.

## Styling

To get started with styling the sorting behavior, we need to import the `index` file, where all the theme functions and component mixins live:

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```

Following the simplest approach, we create a new theme that extends the <ApiLink kind="sass" module="index.html" type="function-grid-summary-theme" label="grid-summary-theme" /> and accepts the `$background-color`, `$focus-background-color`, `$label-color`, `$result-color`, `$pinned-border-width`, `$pinned-border-style` and `$pinned-border-color` parameters.

```scss
$custom-theme: grid-summary-theme(
    $background-color: #e0f3ff,
    $focus-background-color: rgba( #94d1f7, .3 ),
    $label-color: #e41c77,
    $result-color: black,
    $pinned-border-width: 2px,
    $pinned-border-style: dotted,
    $pinned-border-color: #e41c77
);
```

The last step is to **include** the component mixins:

```scss
@include grid-summary($custom-theme);
```

<DocsAside type="info">
If the component is using an [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation, it is necessary to `penetrate` this encapsulation using `::ng-deep`:
</DocsAside>

 ```scss
:host {
    ::ng-deep {
        @include grid-summary($custom-theme);
    }
}
```

### Defining a Color Palette

Instead of hardcoding the color values like we just did, we can achieve greater flexibility in terms of colors by using the <ApiLink kind="sass" module="index.html" type="function-igx-palette" label="igx-palette" /> and <ApiLink kind="sass" module="index.html" type="function-igx-color" label="igx-color" /> functions.

`igx-palette` generates a color palette based on the primary and secondary colors that are passed:

```scss
$blue-color: #7793b1;
$green-color: #00ff2d;

$my-custom-palette: palette($primary: $blue-color, $secondary: $green-color);
```

And then with <ApiLink kind="sass" module="index.html" type="function-igx-color" label="igx-color" /> we can easily retrieve color from the palette.

```scss
$custom-theme: grid-summary-theme(
    $background-color: color($my-custom-palette, "primary", 700),
    $focus-background-color: color($my-custom-palette, "primary", 800),
    $label-color: color($my-custom-palette, "secondary", 500),
    $result-color: color($my-custom-palette, "grays", 900),
    $pinned-border-width: 2px,
    $pinned-border-style: dotted,
    $pinned-border-color: color($my-custom-palette, "secondary", 500)
);
```

<DocsAside type="info">
The `igx-color` and `igx-palette` are powerful functions for generating and retrieving colors. Please refer to [Palettes](../themes/palettes.md) topic for detailed guidance on how to use them.
</DocsAside>

### Using Schemas

Going further with the theming engine, you can build a robust and flexible structure that benefits from [**schemas**](../themes/sass/schemas.md). A **schema** is a recipe of a theme.

Extend one of the two predefined schemas, that are provided for every component, in this case - <ApiLink kind="sass" module="index.html" type="variable-_light-grid-summary" label="_light-grid-summary" />:

```scss
// Extending the light grid summary schema
$my-summary-schema: extend($_light-grid-summary,
    (
        background-color: (igx-color: ('primary', 700)),
        focus-background-color: (igx-color: ('primary', 800)),
        label-color: (igx-color: ('secondary', 500)),
        result-color: (igx-color: ('grays', 900)),
        pinned-border-width: 2px,
        pinned-border-style: dotted,
        pinned-border-color: (igx-color: ('secondary', 500))
    )
);
```

In order to apply our custom schema we have to **extend** one of the globals (<ApiLink kind="sass" module="index.html" type="variable-light-schema" label="light" /> or <ApiLink kind="sass" module="index.html" type="variable-dark-schema" label="dark" />), which is basically pointing out the components with a custom schema, and after that add it to the respective component themes:

```scss
// Extending the global light-schema
$my-custom-schema: extend($light-schema,
    (
        igx-grid-summary: $my-summary-schema
    )
);

// Defining our custom theme with the custom schema
$custom-theme: grid-summary-theme(
    $palette: $my-custom-palette,
    $schema: $my-custom-schema
);
```

Don't forget to include the themes in the same way as it was demonstrated above.
