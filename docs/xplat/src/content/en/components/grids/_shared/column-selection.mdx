# {Platform} {ComponentTitle} Column Selection Overview

The {Platform} {ComponentTitle} Column Selection feature in {ProductName} offers a simplified and Excel-like way to select and highlight an entire column with a single click. It can be enabled through the <ApiLink type="{ComponentName}" member="columnSelection" pkg="grids" /> input. Thanks to the rich API, the feature allows for easy manipulation of the selection state, data extraction from the selected fractions, data analysis operations, and visualizations.

## Basic Usage

The column selection feature can be enabled through the <ApiLink type="{ComponentName}" member="columnSelection" pkg="grids" /> input, which takes <ApiLink type="GridSelectionMode" /> values.

## Interactions

The default selection mode is `None`. If set to `Single` or `Multiple`, all of the presented columns will be <ApiLink type="Column" member="selectable" />. With that being said, in order to select a column, we just need to click on one, which will mark it as <ApiLink type="Column" member="selected" />. If the column is not selectable, no selection style will be applied on the header, while hovering.

<DocsAside type="info">
The [Multi Column Headers](multi-column-headers.md) feature does not reflect on the <ApiLink type="{ComponentName}" member="selectable" pkg="grids" /> input. The <ApiLink type="ColumnGroupComponent" /> is <ApiLink type="{ComponentName}" member="selectable" pkg="grids" />, if at least one of its children has the selection behavior enabled. In addition, the component is marked as <ApiLink type="{ComponentName}" member="selected" pkg="grids" /> if all of its <ApiLink type="{ComponentName}" member="selectable" pkg="grids" /> descendants are <ApiLink type="{ComponentName}" member="selected" pkg="grids" />.
</DocsAside>

## Keyboard Combinations

<DocsAside type="info">
The keyboard combinations are available only when the grid <ApiLink type="{ComponentName}" member="columnSelection" pkg="grids" /> input is set to `multiple`.
</DocsAside>

There are two scenarios for keyboard navigation of the **Column Selection** feature:
- Multi-column selection - holding <kbd>CTRL</kbd> + <kbd>click</kbd> on every **selectable** header cell.
- Range column selection - holding <kbd>SHIFT</kbd> + <kbd>click</kbd> selects all **selectable** columns in between.

## API Manipulations

The **API** provides some additional capabilities when it comes to the **non-visible** columns such that, every **hidden** column could be marked as <ApiLink type="Column" member="selected" /> by setting the corresponding **setter**.

<DocsAside type="info">
The above statement also applies to the <ApiLink type="ColumnGroupComponent" />, except that when the <ApiLink type="{ComponentName}" member="selected" pkg="grids" /> property is changed it changes the state of its descendants.
</DocsAside>

More information regarding the API manipulations could be found in the [API References](#api-references) section.

## Styling

In addition to the predefined themes, the grid could be further customized by setting some of the available [CSS properties](../theming-grid.md).
In case you would like to change some of the colors, you need to set a `class` for the grid first:

<PlatformBlock for="WebComponents">
```html
<{ComponentSelector} class="grid"></{ComponentSelector}>
```
</PlatformBlock>

<PlatformBlock for="React">
```tsx
<{ComponentSelector} className="grid"></{ComponentSelector}>
```
</PlatformBlock>

<PlatformBlock for="Blazor">
```razor
<{ComponentSelector} class="grid"></{ComponentSelector}>
```
</PlatformBlock>

Then set the related CSS properties to this class:


```css
.grid {
    --ig-grid-row-selected-background: #0062A3;
    --ig-grid-row-selected-text-color: #ecaa53;
    --ig-grid-row-selected-hover-background: #0062A3;
    --ig-grid-header-selected-text-color: #ecaa53;
    --ig-grid-header-selected-background: #0062A3;
    --ig-grid-row-selected-hover-text-color: #ecaa53;
    --ig-grid-row-selected-hover-background: #0062A3;
}
```

## Styling

Before diving into the styling options, the core module and all component mixins need to be imported.

```scss
@use "igniteui-angular/theming" as *;

// IMPORTANT: Prior to Ignite UI for Angular version 13 use:
// @import '~igniteui-angular/lib/core/styles/themes/index';
```


<ComponentBlock for="Grid">
>Please note that [row selection](row-selection.md) and [column selection](column-selection.md) can't be manipulated independently. They depend on the same `variables`.
</ComponentBlock>

With that being said, let's move on and change the **selection** and **hover** styles. <br/>

Following the simplest approach, let's define our custom **theme**.

<ComponentBlock for="TreeGrid">
```scss
$custom-grid-theme: grid-theme(
    $row-selected-background: #011627,
    $row-selected-text-color: #ECAA53,
    $row-selected-hover-background: #011627,
    $header-selected-text-color: #ECAA53,
    $header-selected-background: #011627,
    $expand-icon-color: #ECAA53,
    $expand-icon-hover-color: #B64B80
);
```

</ComponentBlock>

<ComponentBlock for="Grid, HierarchicalGrid">
```scss
$custom-grid-theme: grid-theme(
    $row-selected-background: #011627,
    $row-selected-text-color: #ECAA53,
    $row-selected-hover-background: #011627,
    $header-selected-text-color: #ECAA53,
    $header-selected-background: #011627
);
```

</ComponentBlock>

<ApiLink type="function-grid-theme" label="grid-theme" kind="sass" module="index.html" /> accepts several parameters but those are the five responsible for changing the appearance of all selected columns:
- **$row-selected-background** - sets the background of the selected fraction.
- **$row-selected-text-color** - sets the text color of the selected fraction
- **$row-selected-hover-background** - sets the color of the hovered cell or bunch of cells.
- **$header-selected-text-color** - sets the text color of the selected column header
- **$header-selected-background** - sets the background color of the selected column header.


### Using CSS Variables
The last step is to include the custom `{ComponentSelector}` theme.

```scss
@include css-vars($custom-grid-theme)
```

### Overriding the Base Theme
In order to style components for Internet Explorer 11, we have to use a different approach, since it doesn't support CSS variables.

<DocsAside type="info">
If the component is using the [Emulated](../themes/styles.md#view-encapsulation) ViewEncapsulation, it is necessary to `penetrate` this encapsulation using `::ng-deep`. In order to prevent the custom theme from leaking into other components, be sure that you have included the `:host` selector before `::ng-deep`.
</DocsAside>

```scss
:host {
    ::ng-deep {
        @include grid($custom-column-selection-theme);
    }
}
```
