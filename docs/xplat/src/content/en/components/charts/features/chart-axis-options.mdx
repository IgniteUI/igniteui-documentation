## Axis Labels Management & Formatting

The axes of the chart have the ability to perform an enhanced calculation regarding the amount of space available to the labels of the owning axis. This enhanced calculation allows the axis to optimize the amount of space given to it in order to display more labels for the given axis.

This enhanced calculation is something that you need to opt-in to, which you can do by setting the <ApiLink type="Axis" member="useEnhancedIntervalManagement" label="UseEnhancedIntervalManagement" /> property to true. Then, if you prefer to display as many labels as can fit in the dimensions of the axis without manually setting the <ApiLink type="CategoryChart" member="xAxisInterval" label="XAxisInterval" /> property of the axis, you can set the <ApiLink type="Axis" member="enhancedIntervalPreferMoreCategoryLabels" label="EnhancedIntervalPreferMoreCategoryLabels" /> property on the axis to true.

The chart also has the ability to consider auto-rotation of the labels if they will not fit in the allotted space as well as the ability to apply an automatic margin to the plot area to ensure the labels can fit. This is something that can be opted into initially by first setting the <ApiLink type="CategoryChart" member="autoMarginAndAngleUpdateMode" label="AutoMarginAndAngleUpdateMode" /> property on the chart to either `SizeChanging` or `SizeChangingAndZoom`. This will tell the chart when to re-evaluate the auto margin and angle applied to the labels, if desired.

After setting the <ApiLink type="CategoryChart" member="autoMarginAndAngleUpdateMode" label="AutoMarginAndAngleUpdateMode" />, you can set the <ApiLink type="CategoryChart" member="shouldAutoExpandMarginForInitialLabels" label="ShouldAutoExpandMarginForInitialLabels" /> property to true to opt into the automatic margin or set the <ApiLink type="CategoryChart" member="shouldConsiderAutoRotationForInitialLabels" label="ShouldConsiderAutoRotationForInitialLabels" /> property to true for the auto-rotation. You can also further customize the automatic margin that is applied by setting the <ApiLink type="CategoryChart" member="autoExpandMarginExtraPadding" label="AutoExpandMarginExtraPadding" /> and <ApiLink type="CategoryChart" member="autoExpandMarginMaximumValue" label="AutoExpandMarginMaximumValue" /> to provide extra space or a maximum possible margin, respectively.

Custom label formats such as <ApiLink pkg="geo-core" type="NumberFormatSpecifier" /> and <ApiLink type="DateTimeFormatSpecifier" /> can be added to each axis via the <ApiLink type="CategoryChart" member="xAxisLabelFormatSpecifiers" label="XAxisLabelFormatSpecifiers" /> and <ApiLink type="CategoryChart" member="yAxisLabelFormatSpecifiers" label="YAxisLabelFormatSpecifiers" /> collections. Commonly used for applying Intl.NumberFormat and Intl.DateTimeFormat language sensitive number, date and time formatting. In order for a custom format to be applied to the labels, the <ApiLink type="CategoryChart" member="yAxisLabelFormat" label="YAxisLabelFormat" /> or <ApiLink type="CategoryChart" member="xAxisLabelFormat" label="XAxisLabelFormat" /> need to be set to data item's property name on the <ApiLink type="CategoryChart" />, eg. `{Date}`. For the <ApiLink type="FinancialChart" /> the number is the context because it uses a numeric axis, therefore this needs to be set to `{0}`.

The following example formats the yAxis with a <ApiLink pkg="geo-core" type="NumberFormatSpecifier" /> to represent $USD prices for top box office movies in the United States.

## Axis Range Example

In the {Platform} charts, you can define a range minimum and range maximum value of a numeric or time axis. The range minimum is the lowest value of the axis and the range maximum is the highest value of the axis. These are set by setting the <ApiLink type="CategoryChart" member="yAxisMinimumValue" label="YAxisMinimumValue" /> and <ApiLink type="CategoryChart" member="yAxisMaximumValue" label="YAxisMaximumValue" /> options.

By default, charts will calculate the minimum and maximum values for the numeric and time axis range based on the lowest and highest corresponding value points in your data, but this automatic calculation may not be appropriate for your set of data points in all cases. For example, if your data has a minimum value of 850, you may want to set the <ApiLink type="CategoryChart" member="yAxisMinimumValue" label="YAxisMinimumValue" /> to 800 so that there will be a space value of 50 between the axis minimum and the lowest value of data points. The same idea can be applied to the axis minimum value and the highest value of data points using the <ApiLink type="CategoryChart" member="yAxisMaximumValue" label="YAxisMaximumValue" /> property.

## Axis Modes & Scale

In the <ApiLink type="FinancialChart" /> and <ApiLink type="CategoryChart" /> controls, you can choose if your data is plotted on logarithmic scale along the y-axis when the <ApiLink type="CategoryChart" member="yAxisIsLogarithmic" label="YAxisIsLogarithmic" /> property is set to true or on linear scale when this property is set to false (default value). With the <ApiLink type="CategoryChart" member="yAxisLogarithmBase" label="YAxisLogarithmBase" /> property, you can change base of logarithmic scale from default value of 10 to other integer value.

The <ApiLink type="FinancialChart" /> and control allows you to choose how your data is represented along the y-axis using <ApiLink type="FinancialChart" member="yAxisMode" label="YAxisMode" /> property that provides <ApiLink type="FinancialChartYAxisMode" member="Numeric" label="Numeric" /> and <ApiLink type="FinancialChartYAxisMode" member="PercentChange" label="PercentChange" /> modes. The <ApiLink type="FinancialChartYAxisMode" member="Numeric" label="Numeric" /> mode will plot data with the exact values while the <ApiLink type="FinancialChartYAxisMode" member="PercentChange" label="PercentChange" /> mode will display the data as percentage change relative to the first data point provided. The default value is <ApiLink type="FinancialChartYAxisMode" member="Numeric" label="Numeric" /> mode.

In addition to <ApiLink type="FinancialChart" member="yAxisMode" label="YAxisMode" /> property, the <ApiLink type="FinancialChart" /> control has <ApiLink type="FinancialChart" member="xAxisMode" label="XAxisMode" /> property that provides <ApiLink type="FinancialChartXAxisMode" member="Time" label="Time" /> and <ApiLink type="FinancialChartXAxisMode" member="Ordinal" label="Ordinal" /> modes for the x-axis. The <ApiLink type="FinancialChartXAxisMode" member="Time" label="Time" /> mode will render space along the x-axis for gaps in data (e.g. no stock trading on weekends or holidays). The <ApiLink type="FinancialChartXAxisMode" member="Ordinal" label="Ordinal" /> mode will collapse date areas where data does not exist. The default value is <ApiLink type="FinancialChartXAxisMode" member="Ordinal" label="Ordinal" /> mode.

## Axis Gap Example

The <ApiLink type="CategoryChart" member="xAxisGap" label="XAxisGap" /> property of the {Platform} charts, determines the amount of space between columns or bars of plotted series. This property accepts a numeric value between 0.0 and 1.0. The value represents a relative width of the gap out of the available number of pixels between the series. Setting this property to 0 would mean there is no gap rendered between the series, and setting it 1 would render the maximum available gap.

The <ApiLink type="CategoryChart" member="xAxisMaximumGap" label="XAxisMaximumGap" /> property of the {Platform} charts, determines the maximum gap value to allow. This default is set to 1.0 but can be changed depending on what you set <ApiLink type="CategoryChart" member="xAxisGap" label="XAxisGap" /> to.

The <ApiLink type="CategoryChart" member="xAxisMinimumGapSize" label="XAxisMinimumGapSize" /> property of the {Platform} charts, determines the minimum amount of pixels to use for the gap between the categories, if possible.

The following example shows the average maximum temperature in Celsius in New York City's Central Park represented by a [Column Chart](../types/column-chart.md) with an <ApiLink type="CategoryChart" member="xAxisGap" label="XAxisGap" /> initially set to 1, and so there will be a full category's width between the columns. There is a slider that allows you to configure the gap in this example so that you can see what the different values do.

## Axis Overlap Example

The <ApiLink type="CategoryChart" member="xAxisOverlap" label="XAxisOverlap" /> property of the {Platform} charts, allows setting the overlap of the rendered columns or bars of plotted series. This property accepts a numeric value between -1.0 and 1.0. The value represents a relative overlap out of the available number of pixels dedicated to each series. Setting this property to a negative value (down to -1.0) results in the categories being pushed away from each other, producing a gap between themselves. Conversely, setting this property to a positive value (up to 1.0) results in the categories overlapping each other. A value of 1 directs the chart to render the categories on top of each other.

The following example shows a comparison of the highest grossing worldwide film franchises compared by the total world box office revenue of the franchise and the highest grossing movie in the series, represented by a [Column Chart](../types/column-chart.md) with an <ApiLink type="CategoryChart" member="xAxisOverlap" label="XAxisOverlap" /> initially set to 1, and so the columns will completely overlap each other. There is a slider that allows you to configure the overlap in this example so that you can see what the different values do.
