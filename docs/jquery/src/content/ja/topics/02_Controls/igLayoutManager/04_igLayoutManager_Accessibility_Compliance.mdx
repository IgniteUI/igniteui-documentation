# アクセシビリティの遵守 (igLayoutManager)



##トピックの概要


### 目的

このトピックは、`igLayoutManager`™ コントロールのアクセシビリティ機能を説明し、このコントロールを含むページのアクセシビリティ遵守を実現する方法に関する情報を提供します。

### 前提条件

このトピックを理解するために、以下のトピックを参照することをお勧めします。

-	[igLayoutManager の概要](igLayoutManager-Overview.html): このトピックでは、`igLayoutManager` コントロールの概念について説明し、サポートされているレイアウトやその使用についての情報を提供します。

-	[igLayoutManager の追加](igLayoutManager-Adding.html): このトピックではコード例を使用して、HTML 単体、JavaScript または ASP.MVC 実装のいずれかで `igLayoutManager` コントロールを HTML ページに追加する方法を説明します。





##アクセシビリティ準拠のリファレンス

### 概要

すべての %%ProductName%%® コントロールおよびコンポーネントは、1973 年リハビリテーション法第 508 条第 1194 部 22 条を遵守しています。以下の表には、コントロールに関連する第 1194 部 22 条の特定の規則が記載されています。また、`igLayoutManager` コントロールが各規則を遵守する方法も詳細に説明しています。

各アクセシビリティ規則の要件を満たすために、場合によっては、特定のオプションを設定することによってコントロールを操作する必要がありますが、それ以外の場合はコントロール自身がこの作業を行います。

### アクセシビリティ準拠一覧表

以下の表で、`igLayoutManager` コントロールのアクセシビリティ遵守規則を簡単に説明します。

規則|規則内容|準拠方法
---|---|---
(c)* |色によって伝達されるすべての情報が色を使用しなくても理解できるように (たとえば、コンテキストやマークアップ) Web ページを設計するものとします。|igLayoutManager コントロールは、色に依存しません。
(j)|ページは、2 Hz より大きく、55 Hz を下回る周波数で画面がちらつかないように設計するものとします。|周波数が 2 Hz より大きく、55 Hz を下回る場合は、igLayoutManager コントロール自体が画面のちらつきの原因になることはありません。(コントロールの一部として、BLINK または MARQUEE 要素はありません。)


*- コントロールの制限
