##アクセシビリティ準拠のリファレンス

### 概要

すべての \{environment:ProductName\}™ コントロールおよびコンポーネントは、1973 年リハビリテーション法第 508 条第 1194 部 22 条を遵守しています。表 1 には、コントロールに関連する第 1194 部 22 条の特定の規則が記載されています。また、igMap コントロールが各規則を順守するための詳しい方法も含まれています。

各アクセシビリティ規則の要件を満たすためには、特定のプロパティを設定することによって、コントロールとの相互作用が必要になることもありますが、コントロールがこれらのタスクを実行する場合もあります。

>**注:** jQuery コントロールはコントロールのみであり、したがっていくつかのルールはサポートされず制限としてマーク付けされます。

### アクセシビリティ準拠のリファレンスの概要

以下の表は、`igMap` のアクセシビリティ準拠機能についてまとめたものです。

規則|規則内容|準拠方法
---|---|---
(a)|あらゆる非テキスト要素に対してテキスト相当物を提供するものとします（つまり、「alt」、「longdesc」を介して、または要素コンテンツで)。|igMap はグラフィック専用のコントロールであるため、マップ データを、テーブルまたはチャート データを含む igGrid™ コンポーネントに変換することによって、障害を持つ人のために代替ビューを提供することを検討できます。このようにして、スクリーン リーダーは、ユーザーがチャートの情報を読み出すのを支援します。
(b)|マルチメディア プレゼンテーションに相当する代替物をプレゼンテーションと同期するものとします。|この規則を順守するための代替手段の説明については、規則 (a) の「How We Comply（順守方法）」列内のメモを参照してください。
(c)|色によって伝達されるすべての情報が色を使用しなくても理解できるように (たとえば、コンテキストやマークアップ) Web ページを設計するものとします。|igMap のスタイル設定とテーマ設定機能を使用して、マップの配色を制御し、適切な色を提供できます。
(d)\*|関連付けられたスタイル シートがなくても読めるようにドキュメントを構成するものとします。|コントロールのマークアップは、CSS の規則が制御するクライアント側のコントロールであるため、関連付けられたスタイル シートがないと読み取りできません。規則を順守するための代替手段の説明については、メモ (a) を参照してください。

\* - コントロールの制限
