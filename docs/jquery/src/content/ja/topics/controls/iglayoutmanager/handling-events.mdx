##<a id="overview"></a>イベントの処理 - 概要


### <a id="summary"></a>イベント処理の概要

イベント ハンドラー関数の `igLayoutManager` コントロールへのアタッチは、一般的にコントロールの初期化時に行われます。

HTML 内ではイベント ハンドラーを定義できないので、&#123;environment:ProductNameMVC&#125; を使用するときは、実行時にイベント ハンドラーを割り当てる必要があります。

jQuery はイベント ハンドラーの割り当てるための以下のメソッドをサポートします。

-   bind()
-   delegate()
-   live()
-   on()

`igLayoutManager` は、以下のイベントをサポートします。

-   [`itemRendering`](&#123;environment:jQueryApiUrl&#125;/ui.iglayoutmanager#events:itemRendering) - コンテナーで 1 つの項目が描画された後に発生します。
-   [`itemRendered`](&#123;environment:jQueryApiUrl&#125;/ui.iglayoutmanager#events:itemRendered) - すべての項目が描画された後に発生します。
-   [`rendered`](&#123;environment:jQueryApiUrl&#125;/ui.iglayoutmanager#events:rendered) - 項目がコンテナーの幅または高さに完全に対応しようとする前に発生します。

イベントを処理する方法の詳細は、[&#123;environment:ProductName&#125; でのイベントの使用](/general-and-getting-started/using-events-in-igniteui-for-jquery)のトピックを参照してください。

##<a id="example-jquery"></a>コード例: jQuery での初期化時に itemRendered イベントを処理する


### <a id="itemRender-jquery-description"></a>説明

この例は、初期化時に jQuery で [`itemRendered`](&#123;environment:jQueryApiUrl&#125;/ui.iglayoutmanager#events:itemRendered) イベントにイベント処理関数を割り当てます。

### <a id="itemRender-jquery-code"></a>コード

**JavaScript の場合:**

```js
$(".selector").igLayoutManager({
                        itemRendered: function(evt, ui) { 
                              // Handle event
                        }
});
```



##<a id="example-asp-net"></a>コード例: ASP.NET MVC での初期化時に itemRendered イベントを処理する


### <a id="itemRender-mvc-description"></a>説明

この例は、初期化時に &#123;environment:ProductNameMVC&#125; を使用して [`itemRendered`](&#123;environment:jQueryApiUrl&#125;/ui.iglayoutmanager#events:itemRendered) イベントにイベント処理関数を割り当てます。

### <a id="itemRender-mvc-code"></a>コード

**ASPX (MVC) の場合:**

        
```csharp
@(Html.Infragistics().LayoutManager()
        .AddClientEvent("iglayoutmanageritemrendered", "itemRenderedHandler")
        .Render());
```



##<a id="example-run-time-jquery"></a>コード例: jQuery での実行時に itemRendered イベントを処理する
