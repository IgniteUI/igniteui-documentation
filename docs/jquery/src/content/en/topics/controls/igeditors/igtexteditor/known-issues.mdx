# igTextEditor Known Issues


##Known Limitations


-   When the input element is a `<textarea>` (textMode=”multiline”) element the caret position behaviour can be inconsistent. In some browsers, Gecko-based (Firefox) and WebKit-based (Chrome, Opera, Safari) when a is focused, the insertion point is placed at the beginning of the text by default, instead of at the end as observed in Internet Explorer. To provide consistent behavior the <ApiLink type="igTextEditor" member="selectionOnFocus" section="options" label="selectionOnFocus" /> default value is set to `selectAll` instead of `browserDefault`
-   Multi-line text mode not supported in an `<input>` base element. If the base element is `<input>`, setting the <ApiLink type="igtexteditor" member="textMode" section="options" label="textMode" /> option to *multiline* is not supported.
-   If the base element is `<textarea>`, the <ApiLink type="igtexteditor" member="textMode" section="options" label="textMode" /> option is set to *multiline* and cannot be changed.
-   Changing value after initialization for <ApiLink type="igtexteditor" member="buttonType" section="options" label="buttonType" />, <ApiLink type="igtexteditor" member="textMode" section="options" label="textMode" />, <ApiLink type="igtexteditor" member="dropDownOnReadOnly" section="options" label="dropDownOnReadOnly" />, <ApiLink type="igtexteditor" member="visibleItemsCount" section="options" label="visibleItemsCount" />, and <ApiLink type="igtexteditor" member="dropDownAttachedToBody" section="options" label="dropDownAttachedToBody" /> options is not supported
