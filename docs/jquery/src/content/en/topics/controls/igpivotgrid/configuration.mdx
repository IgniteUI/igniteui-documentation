## <a id='basic'></a>  Basic Configuration

The following table lists the basic options that need to be defined in order to display the data in a pivot table.

For the igPivotGrid:

Option| Description | 
------|-------------|
 <ApiLink type="igPivotGrid" member="dataSource" section="options" label="dataSource" />| An instance of $.ig.OlapXmlaDataSource or $.ig.OlapFlatDataSource.

The **$.ig.OlapXmlaDataSource** component handles the communication with a Microsoft® SQL Server Analysis Services (SSAS) server configured with the msmdpump.dll HTTP data provider. 
The relevant basic settings for it are:

Option| Description | 
------|-------------|
<ApiLink pkg="ig" type="OlapXmlaDataSource" member="options.serverUrl" section="options" label="serverUrl" />| The URL of the XMLA server.
<ApiLink pkg="ig" type="OlapXmlaDataSource" member="options.catalog" section="options" label="catalog" />| The catalog name.
<ApiLink pkg="ig" type="OlapXmlaDataSource" member="options.cube" section="options" label="cube" /> | The name of the cube in the data source.
<ApiLink pkg="ig" type="OlapXmlaDataSource" member="options.rows" section="options" label="rows" /> | A list of hierarchy names separated by comma (,). These will be the hierarchies in the rows of the data source.
<ApiLink pkg="ig" type="OlapXmlaDataSource" member="options.columns" section="options" label="columns" /> | A list of hierarchy names separated by comma (,). These will be the hierarchies in the columns of the data source.
<ApiLink pkg="ig" type="OlapXmlaDataSource" member="options.measures" section="options" label="measures" /> | A list of measure names separated by comma (,). These will be the measures of the data source.

You can refer to the following basic sample with an example configuration.
- [Binding to Xmla Data Source](&#123;environment:SamplesUrl&#125;/pivot-grid/binding-to-xmla-data-source)

The **$.ig.OlapFlatDataSource** component allows multi-dimensional (OLAP-like) analysis to be performed on a flat data collection.
The relevant basic settings for it are:


For the OlapFlatDataSource:

Option| Description | 
------|-------------|
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.dataSource" section="options" label="dataSource" />| Specifies any valid data source accepted by $.ig.DataSource, or an instance of an $.ig.DataSource itself.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.metadata" section="options" label="metadata" />| An object containing processing instructions for the $.ig.DataSource data.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.cube" section="options" label="metadata.cube" />|Metadata used for the creation of the cube.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.cube" section="options" label="metadata.cube.measuresDimension" />| An object providing information about the measures' root node.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.dimensions" section="options" label="metadata.cube.dimensions" />|An array of dimension metadata objects.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.rows" section="options" label="rows" />| A list of hierarchy names separated by comma (,). These will be the hierarchies in the rows of the data source.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.columns" section="options" label="columns" />| A list of hierarchy names separated by comma (,). These will be the hierarchies in the columns of the data source.
<ApiLink pkg="ig" type="OlapFlatDataSource" member="options.measures" section="options" label="measures" />|  list of measure names separated by comma (,). These will be the measures of the data source.

### <a id='custom-drag-drop'></a> Drag and Drop custom elements

The following steps will guide you through the process of implementing dragging and dropping of custom elements in the igPivotGrid.

#### Steps
1. Define elements with the data-type and data-name attributes.

 The data-type should be set to the type of element, could be: Dimension, Hierarchy, Measure or Level. The data-name should be the related value.
 For exampe:

```html
         <div class="ui-igpivot-metadataitem ui-state-default ui-corner-all custom-draggable"
             data-type="Hierarchy"
             data-name="[Seller].[Seller]"><span>Hierarchy: Seller</span></div>
        <div class="ui-igpivot-metadataitem ui-state-default ui-corner-all custom-draggable"
            data-type="Measure"
            data-name="[Measures].[UnitsSold]"><span>Measure: Units Sold</span></div>
```

2. Make the defined elements draggable using jQuery UI's [draggable](https://jqueryui.com/draggable/) Interactions.
